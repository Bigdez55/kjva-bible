# OWNER_DECLARATION.md - Legacy Filename

This file is retained under its historical filename so older references do not
break. In the Tokenless Models repository, authority is local and project
portable:

- The repository owner controls this workspace.
- Consuming projects define their own owner/operator authority model.
- Runtime contracts do not require a specific personal title, product name, or
  external system.
- New documentation should use "owner", "operator", or "project authority"
  unless a consuming project deliberately chooses different language.

This file does not create a dependency on any external repository.
