# Member Reconstitution Doctrine

**Status:** Constitutional Law — Immutable
**Governing Principle:** A member is recoverable. The organism heals.

---

## Two-Part Recovery

### Part 1: Resurrection (The Body)

A runtime supervisor restarts the daemon process. The binary loads, the port binds, the TCP listener starts. The process is alive but has no identity, no memory, no authority. This is the body returning — the vessel is empty.

### Part 2: Reconstitution (The Soul)

- **Registry** provides the DNA (who the member IS)
- **SoulManager** provides the memory (what the member KNOWS — never deleted)
- **Heptagon schema** provides the skeleton (what the member DOES)
- **Attestation** proves the member is genuine (not impersonation)
- **Authority mode** starts at RECOMMENDATION and graduates

This is the soul returning — the vessel is filled.

**Together:** Resurrection brings back the body. Reconstitution restores the soul. Both required. Empty vessel = running daemon with no identity. Identity without vessel = seat exists constitutionally but can't execute.

## Authority Graduation

Reconstituted member enters:
1. **RECOMMENDATION** — can suggest but not act
2. **CONDITIONAL** — can act with oversight
3. **FULL** — normal operation

Each graduation requires observed behavioral fidelity across multiple cycles.

## Impersonation Defense

The most dangerous attack is replacement, not removal. Each member requires:
- Signed daemon identity
- Signed schema fingerprint
- Memory lineage hash chain
- Invariant signature set
- Challenge-response attestation from Ahki/Esther/Sarah triad (with vacancy fallbacks)

Restart without attestation = suspicious until proven legitimate.
