## Scope

### Bug #1: Song.isDropbox Never Encoded (Persistence Bug — Silent Data Corruption)
**File:** `Apollo16/MODELS/Song.swift`  
**Severity:** CRITICAL

**Issue:**  
`encode(with:)` persists four fields but omits `isDropbox`. Decoder reads `isDropbox` unconditionally. After first app restart, every Dropbox-sourced song silently loses its flag, causing playback to resolve against nonexistent local paths.

**Fix:**
```swift
func encode(with coder: NSCoder) {
    coder.encode(title, forKey: keys.title)
    coder.encode(url, forKey: keys.url)
    coder.encode(duration, forKey: keys.duration)
    coder.encode(emptySong, forKey: keys.emptySong)
    coder.encode(isDropbox, forKey: keys.isDropbox)  // ADD THIS LINE
}
```

**Test:**
1. Add a Dropbox song to a note
2. Save the note
3. Restart the app
4. Verify the song's `isDropbox` is still `true`
5. Attempt playback; confirm it works

### Bug #2: Note Force-Casts Crash on Archive Corruption
**File:** `Apollo16/MODELS/Note.swift`, `init?(coder:)` (lines ~40-50)  
**Severity:** CRITICAL

**Issue:**  
Seven string fields are force-cast:
```swift
let uid = coder.decodeObject(forKey: keys.uuid) as! String
```
Any missing key crashes the app permanently (EXC_BAD_INSTRUCTION) during `didFinishLaunchingWithOptions`. User is locked out.

**Fix:**
Convert all force-casts to optional casts with defaults:
```swift
let uid = (coder.decodeObject(forKey: keys.uuid) as? String) ?? UUID().uuidString
let title = (coder.decodeObject(forKey: keys.title) as? String) ?? "Untitled"
let content = (coder.decodeObject(forKey: keys.content) as? String) ?? ""
// ... etc. for all String fields
```

**Test:**
1. Load app normally, create and save a note
2. Manually corrupt the archive in UserDefaults (hex-edit the data key)
3. Restart app
4. Confirm: app launches without crash, note is recovered or skipped gracefully, user sees other notes

---

### Bug #3: Main Thread Sleep at Launch (Watchdog Kill Risk)
**File:** `Apollo16/AppDelegate.swift`, line 22  
**Severity:** CRITICAL

**Issue:**  
```swift
Thread.sleep(forTimeInterval: 1)
```
Blocks main thread in `didFinishLaunchingWithOptions`. iOS watchdog terminates apps that don't return within ~5 seconds. Degrades perceived launch speed by 1+ seconds on every cold start.

**Fix:**
Remove the sleep entirely. If splash screen is needed, use `LaunchScreen.storyboard` or show a brief overlay and dismiss async:
```swift
// Remove: Thread.sleep(forTimeInterval: 1)
// Instead, if splash screen needed:
DispatchQueue.main.asyncAfter(deadline: .now() + 0.5) {
    // Dismiss splash overlay if present
}
```

**Test:**
1. Install on device, force-quit app cache
2. Launch cold
3. Measure launch time with Instruments (should be <2 sec, not <3 sec)
4. Confirm no watchdog kills in console logs
