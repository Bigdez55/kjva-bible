### Issue #3: Duplicate Save Logic in resignedActiveState vs onBackPressed
**File:** `Apollo16/VIEW CONTROLLERS/AddNote.swift`  
**Severity:** MEDIUM (DRY violation, bug divergence risk)

**Current Pattern:**
```swift
func onBackPressed() {
    // Check if text changed
    // Save note if changed
    // Update array_notes
    // Post notification
    // Pop VC
}

@objc func resignedActiveState() {
    // Check if text changed (similar code)
    // Save note if changed
    // Update array_notes
    // Post notification
    // (Does NOT pop VC)
}
```

**Problem:** Any bug fix in one path doesn't propagate to the other. The resign path is missing some checks present in the back-press path.

**Fix:**
Extract shared logic:
```swift
private func saveCurrentNoteIfNeeded() {
    guard textView.text != currentNote.content else { return }
    
    currentNote.content = textView.text
    currentNote.lastModifiedDate = Date()
    
    NoteRepository.shared.saveNote(currentNote)
}

@IBAction func onBackPressed(_ sender: Any) {
    saveCurrentNoteIfNeeded()
    navigationController?.popViewController(animated: true)
}

@objc func resignedActiveState() {
    saveCurrentNoteIfNeeded()
}
```

**Test:**
1. Edit a note, press back → verify saves
2. Edit a note, app resigns active → verify saves
3. Edit a note, app resigns active, resume, press back → verify no duplicate save
4. Don't edit note, press back → verify no unnecessary save

### Issue #5: NotificationCenter.post(name: .splashScreenEnded) Called Too Early
**File:** `Apollo16/AppDelegate.swift`  
**Severity:** MEDIUM (Observer misses notification)

**Current Code:**
```swift
func application(_ application: UIApplication, 
                 didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
    
    // ... setup code ...
    
    NotificationCenter.default.post(name: .splashScreenEnded, object: nil)
    return true
}
```

**Problem:** Posted before any scene is ready. If a VC registers an observer in `viewDidLoad`, it misses this notification because no view controller exists yet.

**Fix:**
Post from `SceneDelegate.sceneDidBecomeActive`:
```swift
// AppDelegate.swift - REMOVE the post

// SceneDelegate.swift
func sceneDidBecomeActive(_ scene: UIScene) {
    NotificationCenter.default.post(name: NSNotification.Name("SplashScreenEnded"), object: nil)
}
```

Or delay it:
```swift
// AppDelegate.swift
DispatchQueue.main.asyncAfter(deadline: .now() + 0.5) {
    NotificationCenter.default.post(name: .splashScreenEnded, object: nil)
}
```

**Better Approach:** Use a state flag in AppDelegate/SceneDelegate that VCs can query:
```swift
class AppState {
    static let shared = AppState()
    var isReadyForUI = false
}

// In sceneDidBecomeActive
AppState.shared.isReadyForUI = true
NotificationCenter.default.post(name: .splashScreenEnded, object: nil)
```
