# Alpha Pulse Engine -- Agent Memory

## Signal Architecture Findings (2026-03-08)

### Critical Bugs Identified
- **Dual-singleton bug**: `AutoTradingService._get_signal_gate()` at line 138 creates `SignalGateService(db=db)` directly, bypassing `get_signal_gate()` singleton. Bootstrap calibration from `main.py:130-144` is discarded. Fix: use `from app.services.signal_gate_service import get_signal_gate`.
- **`episode_terminal` corruption**: `outcome_fill_worker.py:300` always sets True (OR with guaranteed-true condition). Fix: change to `action_lc == "sell"` only.
- **Signal fusion dead code**: `SignalFusionService.fuse()` has zero production call sites. The auto-trading loop uses AI-first/rule-based-fallback, not multi-source fusion.

### Architecture Knowledge
- Default fusion weights: llm=0.40, rule=0.30, drl=0.15, sentiment=0.15
- With DRL+sentiment always None: effective weights are llm=0.571, rule=0.429
- Gate threshold: calibrated_confidence >= 0.55
- R^2 regime threshold: 0.15 (too permissive by Alpha Pulse Protocol 2 standard of 0.45)
- 5 regime presets: BULL_LOW_VOL, BULL_HIGH_VOL, BEAR_LOW_VOL, BEAR_HIGH_VOL, SIDEWAYS
- Strong disagreement veto: 2+ sources with conf>0.70 in different directions => HOLD
