# Apex Autonomous Trader -- Agent Memory

## Bot Loop Architecture (verified 2026-03-18)

- **Main file:** `backend/app/services/auto_trading_service.py` (~2400 lines)
- **Loop interval:** 60 seconds
- **Rescan interval:** every 30 cycles (~30 min), no staleness guard
- **Concurrency:** `asyncio.gather()` with `return_exceptions=True` for per-strategy isolation
- **State lock:** `asyncio.Lock` (lazy-init) protects all class-level dict mutations

## Adaptive Learning Loop (implemented 2026-03-18)

- **Trigger:** rolling win rate < 40% over last 20 resolved trades
- **Method:** `_learn_from_losses()` -- queries 30 recent trades, analyzes per-symbol/strategy/regime/confidence-band stats
- **Adaptations:** raises min_confidence (cap 0.85, step +0.05), excludes symbols >60% loss rate (min 3 trades), excludes strategies >65% loss rate (min 4 trades)
- **Safety:** never excludes all symbols (floor 3 remaining), adaptation state persists across bot restarts within same process
- **Decay:** `_decay_adaptations()` runs every cycle; after 60 min without new trigger, lowers conf by 0.02 and reinstates 1 excluded symbol/strategy
- **Cooldown:** 15 min (down from 1 hour) -- bot resumes with adapted parameters
- **Notifications:** ntfy push on every adaptation ("Ruth is learning from mistakes")
- **Ruth analysis:** optional LLM diagnosis of loss patterns (logged, not used for auto-changes)
- **Wiring:** `_adaptation_state` dict checked in `_run_strategy()` closure for exclusions, in `_process_strategy()` for min_confidence override
- **Metrics:** adaptation state exposed via `get_bot_metrics()` under "adaptations" key

## vLLM Integration

- Timeout: 15s default, per-agent override
- Semaphore: 16 concurrent (raised from 4 for L4 GPU)
- Circuit breaker: 2 failures -> 5 min backoff
- Graceful degradation: returns None -> bot falls through to rule-based

## Session Persistence

- AutoTradingSession table with is_active flag
- Cold-start recovery queries is_active=True, calls start_auto_trading() for each
- KILL_SWITCH_OVERRIDE env var blocks restoration
- Gap: ungraceful shutdown leaves is_active=True, causing stale restart
