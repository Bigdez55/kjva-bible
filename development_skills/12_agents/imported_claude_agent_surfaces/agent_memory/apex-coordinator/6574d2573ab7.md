# CI Speed & Efficiency Audit (2026-03-02)

## Audit Scope
- .github/workflows/ci.yml (16 jobs)
- Makefile.rfc002
- build/genos-build.py
- build/iso-assemble.py
- platform/docker/Dockerfile.* (11 files)
- .dockerignore

## Critical Finding (BLOCKER)
DRIFT-DOCKER-03: .dockerignore line 3 (`*`) excludes everything and re-includes via !patterns.
tools/ is NOT in the whitelist. Dockerfile.fabric-energy and Dockerfile.merkle-collective
both COPY from tools/. Actual docker build will fail at COPY step with "file not found".
Fix: add to .dockerignore:
  !tools/
  !tools/fabric_energy/
  !tools/fabric_energy/**
  !tools/merkle_collective/
  !tools/merkle_collective/**
  !tools/bridges/
  !tools/bridges/**

## Speed Findings

### S-01 (HIGH): No pip caching
- 5 setup-python calls, zero have cache: 'pip'
- Affected jobs: policy-and-tests (L18), security-gates (L52), xisc-conformance (L95), poc-python-tools (L217), k8s-manifest-validate (L280)
- Fix: add cache: 'pip' + cache-dependency-path to each setup-python call
- Savings: 4-7 min/run

### S-02 (HIGH): npm ci runs twice, no cache
- typescript-build (L243) and eslint-check (L257) both run npm ci in compositor/genos-shell
- Neither setup-node call has cache: 'npm'
- Fix: add cache: 'npm' + cache-dependency-path to both setup-node calls; consider merging jobs
- Savings: 2-4 min/run

### S-03 (MEDIUM): Duplicate ruff invocations
- policy-and-tests L33: ruff check .  (covers everything)
- poc-python-tools L228: ruff check tools/  (strict subset)
- Fix: remove L227-228 from poc-python-tools
- Savings: 10-20 sec/run

### S-04 (MEDIUM): No -j$(nproc) in Makefile.rfc002
- CI at L155/L167: make -f Makefile.rfc002 run-tests (no -j flag)
- 8 independent C binaries compiled serially; all have zero inter-dependency
- Fix: change to: make -j$(nproc) -f Makefile.rfc002 all && make -f Makefile.rfc002 run-tests
- Savings: 15-30 sec/run

### S-05 (HIGH): mksquashfs uses single-threaded xz
- iso-assemble.py L239: "-comp", "xz" with no -processors flag
- xz is slowest available; zstd at level 19 is 3-5x faster with ~5-10% larger output
- Fix: change to -comp zstd -Xcompression-level 19 -processors $(nproc)
- Savings: 4-12 min/ISO build

### S-06 (HIGH): 11 Docker images built serially in genos-build.py
- genos-build.py L593-602: sequential for loop, blocking subprocess per image
- All 11 share the same base image and pip layer; could parallelize via ThreadPoolExecutor
- Savings: 8-28 min on local full build

### S-07 (HIGH): security-gates gated on policy-and-tests unnecessarily
- ci.yml L47: needs: policy-and-tests
- Bandit, pip-audit, Trivy, npm-audit are all static/FS tools with no test dependency
- Fix: remove the needs: line
- Savings: 3-4 min wall-clock off critical path

### S-08 (MEDIUM): poc-security-barriers and poc-demo-gates recompile binaries
- Both jobs rebuild binaries already built in poc-test-suites-ubuntu22
- Fix: upload build/rfc002/ as artifact from poc-test-suites-ubuntu22; download in dependents
- Savings: 30-60 sec/run per downstream job

### S-09 (MEDIUM): poc-python-tools pip install uncached (resolved by S-01)

### S-10 (MEDIUM): xisc-conformance may duplicate policy-and-tests coverage
- policy-and-tests L38 comment: "includes XISC conformance"
- xisc-conformance L103: python -m pytest xisc/conformance/ --backend=linux -v
- Investigate if --backend=linux adds unique coverage; if not, eliminate the job

## Efficiency Findings

### E-01 (HIGH BLOCKER): .dockerignore excludes tools/ (see DRIFT-DOCKER-03 above)

### E-02 (MEDIUM): No BuildKit cache mounts on pip install
- All 11 Dockerfiles L4: pip install --no-cache-dir
- Layer ordering IS correct (requirements first, app source last)
- Fix (local dev only): use --mount=type=cache,target=/root/.cache/pip without --no-cache-dir
- BuildKit cache mounts do not enter final image layer

### E-03 (MEDIUM): docker build --check sends full context 11 times
- ci.yml L267-271: loop calls docker build --check -f "$df" . for each Dockerfile
- --check mode does not need actual source files; context transmission is wasteful
- Alternative: replace with hadolint (Dockerfile linter, no context needed)

### E-04 (LOW): Vestigial AS base in two single-stage Dockerfiles
- Dockerfile.merkle-collective L1: FROM python:3.12-slim AS base (no second stage)
- Dockerfile.fabric-energy L1: FROM python:3.12-slim AS base (no second stage)
- All other 9 Dockerfiles omit AS alias
- Fix: remove AS base from both files

### E-05 (LOW): Turbo build cache not persisted for ISO builds (future concern)

## Stability Findings

### ST-01 (CRITICAL): Zero timeout-minutes on all 16 jobs
- Default GitHub timeout: 6 hours. Any hung step burns runner for 6h.
- Recommended timeouts: 5-20 min depending on job complexity (see report for table)
- Fix: add timeout-minutes: N to every job definition

### ST-02 (HIGH): Trivy installed via curl|sh pipe
- ci.yml L73-75: curl -sfL https://...trivy.../install.sh | sh
- Version is pinned (v0.50.0) but install SCRIPT is fetched from main branch live
- Supply chain risk: script could be modified upstream between runs
- Fix: use aquasecurity/trivy-action@0.28.0 (pinned action, no curl pipe)

### ST-03 (MEDIUM): npm audit at repo root misses workspace members
- ci.yml L83-87: npm audit --audit-level=high in working-directory: .
- Audits root package.json only; compositor/browser/orange/design-system workspaces unchecked
- Fix: npm audit --workspaces or run per-workspace
- continue-on-error: false at L87 is no-op (same as default); cosmetic cleanup opportunity

### ST-04 (MEDIUM): No apt cache for cmake/build-essential/cppcheck
- xisc-shim-build L112 and poc-static-analysis L176 both apt-get install without caching
- Fix: add actions/cache@v4 for /var/cache/apt/archives keyed on runner image + package list
- Savings: 30-60 sec/run on cache hits

### ST-05 (LOW): Dual ruff invocations may use different effective configs
- Resolved by S-03 (remove redundant invocation)

## Verified Correct (do not change)
- Dockerfile layer ordering: requirements.txt COPY -> pip install -> app COPY (all 11 correct)
- --no-cache-dir on pip install is correct for final image size
- -j$(nproc) already present in xisc-shim-build cmake step (ci.yml L119)
- mksquashfs auto-detects processor count on mksquashfs >= 4.4 but explicit flag is safer
- xorriso call in iso-assemble.py is single-threaded (xorriso does not support parallelism)
- DOCKER_SERVICES in genos-build.py correctly lists 11 services (fabric-energy + merkle-collective confirmed)
