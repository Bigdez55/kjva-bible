## Key Design Decisions

### Hardcoded IP Removal
- `config.py` VLLM_FALLBACK_BASE_URL default was already `""` (removed prior to this session)
- `infra/llm.py:get_llm_base_url()` resolution: VLLM_BASE_URL → VLLM_FALLBACK_BASE_URL → localhost:8000
- NO other file may resolve LLM URL — infra/llm.py is the single source of truth

### Sync vs Async DB Paths
- `app/db/base.py` (sync psycopg2 + SessionLocal) — UNTOUCHED, still the primary path
- `app/infra/storage.py` (async asyncpg + get_async_session) — NEW, for ruth/ and future async routes
- Both coexist; infra/storage.py requires `pip install asyncpg aiosqlite` (not in default requirements)

### InMemoryCache Thread Safety
- Uses asyncio.Lock on every mutating operation (set, delete, flushdb)
- NOT process-safe (each replica has own store) — use redis/valkey for distributed deployments

### SECRETS_PROVIDER Invariant
- `google.*` imports are ONLY allowed in `backend/app/infra/secrets.py`
- Any other file importing google.* is a policy violation
- Verification: `grep -r "from google" backend/app/` should return ONLY infra/secrets.py hits

## docker-compose.local.yml Port Mapping
- Backend: host:8000 → container:8080 (matches Dockerfile EXPOSE 8080)
- Frontend: host:3000 → container:8080 (matches frontend Dockerfile EXPOSE 8080)
- Postgres: 5432:5432
- Valkey:   6379:6379

## Valkey Recommendation
- Use `valkey/valkey:8.1` image in all new deployments
- `CACHE_PROVIDER=valkey` or `CACHE_PROVIDER=redis` — both treated identically in infra/cache.py
- 37% better throughput, BSD licence, Linux Foundation governance
