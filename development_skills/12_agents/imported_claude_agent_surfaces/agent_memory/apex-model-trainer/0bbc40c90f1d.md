## Critical Known Bugs

### episode_terminal Corruption (P0 — ALL 160 DRL records corrupted)

- File: `backend/app/services/outcome_fill_worker.py` line 300
- Bug: `row.episode_terminal = (action_lc == "sell") or (row.price_at_1d is not None)`
- Right operand is ALWAYS True inside that branch (outer guard requires price_at_1d IS NOT NULL)
- Fix: `row.episode_terminal = (action_lc == "sell")`
- SQL repair: `UPDATE trade_decision_logs SET episode_terminal = (LOWER(action) = 'sell') WHERE price_at_1d IS NOT NULL AND episode_terminal IS NOT NULL;`

### outcome_fill_worker Uses Daily Bars (P0 — blocks 1h/4h fill entirely)

- File: `backend/app/services/outcome_fill_worker.py` line 208
- Bug: `mds.get_historical_data(sym, limit=120)` — no timeframe arg, defaults to "1day"
- Fix: `mds.get_historical_data(sym, timeframe="1hour", limit=600)`
- Impact: Switching to hourly bars projects coverage from 4.5% to 65%+ within 24h

## Multi-Round Curriculum Design (2026-03-09)

### Root cause: "signals only got 3 gradient steps"

batch=2, grad_accum=32 → eff_batch=64; 396 examples → 6 steps/epoch; packing collapses further to ~2.
FIX: reduce grad_accum to 8 (eff_batch=16) AND expand corpus to >=4,000 examples → 250 steps/epoch.

### 4-Round Hyperparameter Ladder

| Round | Phase | LR   | Epochs | eff_batch | Corpus | Sujet% | steps/epoch | Scheduler |
|-------|-------|------|--------|-----------|--------|--------|-------------|-----------|
| 1     | A     | 2e-4 | 3      | 16        | 7.5k   | 20%    | 209         | cosine    |
| 2     | B     | 8e-5 | 3      | 16        | 10k    | 25%    | 280         | cosine    |
| 3     | C     | 3e-5 | 4      | 16        | 14k    | 30%    | 392         | cosine_restarts |
| 4     | ALL   | 1e-5 | 2      | 16        | 22k    | 35%    | 617         | linear    |

Loss gate: stop if round-over-round improvement < 0.01. Stop early if loss <= 0.15.
Wealth adapter reference: loss=0.096 at Phase C, 11,927 pairs.
Seeds per round: R1=42, R2=137, R3=271, R4=512 (maximise unique coverage).

## Phase 6 Standalone Training Scripts (2026-03-18)

Three scripts written as self-contained CLI tools (not Django/FastAPI tasks):

| Script | Schedule | Key gate |
|--------|----------|----------|
| `scripts/biweekly_finetune_t1_t2.py` | Every 14 days | +2% improvement vs baseline |
| `scripts/monthly_finetune_all_tiers.py` | Monthly | ALL tiers pass or hold (MAX_REGRESSION -1%) |
| `scripts/quarterly_full_retrain.py` | Quarterly | `--approve` flag or QUARTERLY_TRAIN_APPROVED=true |

Design patterns:
- HF AutoTrain API and H100 SSH calls are STUBBED with TODO comments + print statements
- All three use `structlog` (not stdlib logging) for structured output
- Rolling data paths: `training_data/rolling/14d.jsonl`, `30d.jsonl`, `90d.jsonl`
- Temporal split ALWAYS 80/20 by sort order — no shuffle ever
- `get_baseline_accuracy()` reads `{output_dir}/{tier_lower}_last_summary.json` for prior run accuracy
- Conflict deduplication window: 1 hour — drops BOTH examples when BUY/SELL conflict exists
- ntfy topic: `elson-training` — all three scripts use same topic
- Quarterly script exits early (no training) if `--approve` not set — generates report + ntfy then halts
- Monthly `TRAIN_ORDER = ["T6", "T3", "T5", "T1", "T2"]` — teachers before distillation students
- Monthly curriculum: R1=LR 2e-4/3ep, R2=LR 8e-5/3ep, R3=LR 3e-5/4ep (mirrors multi_round_curriculum_trainer.py ladder)

## Scheduled Training Pipeline (2026-03-18)

Five files written for continuous improvement loop:

| File | Schedule | Gate |
|------|----------|------|
| `backend/app/services/training_data_collector.py` | 11 PM PT nightly | none |
| `backend/app/tasks/weekly_training_review.py` | Friday 6 PM PT | none |
| `backend/app/tasks/biweekly_training.py` | 2nd Sunday 2 AM PT | AUTO_TRAINING_ENABLED |
| `backend/app/tasks/monthly_training.py` | 1st Saturday midnight PT | AUTO_TRAINING_ENABLED |
| `backend/app/tasks/quarterly_review.py` | On demand | QUARTERLY_TRAIN_APPROVED |

Key design decisions preserved:
- `training_data_collector.py` is the ONLY file that writes JSONL — others import `_build_trade_example` and `_write_jsonl` from it
- Output root: `$TRAINING_DATA_ROOT` env or `<repo_root>/training_data/` — daily/ and intel/ subdirs
- `training_data/runs/<prefix>_YYYYMMDD_HHMMSS/` for biweekly/monthly run artifacts
- Temporal split ALWAYS preserved (no shuffling) — oldest → train, newest → val
- `MIN_CONFIDENCE_FILTER=0.55` in biweekly; `MIN_CONFIDENCE=0.50` in daily collector
- Win threshold: 0.5% return counts as a correct directional call (consistent across all files)
- HF AutoTrain and H100 VM calls are clearly stubbed with TODO markers — no real API calls yet
- `_send_*_ntfy()` functions call `ntfy._send()` directly (not the typed wrapper methods)
- `ACCURACY_IMPROVEMENT_GATE = 0.02` (+2%) required to deploy any new adapter
- Monthly training: no regression allowed across ALL tiers (not just net positive)
