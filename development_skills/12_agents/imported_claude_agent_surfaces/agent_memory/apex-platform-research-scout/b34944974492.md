# Apex Platform Research Scout — Persistent Memory

## Evaluated Technologies (2026-03-08)

### Broker APIs

- **Alpaca**: Stay. Best retail algo API. Options stream is msgpack-only (not JSON) — downstream parsers must handle msgpack. Options assignments not on WebSocket — must poll REST.
- **IBKR Web API**: No desktop required (2024). OAuth2. But multi-leg options missing. Only basic order types. Newer and less battle-tested than TWS API.
- **Schwab API**: Single-leg options only. No multi-leg spreads. No futures trading. TD Ameritrade migration ongoing.
- **Tradier**: $10/mo Pro (after 12mo promo at $0). $0.35/contract SPX. Options-focused. Good API. No fractional shares, no crypto.
- **Webull/Robinhood**: No official APIs. Skip.

### Market Data Providers

- **Finnhub**: FREE tier 60 req/min. API key already in Secret Manager. `FinnhubSentimentService` built and NEVER WIRED — P0 fix.
- **Polygon.io**: $29/mo Starter — stocks WebSocket, 5yr history, unlimited calls. Options flow (unusual activity) available on higher tiers. Key already in Secret Manager.
- **Databento**: Pay-per-use ($125 free credit). Sub-microsecond (42us cross-connect, 590us internet). Institutional. $0 for equities live data on subscription plans (Jan 2025 change). Overkill for retail.
- **Alpha Vantage**: Free tier 25 req/day — too low for live scanning. Skip for hot paths.
- **Twelve Data**: $29/mo, no options data.
- **Tiingo**: $10/mo, EOD-focused.

### Redis Alternatives

- **Valkey 8.1**: BSD 3-clause (Linux Foundation). Drop-in redis-py compatible. 37% better SET throughput vs Redis 8.0. 60% lower GET latency. Docker: `valkey/valkey:8.1`. RECOMMENDED for docker-compose.local.yml.
- **DragonFly**: BSL 1.1 (non-OSS). 4.5x throughput vs Valkey on GCP. Better memory efficiency (38% less per item). Skip — BSL licensing risk for open-source use.
- **KeyDB**: Snap-acquired, low development activity. -15-18% vs Redis in benchmarks. Skip.

### Time-Series Databases

- **TimescaleDB**: PostgreSQL extension. `CREATE EXTENSION timescaledb`. 20x insert speed at scale, 14,000x query improvement on time-range scans. 90% compression. RECOMMENDED — no migration, extension only.
- **QuestDB**: Standalone TSDB. 40x faster OHLCV queries (25ms vs 1,021ms vs TimescaleDB). ASOF JOIN for financial data. Separate service — HIGH migration effort. Revisit at 10M+ rows.
- **pg_duckdb**: PostgreSQL extension embedding DuckDB. Analytical queries that timeout in Postgres complete in <10s. Perfect for backtesting. Prototype in Phase 6.
- **InfluxDB**: Flux query language (not SQL). Skip.

### Message Queues

- **NATS JetStream**: Single binary, 17MB Docker image, sub-millisecond latency, Go-native (nats.go), Python client (nats-py). JetStream adds persistence. RECOMMENDED for tick durability.
- **Redpanda**: Kafka-compatible, no ZooKeeper. Better for high-throughput multi-tenant scenarios. Overkill for 100 msg/s Elson volume.
- **Kafka**: Too operationally complex. Skip for single-user platform.
- **RabbitMQ**: 50-100K msg/s cap. Simpler than Kafka but no Kafka compatibility and lower throughput than NATS.

### Cloud Platforms

- **GCP Cloud Run**: Stay for production. L4 GPU is GCP-specific. Cloud SQL mature.
- **Fly.io**: Best cloud-agnostic alternative. Persistent VMs (no cold start). Fly Postgres. ~$15-25/mo. 2-3 day migration effort for `fly.toml`.
- **Railway**: Best for staging/demo. GitHub-native. PostgreSQL + Redis included. $5/mo base. US-only.
- **Render**: Free tier with cold starts. OK for demos.

### Frontend Charts

- **LightweightCharts v5.1.0**: 45KB. OHLCV/candlestick. Already integrated. Keep.
- **Recharts v3**: ~250KB. Standard analytics. Already integrated. Known: formatter callbacks require `as any` cast.
- **Nivo**: Tree-shakeable (separate packages per chart type). Full TypeScript. Stream, treemap, heatmap, calendar, bump, waffle charts. RECOMMENDED for Ruth analytical views.
- **Tremor**: Recharts + Tailwind wrapper. v3.18.7 (Jan 2025). Acquired by Vercel. Best for KPI cards and overview dashboards. Does NOT add new chart engine.
- **Observable Plot**: Good for statistical/research views. Not needed for Ruth.
