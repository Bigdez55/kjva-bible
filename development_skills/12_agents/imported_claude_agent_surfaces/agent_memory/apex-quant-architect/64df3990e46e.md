### 4.3 threading.RLock -> asyncio.Lock Migration Strategy

The circuit breaker uses `threading.RLock()` which blocks the event loop when acquired from async code. Migration plan:

1. **Create `AsyncCircuitBreaker` wrapper** that uses `asyncio.Lock()` internally
2. **Keep sync `CircuitBreaker` unchanged** for sync callers (trading.py, advanced_trading.py service, backtesting)
3. **Dual-mode access:**
   ```python
   # For async callers:
   async_cb = get_async_circuit_breaker()
   allowed, status = await async_cb.check()

   # For sync callers (unchanged):
   cb = get_circuit_breaker()
   allowed, status = cb.check()
   ```
4. **Shared state:** Both wrappers share the same `circuit_breakers` dict and `status_file`
5. **Phase 1:** Convert 11 async callers to use `get_async_circuit_breaker()`
6. **Phase 2:** Eventually merge when all sync callers are async-ified

### 5.2 Missing Columns for IQL Replay Buffer

The Implicit Q-Learning (IQL) replay buffer requires 5 additional columns to transform
TradeDecisionLog into a proper offline RL dataset:

| Missing Column | Type | Purpose |
|----------------|------|---------|
| `state_embedding` | `BYTEA` or `JSON` | Serialized state vector (price, RSI, SMA ratios, volume, portfolio allocation, position P&L) â€” the "observation" in RL terms |
| `reward` | `Numeric(18,8)` | Computed reward signal: `sign(action) * pct_change_1h` for immediate, or risk-adjusted return. Backfilled same time as price_at_1h |
| `next_state_embedding` | `BYTEA` or `JSON` | State vector at t+1 (1h or 4h later), needed for Bellman equation bootstrap |
| `done` | `Boolean` | Episode termination flag: True if position was closed, daily session ended, or stop-loss hit |
| `action_logit` | `Numeric(10,6)` | Raw confidence logit before thresholding â€” needed for importance sampling in offline RL. Currently only `confidence` (post-sigmoid) is stored |

### 5.3 Current Outcome Backfill Pipeline
- **Location:** `_fill_pending_outcomes()` at line 992-1056
- **Trigger:** Called at end of each `_process_strategy()` iteration (line 669-672)
- **Logic:** Finds rows where `outcome_filled_at IS NULL` and `decided_at <= NOW() - 1h`
- **Fills:** `price_at_1h`, `price_at_4h`, `price_at_1d` from live quotes
- **Limit:** Processes max 20 rows per iteration to avoid slowing trading loop
- **Current stats:** 3,559 decisions logged, 160 outcomes backfilled (as of 2026-02-22)

- **CRITICAL:** Adding 5 columns requires `ALTER TABLE trade_decision_logs ADD COLUMN` on Cloud SQL BEFORE deploy
- Reference: Production outage 2026-02-15 caused by schema drift on User model
- All 5 new columns must be NULLABLE to avoid breaking existing rows
- Backfill `reward` column via: `UPDATE trade_decision_logs SET reward = ... WHERE price_at_1h IS NOT NULL`

## 6. Risk Assessment Summary

### 6.1 Blockers for Crypto Support (P0)
1. `time_in_force: "day"` hardcoded (line 232) â€” must be "gtc" for crypto
2. `asset_type: "equity"` hardcoded in get_positions() (line 315) â€” mislabels crypto
3. Quote endpoint uses `/stocks/` path (line 506) â€” wrong for crypto
4. `_is_market_open()` checks NYSE hours only (line 1059-1074) â€” blocks crypto 24/7
5. `list_tradable_assets()` only fetches `us_equity` (line 943) â€” no crypto seeding

### 6.2 Blockers for Options Support (P1)
1. No OCC symbology support in Trade model or order preparation
2. No options-specific data client initialization
3. `get_account_features()` already surfaces approval level (line 565) â€” reusable
4. No greeks storage in TradeDecisionLog or market_context

### 6.3 Architecture Risks (P2)
1. Circuit breaker uses `threading.RLock()` called from async context â€” blocks event loop
2. `circuit_breaker_status.json` file persistence â€” not suitable for Cloud Run (ephemeral filesystem)
3. Singleton class-level state in AutoTradingService â€” no multi-process support
4. No PDT enforcement gate in trading loop
