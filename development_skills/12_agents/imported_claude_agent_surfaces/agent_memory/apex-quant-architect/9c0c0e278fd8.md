- `submit_options_order()` on AlpacaBroker wraps `POST /v2/orders` with OCC validation

## Key Patterns & Invariants

- `portfolio.get_daily_trade_count(session=)` ‚Äî accepts Optional[Session]; ALWAYS pass session
- `portfolio.get_daily_drawdown(session=)` ‚Äî same pattern, ALWAYS pass session
- `circuit_breaker.check()` returns `Tuple[bool, CircuitBreakerStatus]` ‚Äî ALWAYS unpack
- `drawdown_pct` = FRACTION (0.03), NOT percent (3.0); `atr=` required positional
- Alpaca `client_order_id` = UUID per order for idempotent retry dedup
- 7-Tier Router: T1-T5 = HF Router (free), T6 = Ruth 14B (paid), T7 = deterministic Python
- `_FALLBACK_REGIME` re-scoring: signals generated against QUIET/0.30, rescored after regime resolves
- `def` for DB-only endpoints; `async def` only when awaiting external APIs
- Recharts v3: formatter callbacks always cast `as any`
- EFT kwargs: `user_prompt=` (not `prompt=`), `context=` (not `portfolio_context=`)
- Crypto symbols in market_data.py: use `_is_crypto_symbol()` + `_crypto_base()` for format conversion
- Alpaca crypto data: `v1beta3/crypto/us/latest/quotes?symbols=BTC/USD` (NOT `/stocks/`)
- Yahoo crypto: `BTC-USD` (dash), AV crypto: `CURRENCY_EXCHANGE_RATE`, Finnhub: `BINANCE:BTCUSDT`, Polygon: `X:BTCUSD`
