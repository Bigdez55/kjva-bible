## Consensus Gate Thresholds

| Gate   | Threshold        | Description                          |
|--------|------------------|--------------------------------------|
| Gate 1 | conf >= 0.40     | T1 scout passes to T3               |
| Gate 2 | conf >= 0.55     | T3 calibrated passes to T5          |
| Gate 3 | 2/3 free agree   | T5 arbiter confirms consensus       |
| Gate 4 | conf >= 0.65     | T6 Ruth + KellySizer + circuit_breaker |

## Schema Additions (Phase 5)

New columns on TradeDecisionLog:
- `tier_used: str` — which model tier made this prediction
- `tier_confidence: float` — confidence from the specific tier
- `tier_action: str` — BUY/SELL/HOLD from the tier
- `consensus_agreed: bool` — did this tier agree with consensus
- `was_correct: bool` — filled by outcome_fill_worker

New report_type for DailyMarketIntelligence:
- `"overnight_research"` — T4 deep analysis session output

## Risk Mitigations

1. HF serverless cold starts: pre-warm at 5 AM, circuit breakers per tier
2. Consensus latency (15-22s): use limit orders, price at T1 timestamp
3. Model hallucination: all outputs validated against real market data
4. T6 SPOF: allow T3+T5 unanimous for paper trades + tiny positions
5. Regulatory: PDT tracking, position limits, KYC/AML compliance
