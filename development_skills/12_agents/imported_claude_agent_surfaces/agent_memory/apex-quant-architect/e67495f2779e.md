## 11. ai_portfolio_manager.py (Service)

**Path:** `backend/app/services/ai_portfolio_manager.py`
**Lines:** 1288

### Key Class: `AIPortfolioManager`

**Dependencies:**
- `MarketDataService` (yfinance prices)
- `NeuralNetworkService` (price/volatility predictions)
- `MarketDataProcessor` (technical indicators)
- `ElsonFinanceEnsemble` (optional, for ensemble_enhanced method)
- `Redis` (caching, optional)

**Optimization Methods:**
1. `_optimize_efficient_frontier()` - scipy.optimize SLSQP, max Sharpe ratio
2. `_optimize_black_litterman()` - ML predictions as views, blended returns
3. `_optimize_risk_parity()` - equal risk contribution, inverse volatility start
4. `_optimize_ml_enhanced()` - blends efficient frontier + risk parity weighted by ML confidence
5. `_optimize_ensemble_enhanced()` - ElsonFinanceEnsemble signals adjust base allocation

**Market Timing:** Technical score (RSI, MACD, Bollinger) + ML prediction blend (60/40)

**Key Constants:**
- risk_free_rate: 0.02
- min_allocation: 0.05 (5%)
- max_allocation: 0.30 (30%)
- min_trade_impact: 0.02 (2%)
- cache_ttl: 3600s
- lookback_days: 120
- prediction_horizon: 7 days

## Cross-Cutting Observations

### Auth Pattern Inconsistency
- Some files use `from app.api.deps import get_current_active_user, get_db`
- Others use `from app.core.security import get_current_active_user` + `from app.db.base import get_db`
- Both appear to work; likely the same underlying functions

### Schema Location Inconsistency
- ai_portfolio.py, risk_management.py, auto_trading.py: **Inline Pydantic models** (not in schemas/)
- trading.py, wealth_advisory.py, education.py, micro_invest.py: **Centralized schemas** (in schemas/)
- advanced_trading.py: **Inline models** for request/response
