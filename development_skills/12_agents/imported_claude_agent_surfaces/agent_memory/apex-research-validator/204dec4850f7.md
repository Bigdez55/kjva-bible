## Key Patterns Learned

### 1. Circuit Breaker API
- `CircuitBreaker.check()` returns `Tuple[bool, CircuitBreakerStatus]` -- NOT a plain bool
- Code that does `if not circuit_breaker.check()` will NEVER fire because tuples are truthy
- Fixed in `auto_trading_service.py:549-550`, still broken in `trade_executor.py:287,293,303`

### 2. Portfolio.get_daily_drawdown() Requires Session
- Returns `Decimal("0")` when called without a session argument
- `trade_executor.py:572` calls it without session -- daily loss gate is dead

### 3. Two Competing Health Endpoints
- Root `/health` in `main.py:289-335` -- returns 503 for unhealthy (CORRECT for Cloud Run)
- API `/api/v1/monitoring/health` in `monitoring.py:103-145` -- returns 200 for degraded (WRONG)

### 4. win_rate Backend/Frontend Mismatch
- Backend `analytics.py:273` returns fraction (0.0-1.0)
- Frontend `analyticsApi.ts:26` expects percent (0-100)
- `PerformanceDashboard.tsx:525` renders `value={data.win_rate}` with `suffix="%"`

### 5. Credential Exposure Pattern
- `.claude/settings.json` contains production DB password in permission allowlist strings
- Pattern: `PGPASSWORD=<password> psql` commands stored as Claude Code permissions
- Must add to `.gitignore` or remove passwords
