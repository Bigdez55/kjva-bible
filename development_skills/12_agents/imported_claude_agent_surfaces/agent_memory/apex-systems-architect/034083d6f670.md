# Non-Original Dependency Audit — 2026-04-04

## Key Findings

### 100% Original (Zero External Dependencies)
- XSEC: SHA-256/384/512, AES-256-GCM, AES-NI, ChaCha20-Poly1305, X25519, Ed25519, ECDSA-P256, RSA-2048-PSS, TLS 1.3, X.509 — all freestanding C, PAL only
- XNET + XVPN: WireGuard protocol reimplemented (Noise_IKpsk2), uses XSEC crypto via weak symbols
- vDRAM: PAL-only, explicit #include audit in header
- SoftRoCE: Pure Python stdlib sockets (TCP + UDP paths), zero Linux kernel dependency

### External Dependencies (Cloud Tier Only)
- 14 Python packages in `devices/cloud/platform/services/requirements.txt`
- Key: fastapi, cryptography, bcrypt, pyjwt, sqlalchemy, minio, pydantic
- `cryptography` also used by Council Dockerfile (Dockerfile.council line 36)

### Linux Kernel Module (Datacenter Only)
- `devices/cloud/gateway/zc_gw/` — QP Relay uses linux/module.h, linux/slab.h, rdma/ib_verbs.h
- Architecturally correct: runs on datacenter nodes, not XENOS

### Gap: WG-01
- iOS `com.genos.app.tunnel` packet-tunnel extension not in codebase
- WireGuardManager.swift is a config manager using Apple NetworkExtension framework
- Extension implementation determines if mobile VPN is fully original

## Elimination Priority
- E-01: XSEC FFI Python bindings → kills `cryptography` package (2 days)
- E-02: Original JWT encode/decode → kills `pyjwt` (1 day)
- E-03: XSEC bcrypt → kills `bcrypt` (2 days)
- E-04: iOS tunnel extension implementation → closes WG-01 (1 week)
- E-05/E-06: XBLOB/XSTORE Python adapters → kills minio+sqlalchemy (FFI already exists)
