## What Was Implemented

### 1. Data Structure Changes (§3)

### 2. Syscall Handler Functions (§4)
All 7 Phase 2d handlers implemented in kernel/xenos/core/main.c (lines 1190-1656):

| Syscall | Nr | Handler Name | Purpose |
|---------|----|----|---------|
| XK_SYS_PROCESS_CREATE | 7 | `kern_process_create()` | Allocate isolated process (PML4, user stack, guard page) |
| XK_SYS_PROCESS_START | 8 | `kern_process_start()` | Trivial validation of process handle |
| XK_SYS_PROCESS_EXIT | 9 | `kern_process_exit()` | Tear down process, mark threads XF_DONE, free resources |

| XK_SYS_THREAD_KILL | 14 | `kern_thread_kill()` | Force-terminate another thread (safe in cooperative model) |
| XK_SYS_JOB_CREATE | 16 | `kern_job_table_query()` | Create named job slot; returns slot index |

Plus helper: `kern_job_remove_pid()` — deregister process from job's child list

### 3. Syscall Dispatch Table Renumbering (§5)
- **Old dispatch** (Sprint 11): Thread at 7, yield at 8, join at 9, etc.
- **New dispatch** (Phase 2d): Process at 7-10; thread at 11-16; legacy ops shifted to 17-37
- **XKABI convergence**: Kernel now uses XKABI numbers directly (source of truth: xisc/include/xkabi_syscalls.h)
- **Dispatch cases**: Replaced old #define-based cases (e.g., `case XK_SYS_THREAD_YIELD`) with numeric literals (`case 17u`)
- **#defines renamed**: All legacy macros (XK_SYS_THREAD_CREATE, etc.) updated to new numeric values for reference only

### 4. Error Handling & Safety
- All handlers return `int64_t` with negated error codes (XENOS_EINVAL, XENOS_ENOMEM, etc.)
- Memory cleanup on all error paths (no leaks):
  - `kern_process_create()`: frees PML4 + user stack on any failure
  - `kern_thread_create()`: frees kernel stack on PMM failure
  - `kern_process_exit()` / `kern_thread_exit()` / `kern_thread_kill()`: free kernel stacks atomically
- Guard page installed at PROC_USER_STACK_VA - 4096 with prot=0 (generates #PF on access)

### 5. Integration Points
- **Root job**: Pre-initialized at g_jobs[0], state=XJ_ACTIVE (never freed)
- **Fiber mirroring**: g_processes[slot] carries PML4 metadata for scheduler CR3 reloading
- **Scheduler changes**: No changes needed — already skips non-XF_READY states
- **XF_SUSPENDED**: Scheduler-transparent; round-robin only picks XF_READY fibers

## Architecture Diagrams

### Process/Thread Lifecycle
```
kern_process_create()
  → Allocate PML4, user stack, guard page
  → Process slot active=true, fiber state=XF_FREE

kern_thread_create()
  → Allocate kernel stack
  → Thread slot state=XF_SUSPENDED, active=false (kernel fiber)

kern_thread_start()
  → Patch entry/arg in fiber record
  → Transition XF_SUSPENDED → XF_READY
  → Scheduler picks it up next yield

[fiber runs entry(arg)]

kern_thread_exit()
  → Mark XF_DONE, free kernel stack

kern_process_exit()
  → Mark all process threads XF_DONE
  → Free user stack, PML4
  → Deregister from job
```

### Job Hierarchy (Phase 2d Model)
```
g_jobs[0]  ← root job (implicit, always XJ_ACTIVE)
  ├─ g_jobs[0].child_pids[0] = PID 1 (first process)
  ├─ g_jobs[0].child_pids[1] = PID 2 (second process)
  └─ g_jobs[0].child_count = 2

[Future: g_jobs[1..15] = named child jobs with parent_slot=0]
```
