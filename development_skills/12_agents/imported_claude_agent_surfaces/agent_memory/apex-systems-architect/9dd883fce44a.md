# Phase 7 Security Hardening Review -- Detailed Findings

Date: 2026-02-24
Reviewer: Apex Systems Architect

## Summary: 3 HIGH, 10 MEDIUM, 5 LOW, 11 PASS

## HIGH Findings

### H-PROV-01: In-Memory Audit Chain
- File: platform/services/provenance/app.py, line 50
- records: list[ProvenanceRecord] = []
- Pod restart = total provenance data loss
- Fix: Persist to PostgreSQL or append-only SQLite

### H-K8S-01: hostNetwork: true
- File: platform/k8s/09_genesys-ai.yaml, line 16
- Bypasses CNI, exposes all host ports, network policies ineffective
- Fix: Remove hostNetwork, deploy Ollama as sidecar or k8s Service

### H-BROWSER-01: Unauthenticated Automation Bridge
- File: browser/genesys-browser/electron/main.ts, lines 226-300
- HTTP server on 127.0.0.1:8787 with no auth
- Any local process can navigate, click, type, read page content
- Fix: Add SERVICE_TOKEN Bearer auth check on all bridge routes

## MEDIUM Findings

### M-AI-01: Rate Limiter Drift
- api.py line 74: deque not pruned, under-counts at sustained load above 240 req/min

### M-AI-02: Model-Controlled Tool Dispatch
- agent.py lines 173-180: tool_name from model output, browser.navigate accepts arbitrary URLs
- Chain: prompt injection -> tool call -> browser manipulation

### M-ID-01: No Rate Limit on User Registration
- identity/app.py line 301: /users endpoint lacks rate limiting

### M-SYNC-01: Path Traversal
- sync/app.py: path parameter passed directly to MinIO without user-scoped prefix

### M-SYNC-02: No Tenant Isolation
- JWT claims captured but sub claim never used for authorization scoping

### M-PROV-01: No User Scoping on Reads
- /provenance returns all records to any authenticated user

### M-K8S-01: Missing capabilities.drop
- No capabilities drop or seccomp profile in securityContext

### M-K8S-02: Startup Probe Borderline
- 150s window may be tight under memory pressure during cold model loading

### M-BROWSER-01: executeJavaScript with Model Selectors
- CSS selectors from AI model passed to executeJavaScript

### M-BROWSER-02: automationBridge Exposed to Renderer
- preload.ts line 39 exposes full automation API to web page JavaScript

1. Prompt Injection -> Tool Dispatch -> Browser Manipulation

3. Any Authenticated User -> Full File Access
