# Boot Path Wiring Audit — 2026-03-07
## Files Audited
- `init/gensd/main.c` (Sprint 1 gensd entry point)
- `kernel/xenos/core/main.c` (kmain 14-step init sequence)
- `kernel/xenos/include/xenos.h` (kernel type/decl header)
- `init/gensd/include/gensd.h` — DOES NOT EXIST (gap #1)
- `init/gensd/supervisor.c`, `boot_chain.c`, `boot_sequence.c`
- `pal/src/pal_aether.c` (PAL-Aether v1.3 backend)
- All Sprint 8/9 driver headers: xacpi.h, xhpet.h, xlapic.h, xpci.h

## Key Findings Summary
- kmain 14-step sequence: xhpet_init, xlapic_init, xpci_init, xacpi_init ALL MISSING from kmain
- kmain calls old inline apic_init() (not xlapic_init from Sprint 9 xlapic.h)
- xchain_run() is NEVER called from gensd_start() or gensd_run_loop()
- gensd_run_boot_sequence() is NEVER called from gensd_start()
- boot_sequence.c stage_storage/security/network/display/shell/ai are ALL TODO stubs
- pal_pages_alloc/pal_map_pages issue syscalls XK_SYS_PHYS_ALLOC/XK_SYS_VIRT_MAP
  which return XENOS_ENOSYS in xenos_syscall_dispatch — DEAD PATH at runtime
- gensd_start() weak stub in kmain §9 is the one that actually runs (1-liner puts)
- supervisor_watch() called correctly for all 3 essential services
- No gensd.h shared header — supervisor.c duplicates gsd_service_t layout by hand
- Rescue console referenced in boot_chain.c §8 but init/gensd/rescue_console.c does not exist
