# GEN.OS From-Scratch Skills Catalog

Extracted 2026-03-31 from codebase deep-dive across display, storage, network, UI, and AI subsystems.

## Universal Patterns

### P1: Static Pool Allocation (No Malloc)
Every subsystem uses the same pattern: a fixed-size static array + spinlock-protected free list.
- Surfaces: 128 slots (surface.c:103)
- UI Nodes: 2048 slots (main.c:75)
- TCBs: 128 slots (tcp.c:91)
- Text Inputs: 128 slots (text_input.c:117)
- Editors: 8 slots (editor.c:178)
- Spreadsheets: 4 slots (spreadsheet.c:129)
- Slides: 4 slots (slides.c:117)
- CRDT Docs: 64 slots (crdt.c:110)
- Window Chrome: 32 slots (window_chrome.c:62)
- Gossip Members: 256 slots (gossip.c:123)

### P2: Freestanding Stubs
Every .c file redefines memset/memcpy/strlen inline. PAL_FREESTANDING guard at top.

### P3: Single-TU Inclusion (#include .c)
xshell.c includes orange_store.c, settings_init.c, settings_dispatch.c directly.
Used when static BSS must live in the same compilation unit as the consumer.

## Display Domain (10 skills)

### S01: Raw DRM/KMS Mode Setting
- File: devices/desktop/display/xdisp/drm.c
- Pattern: Zero-dependency DRM. All ioctl numbers hand-defined (stable kernel ABI). Opens /dev/dri/card0, enumerates connectors, parses EDID, sets preferred mode, creates dumb buffer, mmaps for scanout.
- Key types: drm_mode_modeinfo_t, drm_mode_card_res_t, drm_mode_get_connector_t, drm_mode_create_dumb_t
- Hot path: DRM_IOCTL_MODE_SETCRTC for page flip
- Target: 1920x1200@60Hz eDP on Intel UHD 620

### S02: Scene Graph Compositor
- File: devices/desktop/display/xcomp/compositor.c
- Pattern: Z-sorted doubly-linked list of surfaces. Double-buffered: render into back, submit back as new front, swap handles. Damage tracking avoids full-screen recomposite (~9MB/frame).
- Key types: xcomp_surface_t, g_surface_pool[128], g_list_head/g_list_tail
- Lock ordering: g_comp_lock before any surface_lock
- GOP fallback: g_use_gop_fallback when DRM unavailable (freestanding kernel)

### S03: Porter-Duff SRC_OVER Alpha Compositing
- File: devices/desktop/display/xcomp/blend.c
- Pattern: Integer-only per-pixel blending. BLEND_MUL255 macro: (a*b + (a*b>>8) + 1) >> 8 — exact for all inputs in [0,255*255]. Three fast paths: skip transparent (sa==0), direct copy opaque (sa==255), full blend otherwise.
- Hot path: Inner pixel loop at blend.c:411-446
- Performance: 60fps at 1920x1200 on UHD 620

### S04: Static Pool Surface Management
- File: devices/desktop/display/xcomp/surface.c
- Pattern: 128-slot static pool with free list through next pointer. Z-sorted insertion O(N). Damage bounding box union for dirty tracking.
- Key types: xcomp_surface_t (id, x, y, z, width, height, opacity, visible, damage, prev, next)

### S05: 2D Vector Graphics Engine
- File: devices/desktop/display/xcomp/vector.c
- Pattern: Pure-C replacement for Cairo/Skia. Path pool (16 slots, ~1.75MB static). De Casteljau curve flattening (tolerance 0.25px). Scanline fill with non-zero winding + 4x vertical supersampling. Stroke expansion with miter/bevel/round joins and butt/square/round caps.
- Key types: xvec_path (cmds[], coords[], n_cmds, n_coords), xvec_edge_t (x0,y0,x1,y1,wind)
- Performance target: 200x200 rounded rect in <0.5ms

### S06: OpenType Font Rasterizer
- File: devices/desktop/display/xcomp/fonts.c
- Pattern: Pure-C sfnt parser (no FreeType/HarfBuzz). Big-endian rd16/rd32 readers. cmap Format 4 segment lookup. Glyph outlines decoded as quadratic B-splines. Scanline rasterizer with 4x supersampling. LRU glyph cache (64x64 max bitmap per glyph). 4MB font data arena. Fallback glyph (border rectangle + center dot) for unmapped codepoints.
- Key types: struct xfont (table pointers into raw data), xf_cache_entry_t (LRU with lru_tick)
- UTF-8 decoder: utf8_decode() handles 1-4 byte sequences
- Text scaling: weak extern g_text_scale_factor from accessibility

## UI Toolkit Domain (14 skills)

### S07: XFRAME Node Tree & Pool
- File: devices/desktop/ui/xframe/runtime/main.c
- Pattern: 2048-node static pool. Free list threaded through next_sibling. Monotonic 1-based IDs. Node types: BOX, TEXT, IMAGE, CUSTOM. Default: flex row, white text 16px, transparent bg.
- Software framebuffer: g_sw_fb[1920*1200] in BSS (~9MB, zero binary impact)

### S08: 60fps Frame Pacing Main Loop
- File: devices/desktop/ui/xframe/runtime/main.c
- Pattern: XFRAME_FRAME_NS = 16,666,667ns. Loop: poll input -> dispatch events -> layout if dirty -> paint if dirty -> xcomp_surface_commit -> pal_thread_sleep_ns(remaining). XCOMP surface created per app.

### S09: CSS Flexbox-Subset Layout Engine
- File: devices/desktop/ui/xframe/runtime/layout.c
- Pattern: Two-pass O(N). Pass 1 MEASURE (post-order): intrinsic sizes from children + padding + gap. Pass 2 ARRANGE (pre-order): distribute free space via flex-grow/flex-shrink, position via justify-content/align-items. Supports direction, gap, padding, margin, min/max width/height. No flex-wrap (deferred).
- Performance: <2ms for 200-node tree

### S10: Widget Paint Engine
- File: devices/desktop/ui/xframe/runtime/paint.c
- Pattern: Pre-order painter's algorithm. Clip stack (32 deep). Per-node opacity via alpha pre-multiplication. Delegates to vector.h (rounded rect backgrounds), fonts.h (text), blend.h (images).
- Key type: xframe_clip_t (x,y,w,h). clip_intersect for nested overflow_hidden.

### S11: Event Dispatch with Hit Testing & Bubbling
- File: devices/desktop/ui/xframe/runtime/event.c
- Pattern: 1024-entry static callback table binding (node_id, event_type) -> (callback, userdata). Hit test: reverse child order traversal (last child = visual top). Bubbling: propagate up parent chain until callback returns non-zero. Focus tracking: single node. Tab-stop table: 64 entries with order field.
- Key types: xframe_cb_entry_t, xframe_tab_entry_t

### S12: Text Input Widget
- File: devices/desktop/ui/xframe/widgets/text_input.c
- Pattern: 128-slot pool. 512-byte UTF-8 buffer per input. Byte-indexed cursor/selection. Caret blink via tick accumulator. Password masking (XINPUT_FLAG_PASSWORD). Horizontal scroll for overflow. on_change/on_submit callbacks. Self-contained memmove for insert/delete.
- Key operations: on_char (insert), on_key (arrows/home/end/backspace/delete), on_pointer_down (click-to-position)

### S13: Rich Text Editor Widget
- File: devices/desktop/ui/xframe/widgets/editor.c
- Pattern: 8-slot pool. 4096 lines x 256 chars per editor = 8MB static arena. Cursor as (line, col). Multi-language syntax highlighting (C, TypeScript, Python, YAML) via xeditor_classify_line(). Token types 0-6 with per-type color mapping. Line gutter with numbers. Vertical scroll integration.
- Language detection: file extension + shebang parsing

### S14: Spreadsheet Cell Renderer
- File: devices/desktop/ui/xframe/widgets/spreadsheet.c
- Pattern: 4-slot pool. 1024 rows x 64 cols x 64 chars = 16MB static arena. Virtual rendering (only visible cells painted). Column/row headers always visible. Inline cell edit mode. Keyboard navigation (arrows, Tab, Enter, Escape).
- Key optimization: viewport intersection test before paint

### S15: Presentation Slide Canvas
- File: devices/desktop/ui/xframe/widgets/slides.c
- Pattern: 4-slot pool. 64 slides max. Thumbnail rail at bottom (72px). Slide transitions: SLIDE_LEFT/SLIDE_RIGHT with animation progress [0..255]. Presentation mode (fullscreen, no rail). Each slide references an XFRAME scene graph subtree via pal_handle_t.

### S16: Settings IPC Dispatch Table
- File: devices/desktop/shell/xshell/src/settings_dispatch.c
- Pattern: Maps (domain, operation) pairs to 35+ backend C functions. Domains: display, power, network, sound, keyboard, mouse, accessibility, security, privacy. All backends are extern declarations — dispatch.c is the central router.

### S17: Window Chrome / Decoration Layer
- File: devices/desktop/shell/xshell/src/window_chrome.c
- Pattern: 32-slot pool. 36px title bar with minimize/maximize/close buttons (Windows-style right-aligned). 6px invisible resize border on 8 edges/corners. Drag-to-move. Double-click maximize toggle. Edge snap (SNAP_ZONE_PX=24). Focus colors update title bar + border. Minimum size enforcement.

### S18: Desktop Shell Singleton
- File: devices/desktop/shell/xshell/src/xshell.c
- Pattern: Global g_shell singleton. Creates XFRAME app at GOP-detected resolution. Scene root is full-screen column container. Sub-components: topbar, dock, launcher, lockscreen. Per-frame tick callback drives notification auto-dismiss (5s timeout), topbar clock, dock auto-hide. Notification ring buffer with card-based overlay display (max 4 visible, color-coded by type).

### S19: Design Tokens System
- Referenced in: layout.c, paint.c, text_input.c, editor.c, spreadsheet.c, slides.c, window_chrome.c
- Pattern: XTOKEN_* constants define all colors, radii, spacing, durations. Single source of truth. Used via #include "design_tokens.h". Ensures consistent visual language across all widgets.

### S20: Accessibility Integration
- Referenced in: fonts.c (text scaling), text_input.c (a11y.h), settings_dispatch.c
- Pattern: Weak extern g_text_scale_factor. Sticky keys, filter keys, caret browse as separate .c modules. Text scale propagates through font rasterizer via xf_get_scale_factor().

## Storage Domain (5 skills)

### S21: B+ Tree KV Store
- File: store/xstore/btree/btree.c
- Pattern: Order-256 leaf, order-128 internal. 4096-byte pages (matches VMM). Variable-length records with offset table growing from page tail upward. Leaf chain via next_leaf pointer at data[0..3]. Split on overflow, push separator up. Free list through page header.
- Lookup: O(log_256 n) — 2-3 pages for 1M keys
- Key types: xstore_leaf_slot_t (key_len, val_len, rec_off, lsn_hi), xstore_int_slot_t (key_len, right_child)

### S22: Write-Ahead Log (WAL)
- File: store/xstore/wal/wal.c
- Pattern: Redo-only ring buffer. XSTORE_WAL_RING_SIZE entries (power-of-two for cheap modulo). Entry types: INSERT, DELETE, COMMIT, CHECKPOINT. CRC32C checksums. Auto-checkpoint at threshold. Crash recovery: replay from checkpoint_lsn, collect COMMIT txn_ids, re-apply their INSERT/DELETE entries. Partial txns discarded.
- Durability: durable after COMMIT entry + io.sync()

### S23: XSTORE Master API
- File: store/xstore/include/xstore.h
- Pattern: PAGE_SIZE=4096, MAX_KEY_LEN=256, MAX_VAL_LEN=4096. MVCC snapshot isolation (optimistic, 64-bit txn IDs). 1024-page LRU cache. 15 error codes. Superblock at page 0 with root_page, free_list_head, next_txn_id, checkpoint_lsn. All buffers caller-provided (no heap inside XSTORE). I/O callbacks for backend abstraction.
- Key types: xstore_page_hdr_t (32B packed), xstore_superblock_t (4096B), xstore_key_t, xstore_val_t, xstore_wal_entry_t

### S24: Content-Addressable Blob Store
- File: store/xblob/include/xblob.h
- Pattern: SHA-256 content addressing. Automatic deduplication (same hash = same blob, refcount++). 1M-slot open-addressing hash table with linear probing + tombstone deletion. 75% load factor threshold. GC via reference counting (saturating uint16_t). GC log ring (16384 entries) for crash-safe refcount recovery. 16MB max blob size. CRC32C integrity.
- Key types: xblob_hdr_t (64B), xblob_index_entry_t (56B), xblob_gc_log_entry_t (56B)

### S25: I/O Callback Abstraction
- Files: xstore.h, xblob.h
- Pattern: Both XSTORE and XBLOB abstract their backing store through function pointer structs (read, write, sync callbacks). Enables NVMe raw block I/O, RAM disk, or test buffers without code changes.

## Network Domain (6 skills)

### S26: Network Interface Abstraction
- File: net/xnet/core/netif.c
- Pattern: 8-slot static array. Registration assigns stable index. IPv4 addr/mask/gateway per interface. Driver tx callback function pointer. Stats tracking. Spinlock-protected. Duplicate name rejection.
- Key type: struct xnet_netif (name, mac[6], ipv4_addr, ipv4_mask, mtu, flags, stats, tx_fn)

### S27: TCP State Machine
- File: net/xnet/transport/tcp.c
- Pattern: Full RFC 793. 128 TCBs. States: CLOSED through TIME_WAIT. 16KB ring send/recv buffers. Retransmit queue (32 segments). RTO via Jacobson/Karels (RFC 6298). Congestion control: simplified CUBIC. Keepalive (2hr idle, 75s probe). Sequence number wraparound via signed cast: SEQ_LT(a,b) = (int32_t)(a-b) < 0. Out-of-order reorder buffer (8 segments). Telemetry counters (conn_established, retransmits, etc.).
- Key types: TCB struct (snd_una, snd_nxt, snd_wnd, rcv_nxt, rcv_wnd), xnet_ring_t, xnet_tcp_rexmit_t

### S28: SWIM Gossip Protocol
- File: net/xnet/mesh/gossip.c
- Pattern: SWIM membership protocol for 256-node mesh. Periodic ping -> ACK timeout -> PING_REQ (indirect) -> SUSPECT -> DEAD. Incarnation numbers for liveness override. Piggybacked state events (infection-style dissemination, ceil(log(N+1)) retransmits). Xorshift64 PRNG seeded from RDRAND/PAL. Fixed-size binary wire format (no serialization library).
- Key types: xnet_gossip_msg_hdr_t (packed), xnet_gossip_event_t (packed), xnet_gossip_member_t

### S29: Ethernet + ARP + IP + ICMP + UDP Stack
- Files: net/xnet/link/ethernet.c, arp.c, inet/ip4.c, icmp.c, transport/udp.c
- Pattern: Full L2-L4 stack from scratch. Ethernet frame parsing. ARP request/reply. IPv4 header construction/validation. ICMP echo. UDP datagram send/receive. All in freestanding C.

### S30: Network Byte Order Helpers
- File: net/xnet/mesh/gossip.c (and throughout xnet)
- Pattern: xnet_htonl/ntohl for 32-bit, xnet_htonll/ntohll for 64-bit. Symmetric implementation. Used consistently for all wire-format fields.

### S31: Ring Buffer for TCP
- File: net/xnet/transport/tcp.c:114-161
- Pattern: uint8_t data[16K] + head/tail/count. Modular indexing. ring_write clips to available. ring_read advances head. ring_peek reads without consuming. Used for both send and receive buffers.

## AI / Inference Domain (9 skills)

### S32: Q4_0 Weight Quantization
- File: ai/xmind/src/quantize.c
- Pattern: Block size 32. scale = max_abs / 7.0f. Quantize: q = clamp(round(w/scale), -8, 7), stored as nibble = q+8 in [0,15]. Pack two nibbles per byte. Dequantize: w = (nibble-8) * scale. Row-level batch quantize/dequantize for entire weight matrices.

### S33: BPE Tokenizer with FNV-1a Hash
- File: ai/xmind/src/tokenizer.c
- Pattern: Dual mode — byte-level fallback (259 tokens: PAD/BOS/EOS + 256 bytes) and full BPE (128K rules, 128K vocab tokens). FNV-1a hash (XM_FNV1A_PRIME=0x01000193, OFFSET=0x811C9DC5) for merge pair lookup. BPE merge rules loaded from XSTORE. Greedy merging: find highest-priority adjacent pair, merge, repeat.
- Key types: xmind_bpe_rule_t (left_id, right_id, merged_id, score), xmind_bpe_token_t (id, str[60])

### S34: Transformer Forward Pass (AVX2)
- File: ai/xmind/src/transformer.c
- Pattern: Per-layer: Q/K/V projection (Q4_0 matmul via tensor.c), RoPE application, KV cache store, multi-head attention (GQA: kv_head = h * n_kv_heads / n_heads), SwiGLU FFN, output projection. Attention uses xmind_dot() which dispatches to AVX2 8-wide VFMADD231PS.
- Key: Fast inverse sqrt via Quake 0x5F3759DF + one Newton-Raphson step for 1/sqrt(head_dim)

### S35: SIMD Runtime Dispatch
- File: ai/xmind/src/tensor.c
- Pattern: CPUID probe at init via xjit_avx2_available(). Function pointers resolved once: s_dot_f32, s_matvec, s_dot_q4. AVX2 path: xjit_dot_f32_avx2, xjit_matvec_f32_avx2, xjit_dot_q4_0_avx2. Scalar fallback if no AVX2.

### S36: No-Libm Math Primitives
- File: ai/xmind/src/tensor.c (expf), inference.c (cosf, sinf, lnf, powf)
- Pattern: expf via 2^(x*log2e) with IEEE 754 bit manipulation for integer exponent + degree-5 minimax polynomial for fractional part. cosf/sinf via degree-10/11 Taylor with angle reduction to [-pi,pi]. lnf via IEEE 754 exponent extraction + minimax polynomial. powf via e^(exp*ln(base)). sqrtf via Quake fast inverse sqrt + 3 Newton-Raphson iterations.

### S37: RoPE Precomputation
- File: ai/xmind/src/inference.c
- Pattern: theta_i = 1.0 / (base ^ (2i/head_dim)), base=500000 (Llama 3.2). cos/sin tables precomputed via Taylor series. Table size: max_seq * (head_dim/2). Angle reduction for large positions.

### S38: Inference Session & Generation Pipeline
- File: ai/xmind/src/inference.c
- Pattern: Session struct holds model back-pointer, sampler state, current position. Pipeline: tokenize prompt -> prefill (forward pass per token, build KV cache) -> autoregressive generation (forward + sample, one token at a time). Preflight check validates weights are loaded. Spinlock added Sprint 45 for thread safety.

### S39: KV Cache Management
- File: ai/xmind/src/transformer.c:51-66
- Pattern: Per-layer k[layer][pos * kv_stride + h * head_dim + d] and v[same]. KV stride = n_kv_heads * head_dim. Freshly computed K/V copied from transient state into persistent cache at each position.

### S40: Softmax with Numerical Stability
- File: ai/xmind/src/tensor.c (xmind_softmax), transformer.c:126-152
- Pattern: Find max, subtract max from all logits (prevents overflow), exponentiate via xm_expf, normalize by sum. Inline in attention hot loop to avoid function call overhead.

## Sync / Collaboration Domain (3 skills)

### S41: CRDT Library (5 types)
- File: devices/desktop/ui/xframe/sync/crdt.c
- Pattern: Pure-C replacement for Yjs/Automerge. 64-doc pool. Five CRDT types: LWW Register (timestamp + writer), G-Counter (per-node counts), PN-Counter (pos/neg per-node), LWW-Element-Set (add/remove timestamps), RGA (Replicated Growable Array for text). Wire format: 16-byte header [type:4][node_id:4][clock:8] + type-specific payload. Per-doc spinlock.

### S42: Offline-First Sync Engine
- File: devices/desktop/ui/xframe/sync/offline_first.c
- Pattern: Version vector per document (16 peers max). Operation log as ring buffer (4096 entries). Merge queue for incoming remote deltas (64 slots). State machine: IDLE -> SYNCING -> MERGING. Local operations applied immediately, synced via CRDT merge when online. Little-endian wire encoding with explicit encode/decode helpers.
- Key types: xsync_vclock_t (per-peer seq tracking), xsync_op_t (immutable oplog entry), xsync_doc_t (doc_id + crdt_type + vclock)

### S43: Version Vector Clock
- File: devices/desktop/ui/xframe/sync/offline_first.c:183-194
- Pattern: Array of (peer_id, seq) entries. vclock_get_peer() with create flag for lazy initialization. vclock_seq_for() returns highest seen seq from a peer. Used to determine which operations are new during sync.

## Cross-Cutting Skills (2 additional)

### S44: GOP Direct Blit Pixel Pipeline
- Files: compositor.c (g_use_gop_fallback, g_gop_softbuf), main.c (g_sw_fb)
- Pattern: XFRAME paint writes to g_sw_fb[1920*1200]. XCOMP compositor blends surfaces into back buffer. If DRM unavailable, compositor uses gop_fb_blit() to write directly to HHDM (Higher Half Direct Map) physical framebuffer. Software double-buffering via PAL-allocated physical page.

### S45: CRC32C Checksums Throughout
- Files: xstore.h (page checksum), wal.c (entry checksum), xblob.h (blob + index entry checksums)
- Pattern: CRC32C (Castagnoli) used for all on-disk integrity verification. Checksum field excluded from its own computation (set to 0, compute, write back). Covers page bytes [8, PAGE_SIZE) for XSTORE pages.
