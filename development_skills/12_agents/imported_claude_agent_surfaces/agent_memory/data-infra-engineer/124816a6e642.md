# Data Infrastructure Engineer — Agent Memory

## Foundation Architecture (Sprint 0)

### Storage Backend
- All Foundation telemetry uses SQLite in WAL mode at `/var/lib/genos/telemetry/foundation.db`
- WAL + NORMAL synchronous = durable against OS crash, fast for 100ms cadence writes
- Single writer connection (Bronze writer daemon); separate read-only connection for dashboard
- PostgreSQL provenance service at `platform/services/provenance/` — DO NOT modify schema
- Archive: nightly gzip of full DB to `/var/lib/genos/telemetry/archive/foundation_YYYY_MM_DD.db.gz`

### ETL Decision — Always ETL for Foundation C components
- `pal_handle_t` (64-bit opaque capability tokens) must NEVER be persisted — strip at FFI boundary
- Thermal readings arrive as `int32_t millidegrees` from `pal_thermal_read_cpu()` — convert to float in Python
- AIT CRC32 (`uint32_t`) must be hex-encoded strings before SQLite storage (sign-extension risk)
- Python Pydantic models own the transformation step, not SQL

### Bitemporal Pattern
- `record_time_ns` = Python daemon wall clock at event receipt
- `valid_time_ns` = `pal_time_now_ns()` embedded in C struct by source component
- All analytics windows use `valid_time_ns` for correct late-arrival handling

### Critical Data Contracts
- vDRAM: `pal_handle_t` NOT stored; `region_id` = slot index (1-4095); `migration_ordinal` = dedup key
- AIT: `AIT_MAGIC = 0x41495432` → hex `"41495432"` in Bronze/Silver; re-verify in Python on ingest
- AIT Strategy IDs: 0=DIRECT 1=CLUSTER 2=SPARSE 3=EXPONENT 4=TRANSPOSE 5=RAW
- XAS: stdout JSON lines from `xas.c`; `is_session_summary=1` for session rows

### Provenance Bridge Rules
- Events go via HTTP POST (NOT direct DB write); chain integrity via Python mutex
- CRITICAL action types: `vdram.integrity_check.fail`, `ait.magic_verification.fail`, `pipeline.schema_drift`
- Provenance emit failure is non-blocking — pipeline must NOT halt on provenance failures

### Idempotency Keys (Silver tier)
- `silver_vdram_tier_metrics`: PK (region_id, scanner_cycle_id)
- `silver_vdram_migrations`: UNIQUE (region_id, migration_ordinal)
- `silver_ait_compression_events`: UNIQUE (session_id, block_index, direction)
- `silver_xas_assembly_events`: UNIQUE (session_id, instruction_index)
- All Gold tables: PK on bucket_time_ns (60-second floor)

### Alert Thresholds
- `vdram.integrity_check.fail` → CRITICAL, halt ingestion
- `bronze_quarantine` count > 0 → WARNING, C ABI may have drifted
- `foundation.db` > 64 MB → trigger early compaction
- Silver watermark unchanged > 30s → ERROR, Bronze writer may be down

### PAL Phase Notes
- Phase 0 (current): PAL-Linux backend, wraps Linux syscalls
- Phase 2 (future): PAL-Aether replaces PAL-Linux; `backend_name` column tracks transition
- When PAL-Aether ships: update `tei_calculator.py` METRIC_REFERENCE_RANGES

## Sprint 3-5 XSTORE Architecture (2026-03-02)

### XSTORE Design Parameters
- B+ tree node = 4096B (NVMe 4K sector aligned); internal fan-out max=255; min=127
- Height for 1M records = 3 levels (proven: 127^2 * 255 * 178 = 730M records at height 3)
- Leaf split threshold: 70% fill (178 entries of max 255 per leaf)
- WAL record: [8B LSN][4B length][1B type][payload][4B CRC32]; checkpoint every 1000 tx or 10MB
- MVCC: 64-bit xid monotonic counter; xmin/xmax per row tuple; snapshot isolation at BEGIN
- WAL types: 0x01=BEGIN, 0x02=COMMIT, 0x03=ROLLBACK, 0x04=INSERT, 0x05=UPDATE, 0x06=DELETE, 0x07=CHECKPOINT
- XSTORE database: /var/genos/xstore/platform.xs; WAL: /var/genos/xstore/platform.wal

### XBLOB Design Parameters
- Root: /var/genos/blobs/{hash[0:2]}/{hash[2:4]}/{hash}
- Chunk size: 4MB (NVMe optimal IO alignment); SHA-256 content hash = object ID
- Multi-chunk: .chunk.NNNN files + .manifest JSON file for objects >4MB
- Metadata in XSTORE xblob_metadata table (Silver tier)
- Integrity check on every GET: re-hash and compare to content_hash key

### XAPI Design
- epoll C event loop; 4-thread Python FFI worker pool; trie-based URL dispatch O(1)
- JSON: recursive descent parser in C (~500 LOC); serializer: C struct to string
- Thread model: each worker thread has own XSTORE connection (per-connection lock, no global lock)

### XSTORE Table → Medallion Tier Map (Sprint 3)
- users, sessions, permissions, documents, sync_ops, provenance, xblob_metadata, gate_policies → Silver
- events (raw), WAL file, xblob files → Bronze
- events_gold_60s, gate_evaluations → Gold

### Migration Key Rules
- Provenance: extract ORDER BY record_time ASC — chain must be replayed in order; audit_token chain integrity verified before cutover
- Sync: MinIO objects (genos-drive, genos-artifacts) → XBLOB first; then register in documents table
- All services: Blue-Green cutover via k3s Service selector; PostgreSQL untouched until validation confirmed
- Timestamp migration: EXTRACT(EPOCH FROM ts) * 1e9 → INTEGER nanoseconds in XSTORE

### Python FFI Functions (xstore_open, xstore_exec, xstore_query, xstore_begin/commit/rollback, xstore_vacuum, xstore_errmsg)
- xstore_open(path) → handle (NOT thread-safe; call once per connection)
- xstore_query(handle, sql, bind_params) → list[dict]; result set copied to Python, C frees C memory
- xstore_query_stream() → generator; uses LIMIT/OFFSET for memory-safe large result sets
- XStorePool: queue-based pool (size=4); per-thread handle checkout; exponential backoff if empty
