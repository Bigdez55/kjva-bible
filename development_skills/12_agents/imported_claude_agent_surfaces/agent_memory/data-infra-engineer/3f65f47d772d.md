## Performance Optimization Patterns (2026-02-21)

### N+1 Query Elimination

Use `joinedload()` (small cardinality) or `selectinload()` (large cardinality).
Impact: 157→12 queries on /trading/batch-data (92% reduction).

### Cursor-Based Pagination

File: `backend/app/core/pagination.py`. Use `paginate_timestamp()`.
Offset pagination is O(n) at depth; cursor pagination is O(1) always.

### Redis Response Caching

File: `backend/app/core/cache.py`. Use `@cache_response(ttl=N, prefix="...")`.
TTL strategy: market quotes 30s, historical 300s, AI responses 300s, user profiles 60s.

### Database Indexing

Migration: `add_performance_indexes_2026_02`. Run BEFORE deploy.
Impact: 187ms → 3ms (98% faster).

## Type Standards (NON-NEGOTIABLE)

- Financial/price columns: `Numeric(18, 8)` — NEVER Float
- Confidence/rate columns: `Numeric(6, 4)`
- Percentage/return columns: `Numeric(10, 6)`
- Basis points: `Numeric(8, 4)`
- JSON in new migrations: `postgresql.JSONB` (not sa.JSON)
- UUID PKs: `postgresql.UUID(as_uuid=True)`
