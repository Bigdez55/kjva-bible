# MinIO → XBLOB Cloud Storage Migration — COMPLETE (2026-04-04)

MinIO (AGPL-3.0) has been fully removed from the GEN.OS cloud platform. All object storage now uses XBLOB (store/xblob/ — 100% original GEN.OS, LicenseRef-Proprietary).

**Why:** MinIO server is AGPL-3.0 which blocks commercial distribution of GEN.OS. The minio Python SDK (MIT) was also removed as it is no longer needed.

**How to apply:** Any future cloud service that needs object storage MUST use `XBlobCloudAdapter` from `devices/cloud/platform/services/common/xblob_cloud_adapter.py`. Never re-introduce the `minio` PyPI package.

## XBlobCloudAdapter API Surface

```python
adapter = XBlobCloudAdapter()
adapter.bucket_exists(namespace)                    # bool
adapter.make_bucket(namespace)                      # None (idempotent)
adapter.list_buckets()                              # list[str] — used for /healthz probe
adapter.put_object(ns, key, data, content_type)    # None (upsert)
adapter.get_object(ns, key)                         # bytes (raises KeyError if absent)
adapter.list_objects(ns, prefix, recursive, limit)  # list[ObjectInfo]
adapter.remove_object(ns, key)                      # None (idempotent)
```

## Key Behavioural Difference from MinIO SDK

- `get_object()` returns `bytes` directly, NOT a streaming response. Call sites that formerly
  called `obj.stream()` must use `io.BytesIO(data)` to produce a StreamingResponse.
- `list_objects()` returns `list[ObjectInfo]`, not a lazy generator. Test mocks must NOT use `iter()`.
- No credentials needed — XBLOB is filesystem-backed; access policy via `.policy.json` files.
