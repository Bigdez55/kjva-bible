## P1 Findings

### P1-DSB-001: dbus_state_bridge system-state.json non-atomic write
**File:** `compositor/genos-shell/system/dbus_state_bridge.py` line 157
```python
STATE_FILE.write_text(json.dumps(snapshot), encoding="utf-8")
```
Same bug fixed in thermal_guardian/thermal_predictor (commit 045f4e6) but missed in bridge.
Fix: replace with tmpfile+fsync+os.replace() pattern identical to thermal_guardian._write_state().

### P1-TLM-001: All telemetry computation is ephemeral — no Bronze sink
Files: `platform/services/telemetry/signal_baseline.py`, `signal_scorer.py`, `tei_calculator.py`, `causal_triples.py`, `variance_decomp.py`
All five modules return Python objects. No database write, no JSONL, no file sink.
No runtime caller exists in production code (embedded _test_*() self-tests only).
Fix: create `platform/services/telemetry/telemetry_store.py` with SQLite sink.
Proposed tables: signal_scores, tei_results, causal_triples (see audit report for schema).

### P1-SYNC-001: sync_entries/sync_conflicts PostgreSQL schema is dead DDL
`platform/services/sync/app.py` routes 100% of document operations to MinIO.
`platform/services/sync/schema.sql` is a well-designed Silver-tier schema (JSONB, optimistic lock, soft-delete, watermark index) that is applied by migrate.py but never written to by any application code.

## P2 Findings

### P2-PROV-001: provenance table missing indexes on audit_token, action_type, target
Needed SQL (add to provenance schema.sql or as migration):
```sql
CREATE INDEX IF NOT EXISTS idx_provenance_audit_token
    ON provenance (audit_token);
CREATE INDEX IF NOT EXISTS idx_provenance_action_type
    ON provenance (action_type, record_time DESC);
CREATE INDEX IF NOT EXISTS idx_provenance_target
    ON provenance (target, record_time DESC);
```

### P2-PROV-002: provenance absent from MIGRATION_SEQUENCE; no schema.sql
`platform/services/migrate.py` MIGRATION_SEQUENCE = [identity, sync, gate] — provenance missing.
Provenance app creates its table via Base.metadata.create_all() at lifespan startup.
This means schema drift between ORM and DDL is invisible to the migration runner.
Fix: create `platform/services/provenance/schema.sql`, add 4th entry to MIGRATION_SEQUENCE,
remove Base.metadata.create_all() from init_db() (rely on migrate.py).

### P2-AI-001: AI /healthz does not probe Ollama
`ai/genesys-ai/src/api.py` GET /healthz returns 200 if circuit breaker is closed.
Circuit only trips after failed calls. k8s liveness check will report healthy when Ollama is down and no calls have arrived yet.

### P2-CONN-001: No explicit connection pool sizing (carry-over from wave2)
identity/app.py and provenance/app.py: create_engine() with only pool_pre_ping=True.
SQLAlchemy default: pool_size=5, max_overflow=10 = 15 connections per process.
3 services x 2 replicas x 15 = 90 connections; leaves only 10 for PostgreSQL admin headroom.

### P2-PROV-003: fsync per fallback append — write amplification at high emit rates
`tools/provenance_client.py` _write_fallback() calls os.fsync() per record.
At 100 events/sec: 100 x 5ms NVMe latency = 500ms/sec I/O budget consumed by provenance alone.
Fix: batch-fsync (group-commit) on N records or T milliseconds with lock held.
