# Privacy & Data Pipeline Audit — 2026-03-02

## Overall Grade: PARTIAL COMPLIANT

The network boundary is correctly enforced and no external cloud services are called
at runtime. However, three substantive findings prevent a COMPLIANT rating.

---

## Audit Results by Component

### 1. platform/services/provenance/ — PARTIAL COMPLIANT

**What is stored:**
- `user_id` (string, max 255 chars — sanitized in agent.py before submission)
- `action_type` (e.g. "chat", "tool_call", "file.upload")
- `target` (resource identifier, max 1024 chars)
- `details` (free-text, max 4096 chars — WARNING: chat messages stored here, see finding F-1)
- `reversible` (bool)
- `action_time`, `record_time` (timestamps)
- `audit_token` (SHA-256 chain hash)

**Storage:** PostgreSQL inside k3s cluster — local device only.
**Access:** JWT-gated (Ed25519 EdDSA), chain-locked writes, paginated reads.

**FINDING F-1 (MEDIUM): User chat message content is persisted to provenance.**
- `agent.py:425` — `self._log_provenance(safe_user, "chat", "assistant", clean_message[:400], reversible=False)`
- `agent.py:459` — same pattern for streaming path
- The first 400 chars of every user chat message are written to the provenance `details` column.
- PII regex patterns ARE applied before storage (`_sanitize_context()`) — SSN, CC, email, phone redacted.
- However: arbitrary conversational content, questions, document topics, etc. are retained.
- Design vision says: "No info-stealer vulnerabilities." Retaining message content in an
  append-only audit log (no user delete endpoint) violates the spirit of this mandate.

**FINDING F-2 (HIGH): No user data deletion endpoint exists on the provenance service.**
- `platform/services/provenance/app.py` exposes: POST /log, GET /provenance, GET /provenance/temporal,
  GET /provenance/{action_id}, GET /healthz.
- There is NO DELETE, purge, or user-scoped erasure endpoint.
- Users cannot delete their own provenance records.
- Design vision mandate: "absolute opt-out toggles" — not achievable without a deletion path.

### 2. tools/provenance_client.py — COMPLIANT

- POSTs only to internal k8s service `http://provenance:8000/log`.
- SSRF protection: `_NoRedirectHandler` blocks all HTTP redirects.
- Fallback: local JSONL at `/var/lib/genos/provenance/audit_fallback.jsonl` (max 20 MB total).
- No external endpoints. No cloud service calls. All traffic stays on device.

### 3. platform/services/telemetry/ — COMPLIANT (with caveat)

- All five modules (signal_baseline, signal_scorer, tei_calculator, causal_triples, variance_decomp)
  are pure in-process computation libraries with no network I/O.
- Data sources are entirely local: `/proc/meminfo`, `/sys/class/thermal/...`, `/sys/block/nvme0n1/stat`,
  `labwc frame timing`, `ollama /api/generate timing`.
- NO telemetry is uploaded anywhere. All values are in-memory only.
- CAVEAT: as noted in prior audits, these modules have no Bronze SQLite sink — computed values
  are discarded on process exit (open P1 issue). This is a data LOSS problem, not a privacy problem.

### 4. tools/bridges/ — COMPLIANT

All three bridge daemons are fully local:
- `entropy_bridge.py`: reads `/sys/class/genos/entropy/` → writes `/run/genos/entropy-state.json`
- `mesh_bridge.py`: reads `/sys/class/genos/mesh/` → writes `/run/genos/mesh-state.json`
- `vdram_bridge.py`: reads `/sys/class/genos/vdram/` → writes `/run/genos/vdram-state.json`
- `dbus_state_bridge.py`: reads UPower/NetworkManager/PulseAudio D-Bus → writes `/run/genos/system-state.json`

Data collected: hardware counters (packet counts, memory tier pages, migration rates), WiFi SSID+strength,
battery level, audio volume. These are SYSTEM METRICS, not user content. No user activity data.

**NOTE on WiFi SSID:** `mesh_bridge.py` does not capture SSID — the D-Bus bridge captures SSID for
display in the top bar (network indicator). SSID is a semi-sensitive network identifier but is:
a) only written to a local volatile file at `/run/genos/system-state.json`
b) never sent anywhere
c) readable by any local process — same exposure as `iwconfig`

### 7. Network Boundary Enforcement — COMPLIANT

`platform/k8s/10_network-policy.yaml`:
- `default-deny-ingress`: blocks all ingress not from within `genos-system` namespace.
- `default-deny-egress`: blocks ALL egress by default.
- `allow-egress-dns-and-namespace`: only permits egress to: (a) other pods in `genos-system`,
  (b) kube-dns on port 53.
- There is NO egress rule permitting traffic to the public internet.
- This is a strong enforcement layer — even if application code attempted an outbound call,
  the network policy would drop it (assuming k3s NetworkPolicy enforcement is active).

## Remediation Recommendations

### R-1 (for F-1): Strip conversational content from provenance `details`

Replace the chat provenance log with a content-free event:
```python
# BEFORE (agent.py:425):
self._log_provenance(safe_user, "chat", "assistant", clean_message[:400], reversible=False)

# AFTER — store only metadata, never message content:
self._log_provenance(safe_user, "chat", "assistant", f"message_length={len(clean_message)}", reversible=False)
```

### R-2 (for F-2): Add DELETE /provenance/user/{user_id} endpoint

```python
@app.delete("/provenance/user/{user_id}")
def purge_user_provenance(
    user_id: str,
    _claims: Annotated[dict[str, str], Depends(decode_bearer)],
    db: Annotated[Session, Depends(get_db)],
) -> dict[str, int]:
    """Purge all provenance records for a user (right-to-erasure)."""
    result = db.execute(
        delete(ProvenanceRow).where(ProvenanceRow.user_id == user_id)
    )
    db.commit()
    return {"deleted": result.rowcount}
```
This endpoint must require the user to hold the same JWT claim as the `user_id` being deleted
(or an admin claim), to prevent one user from deleting another's records.

### R-3 (for F-3): Truncate/summarize tool payload before provenance logging

For `tool_call` events, log only `tool_name + arg_keys`, not full argument values:
```python
safe_payload_summary = {k: f"<{type(v).__name__}>" for k, v in payload.items()}
self._log_provenance(safe_user, "tool_call", tool_name, json.dumps(safe_payload_summary), reversible=True)
```
