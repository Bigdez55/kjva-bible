## Go Microservices Architecture (confirmed 2026-03-08)

### Service Layout
- `backend/go/market-gateway/` -- HTTP REST, chi router, :8080, stub provider only
- `backend/go/order-router/` -- HTTP REST, chi router, :8080, Alpaca REST v2 broker
- `backend/go/risk-engine/` -- HTTP REST, stdlib mux, :8080, 25 golden-vector metrics

### Entry Points (production vs legacy)
- Production: `cmd/server/main.go` in all 3 services
- Legacy skeletons: `cmd/gateway/`, `cmd/router/`, `cmd/risk/` (health-only, superseded)
- Dockerfiles build `cmd/server/` except risk-engine (needs update from `cmd/risk/`)

### Python Integration
- Feature flags: `USE_GO_MARKET_DATA`, `USE_GO_ORDER_ROUTING`, `USE_GO_RISK_ENGINE` (all default false)
- URL config: `MARKET_GATEWAY_URL`, `ORDER_ROUTER_URL`, `RISK_ENGINE_URL` (all default empty)
- Clients exist: `GoMarketGatewayClient`, `GoOrderRouterClient`, `GoRiskEngineClient`
- Risk client missing: `compute_kelly()` method for POST /kelly endpoint
