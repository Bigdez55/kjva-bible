# Provenance Graph Integrity Audit -- Detailed Findings (2026-02-25)

## Chain Hash Vulnerability: Pipe Delimiter Collision
- `build_audit_token()` at app.py:198 uses "|".join() with unescaped fields
- user_id="alice|admin" + action_type="login" == user_id="alice" + action_type="admin|login"
- Fix: length-prefixed encoding e.g. f"{len(field)}:{field}" per field

## Missing Bitemporal Columns
- Schema has action_time and record_time but NOT action_time_end / record_time_end
- Cannot close previous records on corrections (late-arriving data)
- Cannot answer "what did the graph know at time T"
- Migration: ALTER TABLE ADD COLUMN action_time_end TIMESTAMPTZ, record_time_end TIMESTAMPTZ
- Add composite index: (action_time, record_time, action_time_end, record_time_end)

## Chain Lock is Process-Local
- threading.Lock() only serializes within one Python process
- k8s replicas=1 currently prevents fork, but no cluster-wide guard
- Fix: pg_advisory_xact_lock(1) inside the critical section
- Or: startup assertion that aborts if deployment.spec.replicas > 1

## Dead Test Code
- _recompute_audit_token() at test_provenance_app.py:71 mirrors build_audit_token
- It is NEVER called by any test
- Should be used to verify chain N+1 = hash(chain_N + payload)

## Missing API Endpoints
- No GetEventGraphAsOf(action_time, record_time) -- required by Intelligence Lead contract
- No filtering by record_time range, action_time range, or user_id
- No pagination (cursor-based on record_time recommended)

## Causal Triples: In-Memory Only
- extract_from_dmesg() and extract_from_journal() return Python lists
- No PostgreSQL/SQLite persistence
- No graph data structure for traversal/depth queries
- timestamp is extraction-time, not kernel event-time

## NoiseFilter: Not Implemented
- Protocol 3 specifies: anomaly_signal = raw_signal - baseline * system_load_factor
- No NoiseFilter class exists in the codebase
- signal_baseline.py has static ranges but no load_factor adjustment
- Both raw and anomaly signals should be stored; only anomaly forwarded to Intelligence Lead

## Remediation Priority
1. P0: Add action_time_end + record_time_end columns (schema migration)
2. P0: Activate _recompute_audit_token in chain integrity tests
3. P1: Fix pipe delimiter collision (length-prefix encoding)
4. P1: Implement GetEventGraphAsOf endpoint with bitemporal filter
5. P1: Add pagination to GET /provenance
6. P1: Implement NoiseFilter class
7. P2: Add pg_advisory_xact_lock for cluster-wide chain serialization
8. P2: Build causal triple persistence layer
9. P2: Implement adaptive baseline learning (30-min rolling window)
