## Key Architecture Facts
- XSTORE: B+ tree, order 256, 4096-byte pages, CRC32C checksums
- XBLOB: content-addressed, SHA-256 keys, 16 MB max blob, 1M-slot index
- XBLOB dedup: put() increments refcount on duplicate, no new storage
- XBLOB uses pal_sha256() (same as XSEC/XPKG)
- WAL: redo-only, 65K ring, fixed-size entries
- MVCC: snapshot isolation (incomplete), OCC conflict check at commit
- vDRAM FROZEN: stub, returns PAL_ERR_NOT_SUPPORTED, data lost
- PAL_OBJ_TIMER fix: CONFIRMED FIXED in Sprint 3 P2

## Key Namespace Proposal
- xpkg/pkg/{name}/{version} -- package metadata
- xmind/tok/{id} -- tokenizer data
- prov/event/{action_time_ns} -- provenance events
- gensd/svc/{name} -- service state
- orch/workflow/{id} -- orchestration state

## XBLOB Refcount Correctness
- Single-lock design: correct under current spinlock
- Header ref_count (uint16_t) goes stale after ref/delete
- Shared blob scenario: put+dedup+delete chain works correctly
- Missing: leak detection (no way to find orphaned blobs)
