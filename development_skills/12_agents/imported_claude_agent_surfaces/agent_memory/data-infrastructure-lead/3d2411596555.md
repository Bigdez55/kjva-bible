## Capacity Constants (from headers + .c files)
- TCP_TCB_MAX: 128 (tcp.c:87) -- P0, will bottleneck browser at 15+ tabs
- TCP_RCVBUF_SIZE: 16KB, TCP_SNDBUF_SIZE: 16KB (tcp.c:68-69) -- ~33KB/TCB
- SOCKET_MAX: 256 (socket.c:47)
- UDP_SOCKET_MAX: 64 (udp.c:91)
- ARP_CACHE_SIZE: 256 (xnet.h:403) -- OK for workstation
- DNS_CACHE_SIZE: 128 (xnet.h:428) -- P1, too small for browser
- GOSSIP_MAX_NODES: 256 (xnet.h:442)
- XPKG_MAX_INSTALLED: 512 (xpkg.h:58)
- xpkg_package_t: ~79KB per package (files[256] array dominates)
- xpkg_registry_t: ~39.5MB static BSS
- XSEC_AUDIT_RING_SIZE: 1024 (xsec.h:496) -- ring overwrites = provenance violation
- xsec_tls_ctx_t: ~17KB per session (record_buf = 16640 bytes)

## P0 Findings
- TCP 128 TCBs insufficient for browser (need 256+)
- XSEC audit ring silently destroys old entries (no append-only)
- No signed repository index (rollback/MITM attack vector)
- No installation provenance events (package install/remove not recorded)

## P1 Findings
- DNS cache 128 too small (need 512+)
- No timestamps on xpkg_package_t (no installed_at_ns)
- xpkg_index_entry_t.dep_names is comma-separated string (loses version constraints)
- No TIME_WAIT TCB recycling

## Sprint 4 XSTORE Integration Plan
1. XSTORE B+ tree engine (C, PAL-only, libvdram page cache)
2. XPKG registry/keys/manifests -> XSTORE
3. XSEC audit -> append-only XSTORE (no ring overwrite)
4. Bitemporal value headers (action_time + record_time) in XSTORE
5. Python FFI via ctypes for provenance queries
6. Local XSTORE + platform PostgreSQL dual-write

## Provenance Gaps (PG-1 through PG-6)
- PG-1 (P0): Unsigned repo index
- PG-2 (P0): No install Action Nodes
- PG-3 (P1): No timestamps on registry
- PG-4 (P1): No CT-style publisher key log
- PG-5 (P2): No state transition events
- PG-6 (P2): No rollback/removal provenance
