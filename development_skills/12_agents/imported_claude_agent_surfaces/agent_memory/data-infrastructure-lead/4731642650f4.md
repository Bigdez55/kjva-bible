## Findings

### P0-CAP-1: capability_init.c rights mismatch
- aetherboot/src/capability_init.c lines 43-51 define local XKABI_RIGHT_* values
- GPU_EXEC=0x1 vs canonical 1u<<19; STORAGE_READ=0x4 vs 1u<<15
- ALL boot-time cap table entries have WRONG rights bitmasks
- Fix: include kernel/xenos/include/xkabi_rights.h (adjust build include path)

### P0-JOURNAL-1: ring buffer destroys provenance
- 4MB ring, ~16K entries before overwrite
- No export, no WAL, no persistent store
- journal_read_recent() is Sprint 3 STUB

### P1-TOPOSORT-1: cycle fallback starts services in arbitrary order
- service.c line 568-576: appends cycle members in array index order
- Should refuse to start or explicitly break cycle

### P1-PARSE-1: After overwrites Requires
- service.c line 409: both keys call parse_requires()
- parse_requires() resets n_requires=0 at line 296
- Second call erases first call's results

### P1-CAP-2: XISC rights header divergence
- xisc/include/xkabi_rights.h: READ=1<<0, WRITE=1<<1, EXECUTE=1<<2...
- kernel/xenos/include/xkabi_rights.h: GPU_MAP=1<<1, MEM_MAP=1<<3...
- Same include guard name: XKABI_RIGHTS_H

### P1-RKEY-2: undefined crypto_kdf256_expand
- mr_registry.c line 228: unconditional call to undefined symbol
- Will panic if vdram_rkey_rotate() ever invoked

### P1-SPIN-1: sleeping under spinlock in fault handler
- pf_handler.c lines 290-327: page->lock held during attestation+IOMMU check
- T3 slow path would sleep (documented, accepted for validation vehicle only)

### P1-PROV-1: no provenance in vdram page metadata
- vdram_page_meta: has last_access_ns, access_count
- Missing: action_time, record_time, migration history, event ID

### P2-TOPOSORT-2: gsd_dep_order() is dead code
- main.c line 364: "used in Sprint 3"
- Hardcoded start_order[] at line 422 must be manually synced

### P2-RKEY-1: rkey unreachability window
- mr_registry.c: synchronize_rcu() between remove and install
- All lookups return NULL during this window

### ADV-1: missing dependency names silently ignored (no WARN log)
### ADV-2: 9th+ requires silently dropped (no WARN log)
### ADV-3: 7 canonical rights not in GSD parser CAP_NAMES[] table
### ADV-4: 6/12 migration paths unsupported, no multi-hop
