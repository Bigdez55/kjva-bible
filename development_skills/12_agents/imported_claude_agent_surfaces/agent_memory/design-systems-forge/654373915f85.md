Sprint 34 token sync audit (2026-03-20). C source of truth: design_tokens.h.
TS target: design-system/src/tokens.ts (fully rewritten).

**Why:** Sprint 34 audit plan Section 9 required C/TS token parity. Prior tokens.ts used
a product palette (orange primary, different neutrals) that was never aligned with the native
XTOKEN_* macros. All consuming theme files and the dark theme were also updated.

**How to apply:** Any future token additions must be written to design_tokens.h first, then
mirrored to tokens.ts using the ARGB→CSS-hex conversion (strip 0xFF alpha prefix).

### Z-order — entirely missing from tokens.ts (added as zOrder export)

All 9 levels added: wallpaper(-100), desktop(0), window(100), panel(200),
modal(500), dropdown(600), toast(800), tooltip(900), cursor(32767).

### High-contrast danger fails WCAG AAA 7:1

Pure #FF0000 on #000000 = 5.25:1 (only AA). Fixed:
- danger: "#FF6060" — L=0.3046 → CR=7.09:1 on black (AAA)
- dangerText: "#000000" — CR=7.09:1 on #FF6060 (AAA)
- dangerHover: "#FF7777" — L=0.3583 → CR=8.17:1 on black (AAA)

### Color filter matrices (color-filters.ts) — VERIFIED CORRECT

Machado et al. 2009 Table A1 severity-1.0 matrices confirmed correct for
protanopia, deuteranopia, tritanopia. Severity-0.5 matrices for anomalous
trichromacy types also verified. No changes needed.
