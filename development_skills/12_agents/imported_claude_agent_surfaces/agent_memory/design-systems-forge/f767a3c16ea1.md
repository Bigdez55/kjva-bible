## Preload API Exposure Analysis

### go-notes preload (contextBridge "goNotes")
- exposeInMainWorld: aiAction(action, text), sync(docId, payload), load(docId)
- Input sanitization: action allowlist check, stripHtml(), sanitizePii() (SSN/CC/email/phone redaction), length truncation per action type (1500-3000 chars). GOOD.
- Token handling: GENOS_SESSION_TOKEN and GENOS_SERVICE_TOKEN read in preload process only, never forwarded to renderer. GOOD.

### go-docs preload (contextBridge "goDocs")
- Same pattern as go-notes. aiAction, sync, load. Same sanitization pipeline. GOOD.

### go-drive preload (contextBridge "goDrive")
- Exposes: list, upload, download, deletePath, share
- Path sanitization: sanitizePath() rejects "..", leading "/", non-alphanumeric chars. validateUid() regex.
- scope allowlist: ALLOWED_SCOPES = {"file:read", "file:write"} — never taken from renderer as free text. GOOD.

### go-calendar preload (contextBridge "goCalendar")
- Exposes: suggestSchedule, sync, load
- Calendar sync: explicit field extraction (no spread), PII sanitization on notes field. GOOD.

### go-mail preload (contextBridge "goMail")
- Exposes: send, fetch, getMessage, delete, setRead, saveDraft, sync, aiAction
- NOTE: send() does NOT sanitize the `to` field (intentional — redacting email addresses would break delivery). Body is sanitized.
- AI tool mapping: explicit TOOL_MAP, no free-text tool names forwarded. GOOD.
- message.to intentionally exempt from sanitizePii(). Well-documented in code.

### go-sheets preload (contextBridge "goSheets")
- Exposes: sync, export
- export() is a stub returning format string only. GOOD.

### go-slides preload (contextBridge "goSlides")
- Same pattern as go-sheets. GOOD.

### go-files preload (contextBridge "goFiles")
- Exposes: 11 filesystem operations via ipcRenderer.invoke
- Renderer-side validators: assertStr() (length 0-4096), assertContent() (<=1 MiB)
- Main-side validators (assertAbsolutePath) also enforce allowed roots. Double validation. GOOD.
- sandbox: false — necessary for ipcRenderer in preload. Documented.

### genesys-browser preload (contextBridge "genesysBrowser" + "automationBridge")
- Exposes tab management + full automation bridge (navigate, click, type, scroll, fillForm, extractData, screenshot, getPageContent)
- automationBridge passes through raw selectors and text to ipcRenderer without preload-side validation. Validation happens in main process (SAFE_SELECTOR_RE). Acceptable.
- onTabUpdated/onWindowListUpdated: ipcRenderer.on listeners passed to renderer as callbacks. No filtering of event payloads.

### installer preload (contextBridge "genosInstaller")
- Exposes: listDisks, install, status
- ALL calls are fetch() to http://localhost:9980 — local installer daemon only. No external network.
- disk parameter in install() passed as-is to fetch body. No sanitization. Risk: disk string could contain injection if installer daemon doesn't validate. LOW risk (localhost only, hardened daemon).

## Issues Requiring Action

P1:
- [P1-AUDIT-01] go-mail dangerouslySetInnerHTML uses a custom sanitizer (sanitizeHtml()) rather than DOMPurify or a maintained library. Custom sanitizers are historically incomplete. The current implementation strips script/iframe/object/embed/form elements and on* attributes and javascript: URIs — but does NOT strip CSS-based attacks (expression(), background: url(javascript:...)), data: URIs, or SVG-based XSS vectors. Replace with DOMPurify.

P2:
- [P2-AUDIT-01] go-files sandbox:false — functional necessity documented; acceptable with double-validation path/content guards. Monitor for Electron updates where preload-with-sandbox becomes viable.
- [P2-AUDIT-02] genesys-browser/package.json: no package-lock.json generated. build/installer/ui: same. Lock files must be committed.
- [P2-AUDIT-03] ALL 11 package.json files missing "license" field. Add SPDX identifier or proprietary notice.
- [P2-AUDIT-04] orange/common/security.test.ts imports vitest but no package.json in orange/common/ declares it as a devDep. Test runner relies on npx vitest without pinned version.
- [P2-AUDIT-05] orange/backend/src/app.ts imports express but orange/backend/ has no package.json (only go-calendar/backend/ does). If this TypeScript file is ever compiled standalone it will fail. Either remove or add a package.json to orange/backend/.
- [P2-AUDIT-06] installer/ui preload: disk string in install() not sanitized before forwarding to localhost:9980. Low-risk (local daemon) but should add allowlist check (e.g., /dev/sd[a-z] or /dev/nvme[0-9]n[0-9]).

P3:
- [P3-AUDIT-01] @tiptap/* and yjs are non-original runtime dependencies in go-notes and the shared doc-model. These are well-maintained MIT-licensed libraries but represent third-party code in the renderer and shared preload layer. Document explicitly in supply-chain records.
- [P3-AUDIT-02] installer/ui/electron/main.ts has no CSP header, no navigation block, no window.open block. Even though it only loads local content, defense-in-depth requires the same hardening as other apps.
- [P3-AUDIT-03] go-notes filename injection vector (P2 fix from 2026-02-24 already applied — characters stripped, length capped). Verified fixed.
