## CT Branchless Mask Pattern (canonical — never "fix" this)

`mask = (uint8_t)(0u - bit)` is two's-complement negation of {0,1}:
- bit==0 → 0x00, bit==1 → 0xFF. NOT a bug. Seen in aes_gcm.c, aes_sbox_ct.

## Two xkabi_rights.h Files (Critical — always check which one)

- `kernel/xenos/include/xkabi_rights.h` — CORRECT for boot + kernel (XKABI_RIGHT_* hardware bits 12-22)
- `xisc/include/xkabi_rights.h` — DIFFERENT (generic XK_RIGHT_READ/WRITE/EXECUTE object rights)
- Wrong include causes loud compile errors, not silent wrong values.
