## ARM64 CI Parity Audit (2026-04-06)

### Fixed: boot-matrix-md.sh run_arm64() serial capture bug
- Root cause: `-nographic -serial stdio` muxes QEMU monitor onto stdout, capturing "QEMU 8.2.2 monitor" (55 bytes) instead of kernel PL011 UART output
- Fix: `-monitor none -serial file:$serial` (matches x86_64 pattern)
- Also added: `-machine virt,gic-version=3` (kernel expects GICv3 at 0x08000000)

### Fixed: kernel-link.yml ARM64 link job

- ARM64 job: 2 ASM + 5 arch C + 2 driver C (gicv3.c, arm_timer.c) -> xenos-arm64.elf
