## Post-Sprint-13 Final E2E Audit (2026-03-07)

Full report: `post-sprint13-final-e2e-audit.md`

### Verdict: CONDITIONAL GO
- Architecture is sound; P0 SSE/AVX blocker must be resolved
- 2-3 sprints to production inference

### FLOP Count (Exact, Llama 3.2 3B Q4_0)
- Per token: **2.87 GFLOP** (matmul-dominant)
- Attention matmuls: 23.6M/layer x 28 = 660M
- FFN matmuls: 75.5M/layer x 28 = 2.11B
- Logit projection: 98.3M (fp32 dot, not Q4)
- Non-matmul ops: ~2M (negligible)

### Performance (i5-8250U)
- Current x87 scalar: TTFT ~4.6s, TPS ~2.0 -- BROKEN
- After AVX2 + 4-thread: TTFT ~0.2s, TPS ~14 -- TARGET MET
- Memory BW floor: 82ms/token (2.05 GB weights / 25 GB/s)
- Memory BW is the bottleneck AFTER AVX2

### Memory Budget (Validated)
- Weights: 2.05 GB (Q4 layers + fp32 embeddings 393 MB)
- KV cache (full 2048 ctx): 336 MB
- State buffers: 433 KB
- Total: **2.42 GB** (fits 3 GB budget, 16 GB system has 11.6 GB headroom)

### P0 Findings
1. `-mno-sse -mno-sse2` on XMIND (all sprint CIs) -- 23x perf loss
2. Zero weights (Sprint 8) -- no inference output ever validated

### P1 Findings
1. No concurrency protection on singleton model
2. 64 KB stack scratch in xm_ffn (gate[] + up[]) -- overflow risk
3. No streaming/callback API for token generation
4. No thermal-aware throttling
5. Weight pointer NULL not checked before forward()

### P2 Findings
1. KV cache layout suboptimal (position-stride = 3072 bytes)
2. Logit projection in fp32 (not quantized)
3. Attention softmax duplicated inline (not sharing tensor.c)
4. BPE rule lookup O(n_rules) linear scan
5. No context compression/KV eviction

### XJIT SIMD Status: NOT READY
- codegen.c: 22 integer opcodes, GP registers only
- No VEX prefix, no XMM/YMM, no float instructions
- Use compiler intrinsics for SIMD, not XJIT

### iGPU Compute: NOT WORTH PURSUING
- UHD 620 has 24 EUs / 460 GFLOPS but shares DDR4 bandwidth
- No command streamer driver, no GEM, no shader compiler
- ~5,000-10,000 LOC for Gen9 compute driver
- Shared memory = no bandwidth gain over CPU AVX2
- GPU compute reserved for XCOMP display compositing

### Inference Architecture Gaps (updated Sprint 15)
- Singleton model (s_xmind_model) -- no multi-model support
- xmind_generate() NOW EXISTS (Sprint 15) -- synchronous but correct
- No request queue, no mutex, no thread safety
- No streaming token callback (Sprint 16 target)
- Preflight check only validates layer 0 weights (P2)
