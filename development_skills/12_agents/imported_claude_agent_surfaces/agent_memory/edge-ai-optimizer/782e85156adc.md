### P0 BLOCKERS FOR ACTUAL INFERENCE (weights + tokenizer)
1. **Weight data not loaded** -- GGUF loader allocates zero-filled buffers, does not parse tensor data offsets or read actual weight bytes. Forward pass runs but produces zero/garbage logits.
2. **RoPE tables not populated** -- FIXED by xmind_rope_precompute()
3. **Tokenizer vocab mismatch** -- byte-level tokenizer produces IDs 3-258, model expects BPE vocab of 32000 tokens. Inference will index wrong embedding rows.

### Weight Size Calculations (Llama 3.2 3B, Q4_0)
- Per layer Q4 weights: 7 matrices, each ~hidden*hidden/32*20 bytes
  - wq: 3072 * (3072/32) * 20 = 5.625 MB
  - wk: 768 * (3072/32) * 20 = 1.406 MB (n_kv_heads * head_dim = 768)
  - wv: same as wk = 1.406 MB
  - wo: 3072 * (3072/32) * 20 = 5.625 MB
  - w1: 8192 * (3072/32) * 20 = 15 MB
  - w2: 3072 * (8192/32) * 20 = 15 MB
  - w3: same as w1 = 15 MB
  - Per layer total: ~59.06 MB
  - 28 layers: ~1.614 GB
- token_emb: 32000 * 3072 * 4 = 375 MB (fp32)
- RMSNorm: 28*2*3072*4 + 3072*4 = ~685 KB
- RoPE tables: 2048 * 48 * 4 * 2 = ~768 KB
- TOTAL: ~2.05 GB (matches prior FLOP count analysis)

### Bug: weights_loader.c w_size calculation is WRONG
Line 555: `w_size = hidden_dim * (hidden_dim / Q4_BLOCK) * sizeof(q4_block_t)`
This uses hidden_dim for both rows AND cols for ALL matrices.
But wk/wv have rows = n_kv_heads*head_dim = 768, not 3072.
And w1/w3 have rows = ffn_dim = 8192, not 3072.
And w2 has cols = ffn_dim = 8192, not hidden_dim = 3072.
This allocates wrong-sized buffers -- CRITICAL bug in weights_loader.c.
