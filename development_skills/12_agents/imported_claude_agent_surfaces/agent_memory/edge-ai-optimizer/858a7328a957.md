## Sprint 41 XMIND AVX2 Optimization (2026-03-21)

### Changes Made

**1. SIMD Dispatch Infrastructure (tensor.c)**
- Runtime CPUID detection via `xmind_simd_init()` called automatically during RoPE precompute
- Function pointers: `s_dot_f32` (fp32 dot), `s_dot_q4` (Q4_0 dequant+dot), `s_matvec` (reserved)
- AVX2 path: `xjit_dot_f32_avx2`, `xjit_dot_q4_0_avx2` from `jit/xjit/src/avx2_dot.c`
- Scalar fallback: `xjit_dot_f32_scalar` + inline scalar loop for Q4_0
- No CPUID in hot path -- resolved once at init

**2. Q4_0 Matmul AVX2 Acceleration (tensor.c)**
- `xmind_matmul_q4()` now calls `xjit_dot_q4_0_avx2()` per block when AVX2 available
- Fused dequantize + FMA in YMM registers -- no intermediate buffer materialization
- Affects all 7 per-layer weight matmuls: wq, wk, wv, wo, w1, w2, w3

**3. Attention Dot Product Acceleration (transformer.c)**
- `xm_attention_head()` now calls `xmind_dot()` for Q*K dot product per position
- `xmind_dot()` dispatches to AVX2 automatically

**4. Logit Projection Acceleration (transformer.c)**
- `xmind_forward()` logit loop now uses `xmind_dot()` instead of inline scalar
- 32000 x 3072 = 98M MADs per token -- ~8x speedup with AVX2

**5. RMSNorm Acceleration (tensor.c)**
- Sum-of-squares in `xmind_rmsnorm()` uses `s_dot_f32(x, x, n)` when available

**6. Q4_K_M Support (quantize.c + xmind.h)**
- `xmind_q4km_block_t`: 144-byte super-block (256 weights)
- `xmind_dequantize_q4km_block()`: full scale/min extraction with 6-bit packing
- `xmind_matmul_q4km()`: matrix-vector multiply for Q4_K_M weights
- `xm_fp16_to_f32()`: IEEE fp16 to fp32 conversion (software, no FPU)

**7. xjit.h Declaration Fix**
- Added `uint8_t xjit_avx2_available(void)` declaration to xjit.h

**8. Zero-Copy Physical Weight Mapping (weights_loader.c)**
- `xmind_weights_map_phys()` API stub for bootloader-loaded GGUF
- Requires future `pal_map_phys_range()` PAL extension
- Architecture ready -- PAL integration deferred to hardware sprint

### Performance Impact (Predicted, i5-8250U)

| Metric | Scalar (Before) | AVX2 (After) | Improvement |
|--------|-----------------|--------------|-------------|
| Q4_0 matmul per block | 32 scalar FMUL+FADD | 4 VFMADD231PS | ~8x |
| Logit projection (98M MADs) | ~34ms | ~4ms | ~8x |
| Attention dot (96 dim) | ~48ns | ~6ns | ~8x |
| RMSNorm sum-of-squares | ~12ns/3072 | ~1.5ns/3072 | ~8x |
| TTFT (predicted) | ~4.6s | ~0.6s | ~7.7x |
| TPS (predicted) | ~2.0 | ~14 | ~7x |

### Files Modified
- `ai/xmind/src/tensor.c` -- SIMD dispatch, accelerated matmul/dot/rmsnorm
- `ai/xmind/src/transformer.c` -- AVX2 attention dot + logit projection
- `ai/xmind/src/quantize.c` -- Q4_K_M dequant + matmul
- `ai/xmind/src/inference.c` -- Auto SIMD init hook
- `ai/xmind/include/xmind.h` -- Q4_K_M types + xmind_simd_init decl
- `ai/xmind/src/weights_loader.c` -- Zero-copy phys mapping stub
- `jit/xjit/include/xjit.h` -- xjit_avx2_available() declaration

**Why:** On-device inference at ~2 TPS is BROKEN by perception standards. AVX2 gets us to ~14 TPS (RESPONSIVE). The i5-8250U has 256-bit AVX2 execution units that were completely unused.

**How to apply:** These optimizations are compile-time safe (scalar fallback always present). The AVX2 path activates automatically when CPUID detects support. No configuration changes needed.
