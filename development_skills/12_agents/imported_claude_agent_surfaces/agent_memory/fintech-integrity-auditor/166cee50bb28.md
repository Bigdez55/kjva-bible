# 2026-02-12 Connectivity/Contract Audit

## WebSocket Streaming
- Frontend `frontend/src/services/websocketService.ts` sends `type: 'subscribe'` with a single `channel` string (`market_data:AAPL`), but backend `backend/app/routes/websocket/market_feed.py` only checks for `action`/`symbols`. Subscription requests always fall into the `Unknown action` branch, so no streaming data ever flows to the browser.
- Backend streaming messages are shaped as `{ type: 'quote'|'level2'|'trade' }` and trading updates emit `type: 'order_status'`, while the frontend only listens for `{type: 'market_data'|'order_update'|'position_update'}`. Even if the connection opens, no quote/order events reach the UI.

## API Contracts
- `frontend/src/services/api.ts` calls `POST /market/assets` with a JSON body (`symbol`, `name`, `asset_type`, etc.), but `backend/app/api/api_v1/endpoints/market_data.py` defines `create_asset(symbol: str, …)` without a body, so FastAPI expects query parameters. Any frontend attempt receives a 422 immediately.

These mismatches explain why live market data and asset creation paths stall despite both client and server being present.
