## Issues Found

### CRITICAL: Login Rate Limit Parameter Order Bug

**File**: `backend/app/api/api_v1/endpoints/auth.py`
**Lines**: 126, 178

The function `check_login_rate_limit(email, client_ip)` is called with parameters reversed:
- Called as: `check_login_rate_limit(client_ip, user_data.email)`
- Should be: `check_login_rate_limit(user_data.email, client_ip)`

This breaks rate limiting entirely - attackers can brute-force passwords.

### HIGH: TradeResponse Schema Issues

**File**: `backend/app/schemas/trading.py`
**Lines**: 124-136

1. Missing `from_attributes = True` config - will fail on `.model_validate()`
2. `id: int` should be `id: str` - Trade model uses UUID string

### HIGH: Incomplete Pydantic v2 Migration

Files still using deprecated `orm_mode = True`:
- `backend/app/schemas/micro_invest.py`: lines 135, 148, 159, 201
- `backend/app/schemas/trade_advanced.py`: lines 196, 250
- `backend/app/schemas/subscription.py`: lines 108, 120, 143

### MEDIUM: Encryption Salt Weak Default

**File**: `backend/app/core/encryption.py`
**Line**: 186-188

Default salt "ElsonWealthSecureSalt" used if ENCRYPTION_SALT not set.
Should fail in production.
