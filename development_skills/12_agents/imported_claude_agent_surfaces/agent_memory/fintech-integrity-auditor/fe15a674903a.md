## Key Model Facts (Verified)

- **Trade.id** ‚Äî UUID string (`String(36)`), NOT integer
- **TradeResponse.id** ‚Äî `str` (UUID)
- **OrderCancelRequest.trade_id** ‚Äî `int` (schema mismatch with Trade.id ‚Äî known bug)
- **TradeStatus** enum values: `PENDING`, `FILLED`, `PARTIALLY_FILLED`, `CANCELLED`, `REJECTED`, etc. ‚Äî NOT `PARTIAL`
- **Holding** ‚Äî SQLAlchemy ORM model for portfolio positions (fields: `portfolio_id`, `symbol`, `asset_type`, `quantity`, `average_cost`, `current_price`, `market_value`)
- **Position** ‚Äî Lightweight non-ORM class for trading engine only; takes `(symbol, quantity, cost_basis)` ‚Äî NO `portfolio_id`, NO `average_price`
- **AutoTradingStatusResponse** ‚Äî uses `is_active` (bool), NOT `is_running`

## Linked Accounts Schema (Verified)

`POST /api/v1/linked-accounts` expects:
- `institution_name: str`

## Auto-Trading Service ‚Äî Broker Preflight

`AutoTradingService.start_auto_trading()` calls `AlpacaBroker.validate_credentials()` before starting the loop. In tests (no real Alpaca credentials), this returns `(False, "unauthorized")` and the service returns `False`, causing the endpoint to raise HTTP 500.

**Test fix:** Mock `app.services.broker.alpaca.AlpacaBroker.validate_credentials` to return `(True, None)`.

## Trading Service ‚Äî Broker Mock Path

## Known Production Schema Mismatch

`OrderCancelRequest.trade_id: int` vs `Trade.id: String(36)` (UUID). Cancel via API will always return 422 when passing a UUID order ID. The ORM query `Trade.id == trade_id` works with string comparison, but the Pydantic schema blocks it at the API boundary. This is a known bug ‚Äî do NOT fix in tests, accept 422 in cancel assertions.

## Known Service Bug ‚Äî STOP_LOSS stop_price validation gap
`_validate_trade()` checks stop_price only for `OrderType.STOP` and `OrderType.STOP_LIMIT`.
`OrderType.STOP_LOSS` is NOT checked. A SELL+STOP_LOSS without stop_price falls through
to the position DB query (`db.query(Position)`) which crashes (500) because Position is not ORM.
Filed as tech debt ‚Äî do not auto-fix without approval.

## Regression Fixes Applied (2026-03-03)

### Fix 1: Circuit Breaker HALF_OPEN ‚Äî Mixed-Type Dict Bug (CRITICAL)
**Root cause:** P0-4 patch stored half-open probe timestamps as plain `str` values
directly in `self.circuit_breakers` under key `{key}_half_open_last_probe`. All
three iteration methods assumed every value in the dict is a `dict`, causing
`TypeError: string indices must be integers`.

**Files changed:**
- `app/trading_engine/engine/circuit_breaker.py`:
  - `_process_auto_resets` (line ~648): added `if not isinstance(breaker, dict): continue`
  - `get_status` dict comprehensions (lines ~745, ~753): added `isinstance(v, dict)` filter
  - `get_active_breakers_count` (line ~771): added `if not isinstance(breaker, dict): continue`
- `app/tests/test_circuit_breaker.py` (TestCircuitBreakerHalfOpen):
  - Removed `@patch("random.random", ...)` decorators (import removed from production code)
  - Renamed `test_half_open_admits_when_random_above_point_nine` ‚Üí `test_half_open_admits_first_probe` (tests: no prior probe = allow)
  - Rewrote `test_half_open_blocks_when_random_below_point_nine` (tests: recent probe within 15-min window = block)

**DESIGN DEBT:** `self.circuit_breakers` mixes breaker dicts and string sentinels.
Future refactor: move probe timestamps to `self._half_open_probes: dict[str, str]`.

### Fix 2: EFT Agent Config Count ‚Äî Test Not Updated After New Agent (REGRESSION)

### Fix 3: EFT System Prompts Version Mismatch (REGRESSION)
