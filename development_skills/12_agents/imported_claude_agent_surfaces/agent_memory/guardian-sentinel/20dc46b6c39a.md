# XKABI Syscall Security & Completeness Audit — 2026-03-07

## Audit Scope
- Header: `xisc/include/xkabi_syscalls.h` (60 declared syscalls)
- Kernel dispatch: `kernel/xenos/core/main.c` (28 PAL-era syscall numbers, NOT the 60 XISC syscalls)
- Kernel cap table: `kernel/xenos/cap/xkabi_capabilities.c` (handle table only, no syscall dispatch)
- Linux shim: `xisc/runtime/xkabi_linux_shim/` (14 shim files)
- Conformance suite: `xisc/conformance/xkabi_conformance.c` (26 tests, cap table model only)
- Assembly entry: `kernel/xenos/core/syscall.asm` (SYSRETQ non-canonical P0 carry-forward)

## Critical Finding: Dual Syscall Table Mismatch
- XISC header defines 60 syscalls (XK_SYS_HANDLE_CLOSE=1 .. XK_SYS_SYSTEM_TIMESTAMP=60)
- Kernel main.c defines 28 LOCAL syscall numbers (XK_SYS_PHYS_ALLOC=1 .. XK_SYS_HANDLE_CLOSE=28)
- These two tables DO NOT MATCH. Different numbering, different semantics.
- The kernel dispatch has NO routing to the XISC 60-syscall ABI.

## Syscall Implementation Status (of 60 XISC-declared syscalls)

### Kernel (main.c) — 28 PAL-era syscalls
- IMPLEMENTED (3): TIMER_CREATE, TIMER_SET, TIMER_CANCEL (LAPIC one-shot)
- TRIVIAL (2): THREAD_YIELD (returns 0), THREAD_SELF (returns 1)
- ENOSYS STUBS (23): Everything else returns -9

### Linux Shim — covers XISC 60-syscall ABI
- REAL IMPLEMENTATION (11): handle_close, handle_dup, handle_replace, handle_revoke,
  object_get_info (partial), object_signal (partial), vmo_create, vmo_read, vmo_write,
  vmo_get_size, vmo_set_size
- STUB (49): Returns XK_ERR_NOT_SUPPORTED

### Capability Checking
- Kernel dispatch (main.c): ZERO capability checks. No call to xkabi_cap_verify anywhere.
- Linux shim: shim_handle_check_rights() called in VMO read/write/get_size/set_size,
  object_get_info, object_signal, handle_dup (XK_RIGHT_DUP), handle_revoke (XK_RIGHT_REVOKE).
- Kernel cap table (xkabi_capabilities.c): Full create/verify/derive/revoke implemented
  but NOT WIRED into the syscall dispatch path.

## P0 Findings
1. P0-SYSCALL-TABLE-MISMATCH: Kernel uses 28 PAL-era syscall numbers; XISC header declares 60.
   No translation layer exists. User-space calling XISC syscall 7 (PROCESS_CREATE) will hit
   kernel case 7 (THREAD_CREATE) which returns ENOSYS — wrong handler for the syscall number.
2. P0-NO-CAP-CHECK-DISPATCH: xenos_syscall_dispatch() performs ZERO capability verification.
   The 3 implemented syscalls (timer) execute without any rights check.
3. P0-PROCESS-CREATE-STUB: xk_process_create (syscall 7) has no implementation anywhere
   in kernel or shim that actually creates a process (loads ELF, allocates address space).
4. P0-THREAD-CREATE-STUB: xk_thread_create (syscall 11) has no implementation anywhere
   that actually creates a schedulable thread.
5. P0-VMAR-MAP-STUB: xk_vmar_map (syscall 25) returns NOT_SUPPORTED — no mmap equivalent.
6. P0-CHANNEL-IPC-STUB: xk_channel_create/write/read (syscalls 28-30) all stubs — no IPC path.

## P1 Findings
1. P1-SYSRETQ-NONCANONICAL: syscall.asm:205 — no canonical RIP validation before SYSRETQ
   (carry-forward from Sprint 1 audit GS-S01-001)
2. P1-NO-HMAC-HANDLES: Handle encoding uses bit-packing without HMAC authentication.
   User-space can forge handles by guessing slot+rights+generation (carry-forward).
3. P1-TIMER-NO-CAP-CHECK: TIMER_CREATE/SET/CANCEL execute without verifying caller
   holds any capability handle — bypasses the entire XKABI security model.
4. P1-VMO-OP-RANGE-STUB: xk_vmo_op_range (commit/decommit/zero/cache_sync) is a stub.
5. P1-EVENT-ALL-STUB: All 3 event syscalls (create/signal/reset) are stubs.
6. P1-PORT-ALL-STUB: All 3 port syscalls (create/queue/dequeue) are stubs.
7. P1-WAIT-STUB: xk_wait_many is a stub — no multiplexed wait mechanism.
8. P1-EXCEPTION-ALL-STUB: All 4 exception syscalls are stubs.
9. P1-VNODE-ALL-STUB: All 8 vnode (VFS) syscalls are stubs (except vnode_close).
10. P1-IOMMU-ALL-STUB: All 3 IOMMU/driver-pod syscalls are stubs.

## P2 Findings
1. P2-SYSTEM-INFO-STUB: xk_system_info (syscall 59) not implemented.
2. P2-SYSTEM-TIMESTAMP-STUB: xk_system_timestamp (syscall 60) not implemented.
3. P2-CLOCK-NANOSLEEP-STUB: xk_clock_get and xk_nanosleep are stubs.
4. P2-CONFORMANCE-SUITE-DIVERGED: xkabi_conformance.c uses its own local rights model
   (RIGHTS_READ=1<<0, RIGHTS_RDMA_SEND=1<<3) that does NOT match either rights header.
5. P2-VMO-CHILD-STUB: xk_vmo_create_child returns NOT_SUPPORTED (no COW).
6. P2-THREAD-SET-NAME-STUB: xk_thread_set_name returns NOT_SUPPORTED.
7. P2-JOB-CREATE-STUB: xk_job_create returns NOT_SUPPORTED.
