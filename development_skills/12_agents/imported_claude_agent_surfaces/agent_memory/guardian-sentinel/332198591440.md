## Fix 1: D-S1-05 — VERDICT: PASS
- All 8 local `#define XKABI_RIGHT_*` removed from capability_init.c
- `#include "xkabi_rights.h"` at line 46 — single source of truth
- No residual `#define XKABI_RIGHT_*` or `#define NET_SEND` in aetherboot/
- CI: `-Ikernel/xenos/include` added to ABFLAGS in xenos.yml line 160
- Blast radius analysis: old sequential values (0x01-0x80) mapped to bits 0-7 (software rights zone)
  - NET_SEND was 0x10 (bit 4) — canonical is 1u<<17 (bit 17, hardware-enforced)
  - GPU_EXEC was 0x01 (bit 0) — canonical is 1u<<19 (bit 19, hardware-enforced)
  - All 8 old values overlapped with SOFTWARE rights zone (bits 0-11), NOT hardware zone (12-22)
  - Impact: EFI handles got software-zone bits instead of hardware-zone bits
  - Exploitability: LOW — cap table only consumed by kernel at boot, kernel validates magic+layout

## Fix 2: D-S3-P1-003 — VERDICT: PASS (confirmed false positive)
