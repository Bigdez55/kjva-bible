# Maximum Depth Hardware Audit — 2026-03-29

## Summary
- 35 hardware initialization gaps identified
- P0 (boot-blocking): 8 items, ~1,315 LOC
- P1 (daily-driver): 17 items, ~4,680 LOC
- P2 (nice-to-have): 6 items, ~3,290 LOC
- Total: ~9,285 LOC to real-hardware-usable

## P0 BOOT-BLOCKING (EliteBook won't boot to desktop without these)
- GAP-CPU-01: FPU/SSE/AVX state (CR0/CR4/XCR0) -- 120 LOC, any SSE instruction faults
- GAP-BOOT-01: hp_elitebook_x360_init() NEVER CALLED from kmain -- 10 LOC, 700+ LOC dead
- GAP-BOOT-02: NVMe/net poll timer not wired to LAPIC ISR -- 30 LOC
- GAP-STOR-02: GPT partition table parsing -- 200 LOC
- GAP-STOR-03: Block-backed XSTORE persistence -- 500 LOC
- GAP-INPUT-01: I2C-HID touchpad driver -- 400 LOC (EliteBook is I2C, NOT PS/2)
- GAP-PCH-02: I2C device probe not called from boot -- 50 LOC
- GAP-PCH-04: Thunderbolt MMIO uses raw phys addr (needs HHDM) -- 5 LOC

## Top 3 Most Dangerous
1. GAP-CPU-01: SSE fault time bomb -- clang -O2 emits SSE for struct copies
2. GAP-BOOT-01: 700+ LOC EliteBook drivers are dead code (10-line fix)
3. GAP-STOR-02+03: No persistence = live CD mode

## Strongest Areas (well-implemented)
- NVMe: correct register programming, queue management, DMA via PRP
- LAPIC: CPUID calibration, periodic timer, IPI
- HPET: nanosecond-precision hardware counter
- XACPI: RSDP/XSDT/FADT/DSDT + AML scanner + S3/S5
- xHCI: full enumeration, command/event/transfer rings
- I2C LPSS: complete DW_apb_i2c with PCI discovery
- HDA: CORB/RIRB + ALC285 codec init
- EliteBook platform: EC, battery, thermal, camera, Sure Start
