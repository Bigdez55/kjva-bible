# D-S1-05 XKABI Capability Grant Hardware Audit

Date: 2026-03-06

## Fix Summary

capability_init.c replaced local `#define XKABI_RIGHT_*` (sequential 0x01-0x80)
with `#include "xkabi_rights.h"` (canonical bit-mapped values).

## Before/After Table

| Right | OLD (WRONG) | NEW (CORRECT) | Match? |
|-------|------------|--------------|--------|
| GPU_EXEC | 0x01 (bit 0) | 0x00080000 (bit 19) | WRONG |
| GPU_MAP | 0x02 (bit 1) | 0x00000002 (bit 1) | SAME (coincidence) |
| STORAGE_READ | 0x04 (bit 2) | 0x00008000 (bit 15) | WRONG |
| STORAGE_WRITE | 0x08 (bit 3) | 0x00010000 (bit 16) | WRONG |
| NET_SEND | 0x10 (bit 4) | 0x00020000 (bit 17) | WRONG |
| NET_RECV | 0x20 (bit 5) | 0x00040000 (bit 18) | WRONG |
| CONSOLE_OUT | 0x40 (bit 6) | 0x00000010 (bit 4) | WRONG |
| FIRMWARE_VAR | 0x80 (bit 7) | 0x00000020 (bit 5) | WRONG |

Only GPU_MAP accidentally matched (both bit 1). All other 7 rights were wrong.

## Privilege Escalation Paths (OLD, now closed)

1. STORAGE_WRITE=0x08 -> kernel sees MEM_MAP (bit 3) -- arbitrary memory map
2. NET_SEND=0x10 -> kernel sees CONSOLE_OUT (bit 4) -- console takeover
3. NET_RECV=0x20 -> kernel sees FIRMWARE_VAR (bit 5) -- HP BIOS variable R/W

## HP EliteBook Boot-Time Grants (POST-FIX)

|--------|-------------|-------------|-------------------|

| NVMe SSD | Block IO | 0x00018000 | STORAGE_READ + STORAGE_WRITE |
| Intel WiFi AX201 | Simple Network | 0x00060000 | NET_SEND + NET_RECV |
| EFI Console | Simple Text Out | 0x00000010 | CONSOLE_OUT |
