## Deployment Gap Analysis (2026-03-29)

Total driver LOC audited: 15,751 across 16 subsystems.

**Why:** Determine what stands between the current codebase and first boot to graphical UI on real HP EliteBook x360 1030 G4 hardware.

**How to apply:** P0-01 (compositor-to-GOP bridge, ~300 LOC) and P0-02 (NVMe wiring, ~120 LOC) are the two blockers. Everything else is non-blocking for first boot.

### CRITICAL FINDING: Display Pipeline Fork
- Path A (GOP FB via intel_igpu.c): WORKS. AetherBoot sets mode, igpu_init() maps via HHDM, splash renders.
- Path B (DRM/KMS via drm.c + gpu_buf.c): DEAD. Requires open()/ioctl()/mmap() POSIX syscalls not in XENOS.
- xcomp_frame() renders into DRM GEM buffers that can never be scanned out.
- FIX: ~300 LOC bridge to make XCOMP composite directly into GOP framebuffer.

### Subsystem Status Summary
- Display GOP: WORKS (582 LOC)
- Display Compositor: DEAD PATH -- needs GOP bridge (5,410 LOC)
- NVMe: WORKS but nvme_init() not called from boot chain (1,326 LOC)
- PS/2 Keyboard+Mouse: WORKS, wired in kmain Step 10g (605 LOC)
- I2C LPSS: WORKS (517 LOC), no HID driver on top
- WiFi AC 9560: PARTIAL -- FW load done, no 802.11 MAC (927 LOC)
- E1000/VirtIO: WORKS for QEMU (787+746 LOC)
- Audio HDA: PARTIAL -- CORB/RIRB done, no stream DMA (1,206 LOC)
- HP EC Platform: WORKS -- battery/thermal/camera/Sure Start (1,439 LOC)
- xHCI USB: WORKS -- bulk transfers for fingerprint/smartcard (1,875 LOC)
- TPM 2.0: MISSING (0 LOC)

### Open Issues
- P1-02 STILL OPEN: TBT MMIO accessed without HHDM_BASE offset
- P1-03 STILL OPEN: hp_elitebook_x360_init() called but NULL handoff
- HDA s_hda_base never set from PCI BAR0
- E1000/VirtIO poll timer never wired (P2-04 from Sprint 15)
- NVMe not called from boot chain or kmain (only defined, never invoked)
