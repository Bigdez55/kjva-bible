Sprint 36 closed all 14 TODO stubs across 4 files (2026-03-21).

**Why:** Sprint 35 skeleton created static-state implementations with TODO markers for real hardware interaction. Sprint 36 replaced every stub with functional code.

**How to apply:** These implementations are the baseline for Sprint 37+ testing. All compile clean freestanding with -Wall -Wextra -Werror.

## Audio HDA (kernel/xenos/drv/audio.c) -- 6 TODOs CLOSED
1. CORB ring buffer write: s_corb_buf[256] + s_rirb_buf[256] aligned DMA buffers, hda_send_verb() writes to CORB and polls RIRB with MFENCE before CORBWP doorbell write
2. DAC volume: SET_AMP_GAIN_MUTE verb on DAC0+DAC1, 0-100% mapped to 0-87 gain
3. Mute: SET_AMP_GAIN_MUTE with AMP_MUTE bit on speaker+headphone pins
4. Output routing: SET_PIN_WIDGET_CTRL verbs, disable all then enable target pin (speakers=NID14, headphones=NID21, HDMI/BT/USB via subsystem)
5. Input source select: SET_CONNECT_SEL on ADC0 (NID07), mux 0=DMIC, mux 1=headset mic
6. ADC input gain: SET_AMP_GAIN_MUTE on ADC0 input amp, 0-100% to 0-87 gain

## Bluetooth HCI (kernel/xenos/drv/bluetooth.c) -- 4 TODOs CLOSED
1. Inquiry cancel: HCI_Inquiry_Cancel (opcode 0x0402), no params
2. Pairing flow: HCI_Create_Connection (0x0405) + IO_Capability_Reply (0x042B) + PIN_Code_Reply (0x040D), SSP with NoInputNoOutput for audio, KeyboardOnly for HID
3. Unpair: HCI_Delete_Stored_Link_Key (0x0C12) + secure link_key wipe
4. Connect: HCI_Create_Connection (0x0405) with packet type DM1/DH1/DM3/DH3/DM5/DH5
