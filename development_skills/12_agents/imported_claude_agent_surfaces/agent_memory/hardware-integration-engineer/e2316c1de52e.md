## Deployment Critical Path Audit — 2026-03-29

14,381 LOC driver code + 2,977 LOC headers = 17,358 LOC audited across 23 .c + 14 .h files.

### P0 BLOCKER: NVMe not wired into kmain
- nvme.c (1,326 LOC) is COMPLETE and CORRECT
- nvme_init() is NEVER called from main.c or boot_chain.c
- 80 LOC to wire: discover BAR0 from XPCI, call nvme_init, create I/O queues
- **Why:** No wire = no disk access = RAM-only OS

### P0 BLOCKER: No disk filesystem
- xvfs.c backs files to XSTORE (in-memory KV store), not block device
- No FAT32, ext4, or partition table reader exists
- ~2,000 LOC for read-only FAT32

### CORB/RIRB DMA BUG in audio.c
- Lines 262-264: s_corb_buf virtual address written to HDA CORBLBASE register
- Must be physical address: (uintptr_t)s_corb_buf - HHDM_BASE
- Same bug at line 271 for RIRB
- 20 LOC fix

### WiFi firmware delivery gap
- wifi.c has TLV parser + SRAM upload path
- AetherBoot does not load iwlwifi-9000 firmware blob
- No TX/RX DMA rings, no 802.11 MAC layer
- ~4,000 LOC total for basic WPA2-PSK

### Thunderbolt MMIO HHDM bug (P1-02 STILL OPEN since Sprint 11)
- elitebook_x360.c line 292: HP_TBT_MMIO_BASE accessed without HHDM_BASE
- 5 LOC fix

### What WORKS on real hardware today
- UEFI boot, page tables, HHDM, GDT/IDT/TSS, syscall
- PCI enumeration, ACPI parsing, HPET, LAPIC timer 100Hz
- IOMMU VT-d, xHCI USB init + enumeration
- HP EC: battery, thermal fan curve, camera, hinge, kbd backlight
- PS/2 keyboard + mouse, GOP framebuffer 1920x1080
- SMEP/SMAP/IBRS/KASLR, TPM 2.0 PCR extend, GENSD
