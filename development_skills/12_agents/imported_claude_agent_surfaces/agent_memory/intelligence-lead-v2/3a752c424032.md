## Recurring Causal Patterns
- "Compile drift": .ts/.js diverge in Orange suite and companion.
- "EWMA lag bias": (1-alpha)/alpha * interval = steady-state lag per ramp rate.
- "Timeout inversion": Circuit breaker 40s, AbortController 30s still mismatched.
- "SKU bifurcation": G4 i5-8265U = Whiskey Lake (AVX2 ONLY, NO AVX-512). Prior "Ice Lake"
  claim was WRONG. Perf model must use AVX2 ceiling, not AVX-512.
- "Resource arithmetic": k8s+Ollama+Electron=5.6-8.4GB. 8GB=OOM, 16GB=viable.
- "Boot cap table orphan": FIXED. capability_init.c now uses xkabi_rights.h.
- "IST gap": TSS ist[7] defined but never populated. NMI/DF/MC need IST stacks.
- "SWAPGS window": SYSRET ~3 insn window. Fix: IST for NMI + check CS.
- "Dual primitive divergence": PAL vs XSEC SHA-256. Bug isolation but 2x audit.
- "Crypto cascade": sha256->HMAC->HKDF->TLS. p=1.00 propagation. 14 edges.
- "Recursive-update length corruption": FIXED. Save counter before padding.
- "CT false positive from stale code": branchless `0u - bit` masks look like branches.
  Always verify against current source before raising CT finding.
- "Limb width mismatch": ed25519 int32+int64 needs carry. x25519 uint64+u128 safe.
- "Stub-as-drift-marker": `/* Sprint 4: */` = deterministic API drift predictors.
- "Transport callback inversion": TLS fn ptrs enable QUIC addition w/o TCP removal.
- "Residual risk concentration": Single P1 defect (RSA bypass) = 78% of risk.
  Always compute risk-weighted vs raw-count coverage to avoid false confidence.
- "Integer-only JIT": XJIT has 23 opcodes, ALL integer GP (no XMM/FP). Cannot accelerate
  any XMIND op. XJIT + XMIND are architecturally disjoint subsystems.
- "fp32 embedding bloat": token_emb stored as float* not Q4. 375 MB vs 59 MB if quantized.
- "Heap leak cycle": xm_heap_free() is no-op. PAL handles not tracked per-pointer.
  Every init/shutdown leaks KV cache (336 MB). 5 cycles = OOM on 16 GB.
- "Dead BPE dispatch": tokenizer.c has two xmind_tokenize defs. BPE wrapper is
  __attribute__((unused)) and never called. XSTORE vocab loads but never consumed.
- "Weight size mismatch": weights_loader.c uses hidden_dim for ALL 7 weight matrices.
  wk/wv overallocated (4.2 MB waste/matrix/layer), w1/w2/w3 underallocated (will overflow in Sprint 9).
- "Dual boot sequence incoherence": boot_sequence.c marks STORAGE critical; boot_chain.c
  treats it non-critical. These files link together in S8-9 and WILL conflict.
- "XSTORE cascade amplifier": XBLOB->XSTORE, XPKG->XSTORE, XMIND->XBLOB->XSTORE.
  Single XSTORE failure cascades to 3 subsystems. 3-hop chain for AI weights.
- "Missing emergency shell": boot_sequence.c sets emergency_mode flag but no emergency
  shell implementation exists. Critical stage failure = system halt, not recovery.
- "Config drift singleton": boot_chain.c hardcodes XMIND config constants (lines 72-79).
  xmind.c owns the real singleton. No shared struct = guaranteed drift.
- "Attn buffer sizing mismatch": state.attn sized for scores (n_heads*ctx_len) but used as
  concatenated output (n_heads*head_dim). 16x overalloc. Comment in header is stale.
- "Approx nucleus sampling": XMIND top-p uses threshold=(1-p)*max_prob, not sort.
  Conservative (over-includes). Acceptable for freestanding (no heap sort). Document as design choice.
- "Model-vocab coupling": XMIND_VOCAB_SIZE=32000 is LLaMA-specific constant. Runtime uses
  cfg.vocab_size. Switching to Qwen3 (151936) works at runtime but constant is misleading.
