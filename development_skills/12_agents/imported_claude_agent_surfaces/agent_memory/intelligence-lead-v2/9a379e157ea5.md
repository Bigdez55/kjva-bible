# XMIND Inference Engine Design Analysis (2026-03-02)

## Summary
Full causal analysis of Sprint 4 XMIND inference engine dependencies:
- vDRAM x KV Cache fit assessment: 5 API gaps identified, proposed C signatures
- GEMM optimization: AVX-512 tiles 6x16x128, memory-bound on iGPU
- KV cache sizing: 28 KB/token Q4, 57.4 MB resident for 4096 context
- Migration plan: 4 phases (8 weeks), ctypes preferred over HTTP
- Scheduler integration: XKABI_SCHED_HINT_AI_INFERENCE proposed
- Performance: 75% decode improvement [CI: 40%, 120%] on Ice Lake

## Key Defects Found

### D-XMIND-01: vDRAM RLE compression unsuitable for Q4 KV data (P1)
- RLE in vdram.c (lines 114-153) requires byte runs >= 3
- Q4 nibble pairs: P(3 identical bytes) ~ 0.01
- Compression ratio on Q4 KV: ~1:1 (no compression)
- Fix: integrate AIT compressor from legacyforge/src/ait_compress.c
- Comment on line 113 already says "placeholder for AIT integration"

### D-XMIND-02: vDRAM lacks sub-region access for KV cache (P1)
- vdram_access() returns pointer to entire region
- KV cache attention needs [start_pos, end_pos] sub-range
- Without this, entire tensor must be HOT even for 128-token attention window
- Proposed: vdram_access_range() with offset+length parameters

### D-XMIND-03: No batch prefetch API (P2)
- vdram_prefetch() takes spinlock per call
- Prefetching 28 layers sequentially: 28 * spinlock_overhead
- Proposed: vdram_prefetch_batch() for bulk prefetch

### D-XMIND-04: No eviction callback (P2)
- Scanner demotes regions silently
- XMIND cannot know when a KV block was demoted from HOT
- Risk: stale pointer after lease release + scanner demotion
- Proposed: vdram_set_eviction_callback()

### D-XMIND-05: No sliding window hint (P2)
- KV cache has a moving head pointer (newest tokens)
- Scanner demotes by recency, which conflicts with sliding window
- Proposed: VDRAM_HINT_SLIDING_WINDOW + vdram_hint_window_head()

### D-XMIND-06: SKU uncertainty (P1)
- EliteBook x360 1030 G4 ships as Ice Lake or Comet Lake
- Ice Lake: AVX-512, Iris Plus 64EU, LPDDR4X-3733
- Comet Lake: AVX2 only, UHD 620 24EU
- 40% performance gap in GEMM throughput
- Must: cpuid check at boot, dual kernel compilation (avx512 + avx2)

### D-XMIND-07: FROZEN tier NVMe not functional (Known, Sprint 2)
- vdram_freeze_region() returns PAL_ERR_NOT_SUPPORTED (line 392)
- Context beyond RAM capacity loses data on FROZEN eviction
- Acceptable: 430K tokens fit in 11.7 GB. No consumer workload exceeds this.

### D-XMIND-08: Contra-rotation scanner no AI client registration (P1)
- cr_sched_register_device() exists but no AI client registration
- No phase transition callbacks for XMIND coordination
- XMIND cannot request thermal burst scheduling
- Proposed: cr_sched_register_ai_client() + cr_sched_request_thermal_burst()
