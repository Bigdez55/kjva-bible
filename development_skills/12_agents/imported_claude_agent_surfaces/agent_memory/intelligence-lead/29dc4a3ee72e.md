## Contra-Rotation Scheduler — Grade B+

State machine is logically correct and fail-safe. Sensor error forces COOLDOWN (correct).

DEFECTS:
- CR-1 (MEDIUM): Hysteresis gap at 60C is zero-width. COOLDOWN->LOAD fires at temp<=60,
  COMPUTE->COOLDOWN fires at temp>=60. Under ±2C sensor noise this causes 0.4 Hz oscillation
  in 58-62C band, ~20% compute throughput loss. Fix: lower COOLDOWN->LOAD threshold to 55C.
- CR-2 (LOW): pal_thermal_read_cpu() called while spinlock is held. Acceptable on PAL-Aether
  (<1 us), borderline on PAL-Linux (<100 us). Acceptable at 500ms poll interval.
- CR-3 (MEDIUM): Scanner thread detached immediately (line 395). No join on shutdown.
  Up to 500ms delayed exit after running=false. Sprint 2: store handle, add join.
- CR-4 (ARCH): NO EDF COMPLIANCE. Scheduler is a thermal-reactive FSM, not EDF.
  No per-task deadline timestamps, no priority queue, no preemption. As designed per RFC-002.
- CR-5 (NOTE): NCCL idle threshold 50ms is too short. Expect RDMA lane churn.
  Correct statistical baseline: 200-500ms. Will need calibration on real NCCL traces.

## TPM / SHA-256 — Grade A- (SHA-256 correct; two wire format issues)

SHA-256 FIPS 180-4 COMPLIANCE: FULLY VERIFIED
- All 8 initial hash values correct (H0=0x6a09e667 .. H7=0x5be0cd19)
- All 64 round constants correct (K[0]=0x428a2f98 .. K[63]=0xc67178f2)
- CH, MAJ, SIG0, SIG1, sig0, sig1 macros match FIPS equations (4.2)-(4.7) exactly
- Padding: 0x80 + zero-fill to 56 + big-endian 64-bit bit_len — correct per FIPS 5.1.1
- Message schedule expansion (w[i] = sig1+sig0+w terms): correct

DEFECTS:
- TPM-5 (LOW): Line 362: `(UINT32 *)&cmd[off]` cast for commandSize back-fill.
  cmd is UINT8[] on stack, offset 2 — unaligned write. Benign on x86 but C11 UB.
  Fix: use `put_be32(&cmd[2], off)` directly.
- TPM-6 (MEDIUM): tpm2_get_pcr() uses digest_offset=28 (line 491).
  Correct value is 30 (TPML_PCR_SELECTION = 10 bytes, not 8).
  Result: PCR log display shows wrong bytes. Non-security-critical (readback is display only)
  but auditability is compromised. Fix: change digest_offset to 30.

## PMM Buddy Allocator — Grade A-

XOR FORMULA VERIFICATION: CORRECT
  `phys ^ ((uint64_t)PAGE_SIZE << order)` — toggles bit [11+order] of address.
  Standard Linux kernel buddy formula. Provably correct for aligned inputs.

ZONE GUARD: CORRECT — buddy_zone != this_zone stops coalescing across 4GB boundary.

DEFECTS:
- PMM-3 (MEDIUM): Alloc fallback duplicates split-and-return logic at lines 346-351
  and 367-372. Manual DMA32 fallback inner loop when zone=NORMAL. Code divergence risk.
  Fix Sprint 2: extract helper pmm_try_alloc_from_zone() used by both paths.
- PMM-4 (LOW): g_free_pages incremented before ownership verified. Double-free inflates
  counter. No ownership tracking. Acceptable for Sprint 1.
- PMM-5 (ARCH): page_frame_t stored in first 64B of physical page. Overwritten by
  allocator caller. No use-after-free detection at physical level. Sprint 2: migrate to
  heap-allocated descriptors.

## Supervisor Backoff — Grade A

BACKOFF FORMULA: CORRECT AND OVERFLOW-SAFE
  `next = current_ms * 2; return (next > 30000 || next < current_ms) ? 30000 : next`
  Overflow guard `next < current_ms` is correct uint32 wraparound detection.
  Sequence: 1000->2000->4000->8000->16000->30000(cap)->30000...
  Overflow cannot reach guard (cap fires at 30000, far below uint32 overflow at ~2B).

DEFECTS:
- SV-2 (P1 KNOWN): supervisor_restart() calls pal_thread_sleep_ns() for up to 30s.
  Blocks ALL service liveness probes during sleep. Worst case: N services * 30s blackout.
  Acknowledged in TODO comment (lines 207-213), tracked event-horizon-analysis.md.
- SV-3 (FRAGILITY): supervisor_watch() uses void* + manual svc_layout_t reinterpretation.
  If gsd_service_t layout changes, silent ABI break. Fix Sprint 3: shared gensd_types.h.

## XKABI Capability Management — Grade B+

HANDLE ENCODING (64-bit): STRUCTURALLY SOUND
  [slot:24][rights:24][generation:16] = 64 bits, non-overlapping.
  Slot field allows up to 16M slots (250x headroom over current 64K).

RIGHTS-NARROWING CHECK: MATHEMATICALLY CORRECT
  `(new_rights & ~parent_rights) != 0` — correct Bell-La Padula subset test.

DEFECTS:
- CAP-2 (MEDIUM): Generation field is 16-bit in handle, 32-bit in cap struct.
  After 65536 bulk revocations, old handles can pass Check 3a (gen match) and must
  rely on Check 3b (epoch comparison) for rejection. Defense-in-depth is sound but
  the truncation is a latent risk. Sprint 2 HMAC makes this irrelevant.
- CAP-3 (KNOWN): xkabi_cap_create() linear scan under spinlock. Worst case ~75 us
  at full 64K occupancy. Acceptable for Sprint 1. Sprint 2: embed free-list.
- CAP-4 (DOCUMENTATION ERROR — MEDIUM): Comment states "96 bytes/entry = 6 MB BSS".
  ACTUAL sizeof(xkabi_cap_t): lineage[16][16]=256B + object_id(8) + rights(4) +
  generation(4) + lineage_depth(4) + revoked(1) + valid(1) + padding ~= 280-288 bytes.
  Actual BSS: 65536 * ~284 bytes = ~18.6 MB, NOT 6 MB. 3x misstatement.
  Fix: add _Static_assert(sizeof(xkabi_handle_table_t) < 25*1024*1024, "BSS budget");
  and correct the comment.
- CAP-6 (KNOWN): xkabi_cap_revoke() marks revoked=true but does NOT set valid=false
  or free the slot. Slot permanently occupied until Sprint 2 free-list.
  At 64K slots: exhaustion requires 64K create+revoke cycles. Acceptable Sprint 1.
- CAP-SECURITY (P0 for Sprint 2): No HMAC. Handles forgeable by guessing slot+rights+gen.
  Forgery cost: O(2^16) for generation field = 65536 guesses. Must add pal_sha256 HMAC.
