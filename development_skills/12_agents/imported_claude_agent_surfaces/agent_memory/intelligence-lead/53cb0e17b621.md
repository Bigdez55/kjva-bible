## Critical ML Patterns (Confirmed)

### Cloud Run Hardware Constraints
- FinBERT (ProsusAI/finbert) CANNOT run on Cloud Run — weights ~440MB + torch ~800MB exceeds 2GB RAM
- All torch/transformers inference MUST route to vLLM VM via VPC connector
- VADER is the only viable synchronous sentiment fallback on Cloud Run
- onnxruntime quantized FinBERT is viable (~300MB) if truly needed on Cloud Run

### Feature Engineering
- Rolling z-score must be fit ONLY on training window inside each walk-forward fold
- Multi-timeframe features: always use prior day's close for daily features at intraday decision points
- 12-month momentum features require 252-day embargo, not 5-day
- Embargo = max(all feature lookback windows) — never a fixed small constant

### DRL Architecture
- PPO portfolio allocation requires explicit budget constraint (Σwᵢ = 1) — MlpPolicy does NOT enforce this
- Options: softmax output (long-only), Dirichlet policy, or manual normalization action transform
- CQL REQUIRES (s, a, r, s') tuples from a behavior policy — OHLCV data alone is invalid for CQL
- Use Decision Transformer or Behavior Cloning from rule-based strategy logs as offline bootstrap instead
- SAC execution reward must be dense (per-tranche) not sparse (terminal only)
- For financial data (limited samples), SAC/TD3 preferred over PPO due to off-policy sample efficiency

### Reward Function Design
- All reward terms must be normalized to common scale (% of portfolio NAV)
- Anti-HOLD constraint required: agent must be penalized for inactivity vs benchmark
- CVaR estimation needs ~200 tail samples to be stable — use rolling window with exponential decay

### Validation Rigor
- 30-day paper trading validates operational correctness ONLY — not statistical edge
- Deflated Sharpe Ratio: must count ALL experiments in MLflow, not just final grid search
- Adversarial validation (AUC > 0.6 = distribution shift) must gate all walk-forward results
- Survivorship bias: Alpaca historical data only has currently listed symbols — use Polygon/Norgate for backtests
- Stress test scenarios must include pre-training-period events (2008-2009, 2000-2002) to be truly OOS

### HMM Regime Detection
- Minimum 750 daily observations (3 years) for reliable Baum-Welch convergence on 2-state HMM
- HMM is descriptive (lagging), not predictive — 1-2 day lag in state assignment changes
- 3-state model (tranquil/normal/crisis) empirically better than 2-state for US equities
- Fuse VIX threshold and HMM state BEFORE feeding to DRL — do not let DRL arbitrate between them

### Inference Latency
- PPO/SAC MlpPolicy forward pass: ~0.5-2ms on 1 CPU (136-dim obs, 2-layer MLP)
- 50ms total budget requires Redis feature cache — rolling z-scores must be precomputed
- Feature computation must complete in <15ms; FinBERT MUST be pre-routed to vLLM

## ML Audit Findings (Confirmed 2026-03-08)

### Feature Engineering Scale Issues (CRITICAL)

- 46-dim DRL obs vector has NO normalization — raw price ($150) vs RSI (0-100) vs OBV (billions)
- Python RSI uses SIMPLE rolling mean (wrong); WASM uses correct Wilder's EMA — 5-15 RSI point divergence
- Python ATR uses SIMPLE rolling mean (wrong); WASM uses Wilder's smoothing — 10-20% ATR error propagates to Kelly sizer
- Python Bollinger Bands uses ddof=1 (sample std); WASM uses population std — 2.5% band-width divergence
- MACD is price-denominated (not normalized); must divide by ATR before feeding to ML models

### Outcome Fill Worker (CRITICAL BUG)

- `episode_terminal` set to True for ALL rows with price_at_1d filled (line 300, outcome_fill_worker.py)
- Root cause: `row.price_at_1d is not None` is ALWAYS True after fill; invalidates all DRL training tuples
- 1h/4h slots cannot be filled by daily bars (tolerance 30min/2h vs daily bar granularity) — switch to hourly bars
- Outcome coverage: 160/3,559 = 4.5% — PRIMARY cause is bar granularity mismatch

### Calibration Status

- Only ~48 usable calibration samples (160 outcomes * 30% BUY/SELL fraction)
- _MIN_CALIBRATION_SAMPLES lowered to 10 for bootstrap; still insufficient for reliable ECE
- No automatic recalibration on drift — ModelEvaluationService.detect_drift() does NOT call SignalGateService.refresh_calibration()
- ECE is computed on-demand only; never persisted or alerted on

### Anomaly Detector

- anomaly_detector.py: Z-score + threshold based (NOT Isolation Forest + LOF as designed)
- DEAD CODE in auto-trading loop — zero integration in auto_trading_service.py or any API endpoint
- Module-level singleton exists but is never called during live trading

### DRL Readiness Score: 18/100

- episode_terminal bug corrupts all existing labeled rows for multi-step RL
- next_observation_vector approximation (only first dim scaled by realized return) is invalid
- No replay buffer persistence, no DRL env wrapper, no training loop
