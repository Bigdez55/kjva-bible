## Key Modifications Required

### Feature Engineering
- Implement LeakageAuditStep: assert feature.shift(1) correlation within expected range
- Multi-timeframe alignment: use prior day close, never current day close, for daily features at intraday points
- compute_embargo_per_feature() function returning max lookback per feature

### PPO Portfolio Allocation
- Must enforce budget constraint Σwᵢ = 1 explicitly (softmax, Dirichlet, or manual normalization)
- Prefer SAC/TD3 over PPO for financial data (off-policy, sample efficient)
- Define step unit precisely (daily vs intraday) before sizing n_steps=2048

### SAC Execution
- Dense per-tranche reward: r = -slippage_bps - market_impact + participation_rate_bonus
- Verify Alpaca data tier provides L2 order book depth (not just NBBO)

### HMM Regime
- Use 3-state model (tranquil/normal/crisis)
- Minimum 750 daily obs for Baum-Welch convergence
- Treat as descriptive (lagging), not predictive signal
- Fuse with VIX thresholds before DRL observation, not after

### Reward Function
- Normalize all 5 terms to % of portfolio NAV
- Add benchmark-relative underperformance penalty (anti-HOLD)
- Use rolling window CVaR with exponential decay weighting (not episodic CVaR)
- Tune {α,β,γ,δ,ε} via Optuna with DSR as objective

### Deflated Sharpe
- Log ALL experiments to MLflow from day 1 (including informal tests)
- Count N_trials = all unique (model, hyperparams, feature_set) combinations tested
- DSR < 0 = do not deploy, regardless of raw Sharpe

### LLM Signal Injection
- Use at daily portfolio layer only (PPO), not intraday execution layer (SAC)
- One-hot encode categorical fields (strategy_bias, risk_appetite) — NOT label encoding
- Freeze the LLM-to-observation projection during DRL training

### Validation
- Add adversarial validation gate: AUC > 0.6 = distribution shift detected = investigate before proceeding
- Stress tests must include pre-training events (2008-2009, 2000-2002 dot-com)
- Use survivorship-bias-free data source (Polygon.io, Norgate) for backtesting
- 30-day paper trading validates pipeline, NOT statistical edge

## Statistical Justifications

### Why 252-day embargo (not 5-day)
12-month momentum feature at t=230 uses prices from t=230-252 to t=230.
If test set includes t=252 onward, training label at t=230 has a feature that looks
22 bars forward into test window. 5-day embargo does not prevent this.

### Why CQL requires behavior policy
CQL minimizes Q(s,a) for OOD actions relative to behavior policy distribution.
Without knowing what actions the behavior policy took, the conservative penalty
has no reference point. Result: Q-function learns spurious correlations between
favorable price movements and arbitrary pseudo-actions.

### Why 30 days is not statistical validation
To distinguish SR=1.0 from noise at 95% CI:
T = (1.96)^2 * (1 + SR^2/2) ~= 6 years of annual Sharpe data
= ~1,500 trading days
22 trading days gives 95% CI of approximately ±4.8 Sharpe units.
