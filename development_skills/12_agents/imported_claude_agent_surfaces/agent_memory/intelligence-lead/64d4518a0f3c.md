## Methodology

### Test Design Principles Applied

1. **Parsimony Principle:** Started with simplest test cases, built complexity incrementally
2. **Statistical Rigor:** Each test has clear pass/fail criteria with no ambiguity
3. **Edge-Case Dictionary:** Systematically covered:
   - Boundary values (zero, negative, maximum)
   - Error conditions (network, auth, validation)
   - Race conditions (concurrent operations)
   - State transitions (pending → filled → cancelled)

4. **Hypothesis Testing:** Each test validates a specific behavioral hypothesis:
   - H₀: System handles edge case correctly
   - H₁: System fails on edge case
   - Test either rejects or fails to reject H₀

### Mock Strategy

- **Alpaca Broker:** Full API mocking with realistic response structures
- **Trading Service:** Isolated broker interactions, real database operations
- **Auth Flow:** Mocked external services (email, SMS), real token logic
- **Frontend:** Mocked RTK Query, real Redux store
