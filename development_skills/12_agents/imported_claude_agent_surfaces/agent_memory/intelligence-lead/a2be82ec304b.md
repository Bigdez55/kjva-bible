# Intelligence Lead — Analytical Patterns

## GEN.OS AI Pipeline Analysis Pattern

When analysing an end-to-end AI feature (renderer → preload → agent → model):

1. Read all four layers before forming conclusions — the routing gap between
   preload tool names and agent `_tool_dispatch` registrations is invisible
   unless you read both sides.
2. Timeout values must be compared against the FULL call chain latency, not
   just the network RTT. The OllamaModelManager has its OWN 45 s timeout;
   the preload AbortController is a separate, shorter gate.
3. For CPU-only LLM inference: P95 latency = (output_tokens / tok_per_second).
   Always check thermal state; EliteBook x360 can drop to 4–6 tok/s under load.
4. The agent's `_sanitize_context` is only applied in `run()`, not
   `_tool_dispatch()` — any future tool that accepts free-text payloads must
   explicitly call `_sanitize_payload_text()`.

## Token Expiry Pattern

GENOS uses a static token read at preload startup. For all Orange Suite
apps, `SyncClient` is the correct place to detect 401/403 and throw
`TokenExpiredError`. The preload is the correct place to catch it.
The renderer should NEVER see raw HTTP status codes.

## AI Response XSS Boundary

Model output must NEVER be inserted into the DOM via `innerHTML` or
`dangerouslySetInnerHTML`. The correct insertion path for TipTap editors is
`editor.commands.setContent(text, true)` — ProseMirror validates the content
against the schema and strips unknown nodes before they reach the DOM.

## Error Classification Taxonomy (AI calls)

| HTTP status / exception | Kind |
|---|---|
| AbortError (DOMException) | "timeout" |
| 503 | "circuit_open" |
| 500 | "model_unavailable" |
| result.status === "ignored" | "routing_failure" |
| fetch throws (non-Abort) | "network" |

Always check for `routing_failure` by inspecting the response body's
`result.status` field AFTER a 200 response — the agent returns 200 with
`{"status": "ignored"}` for unknown tools, which would otherwise be silently
treated as a successful but empty response.

## Input Truncation Quality Guidelines

For Llama 3.2 3B Q4 (4096-token context):
- 500 chars = ~100 words = inadequate for any real summarisation task
- 3 000 chars = ~600 words = sufficient for meeting notes / project outlines
- Never use a single truncation limit for all action types
- Truncation is NOT a PII gate — use regex redaction instead

## Circuit Breaker Alignment

The GENESYS AI circuit breaker resets after 30 s. The client AbortController
timeout should equal or slightly exceed this so that:
1. A timed-out client retrying after 30 s hits an open circuit (clean error)
   rather than another timed-out request
2. The model has enough wall-clock time to complete before abort fires
