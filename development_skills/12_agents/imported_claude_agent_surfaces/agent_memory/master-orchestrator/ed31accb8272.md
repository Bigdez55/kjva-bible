# Sprint 16 Plan -- "First Light" (QEMU Visible Boot)

## Date: 2026-03-07
## Milestone: GEN.OS produces verifiable serial output in QEMU CI

## Root Cause of Sprint 15 QEMU Soft-Fail

**BLOCKER-1 (P0)**: CI sprint15.yml line 276 links BOOTX64.EFI with `-entry:efi_main`.
- D-04 fix renamed `efi_main` -> `aetherboot_main()`.
- Real PE32+ entry is `_abi_entry` in `aetherboot/src/stage1.asm`.
- Makefile is correct: `-entry:_abi_entry`. CI is wrong.

**BLOCKER-2 (P0)**: CI does NOT assemble `stage1.asm` at all.
- Only 3 C files compiled. No `stage1.o` in link step.
- Without stage1.o, `_abi_entry` symbol doesn't exist.

**BLOCKER-3 (P1)**: Potential struct mismatch `xenos_handoff_t` vs `xenos_boot_info_t`.
- AetherBoot writes `xenos_handoff_t`; kmain reads `xenos_boot_info_t`.
- Magic field must be at offset 0 in both for boot to proceed.
