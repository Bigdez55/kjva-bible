# Advisory-03 Routing Audit: XSEC Audit Log -> GENSD Journal

## Date: 2026-03-06
## Scope: Audit log routing gap analysis + system observability audit

### Summary

Advisory-03 states: "xsec_audit_log events (HANDSHAKE_FAILED, CERT_REJECTED)
generated in x509.c and tls13.c have no routing path to the GENSD journal yet.
These are currently write-only audit records."

**Status: CONFIRMED STILL MISSING. No routing exists.**

### Evidence

1. GENSD (init/) has ZERO references to xsec_audit, xsec_g_audit, audit_query,
   audit_dump, or audit_get_dropped. Verified via exhaustive grep.

2. XSEC (sec/xsec/) has ZERO references to journal_write, journal, or gensd.
   Verified via exhaustive grep.

3. XNET (net/xnet/) has ZERO references to xsec_audit or journal_write.
   Verified via exhaustive grep.

4. supervisor.c does NOT poll the XSEC audit ring. It only monitors service
   liveness via running flag checks.

5. gensd main.c run_loop has a 30s periodic journal_flush_serial() call,
   but this only flushes the GENSD journal ring -- it never reads the XSEC
   audit ring.

### Two Isolated Ring Buffers

System A: XSEC Audit Ring (sec/xsec/audit/audit.c)
  - 1024-entry fixed struct ring (xsec_audit_entry_t)
  - Entry size: ~84 bytes (8B timestamp + 4B event + 4B module + 64B detail + padding)
  - Lock-free writes via atomic FAI on write_idx
  - Read API: xsec_audit_query_recent() (spinlock-protected)
  - Dump API: xsec_audit_dump() (serial console only)
  - Output: pal_console_puts() -- serial console, not structured

System B: GENSD Journal (init/gensd/journal.c)
  - 4MB byte-stream ring buffer in BSS
  - Variable-length entries: 16B header + msg[0..255]
  - Spinlock-protected writes
  - Read API: journal_read_recent() -- STUB (returns nothing)
  - Dump API: journal_flush_serial() -- serial console scan

### The Gap

There is NO bridge function, NO periodic poll, NO callback, and NO shared
memory region that connects System A to System B. They are two completely
independent observability subsystems that do not know about each other's
existence.

### Recommended Minimum Routing (Sprint 4)

A bridge function in GENSD that periodically drains XSEC audit events
into the GENSD journal:

  void audit_bridge_tick(void) {
      xsec_audit_entry_t recent[32];
      uint32_t count = 0;
      xsec_audit_query_recent(recent, 32, &count);
      for (uint32_t i = 0; i < count; i++) {
          // Map audit event to journal entry
          journal_write("xsec", level_from_event(recent[i].event_type),
                        recent[i].detail);
      }
  }

Called from the supervisor_tick() loop or the main 30s flush loop.

### Challenges

1. journal_read_recent() is a STUB -- returns nothing. The GENSD journal
   reader is not implemented. This means even GENSD's own events cannot
   be queried programmatically (only flushed to serial).

2. xsec_audit_query_recent() returns most-recent-N but has no cursor or
   "since last read" mechanism. A bridge would need to track the last
   total_events value to avoid re-processing.

3. xsec.h is not included by any GENSD file. Adding the dependency
   requires an include path addition (-Isec/xsec/include) to the
   GENSD compile command.

### TCP Silence Confirmation

TCP (net/xnet/transport/tcp.c, 1347 LOC) has ZERO observability:
- 0 pal_console_printf calls
- 0 xsec_audit_log calls
- 0 journal_write calls
- Complete silence on connection state changes, RST events, retransmits
