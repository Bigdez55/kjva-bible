# Sprint 3 Observability Gap Analysis — 2026-03-05

## Scope
XNET (11 files), XSEC (9 files), XPKG (7 files), GENSD journal (1 file)

## Existing Instrumentation

### GENSD Journal (init/gensd/journal.c)
- 4MB BSS ring buffer, binary packed entries
- Fields: timestamp_ns, service_hash (fnv1a32), level (0-4), msg (max 255B)
- Thread-safe via spinlock
- journal_read_recent() is STUBBED (Sprint 1 comment says Sprint 3 implementation)
- journal_flush_serial() scans ring and dumps to console (cap 64 entries)

### XSEC Audit Ring (sec/xsec/audit/audit.c)
- 1024-entry ring buffer, lock-free write (atomic fetch-add), spinlock for dump
- Fields: timestamp_ns, event_type (enum), source_module (enum), detail[64]
- Events logged: KEY_GENERATED, HANDSHAKE_COMPLETE/FAILED, CERT_VERIFIED/REJECTED,
  DECRYPT_FAIL, AUTH_TAG_MISMATCH, TLS_ALERT_RECV, SESSION_CLOSED
- xsec_audit_query_recent() and xsec_audit_dump() exist

### XNET Interface Stats (net/xnet/core/netif.c)
- xnet_if_stats_t: rx/tx_packets, rx/tx_bytes, rx/tx_errors, rx/tx_dropped
- Atomic increments on each send/input path
- xnet_netif_get_stats() retrieves snapshot

### XNET Console Logging
- netif.c: register, unregister, set_ipv4, up, down, init, shutdown
- arp.c: gratuitous ARP, duplicate IP warning, resolution timeout
- gossip.c: member join/dead/leave, init, suspect
- dns.c: truncated response warning
- mdns.c: bind failure, probe, announce, conflict
- icmp.c: dest unreachable, time exceeded

### TCP (net/xnet/transport/tcp.c)
- RTT tracking: srtt_ns, rttvar_ns, rto_ns (Jacobson/Karels)
- Congestion: cwnd, ssthresh, dup_acks
- NO console logging at all in tcp.c (completely silent)

### XPKG Logging
- All via pal_console_printf — install, remove, upgrade, gate results
- refusal_gate.c logs each gate PASS/WARN/FAIL with detail string
- pkg_verify.c logs signature/integrity results
- NO integration with XSEC audit log (security events not audited)
- NO integration with GENSD journal

## Critical Gaps Identified

### GAP-1: TCP is completely silent (P0)
tcp.c has ZERO logging. Connection state transitions, retransmits, resets,
checksum failures, OOO segments — all invisible.

### GAP-2: No counters/gauges beyond netif stats (P0)
Missing: TCP connections active, TCP retransmit count, SYN flood count,
DNS query count/latency, ARP cache size, gossip member count over time.

### GAP-3: journal_read_recent() is stubbed (P1)
Cannot programmatically read journal entries. Only serial flush works.

### GAP-4: XSEC audit not connected to GENSD journal (P1)
Two independent ring buffers with no bridge. Security events are invisible
to the system-wide journal.

### GAP-5: XPKG has no audit trail integration (P1)
Package install/remove/verify events use console only. No XSEC audit log.
No GENSD journal writes.

### GAP-6: No packet trace correlation (P2)
Cannot trace a packet from NIC input through eth->ip->tcp/udp to socket.
No trace IDs, no span-like correlation.

### GAP-7: No histogram/percentile tracking for latencies (P2)
TCP RTT is tracked per-connection but not aggregated. DNS latency not tracked.
TLS handshake duration not tracked.

### GAP-8: No alert conditions defined (P2)
No threshold checks for SYN flood, cert rejection rate, retransmit spike,
ARP storm, etc.

### GAP-9: Crypto operations not audited (P2)
SHA-256, HMAC, AES-GCM, ChaCha20, X25519 — none log to audit ring.
Only TLS and X.509 call xsec_audit_log().

### GAP-10: No SLO framework for kernel networking (P3)
No error budget tracking, no latency percentile targets defined.
