# Sprint 3 Security Fix Performance Impact Analysis (2026-03-05)

## 1. AES S-box Constant-Time (P0 fix: GS-S3-P0-001)

File: `sec/xsec/crypto/aes_gcm.c:58-69`

### Iteration Count Per AES-256 Block

- SubBytes: 16 bytes * 14 rounds = 224 sbox calls
- Key schedule sub_word: (7 + 6) * 4 = 52 sbox calls
- Total: 276 sbox calls/block * 256 iterations/call = 70,656 iterations/block

### Throughput

| Path | Cycles/sbox | Cycles/block | Cycles/byte |
|---|---|---|---|
| Direct table | ~4 | ~1,104 | ~69 |
| CT scan (256) | ~1,280 | ~353,280 | ~22,080 |
| AES-NI | ~4/round | ~56 | ~3.5 |

Software CT path: ~200 KB/s at 3.4 GHz -- UNACCEPTABLE for TLS records.
AES-NI path: ~1 GB/s -- required for production.

### Additional bottlenecks in same file

- gf_mul (line 84-96): generic 8-iteration loop with branches. ~260 cycles/byte.
  Fix: use xtime() inline for multiply-by-2/3 (MixColumns only needs 0x02, 0x03).
- ghash_mul (line 251-282): bit-by-bit O(128). ~160 cycles/byte.
  Fix: PCLMULQDQ carry-less multiply = ~1 cycle/byte.

## 2. Ed25519 CertificateVerify (P1 fix: GS-S3-P1-001)

File: `sec/xsec/crypto/ed25519.c:953-1000+`
Called from: `sec/xsec/tls/tls13.c:830-918`

- ge_frombytes (point decode + field inversion): ~50K cycles
- SHA-512 (R || pub || msg): ~4K cycles
- Scalar reduction mod l: ~2K cycles
- Double scalar multiplication: ~500K cycles
- Total: ~550-600K cycles = ~175 us at 3.4 GHz

Verdict: ACCEPTABLE. One-time cost per TLS connection. Dominated by network RTT.

## 4. PAL-Aether Syscall Wiring (Sprint 3)

File: `pal/src/pal_aether.c:388-400` (_xk_syscall4)

- SYSCALL round-trip: ~250-400 cycles bare metal
- With Spectre mitigations (KPTI/retpoline): ~450-800 cycles
- At 3.4 GHz: ~75-235 ns per syscall

Affected functions: pal_timer_create/set/cancel, pal_thread_create/join,
pal_sg_build, pal_pages_alloc, pal_map_pages, pal_channel_create/send/recv.

Verdict: ACCEPTABLE. Standard syscall overhead. Monitor IPC hot paths.
Note: pal_time_now_ns correctly uses RDTSC (no syscall) -- good.
