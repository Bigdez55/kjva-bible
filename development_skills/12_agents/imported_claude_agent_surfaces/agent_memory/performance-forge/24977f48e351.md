## Boot Deep Analysis (2026-03-28)

**Why:** User measured 146-object boot in QEMU/TCG with per-step TSC cycle counts. Need sub-1ms kernel init on bare metal (i7-8565U @ 4.6 GHz).

**How to apply:** Use these findings as the definitive boot performance baseline. All future boot optimizations should reference this analysis. The priority order is: PMM bulk insert > CPUID LAPIC > serial removal > GENSD deferral > demand-paged BSS.

### Key Findings

1. **QEMU/TCG vs bare metal scaling: 657x** (7 MHz virtual TSC vs 4.6 GHz real)
2. **Projected bare-metal kernel init: ~17.2ms** (79M cycles / 4600 MHz)
3. **Top 3 bottlenecks consuming 77% of kernel init:**
   - Step 4 PMM: 7,222 us (42%) -- buddy allocator inserts pages one at a time (O(N/page_size))
   - Step 12 APIC+HW: 5,435 us (32%) -- 10ms HPET-timed LAPIC calibration delay
   - Step 14 cap-table: 3,913 us (23%) -- per-entry cap_create calls + serial I/O

4. **Serial I/O dominates every step**: 80-96% of each step's time is serial_early_puts/pal_console_printf. Removing serial alone saves ~3.5ms.

5. **BSS is 209MB, zeroed TWICE**: AetherBoot zeros full 209MB, entry.asm re-zeros first 4MB. AetherBoot zero costs ~8.4ms at 25 GB/s.

6. **GENSD top bottlenecks**: Shell (552ms), Network (485ms), Display (441ms), Compositor (242ms), Security KAT (213ms).

7. **Network + AI can be deferred** to post-desktop: saves 683ms off critical path.

### Optimization Projections

- PMM bulk buddy insert: 7,222 us -> 22 us (99.7% reduction)
- CPUID LAPIC frequency: eliminates 10ms calibration delay
- Serial ifdef: saves ~3,500 us across all steps
- **Projected post-optimization kernel init: ~795 us (sub-1ms ACHIEVED)**
- **Projected post-optimization total boot: ~250-550ms bare metal**
