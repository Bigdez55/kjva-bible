## Full-System Benchmark (2026-04-11)

**Why:** Complete performance characterization needed to prioritize optimization work and validate resource budgets.
**How to apply:** Use these numbers as baselines for any future optimization proposals. All numbers computed from source code + Intel Whiskey Lake architecture specs.

### Key Performance Numbers

- Syscall round-trip: ~40 ns null, ~80-150 ns typical (CVE-2012-0217 + MDS mitigated)
- kmalloc: ~25 ns best (freelist pop), ~300-500 ns cold (new slab from PMM)
- PMM alloc order-0: ~80-150 ns (buddy allocator)
- PMM free (with coalescing): O(N) buddy search per order -- needs bitmap fix
- Context switch: ~1-2 us typical, ~10-15 us with SMP IPI
- XMIND 3B tok/s: ~9.4 (AVX2), ~1.8 (scalar) on single core
- XMIND 3B memory: 2.53 GB (weights + KV@2048 + state)
- XMIND 13B memory: 10.6 GB (marginal fit in 16 GB)
- XMIND 70B: ~0.13-0.18 tok/s (NVMe bound, impractical)
- TCP throughput: ~250 MB/s (byte-by-byte ring -- P0 fix needed)
- AES-256-GCM (AES-NI): ~650-750 MB/s
- TLS 1.3 handshake: ~400-600 us (AES-NI + Ed25519)
- XSTORE B-tree lookup: ~500 ns (cached), ~40-50 us (NVMe miss)
- Boot to interactive: ~3.5-6.1 sec total (WITHIN 12s budget)
- Idle OS footprint: ~58 MB (without AI)
- AI active footprint: ~2.53 GB (3B @ 2048 ctx)

### TOP 10 Bottlenecks (Priority Order)

1. P0: TCP ring byte-by-byte copy (12x slower, 20 LOC fix)
2. P0: XMIND silu/softmax/rmsnorm not AVX2 vectorized
3. P1: Logit projection sequential O(128K x 3072)
4. P1: PMM free buddy search O(N) per order
5. P1: TCP cwnd capped at 16KB SNDBUF (limits WAN to ~13 Mbps)
6. P1: TCP TCB table 11.4 MB static (reduce to 32 TCBs = 768 KB)
7. P1: XMIND attention V accumulation not vectorized
8. P1: XSTORE WAL checkpoint threshold (48000) > ring size (4096) -- BUG
9. P2: Kernel heap single global lock (SMP contention)
10. P2: Software CRC32C in XSTORE (SSE4.2 CRC available but unused)

### Untapped Hardware: SSE4.2 CRC32C, AVX2 for silu/softmax/rmsnorm, GPU compute
