# Sprint 15 -- XJIT AVX2 Matmul Performance Audit

Date: 2026-03-07

## XJIT Current State (Sprint 6 baseline)
- 22 IR opcodes: NOP, MOV_RI, MOV_RR, ADD, SUB, IMUL, AND, OR, XOR, SHL, SHR,
  CMP, JMP, JEQ, JNE, JLT, JGT, CALL, RET, LOAD, STORE, LABEL, PHI
- 14 allocatable GP registers (no YMM/XMM), linear scan regalloc
- 64KB code buffer, 4096 IR instructions max, 256 vregs, 64 spill slots
- Integer-only: zero SIMD/FP support in codegen

## Sprint 15 Deliverables
- avx2_matmul.c: 593 LOC, compiles clean (clang -Werror -fsyntax-only)
- 7 new AVX2 instruction emitters: vmovups load/store, vfmadd231ps,
  vaddps, vmulps, vpxor, vbroadcastss
- Plus helper emitters: vzeroupper, vextractf128, vhaddps, vmovss, vaddps_128
- CPUID detection: AVX2 (leaf 7, EBX.5) + FMA3 (leaf 1, ECX.12)
- JIT matmul generator: xjit_emit_matmul_f32(fn, m, k, 1)
- 4-accumulator FMA inner loop (32 floats/iter)
- Horizontal reduction: vextractf128 + vaddps_128 + 2x vhaddps

## Performance Projections (i5-8265U Whiskey Lake @ 3.9 GHz)
- Scalar Q4_0 matmul: ~1.5 GFLOPS (tensor.c, includes dequant overhead)
- AVX2 float32 matmul: ~50 GFLOPS theoretical, ~25-30 GFLOPS sustained
- Speedup: 16-20x on dequantized float32 path
- Token/sec estimate: scalar ~2-3 tok/s, AVX2 ~10-15 tok/s (Llama 3.2 3B)

## Boot Chain Analysis (boot_chain.c, 740 LOC)
- 10 stages, strictly sequential
- Parallelizable pairs: storage+security, network+packages, display+compositor (deps met)
- Estimated savings: ~200-400ms from parallel stage execution
