## Sprint 40 XISC Native Layer Performance Audit (2026-03-20)

### Architecture
- 14 source files, ~1600 LOC freestanding C
- Global state: g_native (native_state_t) + g_mappings (512 entries)
- Handle table: 4096 entries * 48B = 192 KB
- Object store: 4096 entries * ~72B = 288 KB
- Audit ring: 65536 entries * 48B = 3 MB (dominates memory)
- Total static: ~3.5 MB (35% of 10 MB budget)

### Top 5 Hotspots
1. P0-PERF-01: native_object_lookup is O(N) linear scan (1-3 us worst case, target < 100 ns)
2. P0-PERF-02: native_handle_alloc is O(N) linear scan under spinlock (2-5 us, target < 500 ns)
3. P0-PERF-03: native_object_alloc is O(N) linear scan under spinlock (2-5 us, target < 500 ns)
4. P1-PERF-04: channel write/read uses byte-by-byte copy with per-byte modulo (60-120 us for 1KB)
5. P1-PERF-05: channel write has STACK-LOCAL spinlock (correctness bug, zero sync)

### Key Optimization Plan
- OPT-01: Object lookup -> store obj_index in handle entry (O(1), ~20 LOC)
- OPT-02: Handle alloc -> freelist (O(1), ~40 LOC)
- OPT-03: Object alloc -> freelist (O(1), ~40 LOC)
- OPT-04: Channel copy -> two-phase __builtin_memcpy (~15 LOC)
- OPT-05: Remove stack-local spinlock in channel_write (3 lines deleted)
- OPT-06: Audit ring 65536 -> 4096 entries (saves 2.88 MB)

### SLO Targets (bare metal)
- handle_close p99 < 100 ns
- handle_dup p99 < 200 ns
- channel_write(64B) p99 < 500 ns
- event_signal p99 < 100 ns
- clock_get p99 < 30 ns
- Channel throughput > 1M msg/sec (8B)
- Total static memory < 700 KB (after OPT-06)

### Deliverables
- ADR: docs/adr/ADR-S40-PERF-01-native-layer-performance.md
- Benchmark: xisc/benchmarks/bench_native_layer.c
- Total P0+P1 effort: ~120 LOC changes, predicted 10-100x improvement on hot paths

**Why:** The native layer is the syscall fast-path for all XKABI operations. Every kernel
object operation goes through handle_lookup + object_lookup. O(N) scans at 4096 capacity
are 10-30x over budget and will cause visible latency at scale.

**How to apply:** OPT-01 through OPT-05 must land before Sprint 40 integration testing.
OPT-06 can land independently. All optimizations are low-risk with the regression suite
in bench_native_layer.c providing correctness verification.

### IMPLEMENTATION STATUS: ALL 6 OPTs COMPLETE (2026-03-20)

All optimizations implemented and compilation-verified (14 files, 0 errors, -Werror).

**Files modified:**
- native_internal.h: obj_index + next fields in handle_entry, next_free in object,
  freelist_head in handle_table + state, audit ring 65536->4096,
  ring_lock removed from channel, native_object_lookup_fast() inline helper
- native_handles.c: O(1) freelist alloc/free, O(1) object lookup in close/dup/signal/revoke
- native_objects.c: O(1) freelist alloc/free, release pushes to freelist
- native_channel.c: spinlock removed from write+read, two-phase memcpy for both paths,
  O(1) object lookup via obj_index
- native_main.c: freelist initialization for both handles and objects
- native_vmo.c: native_object_lookup_fast() in all 4 callers
- native_event_timer.c: native_object_lookup_fast() in all 5 callers
- native_wait.c: native_object_lookup_fast() in 1 caller
- native_vmar.c: native_object_lookup_fast() in 1 caller
- bench_native_layer.c: updated memory layout analysis
