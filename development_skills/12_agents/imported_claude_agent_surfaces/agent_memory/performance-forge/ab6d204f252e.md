# Sprint 44 Performance Audit (2026-03-21)

## Scope
- 64 new TSX files: MissingApps(20), SystemInfra(16), SecurityPrivacy(8), DisplayFeatures(9), CoreApps(10), Accessibility(7)
- 1 new C test file: kernel/xenos/tests/phase2e_tests.c
- Total new LOC: 28,347 TypeScript + ~1,700 C

## P0 Issues (ALL FIXED)

### P0-REACT-01: Orphaned setInterval in NearbyShare.tsx (FIXED)
- sendFile() and handleAccept() created intervals outside useEffect
- Fix: Added transferIntervalsRef Set, cleanup on unmount
- File: compositor/genos-shell/src/renderer/components/DisplayFeatures/NearbyShare.tsx

### P0-REACT-02: Orphaned setTimeout in NetworkDiagnostics.tsx (FIXED)
- runTraceroute() fired 5 timeouts via forEach with no cleanup
- runDNS() fired orphaned setTimeout
- Fix: Added pendingTimeoutsRef Set, cleanup on unmount + tool switch
- File: compositor/genos-shell/src/renderer/components/SystemInfra/NetworkDiagnostics.tsx

### P0-REACT-03: Orphaned setTimeout in ScreenCast.tsx (FIXED)
- handleConnect() had untracked 1500ms setTimeout
- Fix: Added connectTimeoutRef, cleanup on unmount
- File: compositor/genos-shell/src/renderer/components/DisplayFeatures/ScreenCast.tsx

## P1 Issues (DOCUMENTED, NOT FIXED)

### P1-REACT-01: Zero React.memo across 64 components
- Highest impact: SystemMonitor.tsx, Clock.tsx (1s update cycles)
- Leaf components Sparkline, GaugeCard, TabBtn should be memoized

### P1-REACT-02: 2,056 inline style={{}} objects
- Worst: ColorProfileManager(66), InstallerUI(61), UserAccountManager(58)
- Creates GC pressure on every render

### P1-REACT-03: Clock.tsx dual 1-second intervals
- Parent Clock and child WorldClock both run independent setInterval(1000)
- Should consolidate to single parent timer

### P1-ANIM-01: backdrop-filter blur on 4 components
- ActivationLock blur(24px) full-screen: ~4-8ms GPU on UHD 620
- Acceptable for static overlay, would be P0 if animated

### P1-ANIM-02: Direct style.boxShadow mutation on focus/blur
- Pattern across ~40 files triggers reflow vs CSS :focus-visible

## P2 Issues (ADVISORY)
- Bundle: ~250KB minified (acceptable, but code-split recommended)
- XFRAME node pool: UNAFFECTED (TSX renders in Chromium DOM, not XFRAME)
- C test file BSS: 33.4KB total (reasonable for test mock)
- C stack allocations: all under 100 bytes (safe)
- C bounds checks: comprehensive, no issues found

## Why
Sprint 44 added 28K+ LOC of UI components. Timer cleanup bugs cause memory leaks in Electron on 16GB RAM. The pattern of orphaned setInterval/setTimeout in event handlers (not useEffect) is the most dangerous anti-pattern found.

## How to apply
Future UI component reviews must check: (1) every setInterval/setTimeout has a cleanup path on unmount, (2) intervals created in event handlers are tracked in refs, (3) leaf components with stable props should use React.memo.
