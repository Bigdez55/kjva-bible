## BSS Crisis Analysis (2026-03-29)

ELF PT_LOAD MemSiz = 0x11F0B000 = 286.2 MB. AetherBoot kern_span = 256 MB. EXCEEDED.

### Root Cause

XSTORE WAL ring: 65,536 entries * 4,384 bytes (inline key[256] + val[4096]) = 274 MB.
This single array is 95% of the BSS.

### Top BSS Contributors

1. XSTORE WAL ring: 274 MB -- store/xstore/include/xstore.h:86
2. TCP g_tcbs[128]: 11.5 MB -- net/xnet/transport/tcp.c:89,251
3. Font g_arena: 4.0 MB -- devices/desktop/display/xcomp/fonts.c:140
4. vDRAM tables: 2.5 MB -- kernel/xenos/mm/vdram/
5. Vector edges: 1.9 MB -- devices/desktop/display/xcomp/vector.c
6. XFRAME pools: 1.8 MB -- devices/desktop/ui/xframe/runtime/
7. ts_bridge file_buf: 1.0 MB -- devices/desktop/ui/xframe/runtime/ts_bridge.c:799
8. XISC native+linux: 1.7 MB -- xisc/runtime/

### Fix Plan (3 changes, -270 MB)

- OPT-1A: XSTORE_WAL_RING_SIZE 65536->4096 (-257 MB) + threshold 48000->3072
- OPT-2: TCP_TCB_MAX 128->32 (-8.6 MB)
- OPT-3: Font arena static->heap (-4.0 MB)

### XBLOB INDEX WARNING

XBLOB_INDEX_SLOTS = 1M * 56B = 56 MB. Currently caller-provided (pointer, not BSS).
But if any static allocation exists, it adds 56 MB. Reduce to 16K slots (896 KB).

**Why:** AetherBoot cannot load the kernel -- kern_span exceeded by 30 MB.
**How to apply:** Any time XSTORE/XBLOB/TCP constants are touched, verify BSS impact.
