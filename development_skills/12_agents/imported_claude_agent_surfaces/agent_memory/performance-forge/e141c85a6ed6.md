# Boot Speed Wave 1 — AetherBoot Optimizations (2026-03-27)

## OPT-1.1: CPUID-Based TSC Calibration (-100ms)
- **Before**: `BS->Stall(100000)` = 100ms blocking stall
- **After**: CPUID leaf 0x15 (crystal ratio, 0ms) -> leaf 0x16 (base freq MHz, 0ms) -> fallback 10ms stall
- **File**: `devices/desktop/aetherboot/src/aetherboot.c` function `measure_tsc_hz()`
- **i5-8265U**: Whiskey Lake supports both leaf 0x15 and 0x16, so 0ms path taken
- **Savings**: 100ms (CPUID path) or 90ms (fallback 10ms stall)

## OPT-1.2: Deferred Serial Output (-80-120ms)
- **Before**: ~47 blocking serial_puts() calls at 115200 baud (~87us/byte, ~2KB total = ~174ms)
- **After**: 39 calls routed to 4KB ring buffer `g_serial_defer_buf[]`, 17 FATAL/error calls stay blocking
- **Handoff**: `deferred_serial_phys` + `deferred_serial_len` added to `xenos_handoff_t` and `xenos_boot_info_t`
- **Kernel drain**: kernel reads buffer after `_serial_init_16550()` (Step 1 in kmain) -- NOT YET WIRED
- **Recovery**: `serial_defer_flush()` called before recovery mode entry to dump all buffered output
- **Files modified**: `aetherboot.c` (buffer + 39 call conversions), `xenos.h` (handoff struct)
- **Savings**: ~80-120ms (depending on how many info paths are taken per boot)

## OPT-1.3: rep movsq/stosq for ELF Copy (-5-15ms)
- **Before**: byte-at-a-time loops for BSS zero + PT_LOAD copy + BSS gap zero
- **After**: `rep stosq` for zeroing, `rep movsq` for copy, with tail-byte loops for alignment remainder
- **Three sites**: (1) full kern_span pre-zero, (2) per-segment file copy, (3) per-segment BSS gap zero
- **Throughput**: ~12 GB/s (rep stosq) vs ~1.5 GB/s (byte loop) on Whiskey Lake
- **Savings**: ~2.5ms for 4MB kernel span (pre-zero) + ~1-2ms per segment copy
