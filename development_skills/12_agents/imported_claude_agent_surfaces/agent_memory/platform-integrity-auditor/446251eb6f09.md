## CRITICAL Findings

### C1: vDRAM stdint.h includes violate freestanding contract
Files: vdram_pf.c:10-11, vdram_init.c:8-9, vdram_pf.h:10-11, vdram_advisory.h:10-11
Risk: Hosted headers may pull in libc symbols, break freestanding build on non-GCC toolchains
Fix: Replace `<stdint.h>/<stddef.h>` with `#include "../../../include/xenos.h"`

### C2: xkabi_cap_verify() always returns 0 — capability gate non-functional
File: main.c (cppcheck confirmed at lines 2330, 2511, 2519, 2533)
Risk: All syscalls requiring capability verification are unconditionally denied
Fix: Implement real capability verification in xkabi_capabilities.c

## HIGH Findings

### H1: xblob_get ABI mismatch
- boot_stubs.c:223 declares 6-param: `(void*, const void*, uint32_t, void*, uint32_t, uint32_t*)`
- supervisor.c:246 declares 5-param: `(void*, const uint8_t[32], void*, uint32_t, uint32_t*)`
- xblob.h:332 declares 5-param inline wrapping xblob_get_impl
Risk: Undefined behavior if wrong parameter count is passed at link boundary

### H2: supervisor_council_port() buffer overread
File: supervisor.c:189-207
The string matching reads exec[i+1]..exec[i+7] without verifying those indices are within bounds
Risk: Reads past exec[256] buffer if pattern chars appear near end of string

### H3: boot_stubs.c POSIX shims create implicit external dependency
File: boot_stubs.c:586-599 (mmap, munmap, epoll_create1, epoll_ctl, epoll_wait, open, close, read, ioctl)
These exist because xdisp/drm.c assumes hosted POSIX. Weak but still pollute symbol table.
Risk: Any translation unit that accidentally includes real POSIX headers would override these
