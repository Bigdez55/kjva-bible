## Critical Issues

### 1. CRITICAL: Buffer Overflow in native_vnode.c:58
**File:** `xisc/runtime/xkabi_native/native_vnode.c` (Lines 49-58)
**Issue:** Array bounds validation is logically incorrect. The condition checks `path_len > 4096` and returns error, BUT then unconditionally writes to `pbuf[path_len]` where `pbuf` is declared as `char pbuf[4096]`. If `path_len == 4096`, the write at line 58 accesses index 4096 (out of bounds).

**Root Cause:** Off-by-one in validation logic. The array has indices 0-4095 (4096 total), but the code allows `path_len == 4096`, then writes to `pbuf[4096]`.

**Risk:** Stack buffer overflow on vnode_open() calls where path_len == 4096. This corrupts the stack frame and creates a potential code execution vector.

**Fix:** Change condition on line 49 from `>` to `>=`:
```c
if (path_len >= 4096) {  // CRITICAL FIX: >= instead of >
    return XK_ERR_INVALID_ARGS;
}
```

### 2. HIGH: Incomplete Timeout Logic in native_channel.c:283-302
**File:** `xisc/runtime/xkabi_native/native_channel.c` (Lines 283-302)
**Issue:** Timeout measurement is non-functional. Variable `start_ns` is declared but never initialized with actual time. Variable `elapsed_ns` is hardcoded to 0 on line 298, making the timeout check at line 299 always false (deadcode).

**Root Cause:** PAL timer integration not completed. The comment on line 298 admits this: `/* Would need PAL timer for real measurement */`.

**Risk:** Any code calling `xk_channel_read_with_timeout()` will hang indefinitely because the timeout never triggers. This blocks all channel operations expecting timeout behavior.

**Impact:** Functional regression in IPC layer; violates xkabi contract for timeout semantics.

**Fix:** Implement actual timer measurement using PAL:
```c
int64_t start_ns = pal_timer_monotonic_ns();  // Get actual start time
while (1) {
    response->data_len = response->data_capacity;
    rc = xk_channel_read(ch, response->data, &response->data_len,
                         response->handles, &response->handle_count);
    if (rc == XK_OK) {
        return XK_OK;
    }

    if (rc != XK_ERR_SHOULD_WAIT) {
        return rc;
    }

    /* Check timeout (if specified) */
    if (timeout_ns > 0) {
        int64_t elapsed_ns = pal_timer_monotonic_ns() - start_ns;
        if (elapsed_ns > timeout_ns) {
            return XK_ERR_TIMED_OUT;
        }
    }

    /* Yield and retry */
    pal_thread_yield();
}
```

### 3. HIGH: Dead Code - Unused Variable in native_channel.c:283
**File:** `xisc/runtime/xkabi_native/native_channel.c` (Line 283)
**Issue:** Variable `start_ns` is initialized but never read. The loop never uses `start_ns` to measure elapsed time, making it dead code.

**Risk:** Misleads maintainers; suggests timeout logic exists when it doesn't.

**Fix:** Remove `int64_t start_ns = 0;` line and implement proper timer as shown above.

## Medium Issues

### 4. MEDIUM: Redundant Conditional Assignment in native_objects.c:57-59
**File:** `xisc/runtime/xkabi_native/native_objects.c` (Lines 57-59)
**Issue:** The condition `if (obj->backend.vmo.mapped_vaddr != 0)` is redundant. The assignment `obj->backend.vmo.mapped_vaddr = 0` happens regardless. The code can be simplified by removing the condition.

**Pattern:** This appears twice (lines 57 and 76).

**Impact:** Code clarity issue; no functional impact, but violates "simplicity" axiom.

**Fix:**
```c
// BEFORE
if (obj->backend.vmo.mapped_vaddr != 0) {
    obj->backend.vmo.mapped_vaddr = 0;
}

// AFTER
obj->backend.vmo.mapped_vaddr = 0;
```

### 5. MEDIUM: Const Qualifier Missing - Multiple Files
**Files Affected:**
- native_channel.c (lines 139, 210): `entry` should be `const *`
- native_event_timer.c (lines 73, 103, 134, 139, 209, 214, 237, 242): `he`, `obj` should be `const *`
- native_handles.c (lines 67, 94, 125, 200): `e` should be `const *`
- native_vmar.c (lines 148, 150, 228): `vmo_e`, `vmo_obj`, `entry` should be `const *`
- native_vmo.c (lines 122, 149, 177, 194, 196): `e`, `obj` should be `const *`
- native_wait.c (lines 48, 53): `he`, `obj` should be `const *`

**Issue:** These variables are read-only (result of lookup functions) but are declared as mutable pointers. Violates const-correctness.

**Impact:** Misses compiler optimization opportunities; violates API contract that lookups return read-only references.

**Example Fix (native_channel.c:139):**
```c
// BEFORE
native_handle_entry_t *entry = native_handle_lookup(ch);

// AFTER
const native_handle_entry_t *entry = native_handle_lookup(ch);
```
**NOTE:** This requires the `native_handle_lookup()` function signature to return `const *`. Check `native_internal.h` for function prototype and update it accordingly.

### 6. MEDIUM: Parameters Missing Const in native_vnode.c
**File:** `xisc/runtime/xkabi_native/native_vnode.c`
**Issues:**
- Line 81: `void *buf` should be `const void *buf` in `xk_vnode_read()`

**Impact:** API design violates const-correctness; unclear intent to callers.
