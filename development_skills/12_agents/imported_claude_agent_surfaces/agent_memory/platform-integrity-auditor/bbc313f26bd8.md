## 2026-04-24 — QEMU Replacement Status (EXHAUSTIVE DEEP DIVE)

### Verdict
GEN.OS has NO system-level QEMU replacement. The only emulator in tree is XEMU (user-mode Linux ABI compat, scoped narrowly). QEMU `qemu-system-*` is required for ALL boot testing.

### XEMU = USER-MODE only (devices/desktop/vm/xemu/, 2,093 LOC)
- xemu.h:9-10 explicitly: "It replaces QEMU for running legacy Linux binaries on GEN.OS without POSIX" — i.e. replaces qemu-USER-mode, NOT qemu-system
- 5 sources: elf_loader.c (293), decoder.c (263), rewriter.c (372), xjit_backend.c (376), xemu.c (496)
- Capabilities: ELF64 ET_EXEC + ET_DYN load, SYSCALL→INT3+NOP rewrite, 25-syscall Linux→XENOS gate, x86-64 interpreter (~15 instruction families: NOP, INT3, RET, PUSH/POP, MOV, ADD/SUB/CMP, XOR, JMP/Jcc/CALL), 256-entry block cache lifting RAX-only sequences
- NO device emulation (no virtio, no UART emulator, no block dev emulator, no network dev emulator)
- NO MMIO trap, NO BIOS/UEFI emu, NO multi-VM management
- Cannot boot xenos.elf

### XJIT qemu_backend.c (devices/desktop/jit/xjit/src/qemu_backend.c, 241 LOC)
- WRAPS `qemu-x86_64-static` user-mode via posix_spawn (XJIT_HOST_TEST mode)
- Comments line 7-8 + 15-16: "Sprint 31: stub backend ... for host-side testing. Production path: Sprint 32 wires the native CPU backend via XENOS XK_SYS_EXEC syscall (Sprint 19 implementation)"
- This is a host-side TEST harness — uses qemu, doesn't replace it

### XISC translator (xisc/runtime/translator_service/backends/)
- qemu_user_mode.py:18-19 explicit: "DEV/TEST ONLY — This backend is NOT included in production. Production uses the XKABI native layer. See ADR-001."
- native_exec.py uses subprocess.run + LD_PRELOAD libxkabi_shim.so for syscall interception
- xisc/spec/POLICIES.md Policy 3: "The translator (QEMU user-mode initially, custom JIT later)..."
- ADR-S39-01: Mode 1 (XENOS native) is target; Mode 2 (Linux shim) was scaffolding (deleted pal_linux.c)

### genvirt — XISC bytecode JIT, NOT a CPU emulator
- devices/console/genvirt/jit/xisc_llvm_jit.c (410 LOC) — LLVM C API JIT for XISC bytecode
- BUILD_STATUS.md: "explicitly excluded from CI compilation" — requires LLVM headers unavailable in kernel build
