## Phase 2e Vnode Syscall Audit (2026-03-21) — CRITICAL FINDINGS

**Status:** AUDIT COMPLETE — FAIL (readiness gate: NOT READY FOR PHASE 2F)

### Critical Blocking Issues

1. **Syscall 54 Routing Mismatch**
   - Kernel dispatcher (main.c:2984) routes 54 → `kern_vnode_ioctl()`
   - Native spec (native_vnode.c:146) defines 54 as `XK_SYS_VNODE_CLOSE`
   - **Result:** No vnode_close handler in kernel; FD table cannot be cleaned up
   - **Risk:** Resource exhaustion + DoS (64 FD limit per process)
   - **Fix:** Implement `kern_vnode_close()` using existing `_fd_free()`, move ioctl to syscall 55

2. **Missing Syscall 55 IOCTL Routing**
   - Native layer expects syscall 55 for ioctl
   - Kernel dispatcher has no case 55u
   - **Result:** Terminal control ops (TIOCGWINSZ, FIONREAD) not routed
   - **Fix:** Add case 55u dispatcher case

3. **No User Buffer VA Validation**
   - `kern_vnode_read()` / `kern_vnode_write()` apply HHDM offset without bounds check
   - Malicious code could pass kernel VA
   - **Fix:** Add `_is_user_va(va)` check (canonical address validation)
