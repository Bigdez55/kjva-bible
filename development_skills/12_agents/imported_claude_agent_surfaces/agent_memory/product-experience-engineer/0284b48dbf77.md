# Apex Trading UI — Pre-Implementation Baseline Audit
**Date:** 2026-02-26

## Critical Gaps Found in InvestPage (for Apex overhaul)

### WebSocket Data Is Stranded
- `LiveDataProvider` connects WebSocket at app mount and stores `marketData`, `positions`, `portfolio`, `recentOrders` in Redux (`state.websocket.*`)
- `InvestPage` reads ZERO WebSocket state — prices are REST-polled only (30s TTL from RTK Query)
- `LiveOrderUpdates.tsx` component exists and consumes `selectRecentOrders` from Redux, but is NOT rendered anywhere in InvestPage
- Channel format for price subscription: `subscribeToChannel('market_data:AAPL')` → dispatched via `useWebSocketControl().subscribeToSymbol(symbol)`
- **Apex must:** On Trade tab mount, subscribe to `market_data:{selectedSymbol}`, read `state.websocket.marketData[symbol]` for real-time price tick overlay on chart

### Portfolio Chart Is Synthetic
- Portfolio tab chart uses interpolated data: `startValue = total * 0.83`, `endValue = total`, with 2% noise
- NO real portfolio equity curve from backend — the API endpoint may or may not exist
- **Apex must:** Verify if `/trading/portfolio/history?mode=paper&period=1D` exists or needs to be created. Until then, the synthetic approach is what ships.

### Historical Data Includes OHLCV But UI Only Uses Close
- `HistoricalData.data[]` from `/market-enhanced/historical/{symbol}?period={period}` has `open, high, low, close, volume`
- `LineChart` mapping in InvestPage discards all except `close` → `{ value: point.close, timestamp: point.timestamp }`
- `lightweight-charts` CandlestickSeries needs `{ time, open, high, low, close }` — all fields are already in the backend response
- **Apex must:** Pass full OHLCV to new chart component when `lightweight-charts` is added

### Stock List Virtualization Is Absent
- `sortedStocks.map()` renders up to 500 `<button>` elements directly in the DOM
- Each cell has `backdropFilter: blur(12px)` — GPU compositing cost multiplied by 500 nodes = mobile lag
- **Apex must add:** `@tanstack/react-virtual` with `windowVirtualizer` for the stock grid in Discover tab
- Estimated DOM savings: 500 nodes → ~15 visible nodes at a time

### RTK Subscription Count Is High
- 13+ simultaneous RTK Query subscriptions when Invest tab is fully open
- P2 consolidation opportunity noted in project MEMORY: consolidate using `useGetBatchDataQuery` to reduce 10→7
- **Apex planning note:** New order book feature will add 1-2 more subscriptions; consider RTK Query normalization to avoid cache bloat

### CandlestickChart.tsx Still Exists
- `frontend/src/components/elson-v2/charts/CandlestickChart.tsx` — deprecated, archived, not imported
- CRA validates ALL `.ts`/`.tsx` files in `src/` — if it has TypeScript errors it WILL fail the build
- **Before Apex:** Verify `CandlestickChart.tsx` compiles clean, or add `// @ts-nocheck` at top
- Same applies to `PortfolioChart.tsx`

### Crypto Coverage Is 3 Symbols Hardcoded
- `CRYPTO_SYMBOLS = ['BTC', 'ETH', 'SOL']` in `useInvestData.ts` line 118
- `CRYPTO_NAMES = { BTC: 'Bitcoin', ETH: 'Ethereum', SOL: 'Solana' }` — also hardcoded
- `extractCryptoBaseSymbol()` strips USD pairs but only matches against these 3 known symbols
- **Apex must expand:** Add more symbols to array; update `CRYPTO_NAMES` map; consider fetching from `/market/assets?asset_type=crypto` instead

### Features That Are Toast-Only Placeholders (need real implementation or proper disabled state)
- Crypto Swap — `toast.info('Swap', 'coming soon')`
- Crypto Send — `toast.info('Send', 'coming soon')`
- Trending "See All" — `toast.info('Trending', 'coming soon')`
- Market News "More" — `toast.info('Market News', 'coming soon')`
- Crypto News "More" — `toast.info('Crypto News', 'coming soon')`

## Architecture Decisions Confirmed by Audit

### Tab Navigation Is State-Based, Not URL-Based
- All authenticated pages render at `/` — tab is `useState<TabId>` in `MobileLayout`
- Sub-tabs within InvestPage are also `useState<InvestSubTab>` — no URL sync
- Implication: no back button, no deep-link, no browser history for sub-tab state
- For Apex: consider adding `?tab=trade&symbol=AAPL` URL search params for shareable trade links

### Responsive Breakpoint Is 1024px
- `useIsDesktop()` in `MobileLayout` — boolean state, checked on resize
- Desktop: 260px sidebar + `maxWidth: 1200` centered main, `padding: 24px 32px 48px`
- Mobile: no sidebar, bottom nav, `padding: 20px 16px calc(80px + env(safe-area-inset-bottom))`
- InvestPage has NO internal breakpoints — same layout at all widths within main area
- Apex workstation grid: only render multi-column panels at desktop breakpoint

### InvestPage Renders StockProfilePage as Full Overlay
- `profileSymbol: string | null` state — when set, early return renders `<StockProfilePage>`
- Entry via Discover grid: `onClick={() => setProfileSymbol(s.symbol)}`
- Exit: `onBack` sets `profileSymbol = null`; `onTrade` sets null + calls `selectStock` + `setSub('trade')`
- This overlay pattern may conflict with Apex's multi-panel layout — consider converting to a slide-in drawer instead
