## Sprint 46 — GOP Display Pipeline Fallback (2026-03-30)

**Problem:** `open("/dev/dri/card0")` returns -ENOENT in the freestanding XENOS kernel (no VFS, no /dev tree). DRM cascade failure → black screen.

**Solution:** Three-layer fix.

### Files Modified

- `init/gensd/boot_chain.c`
- `devices/desktop/display/xcomp/compositor.c`

### Pattern: GOP Fallback in boot_chain.c xchain_display()

After `xdisp_init()` fails, call `gop_fb_init()` (declared `__attribute__((weak))` inline so CI builds link without gop_fb.c). Guard with null pointer check for the weak symbol. Return `PAL_OK` — display is degraded but functional. The compositor detects `gop_fb_is_initialized() == 1` on its own init.

```c
extern void gop_fb_init(...) __attribute__((weak));
if (gop_fb_init != (void *)0) { gop_fb_init(...); }
return PAL_OK;  /* boot continues */
```

### Pattern: GOP Fallback in xcomp_init()

After `xdisp_drm_init()` fails:
1. Check `gop_fb_is_initialized()` — fail hard if neither DRM nor GOP is available.
2. Get dimensions from `gop_fb_get_width/height()`.
3. Allocate PAL soft backbuf: `width × height × 4` bytes, rounded to pages.
4. Map it with `pal_map_pages()` — keep the mapping open for the compositor lifetime.
5. Set `g_use_gop_fallback = 1u`.
6. Use `goto xcomp_init_scene_graph` to skip DRM Steps 2-4 (no local var declarations between goto and label — safe in C11).

### Pattern: xcomp_frame() GOP branches

- **Phase B (buf map):** If GOP mode, use `g_gop_softbuf` directly with `back_stride_bytes = g_output_w * 4u` (tight packing). Skip `xdisp_buf_map()`.
- **Phase E (unmap):** If GOP mode, skip `xdisp_buf_unmap()` — persistent mapping, no per-frame remap.
- **Phase F (submit):** If GOP mode, call `gop_fb_blit(0, 0, w, h, back_pixels, w*4)`. Skip `xdisp_buf_submit()` and buffer swap (single buffer, direct HHDM target).

### Pattern: Fiber Trampolines for Blocking Subsystems

`xcomp_run()` and `xshell_start()` both enter infinite 60 Hz loops. Calling them inline blocks the boot chain permanently.

**Rule:** Never call a blocking loop inline in the boot chain. Always spawn via `pal_thread_create()`.

**C11 constraint:** No nested function definitions. Trampolines MUST be at file scope:

```c
/* Sprint 46 fiber trampolines — file scope, BEFORE the xchain_* functions */
extern void xcomp_run(void);

static void xcomp_run_fiber(void *arg) { (void)arg; xcomp_run(); }
static void xshell_fiber(void *arg)    { (void)arg; xshell_start(); }
```

xcomp_run → PAL_THREAD_REALTIME (display must not be preempted by shell work)
xshell_start → PAL_THREAD_NORMAL (yields to compositor)

### GOP soft backbuf sizing

```c
uint64_t bytes = (uint64_t)w * (uint64_t)h * 4u;
uint64_t pages = (bytes + PAL_PAGE_SIZE_4K - 1u) / PAL_PAGE_SIZE_4K;
```

For 1920×1200: 9,216,000 bytes → 2250 pages (~8.8 MB). PAL_HANDLE_INVALID = 0, so zero-initialized statics are correct for handle fields.

### Why: gop_fb.h include placement

Include `gop_fb.h` AFTER `compositor.h` in compositor.c so XCOMP types are already defined. Path from compositor.c: `"../../../kernel/xenos/drv/gop_fb.h"`.
