## Auth Duplication (authSlice + userSlice)

Both slices:
- Read `localStorage.getItem('token')` on init
- Have their own `login`, `register`, `logout` thunks
- Store `user`, `token` in state
- Write/remove token on login/logout

userSlice adds: `requires2FA`, `twoFactorEmail`, `currentUser` (duplicate of user), `portfolios`
authSlice has: `checkAuth` thunk, `isAuthenticated` flag

LoginPage uses userSlice/login. App.tsx uses authSlice/checkAuth.

Consolidation plan: merge into single authSlice. See audit doc.

## localStorage Token Scope

33 HIGH risk `localStorage.getItem('token')` calls across 26 files.
The cookie migration is NOT complete — every service file still reads from localStorage.
`credentials: 'include'` has NOT been added to RTK Query baseQuery calls yet.

Key files still reading raw localStorage token:
- `services/api.ts` (Axios instance — for refresh token too)
- `services/auth.ts` (authService, duplicate of api.ts)
- Every RTK Query service: marketDataApi, tradingApi, aiTradingApi, wealthAdvisoryApi, etc.
- Inline fetch calls in: AdvancedOrderEntry, TradeJournal, AlertManager

## WASM Gap — New Rust Modules Have No TS Wrapper

The `wasm.ts` interface ONLY covers: calculator, charts, risk modules.
The following Rust modules were added in sprint but have NO TypeScript wrapper and NO JS fallbacks:
- `options.rs` — Black-Scholes, Greeks, IV solver
- `statistics.rs` — RSI, MACD, ATR, ADX, Bollinger Bands
- `portfolio.rs` — MVO, efficient frontier, correlation matrix
- `backtest.rs` — backtesting engine

If WASM fails to load (which it does in dev without built .wasm binary), calls to these
functions fail silently with no fallback.

## AdvancedOrderEntry — No Paper/Live Warning

The AdvancedOrderEntry component has no knowledge of trading mode.
No paper badge, no live warning in confirm dialog.
`paperMode: boolean` prop is not in AdvancedOrderEntryProps.
This is a HIGH severity safety issue for live trading users.
