## Form Validation Patterns (Phase 2 P1 — 2026-02-21)
- **Phone:** `/^\+?1?\d{10}$/` — 10 digits, optional +1 prefix
- **ZIP:** `/^\d{5}(-\d{4})?$/` — 5 or 9 digits
- **Beneficiary percentage:** 1-100%, total per type (primary/contingent) ≤ 100%
- **Pattern:** Validate before API call, show `toast.error()`, `return` early
- **Phone normalization:** Strip non-digits before sending to API: `phone.replace(/\D/g, '')`
- **Beneficiary logic:** Calculate `existingTotal` by filtering `beneficiaries` by type, sum `percent`, check `existingTotal + newPercent <= 100`

## Search Debouncing Pattern (Phase 2 P1 — 2026-02-21)
- **Use case:** Prevent API spam when user types search query
- **Implementation:** `useRef<NodeJS.Timeout | null>` + `setTimeout` for debouncing
