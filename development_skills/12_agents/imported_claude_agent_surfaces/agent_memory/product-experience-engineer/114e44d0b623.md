## P1 Defects

### P1-UX-001: prefers-reduced-motion missing from ALL Orange Suite CSS
- `orange/go-notes/src/styles.css` — shimmer animation has no reduced-motion override
- `orange/go-mail/src/styles.css` — no @media block at all
- go-calendar, go-drive — no CSS files at all (inline styles only, no override possible)
- Shell has global override at styles.css:424-434 — but Orange Suite apps are separate Electron windows

### P1-UX-002: color-scheme: dark missing from 5 app surfaces
- Missing from: go-docs/src/styles.css, go-calendar (no CSS), go-drive (no CSS), go-sheets (no CSS), go-slides (no CSS)
- Result: Native form controls (inputs, selects, scrollbars) render in OS default (white in light mode)

### P1-UX-003: Button.tsx loading spinner reduced-motion override is a no-op
- File: `design-system/src/components/Button.tsx:99-102`
- The @media block duplicates `transform: rotate(360deg)` — spinner still rotates

### P1-UX-006: companion/src/styles.css missing prefers-reduced-motion for breathe/thinking-pulse/celebrate

## P2 Defects

### P2-UX-001: go-sheets, go-slides use wrong font stack
- `go-sheets/src/main.tsx:19`, `go-slides/src/main.tsx:16`: `"Noto Sans, sans-serif"`
- Correct: `typography.families.sans` = `"Inter", "Noto Sans", sans-serif`

### P2-UX-002: Browser tab strip lacks roving tabindex
- `browser/genesys-browser/src/renderer/App.tsx:179-203`
- Every tab button is independently in Tab sequence — poor keyboard UX with many tabs

### P2-UX-003: Dock hover transform bypasses CSS reduced-motion override
- `Dock.tsx:28-33` — inline `style.transform` assigned via JS
- CSS `@media (prefers-reduced-motion: reduce)` cannot suppress inline style changes
- Fix: Check `window.matchMedia('(prefers-reduced-motion: reduce)').matches` before applying

### P2-UX-004: LockScreen password input has aria-label but no visible label element
- `LockScreen.tsx:66-86` — `aria-label="Unlock password"` overrides any visible label
- Username shown as a `<p>` (not a form label)

## Surfaces with NO prefers-reduced-motion (to fix)

Add to each:
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

Files needing this:
- orange/go-notes/src/styles.css
- orange/go-mail/src/styles.css

## What is genuinely COMPLIANT (do not regress)

Shell:
- Global prefers-reduced-motion blanket override (styles.css:424-434)
- launcher-item-in reduced-motion fade override (styles.css:222-227)
- Skip-to-content link (App.tsx:99-101 + styles.css:61-87)
- Global :focus-visible ring (styles.css:102-105)
- All ARIA roles on shell components (header, nav, dialog, menu, menuitem, alert, status)
- Roving tabindex in ContextMenu
- Focus trap in Launcher and Dialog
- Error pattern: role="alert" + aria-live="assertive" + aria-atomic="true" on all error messages
- Battery danger color + aria-label both updated when level < 20%
- ThermalBadge: role="alert" for critical, role="status" otherwise
- Tablet mode expands touch targets to 56x56px via .tablet-mode .touch-target
- color-scheme: dark on shell, go-notes, go-mail, browser, companion

Design system:
- Input.tsx: useId, aria-invalid, aria-describedby linking to error + helper
- Dialog.tsx: full focus trap + Esc + role="dialog" + aria-modal + aria-labelledby
- Toast.tsx: role/aria-live variant-aware, prefers-reduced-motion override
- Tooltip.tsx: WCAG 1.4.13 compliant hover+focus, Esc dismissal, prefers-reduced-motion

Orange Suite (compliant apps):
- go-notes: role="toolbar" + aria-pressed on all toggle buttons + skeleton loader + aria-live confirmations
- go-mail: three-panel aria-labels, status toast, keyboard shortcuts
- go-calendar: role="grid" + role="gridcell" + role="columnheader", focus trap on event dialog
- go-drive: share modal backdrop click, file list aria-label
