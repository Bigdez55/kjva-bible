# Orange Suite — UX Patterns & Component Conventions

## Proposed Token File Location
`orange/shared/tokens.css` — import in each app's main CSS file.

## Correct Toggle Button Pattern (aria-pressed)
All view switchers, toolbar format buttons, grid/list toggles:
```tsx
<button
  aria-pressed={isActive}
  className={isActive ? "btn btn--active" : "btn"}
  onClick={handleToggle}
>
  Label
</button>
```

## Correct Modal Pattern (focus trap + Esc + role)
```tsx
// useEffect auto-focuses first focusable element on open
// onKeyDown on outer overlay handles Esc
<div
  role="dialog"
  aria-modal="true"
  aria-labelledby="dialog-title"
  style={{ position: "fixed", inset: 0, ... }}
  onKeyDown={(e) => { if (e.key === "Escape") onClose(); }}
>
  <div style={{ ... }}>
    <h3 id="dialog-title">...</h3>
    ...
    <button onClick={onClose}>Cancel</button>
  </div>
</div>
```

## Correct Toolbar Pattern (role + keyboard nav)
```tsx
<div
  role="toolbar"
  aria-label="Text formatting"
  onKeyDown={(e) => {
    const btns = Array.from(e.currentTarget.querySelectorAll("button"));
    const idx = btns.indexOf(document.activeElement as HTMLButtonElement);
    if (e.key === "ArrowRight") btns[(idx + 1) % btns.length]?.focus();
    if (e.key === "ArrowLeft") btns[(idx - 1 + btns.length) % btns.length]?.focus();
  }}
>
```

## Correct Skeleton Loader Pattern
Mirror the layout of the expected content, not a generic spinner:
```tsx
if (!dataReady) {
  return (
    <main aria-busy="true" aria-label="Loading [context]">
      <div className="skeleton skeleton--[shape]" />
    </main>
  );
}
```
CSS:
```css
.skeleton {
  background: linear-gradient(90deg, #1b242d 25%, #2e3944 50%, #1b242d 75%);
  background-size: 200% 100%;
  animation: shimmer 1.4s infinite;
  border-radius: 6px;
}
@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

## Correct useCallback + useEffect reload pattern
```tsx
const reload = useCallback(async (): Promise<void> => {
  setSyncStatus("syncing");
  const data = await window.someApi.list(currentParam);
  // process data...
  setSyncStatus("synced");
}, [currentParam]);

useEffect(() => { void reload(); }, [reload]);
```

## Known Bug Patterns to Watch
1. `btoa()` on binary file content crashes for non-Latin-1 bytes — IPC must return base64 strings
2. Download buttons using shared `preview` state download wrong file when multiple previewed
3. `prompt()` is synchronous and freezes Electron renderer — always replace with modal
4. TipTap toolbar `min-height: 32px` overrides global 44px — explicitly re-declare 44px in toolbar rule
5. `persist(allNotes)` writes every note when only one changed — only write the mutated note
6. Export as `.md` when content is TipTap HTML — convert with turndown or use `.html` extension
7. TipTap `onUpdate` callback `{ editor: ed }` causes TS7031 implicit any — type as `{ editor: import("@tiptap/core").Editor }`
8. Per-item async button states require `Map<id, state>` not a single boolean — prevents state collisions
9. Filename injection via note title in export — sanitize with `.replace(/[/\\:*?"<>|]/g, "_").slice(0, 128)`

## Correct Per-Item Async State Pattern (download, delete, upload)
```tsx
const [itemState, setItemState] = useState<Map<string, "idle" | "loading" | "error">>(new Map());
const getState = (id: string) => itemState.get(id) ?? "idle";
const setState = (id: string, s: "idle" | "loading" | "error") =>
  setItemState(prev => new Map(prev).set(id, s));

// Usage:
disabled={getState(item.id) === "loading"}
aria-busy={getState(item.id) === "loading"}
```

## Correct Error Alert Region Pattern
```tsx
// Place this container where errors can appear — always render it, never conditionally render the container
<div role="alert" aria-live="assertive" aria-atomic="true" style={{ minHeight: "1.375rem" }}>
  {error && <p style={{ color: "#f47171", margin: 0, fontSize: "0.875rem" }}>{error}</p>}
</div>
```
If you conditionally render the entire container, the live region will not be announced on first appearance.

## Correct Focus Management Pattern for Modals + Panels
```tsx
const triggerRef = useRef<HTMLButtonElement>(null);
const firstInputRef = useRef<HTMLInputElement>(null);

useEffect(() => {
  if (isOpen) {
    setTimeout(() => firstInputRef.current?.focus(), 0);
  } else {
    triggerRef.current?.focus();
  }
}, [isOpen]);
```

## Empty State Requirements
Every list must handle the empty case with:
- A meaningful headline (not "No items")
- Supporting copy explaining why it is empty OR what to do
- A primary CTA button pointing to the creation flow

## Focus Management
- After modal opens: move focus to first interactive element inside modal
- After modal closes: return focus to the button that triggered the modal (store `triggerRef`)
- After selecting a note in GO Notes: call `editor.commands.focus()` after `setContent()`
- After creating a new note: focus the title input

## TopBar Kernel-State Badge Pattern (Movement 4.4)

New bridge daemons (entropy, mesh, vDRAM) write to /run/genos/*.json, aggregated into
system-state.json by dbus_state_bridge.py as entropyState / meshState / vdramState.

Pattern for adding a new TopBar status indicator:
1. Add interface to `compositor/genos-shell/src/renderer/types.ts` AND duplicate locally in `electron/preload.ts`
2. Add optional field to `SystemState` in BOTH types.ts AND preload.ts
3. Add optional field to FALLBACK_STATE in `electron/main.ts` (as `undefined as Record<string,unknown>|undefined`)
4. Add badge component to `compositor/genos-shell/src/renderer/components/TopBar.tsx`
5. Wire badge into status-cluster JSX after ThermalBadge
6. Run `cd compositor/genos-shell && npx tsc --noEmit` — must be zero errors

Badge component invariants:
- Props type: `{ fieldName: StateType | undefined }` — return null when undefined
- role="status" for informational, role="alert" for error/offline states
- aria-label with human-readable text including key value
- minHeight: "24px" — compact, not a touch target (status indicator, not interactive button)
- Hover tooltip: `useState(false)` + `useRef<ReturnType<typeof setTimeout>>` + 120ms close delay
- tooltip: position absolute, top: "calc(100% + 6px)", right: 0, zIndex: 9999
- Use colors.secondary (blue) for IB/fast, colors.success (green) for healthy/RoCE,
  colors.warning (amber) for degraded, colors.danger (red) for offline/overflow
- Background: `accentColor + "26"` (15% opacity) — matches ThermalBadge convention
- Border: `1px solid accentColor` — matches ThermalBadge convention
- Pulsing animation (thermal-pulse keyframe — already in styles.css) for error/offline states
- All optional chaining on nested fields (?.) since bridges may not be running

EntropyBadge: shows current_routing (IB/RoCE/UET), hover reveals per-protocol classification_counts + throughput_bps
MeshBadge: shows "Mesh: N" where N=peer_count; severity=healthy/degraded/offline from peers[].health values
VdramBadge: shows proportional tier bar (T0–T3 colored segments) + total MB; hover shows tier breakdown + migration stats

preload.ts CANNOT import from renderer src/ — types must be duplicated verbatim locally.

```
orange/
  go-[app]/
    electron/         — main process, preload, IPC handlers
    src/

```
