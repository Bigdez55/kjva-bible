# Sprint 3 API Ergonomics Review (2026-03-05)

Sources: sec/xsec/include/xsec.h, net/xnet/include/xnet.h, pkg/xpkg/include/xpkg.h

## XSEC — Critical Findings

### F1 — xsec_tls_ctx_t stack size (CRITICAL UX / safety hazard)
sizeof(xsec_tls_ctx_t) ≈ 17,400 bytes due to inline record_buf[16640].
No SIZE WARNING in the header. A kernel developer declaring this on the stack causes
instant stack overflow in contexts where stack depth is 8–16 KB.
FIX: Add SIZE WARNING comment at typedef. Document: declare in BSS/static/heap region only.

### F2 — hostname field invisible from xsec_tls_init (CRITICAL — silent security bypass)
xsec_tls_init() has no hostname parameter. ctx->hostname (line 363) is added as a P1 fix
but is undocumented in the init function. A caller who skips manual hostname assignment
will cause xsec_x509_verify_hostname to compare against empty string → vacuous match
→ complete bypass of hostname validation with no compile or runtime warning.
FIX: Add xsec_tls_set_hostname(ctx, hostname) helper + document required 3-step call order:
  1. xsec_tls_init(ctx, transport, true)
  2. xsec_tls_set_hostname(ctx, "hostname")  ← REQUIRED for client
  3. xsec_tls_handshake(ctx)
Returning XSEC_ERR_STATE if xsec_tls_handshake is called on a client with empty hostname
would make the omission detectable at runtime.

### F3 — xsec_audit_init() call order undocumented (MEDIUM)
xsec_g_audit extern is declared but no doc states "call xsec_audit_init() before any XSEC
function." SHA-256/AES/TLS internals call xsec_audit_log() which writes to uninitialized
ring buffer with uninitialized spinlock.
FIX: Add init-order block to file header; add comment at extern.

### F4 — XSEC_ERR_TLS_HANDSHAKE (-7) too coarse + alignment gap (LOW)
Comment says "TLS handshake failed" — does not distinguish key-exchange failure from
cert failure (XSEC_ERR_CERT_INVALID) or alert failure (XSEC_ERR_TLS_ALERT).
Hostname mismatch inside handshake should surface as XSEC_ERR_CERT_INVALID not
XSEC_ERR_TLS_HANDSHAKE — callers cannot distinguish root causes.
Column alignment also breaks at -7 (cosmetic — added post-facto).
FIX: Comment "Key exchange failure (check cert/alert codes first)." Map hostname
mismatch path to XSEC_ERR_CERT_INVALID in handshake implementation.

### F5 — xsec_x509_verify_hostname API shape (DESIGN)
Current: xsec_x509_verify_hostname(const xsec_x509_cert_t *cert, const char *hostname)
Verdict: CORRECT as standalone predicate. WRONG as the only hostname API — callers have
no hook into the in-progress handshake to customize behavior or receive the cert.
FIX: Keep standalone predicate. Add xsec_tls_set_hostname() as the TLS-integrated path.
Do NOT replace standalone with context-taking variant — testability as pure predicate is valuable.

## XNET — Findings

### F6 — xnet_tick() documentation critically insufficient (HIGH)
Current: `void xnet_tick(void);    /* Periodic maintenance — call at ~1 Hz */`
Drives: ARP expiry, DNS cache expiry, TCP retransmit/keepalive, mDNS, gossip detection.
Failing to call it: ARP/DNS stale, TCP connections timeout, gossip freezes.
NOT reentrant — no doc. No consequence of missing documented.
FIX: Expand to 6-line comment documenting what it drives, consequences of omission,
non-reentrancy requirement.

### F7 — XNET_IPV4 macro byte-order semantics silent (HIGH)
XNET_IPV4(192,168,1,1) produces 0xC0A80101 in host byte order.
xnet_sockaddr_in_t.addr is documented as "network byte order."
On little-endian x86: passing XNET_IPV4() directly to sockaddr.addr is wrong unless
XNET internally applies xnet_htonl() when reading addr.
XNET_IPV4_LOOPBACK = 0x7F000001U — consistent with macro but silent on byte order.
FIX: Either wrap macro with xnet_htonl() OR document "produces host-byte-order; call
xnet_htonl() before placing in sockaddr." Current silence causes class of addressing bugs.

### F8 — Mixed int / xnet_status_t return convention across §23 and §24 (MEDIUM)
BSD socket layer (§24): returns int, -1 on error (POSIX compatible — correct).
Protocol APIs (§22-23): returns xnet_status_t (XNET_ERR_* values, -1 through -20).
A developer writing `if (xnet_tcp_connect(...) == -1)` misses XNET_ERR_CONNREFUSED (-10).
FIX: Add section boundary comment in §24 header block documenting the convention split:
"§24 socket functions return POSIX-style int (-1 on error). All other XNET functions
return xnet_status_t (XNET_ERR_* on error). Do not mix in the same error-handling path."

## XPKG — Findings

### F9 — xpkg_install buffer lifetime undocumented (MEDIUM)
xpkg_install(archive_buf, archive_len, config) — ownership/lifetime of archive_buf
not documented. xsec_x509_cert_t has zero-copy pointers into caller buffers (tbs_data,
der_data). If XPKG follows same pattern for file extraction, caller must keep buffer live.
FIX: Add lifetime comment: "archive_buf must remain valid for duration of install. XPKG
does not retain any pointer after return." If XPKG does retain pointers, document prominently.

### F10 — xpkg_index_entry_t.dep_names is CSV string breaking typed-array convention (MEDIUM)
All other dependency relationships use xpkg_dep_t[] with dep_count. Index entries use
char dep_names[256] — comma-separated, no version constraints, no type safety.
Caller must write a CSV parser to enumerate index dependencies. Cannot express version
ranges that xpkg_dep_t already supports.
FIX: Replace dep_names[256] with dep_names[XPKG_MAX_DEPS_PER_PKG][64] + dep_count uint32_t.
Size delta: 16*64=1024 bytes vs 256 bytes per entry — acceptable for 512-entry index.

## Priority Action Summary

P0 — xsec_tls_set_hostname() must exist before any integration test of TLS client code.
     Without it, hostname validation is silently bypassable with no compile or test signal.

P1 — xsec_tls_ctx_t SIZE WARNING comment. First kernel developer to declare this on the
     stack will hit a crash with no useful error message.

P1 — xnet_tick() documentation expansion. Missing ticks cause time-delayed failures
     (ARP staleness, TCP timeouts) that are extremely hard to attribute.

P2 — XNET_IPV4 macro byte-order clarification. Class of silent addressing bugs.
P2 — xpkg_index_entry_t.dep_names CSV → typed array replacement.
P3 — Audit init order, mixed return convention comment, buffer lifetime docs.

## Positive Patterns to Preserve

- §12 XSEC utility block (xsec_ct_eq, xsec_secure_zero) — excellent placement, correct
  volatile annotation, "NOT constant-time" warning on xsec_memcmp.
- xsec_tls_state_t enum — makes state-machine debugging tractable.
- BSD socket layer in §24 — near-identical to POSIX, zero learning curve.
- 9-gate refusal machine enum + xpkg_gate_check_t.detail[128] — structured actionable failure.
- xpkg_version_cmp / xpkg_version_in_range as inline header functions — consistent with
  xsec §12 utility pattern. Codify as GEN.OS C library convention: utility functions
  that operate purely on structs defined in the same header belong in the header as inlines.
- xnet_gossip_node_t with 128-bit UUID + incarnation counter — correct SWIM design.
- __attribute__((packed)) applied only to wire-format structs (§5-9), not in-memory structs.
