## CoreApps Suite (Sprint 43+, 2026-03-21)

All 9 core OS apps live in `compositor/genos-shell/src/renderer/components/CoreApps/`:
Calculator, Camera, Clock, Contacts, Maps, Music, SystemMonitor, Video, Weather.

Each is a self-contained single-file TSX component using only `@ds/tokens` and local state.
Launched from App.tsx via `activeCoreApp` state — IDs prefixed `core-` (e.g. `core-calculator`).
`CoreAppWindow` component in App.tsx wraps every app in a modal overlay with title bar + Esc dismiss.
Pre-existing TS errors in AppStore.tsx/CouncilDashboard.tsx/SystemDialogs are NOT from CoreApps work.
`zOrder` and `shadows` are both exported from `@ds/tokens` — import them as needed.

## Settings Panels (Sprint 40)

- All TSX panels: `compositor/genos-shell/src/renderer/components/Settings/`
- Registry: `Settings/index.tsx` — CATEGORIES array (id, label, icon, keywords[], component)
- Primitives: `SettingsPrimitives.tsx` — SettingsSectionCard, SettingsRow, SettingsToggle, SettingsSlider, SettingsSelect, SettingsRadioGroup, SettingsButton
  - NOTE: SettingsToggle/Slider/Select/RadioGroup/Button export as `export const X = React.memo(...)` NOT `export function X`
- Sprint 40 new panels: SettingsPower.tsx, SettingsHotspot.tsx (both registered in index.tsx)

### Settings Panel Registry Pattern
- Add import to index.tsx: `import { SettingsX } from "./SettingsX";`
- Add entry to CATEGORIES array: `{ id: "x", label: "X", icon: "...", keywords: [...], component: SettingsX }`
- All controls delegate to SettingsPrimitives — never raw HTML inputs in panel files

## Persistent C-layer Architecture Facts

- Orange apps: `apps/orange/{notes,calendar,mail,drive}.c` — XFRAME-native C, zero Electron
- xshell_tick(): __attribute__((unused)) — never registered — notifications/animations dead
- Greeter input fields: plain BOX nodes, not text_input.c widget — no keyboard routing
- Greeter login_btn: gr_on_login_click() exists but never registered as click handler
- XFRAME has NO Tab-order API — keyboard nav impossible without xframe_node_set_tab_stop()
- xframe_app_t has both surface_id (uint32_t pool index) AND surface (void* pointer)
- g_sw_fb[1920*1200] lives in BSS — 8.8MB. Dead code once FIX-01/03 applied.

## Key Shell Files

- XSHELL: `shell/xshell/src/` (xshell.c, greeter.c, topbar.c, dock.c, launcher.c, lockscreen.c)
- XSHELL header: `shell/xshell/include/xshell.h`
- XFRAME runtime: `ui/xframe/runtime/` (main.c, paint.c, layout.c, event.c, ts_bridge.c, xframe.h)
- XCOMP compositor: `display/xcomp/compositor.c` + `display/xdisp/drm.c` + `display/xdisp/gpu_buf.c`
- Boot chain: `init/gensd/boot_chain.c`
- Design tokens (C): `ui/xframe/runtime/design_tokens.h` + XTOKEN_COLOR_* constants
