## Key Findings (Sprint 5 Audit)

### Security — CRITICAL

- S5-SEC-001 (P0): lockscreen.c PIN uses XOR-accumulation stub, NOT XSEC SHA-256.
  Comment explicitly says "Replace with XSEC SHA-256 in Sprint 6."
  Constant-time comparison (ls_hash_equal) IS present and correct (XOR-accumulate all bytes).
  Attack surface: XOR with fixed 6-byte key is trivially reversible. Must be replaced Sprint 6.

### Greeter — OOBE Missing

- S5-UX-001 (P1): greeter.c has login form (user/pass/session) but NO OOBE/first-run wizard.
  First-boot detection, account creation, timezone/locale selection all absent.
  This is a login greeter, not an OOBE flow. Dedicate a separate component in Sprint 6.

### Context Menu — Click Events Unwired

- S5-UX-002 (P1): context_menu.c cm_item_click is defined and click contexts are wired,
  but BOX nodes have no XFRAME event API in Sprint 5 — click callbacks are NOT connected.
  Comment at line 129: "no XFRAME event API on BOX nodes in current sprint."
  Menu items are display-only; keyboard/click activation is non-functional.

### ts_bridge — G-02 STILL OPEN

- S5-GAP-G02: ts_bridge.c is a well-specified stub. QuickJS NOT linked (#ifdef QUICKJS_AVAILABLE).
  The API surface (createElement, destroyElement, appendChild, setStyle, setLayout, setText,
  addEventListener, removeEventListener) is defined and ready for wiring.
  All bodies are documented no-ops pending Sprint 4 QuickJS linkage.
  G-02 is UNRESOLVED as of Sprint 5.

### editor.c — G-04 STILL OPEN (partial)

### text_input.c — Fuzz Safety

- Null byte: xwidget_input_on_char rejects c < 0x20 || c > 0x7E. Null byte (0x00) rejected. Safe.
- Unicode/emoji: Documented limitation — ASCII + Latin-1 only for cursor movement in Sprint 2.
  Multi-byte UTF-8 grapheme cluster walk deferred to Sprint 3. Buffer stores raw bytes (512B).
  Risk: multi-byte UTF-8 chars accepted as byte sequence but cursor will split at byte boundaries.
  Sprint 3 must add grapheme cluster boundary detection.
