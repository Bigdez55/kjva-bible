## Sprint 43 Screen Reader TTS Integration (2026-03-21)

**Why:** WCAG AAA certification requires end-to-end screen reader pipeline: focus moves
and widget state changes must trigger spoken announcements via the TTS engine.

**How to apply:** Use this as the reference for any future a11y work on announcement
dispatch, settings persistence, or widget hook patterns.

### Screen Reader API (a11y.h)
- `xa11y_announce_text(text, priority)` — A11Y_PRIORITY_LOW/NORMAL/INTERRUPT
- `xa11y_announce_widget_focused(widget, custom_label)` — dedup on same node pointer
- `xa11y_announce_button_pressed(button_label)` — "Pressed: " prefix, polite
- `xa11y_announce_value_changed(widget_name, old_val, new_val)` — "name: new_val" in FULL mode
- `xa11y_announce_error(message)` — "Error: " prefix, ASSERTIVE
- `xa11y_announce_success(message)` — "Done: " prefix, polite
- `xa11y_announce_modal_opened(title)` — "Dialog: " prefix, ASSERTIVE, resets dedup
- `xa11y_announce_modal_closed()` — "Dialog closed.", polite, resets dedup
- `xa11y_set/get_screen_reader_enabled(int)` — global on/off toggle
- `xa11y_set/get_screen_reader_mode(int)` — A11Y_MODE_FULL(0)/BRIEF(1)/OFF(2)
- `xa11y_dequeue_announcement()` — compat shim returning `const char *` for tts_engine.c

### TTS Integration Architecture
- `a11y.c` enqueues text into `g_announce_queue[16]` ring buffer
- After unlock, `xa11y_announce()` calls `xtts_process_a11y_queue()` via weak extern
- `xtts_process_a11y_queue()` in tts_engine.c calls `xa11y_dequeue_announcement()` (old API)
- `xa11y_dequeue_announcement()` is a shim in a11y.c wrapping `xa11y_announce_dequeue()`
- The dequeue shim uses a static buffer `g_dequeue_compat_buf[256]` — not thread-safe across callers

### Settings Persistence (XSTORE keys)
- `settings:a11y:sr_enabled` — uint32 LE (0=off, 1=on)
- `settings:a11y:sr_mode` — uint32 LE (0=full, 1=brief, 2=off)
- `settings:a11y:tts_voice` — uint32 LE (xtts_voice_t cast)
- `settings:a11y:tts_speed` — uint32 LE (WPM, 80-300)
- `settings:a11y:tts_volume` — uint32 LE (0-100)

### Widget Announcement Conventions
- `button.c`: only calls `xa11y_announce_text()` directly (no xframe_node_t pointer on widget struct)
- `text_input.c`: announces per-keystroke (last char or "dot" in password mode) to avoid reading full buffer
- `dialog.c`: show() is ASSERTIVE (interrupts), hide() is polite; both reset `g_last_announced_node`
- Dedup: `g_last_announced_node` static pointer — suppress if same node announces twice consecutively

### C Test Conventions (test_a11y_integration.c)
- Each TC: drain queue with `while (xa11y_announce_dequeue(...))` before asserting
- Stubs: pal_spin_lock/unlock as no-ops; pal_console_printf as no-op; xtts_process_a11y_queue as no-op
- All 16 tests pass under clang-18 -fsyntax-only; main() returns 0 on all pass

### Python Test Conventions (test_a11y_e2e.py)
- ruff-clean: no unused imports; use `random.SystemRandom()` not `random.Random(hex_seed)` (S311)
- 108 assertions, 11 categories: Smoke(12) Functional(14) Integration(12) Regression(10) Load(8) Stress(8) Security(10) UI(8) Fuzz(8) Reliability(10) Static(8)
- Tests are source-level (read file content with `_read()`), not runtime execution
