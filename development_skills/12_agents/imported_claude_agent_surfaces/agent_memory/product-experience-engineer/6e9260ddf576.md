# Shell UI Performance Audit — 2026-03-02

## Findings Summary (16 items)

### HIGH severity
- H1: TopBar re-renders wholesale on every 2s state tick — no React.memo on TopBar or badge sub-components
- H2: EntropyBadge/MeshBadge/VdramBadge have internal useState for hover — they cause independent re-renders on hover AND on every state push simultaneously
- H3: App.tsx onClick closes context menu via setState spread `{ ...contextMenu, visible: false }` — creates new object every click, breaks referential equality
- H4: thermal-pulse keyframe animates `opacity` — GPU composited (OK), but MeshBadge and ThermalBadge BOTH attach the same named animation ("thermal-pulse") defined in styles.css; Launcher inlines a duplicate @keyframes block inside a <style> tag that is re-injected on every render
- H5: main.ts readSystemState() does synchronous fs.readFileSync on the Electron main thread every 2 seconds — blocks the main thread (IPC deadlock risk under disk pressure)
- H6: No `React.memo` on Dock — apps array is stable (useMemo) but Dock re-renders every 2s because App re-renders when state changes

### MEDIUM severity
- M1: IPC payload is the full SystemState object sent via structured clone — no diff, no selective fields
- M2: Launcher injects a raw `<style>` block with @keyframes on every render cycle (when open)
- M3: EntropyBadge/MeshBadge/VdramBadge create new inline handler functions (handleMouseEnter/Leave) on every parent render — no useCallback
- M4: timeoutRef cleanup missing in EntropyBadge, MeshBadge, VdramBadge — useEffect cleanup not called on unmount
- M5: Launcher focus-trap useEffect re-queries the DOM `panel.querySelectorAll(...)` on every open (expensive if app list is large)
- M6: No battery-saver / display-off throttle — pollTimer runs at fixed 2s regardless of power state

### LOW severity
- L1: `contextMenu.visible` spread in App.tsx onClick creates new state object each click unnecessarily
- L2: No `will-change: opacity` hint on thermal-pulse animated elements — minor GPU layer promotion miss
- L3: Apps list in App.tsx is `useMemo([], [])` with empty dep array — correct, but Launcher and Dock both receive the same reference, so this is fine; doc note only
- L4: LockScreen `void unlock()` pattern correct, but no abort controller — if user closes lock screen during network call, fetch continues in background

## Recommended Patterns

### Selective state slices (replaces full SystemState push)
Instead of sending the full SystemState snapshot, main.ts should send only the changed sub-keys.
Track `lastBattery`, `lastWifi`, `lastThermal`, etc. separately, send only delta.

### React.memo boundaries
```
export const TopBar = React.memo(function TopBar(...) { ... }, (prev, next) => {
  return prev.state.battery.level === next.state.battery.level &&
         prev.state.battery.charging === next.state.battery.charging &&
         prev.state.wifi.ssid === next.state.wifi.ssid &&
         prev.state.volume.level === next.state.volume.level &&
         prev.state.thermal === next.state.thermal &&
         prev.state.entropyState === next.state.entropyState &&
         prev.state.meshState === next.state.meshState &&
         prev.state.vdramState === next.state.vdramState &&
         prev.onLock === next.onLock;
});
```

### Split SystemState into slices in App.tsx
```
const [batteryState, setBattery] = useState(...)
const [thermalState, setThermal] = useState(...)
// etc.
// Each badge component receives ONLY its slice, React.memo comparison is cheap
```

### Async readSystemState (avoid main-thread block)
```
// main.ts — use fs.promises.readFile inside pollTimer
pollTimer = setInterval(async () => {
  const snapshot = await readSystemStateAsync();
  ...
}, 2000);
```

### Battery-saver throttle
```
// In main.ts pollTimer: read system power source, increase interval to 10s on battery
const interval = isOnBattery() ? 10_000 : 2_000;
```

### Launcher keyframe deduplication
Move launcher-item-in @keyframes to styles.css, remove inline <style> block from Launcher.tsx.

### useCallback for badge hover handlers
All three badge hover handlers should be wrapped in useCallback with [] deps.

### will-change on animated elements
Add `willChange: "opacity"` to the span style object on warn/critical ThermalBadge and offline MeshBadge.
