# Frontend Data Integrity Audit ‚Äî Fixes Applied (2026-03-02)

## Audit Summary
Comprehensive static audit of all pages/tabs. Found 10 Broken items and 19 Partial items.

## Fixes Implemented

### B8 ‚Äî InvestPage Portfolio Chart (FIXED)
- **Was:** `total * 0.83 + Math.random()` synthetic interpolation
- **Fix:** `useGetPortfolioHistoryQuery({ mode, period: portfolioPeriodMap[invTimeRange] })`
- **Period map:** `'1D'‚Üí'1W', '1W'‚Üí'1W', '1M'‚Üí'1M', '3M'‚Üí'3M', '1Y'‚Üí'1Y', 'ALL'‚Üí'1Y'`
- **Skip guard:** `skip: sub !== 'portfolio'` ‚Äî only fetches when portfolio tab is active
- **Fallback:** flat line at `total` when no history data available
- **File:** `frontend/src/pages/InvestPage.tsx` lines 292-329

### Trade Tab OHLC Fields (FIXED)
- **Was:** `sel.price - sel.change` (fabricated Open/Previous Close), `price ¬± change * 0.5` (fabricated Day Range)
- **Fix:** Extended `SelectedTradeStats` interface with `open`, `high`, `low`, `previousClose: number | null`
- **Source:** `selectedQuoteData.open/high/low/previous_close` from `useGetQuoteQuery`
- **Display:** shows `‚Äî` when null, never a fabricated number
- **Files:** `useInvestData.ts` (interface + useMemo), `InvestPage.tsx` (JSX display)

### P4 ‚Äî Dashboard Time Range Pills (FIXED)
- **Was:** `setTimeRange(t)` changed UI state but chart never re-fetched (always showed 1W data)
- **Fix:** Added dedicated `useGetPortfolioHistoryQuery` in `DashboardPage.tsx` driven by `timeRange` state
- **Period map:** `CHART_PERIOD_MAP` constant at top of file (`'1D'‚Üí'1W', '1W'‚Üí'1W', '1M'‚Üí'1M', '3M'‚Üí'3M', '1Y'+'ALL'‚Üí'1Y'`)
- **File:** `frontend/src/pages/DashboardPage.tsx`

### P2 ‚Äî weeklyPnl Dollar/Percentage Mismatch (FIXED)
- **Was:** `dailyReturns.reduce((sum, d) => sum + toNumber(d.return), 0)` ‚Äî sums `profit_loss_pct` values (%) and displays as $
- **Fix:** `validEquityPoints[last] - validEquityPoints[first]` ‚Äî actual dollar delta from equity series
- **File:** `frontend/src/components/elson-v2/hooks/usePortfolioData.ts`

### B6 ‚Äî Market News Hardcoded Text (FIXED)
- **Was:** `<Txt c="gray"...>Market news feed is not connected to a backend endpoint in this build.</Txt>`
- **Fix:** Proper empty state with icon, description, "COMING SOON" badge ‚Äî no hardcoded dev notes in production UI
- **File:** `frontend/src/pages/InvestPage.tsx`

### B1-B5 ‚Äî BotDashboard Mock Data (FIXED)
- **Was:** `generateMockSignals(25)`, `generateMockPnL(78)`, `generateMockBreakers()`, `generateMockFeatures()`, hardcoded `regime = 'trending'`
- **Fix:** Replaced entire BotDashboard with honest UI ‚Äî shows REAL data from `/auto-trading/status` (strategies, scanned symbols, win rates) and `PendingPanel` empty states for the 5 metrics panels
- **PendingPanel pattern:** Gold "COMING SOON" badge + icon + descriptive text explaining what will appear
- **Real data displayed:** Active strategies (name, symbol, is_active, win_rate, total_return), scanned symbols strip, strategy count in header
- **File:** `frontend/src/components/trading/BotDashboard.tsx` (fully rewritten)

### P3 ‚Äî Risk Level False "Low" Signal (FIXED)
- **Was:** Returns "Low" when 0 positions (no portfolio = no risk, but displayed as "Low Risk" false positive)

- **File:** `frontend/src/components/elson-v2/hooks/useAIInsights.ts`

### P5 ‚Äî Sector Change Hardcoded 0% (FIXED)

## Key Patterns Established
- `useGetPortfolioHistoryQuery` period constraint: only `'1W' | '1M' | '3M' | '1Y'` are valid ‚Äî map '1D' ‚Üí '1W', 'ALL' ‚Üí '1Y'
- `PortfolioHistoryEntry`: fields are `timestamp: number` (Unix seconds), `equity: number`, `profit_loss: number`, `profit_loss_pct: number`
- "Honest empty beats dishonest data" ‚Äî `PendingPanel` pattern for features awaiting backend
- `SelectedTradeStats` now includes OHLC: `open`, `high`, `low`, `previousClose: number | null`
- Sector `change` field is `number | null` in `UseAIInsightsReturn` ‚Äî always render `‚Äî` when null
