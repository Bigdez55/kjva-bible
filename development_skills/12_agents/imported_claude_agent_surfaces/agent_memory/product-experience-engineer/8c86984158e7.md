## SKILL CATALOG

### SKILL-01: Electron Process Isolation (contextBridge + contextIsolation)
- **Domain:** Electron Security Architecture
- **Pattern:** `contextBridge.exposeInMainWorld("genosShell", {...})` — whitelist-only IPC surface. No `nodeIntegration`. Two named worlds: `genosShell` (full system state API) and `ipcRenderer` (restricted rotation bridge with `ALLOWED_ROTATION_CHANNELS` allowlist).
- **UX/A11y impact:** Prevents renderer code from accessing Node.js APIs. All hardware calls must pass through typed IPC contracts. Security boundary is transparent to the user.
- **File:** `electron/preload.ts:82`

### SKILL-02: inotify-based System State Watcher (directory-level)
- **Domain:** Electron / Real-time State Push
- **Pattern:** `fs.watch(dir)` on the *directory* containing `system-state.json`, not the file itself. Atomic renames (write to `.tmp` then `rename`) survive inode changes that kill file-level watchers. Serialized de-duplication prevents redundant renders.
- **UX/A11y impact:** Zero-CPU live system state (battery, WiFi, volume, thermal, vDRAM) without polling. Instant topbar updates on any hardware change.
- **File:** `electron/main.ts:320`

### SKILL-03: Prototype Pollution–Safe Settings Dot-Path
- **Domain:** Electron Settings Persistence
- **Pattern:** `deepGet` / `deepSet` with a `BANNED_SEGS = new Set(["__proto__", "constructor", "prototype"])` guard. Dot-path key like `"power.screenTimeout"` splits into segments and traverses a JSON tree. Atomic write via `.tmp` rename.
- **UX/A11y impact:** Settings panels can independently read/write their own slice without coupling to the full schema.
- **File:** `electron/main.ts:21–70`

### SKILL-04: JWT Session Token Validation (client-side expiry guard)
- **Domain:** Identity / Auth
- **Pattern:** `decodeJwtPayload()` base64url → JSON, reads `exp` claim, adds 5s grace. `getValidSessionToken()` clears and returns null on expiry — no expired tokens leak to the renderer.
- **UX/A11y impact:** Prevents ghost sessions from appearing authenticated. Forces fresh login when token expires.
- **File:** `electron/main.ts:204–237`

### SKILL-05: iio-sensor-proxy Orientation Bridge (D-Bus via spawn)

### SKILL-16: Filter Keys Three-Stage Pipeline (C — debounce + slow + repeat)
- **Domain:** Shell Accessibility / Input
- **Pattern:** 512-slot hash table indexed by `keysym & 0x1FF`. Stage 1: BOUNCE — re-press within `bounce_ms` after key-up is dropped. ### SKILL-23: x86-64 Code Generator with REX Prefix Encoding
- **Domain:** JIT Compiler / Code Generation
- **Pattern:** Hardware register encoding table `XJ_HW_ENC[14]`. REX prefix formula: `0x40 | (W<<3) | (R<<2) | (X<<1) | B`. ModRM: `(mod<<6) | (reg<<3) | rm`. `mod=3` → register-register `0xC0`. - **Pattern:** `k_digit_glyphs[10][7]` — 5-bit row encoding per digit (1=dark pixel). `esos_paint_digit()` renders at configurable `scale` factor. Emergency SOS uses scale=8 → 40x56px per digit. Integrated directly into ARGB32 framebuffer via `esos_fill_rect()`.
- **UX/A11y impact:** Countdown digit is 40x56px (>> WCAG 44px) and readable at arm's length. Zero dependencies — works in bare-metal panic state.
- **File:** `devices/desktop/shell/xshell/src/emergency_sos.c:443–470`

### SKILL-43: XFRAME Node Flex Layout API (C)
- **Domain:** C Shell / Layout
- **Pattern:** `xframe_node_set_direction(XFRAME_DIR_COLUMN/ROW)`, `set_flex_grow`, `set_align_items(XFRAME_ALIGN_CENTER)`, `set_gap`, `set_padding`, `set_size`, `set_border_radius`, `set_border_color`, `set_border_width`. Used in factory_reset.c, emergency_sos.c, captive_portal, etc.
- **UX/A11y impact:** Consistent flex-based layout in C layer without libc. Gap/padding from design tokens.
- **File:** `devices/desktop/shell/xshell/src/factory_reset.c:183–270`

### SKILL-44: App Launch Map (Shell → Process Spawn)
- **Domain:** Shell / Application Management
- **Pattern:** `APP_LAUNCH_MAP: Record<string, string>` — maps app IDs (`"genesys-browser"`, `"go-notes"`, `"go-docs"`, `"go-drive"`, `"go-calendar"`, `"go-mail"`, `"go-sheets"`, `"go-slides"`, `"genesys-companion"`) to executable names. IPC `shell:launch-app` invokes spawn.
- **UX/A11y impact:** App launches are typed and enumerated — no arbitrary process spawning from renderer.
- **File:** `electron/main.ts:242–252`

### SKILL-45: Weak Extern Hook Pattern for Optional XNET Integration
- **Domain:** C Shell / Link-Time Optionality
- **Pattern:** `__attribute__((weak)) void xnet_voip_dial(const char *number) { (void)number; }` and `xnet_wifi_get_location()`. Emergency SOS compiles and links without XNET. When XNET is linked, strong definitions override. No conditional compilation needed.
- **UX/A11y impact:** Emergency SOS works even in partial system states where XNET is not yet initialized.
- **File:** `devices/desktop/shell/xshell/src/emergency_sos.c:211–215`

### SKILL-46: Rotation Bridge Channel Allowlist in preload.ts
- **Domain:** Electron Security / IPC
- **Pattern:** Second `contextBridge.exposeInMainWorld("ipcRenderer", {...})` expose. `ALLOWED_ROTATION_CHANNELS = new Set(["display-rotation"])`. `on()` and `removeListener()` silently return for unlisted channels. Only `simulateRotation` and `getRotation` invoke methods exposed.
- **UX/A11y impact:** Tablet mode gets rotation events without exposing full IPC surface. Prevents renderer from subscribing to arbitrary kernel events.
- **File:** `electron/preload.ts:274–293`
