## orange_docs.c G-09 Fix Details

- Problem: `s_editor_title` and `s_editor_body` were `xframe_node_t*` created with `XFRAME_NODE_TEXT` — read-only, cannot receive keyboard input.
- Fix: Replaced with `xwidget_input_t*` (opaque typedef void) + full API call pattern.
- `typedef void xwidget_input_t;` at top of file, then forward-declare all API functions.
- Timestamp API: `pal_time_now_ns() / 1000000ULL` — NOT `pal_time_ms()` (which does not exist).
- Pattern from notes.c: `n->modified_ms = pal_time_now_ns() / 1000000ULL;`

## XBLOB Drive API (drive_blob.c)

- drive_blob_set_stores(xstore, xblob) — inject both handles
- drive_file_upload(path, data, len) → xblob_put + xstore_put("drive:v1:blob:<path>", hash)
- drive_file_download(path, buf, buflen, out_len) → xstore_get hash → xblob_get(hash)
- drive_file_delete_blob(path) → xstore_delete + xblob_delete(hash)
- drive_file_rename(old, new) → read hash → write new key → delete old key (O(1), no data copy)
- drive_file_exists(path) → xstore_exists (O(log N))
- drive_file_list_prefix(dir, out, max) → xstore cursor scan over "drive:v1:blob:<dir>"
- Upper bound for cursor scan: append '~' (0x7E) — covers all printable ASCII paths

## orange_store.c (apps/orange/) Injection Layer

- Distinct from shell/xshell/src/orange_store.c (Sprint 42, RAM-disk only for XSTORE)
- apps/orange/orange_store.c adds: XBLOB lifecycle + drive_blob_set_stores() injection
- RAM-disk: s_store_disk[16MB] for XSTORE pages, s_blob_disk[8MB] for XBLOB content
- Cache/WAL: static BSS (xstore_cache_entry_t[1024], xstore_wal_entry_t[65536])
- XBLOB index: xblob_index_entry_t[1M slots] = 56 MB BSS (demand-paged in XENOS)
- orange_store_init() returns -1 on XSTORE open failure; XBLOB failure is non-fatal
- orange_store_close() checkpoints before close, revokes all injected pointers to NULL

## XSTORE Convenience API Pattern (freestanding)

- xstore_put(store, &key, &val) — auto-commit, retries OCC conflicts 3×
- xstore_get(store, &key, &val_out) — val_out.buf caller-provided
- xstore_delete(store, &key) — XSTORE_ERR_NOT_FOUND is not an error on delete
- xstore_exists(store, &key) — bool, O(log N), no alloc
- XSTORE_KEY_STR(s) macro — uses xstore_strlen() not strlen()

## Orange Notes Persistence Schema (reference for all Orange apps)

- XSTORE key schema: `notes:v1:meta:<10-digit-id>` + `notes:v1:body:<10-digit-id>`
- Body stored separately because orange_note_t.body = 4096B exceeds XSTORE_MAX_VAL_LEN
- Struct notes_meta_t = 160 bytes (safely under XSTORE_MAX_VAL_LEN=4096)
- _Static_assert guards struct size vs XSTORE_MAX_VAL_LEN — always add this guard
- Validates magic (NOTES_META_MAGIC="NOTE") + version on load; corrupt = XSTORE_ERR_CORRUPT
- Empty body stored as 1 byte (not 0) to keep key alive in XSTORE
- notes_persist_load_all(): sequential ID probe 1..NOTES_ID_SCAN_MAX using xstore_exists() fast-path
