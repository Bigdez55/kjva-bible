# XFRAME Sprint 15 Display Pipeline Audit
# Date: 2026-03-07

## Summary

Full audit of the XFRAME/XCOMP/XDISP/XSHELL display pipeline to determine the minimum
wiring needed to produce a live UI frame in QEMU.

## 1. XFRAME State: Real vs Stub

### xframe_init() — REAL (main.c)
- Builds 2048-node free list via next_sibling threading. O(1) alloc/free.
- Spinlock-protected. Double-init guard via pal_atomic_load.
- Does NOT call xcomp_init() or xdisp_init(). XFRAME init is independent of XCOMP.
- Status: REAL — fully implemented.

### xframe_app_create() — PARTIAL STUB
- Creates app context in static pool, sets width/height.
- CRITICAL GAP: xcomp_surface_create() is commented out with a "Production integration"
  note (main.c lines 252-265). Uses a synthetic surface_id (pool index + 1) instead.
  xframe_app does NOT hold a real XCOMP surface pointer.
- Status: STUB at the XCOMP integration boundary.

### xframe_run() — REAL LOOP, STUBBED PAINT TARGET
- 60fps loop: event_dispatch → layout → paint → frame_pace. Structure is correct.
- CRITICAL GAP: paint call passes `buf=NULL` (main.c line 639):
  `xframe_paint(app, NULL, app->width, app->width, app->height);`
  paint.c handles NULL buf by skipping the clear step but still traverses the tree.
  paint_fill_rect() returns immediately when buf==NULL (line 249 check).
  Result: the loop runs and calls paint_subtree but writes ZERO pixels to any buffer.
  XCOMP surface_commit is also commented out — no frame ever reaches XDISP.
- Status: Loop is real. Pixel output is completely disabled.

### xframe_paint() — REAL (paint.c)
- paint_subtree: real pre-order tree traversal with clip stack. Correct.
- paint_fill_rect: real pixel writer — iterates rows/cols, writes argb to buf[row*stride+col].
  When buf is non-NULL this actually writes pixels. This is a genuine software rasterizer.
- paint_text: STUB — logs to pal_console_printf, writes NO glyphs into the buffer.
  Text rendering depends on xfont_render_string (fonts.h) which is not wired.
- paint_rounded_rect: falls back to paint_fill_rect (ignores radius). Functionally writes pixels.
- Status: BOX painting is REAL. TEXT is a logging stub. No glyph rasterizer linked.

### xframe_event_dispatch() — REAL (event.c, referenced via extern)
- Infrastructure exists for XFRAME_EVENT_KEY, g_focused_node.
- No xinput_poll() call in the main loop (commented out in main.c line 617).
- Status: Infrastructure exists; input polling not connected.

## 2. XCOMP State

### xcomp_init() — REAL (compositor.c lines 693-811)
- Calls xdisp_drm_init() → allocates two ARGB8888 output buffers (back + front) via
  xdisp_buf_alloc() → submits a blank front buffer as the initial scanout frame.
- Proper double-buffer setup. Guard against double-init.
- Status: REAL. This is a proper compositor init.

### xcomp_frame() — REAL (compositor.c line 1197)
- Iterates z-sorted scene graph, SRC_OVER blends damaged surfaces into back buffer,
  calls xdisp_buf_submit() to present, then swaps back/front handles.
- Integer-arithmetic alpha compositing (no FP), damage-region-aware.
- Status: REAL render loop. The render loop is complete.

### xcomp_surface_create() — REAL (compositor.c line 847)
- Allocates a slot from the 128-surface static pool. Inserts into z-sorted list.
- Status: REAL.

### XCOMP ↔ XFRAME integration gap:
- xframe_app_create() never calls xcomp_surface_create() (commented out).
- xframe_run() never calls xcomp_surface_commit() (commented out).
- These are the two wiring points needed for Sprint 15.

## 3. XDISP State

### xdisp_init() (boot_chain.c weak stub, drm.c real)
- boot_chain.c declares weak stub for xdisp_init(fb_phys, width, height, pitch, fmt).
- drm.c: xdisp_drm_init() opens /dev/dri/card0 — Linux DRM ioctl interface.
- For QEMU: /dev/dri/card0 is present with -vga virtio (virtio-gpu exposes DRM).
  With -vga std QEMU does NOT expose DRM — xdisp_drm_init() will fail (no /dev/dri/card0).
- Status: Real DRM code exists. Requires QEMU virtio-gpu for DRM access.

## 4. XSHELL / greeter.c State

### xshell_init() — REAL (xshell.c lines 161-229)
- Calls xframe_init() → xframe_app_create(1920, 1200) → creates scene root.
- Creates topbar, dock, launcher, lockscreen, greeter, context_menu sub-components.
- Attaches topbar and dock root nodes to scene root.
- Status: REAL scene graph construction. All sub-component creates are wired.

### xshell_run() — REAL (xshell.c lines 235-247)
- Sets running=1, calls xframe_run(g_shell.xframe_app).
- Status: REAL — this is the entry to the 60fps loop.

### xshell_start() — MISSING
- boot_chain.c declares it as a weak stub returning PAL_OK (line 89).
- xshell.c does NOT define xshell_start(). The function that exists is xshell_run().
- CRITICAL GAP: boot_chain calls xshell_start() but xshell only provides xshell_run().
  A real xshell_start() must call xshell_init() + xshell_run().
  Without it, the weak stub fires and xshell_init/run are never called.

### greeter.c — REAL node construction, MISSING click handler
- xshell_greeter_create(): builds full node tree: overlay, logo, card, username field,
  password field, session selector, login button.
- All nodes use XFRAME_NODE_BOX (not text_input.c widget) for input fields — no
  on_change callback, no keyboard focus routing.
- gr_on_login_click() exists but is only referenced in the _gr_unused() suppressor.
  It is NEVER registered on g->login_btn as a click handler.
- xshell_greeter_show(): sets visible=1, marks dirty. Called nowhere in the boot sequence.
- CRITICAL GAP: greeter is created in xshell_init but greeter_show() is never called.

### xshell_tick() — UNUSED
- Declared static with __attribute__((unused)) at line 98.
- Never registered with XFRAME as a per-frame callback.
- Notifications never auto-dismiss. Dock/topbar animations never tick.

## 6. ts_bridge.c — QuickJS Removal Analysis

### What JSCFunction bindings exist (all inside #ifdef QUICKJS_AVAILABLE):
- js_createElement(type: string) → int nodeId
- js_destroyElement(nodeId: int) → void
- js_appendChild(parentId, childId) → void
- js_setStyle(nodeId, property, value) → void  [7 properties planned, all stubbed]
- js_setLayout(nodeId, property, value) → void  [10 properties planned, all stubbed]
- js_setText(nodeId, text) → void
- js_addEventListener(nodeId, type, handler) → void
- js_removeEventListener(nodeId, type) → void
- ts_register_xframe_api(): wires all 8 JSCFunctions onto a global "xframe" JS object
- ts_inject_design_tokens(): injects 21 token constants as JS globals
- ts_interrupt_handler(): runaway-script guard (stub body)

### Minimal change to compile WITHOUT QuickJS:
The file already compiles without QuickJS because ALL QuickJS code is inside
`#ifdef QUICKJS_AVAILABLE ... #endif`. The public API (xframe_ts_init, xframe_ts_eval,
xframe_ts_eval_file, xframe_ts_shutdown) has stub implementations in the #else branches.

No code change is needed. Simply: do NOT define QUICKJS_AVAILABLE in the build.

To make the intent explicit for Sprint 15, the recommended change is:
1. Delete the entire #ifdef QUICKJS_AVAILABLE block (lines 165-407 of ts_bridge.c).
2. Keep only the 4 public functions with their stub/no-op bodies.
3. Remove the g_js_rt, g_js_ctx, g_xframe_obj, g_tokens_obj static variables.
4. Remove ts_strlen() (only used in the eval stub preview path — can be inlined).

This reduces ts_bridge.c from 625 lines to ~100 lines with no functional regression
since QuickJS was never linked.

The 8 JSCFunction bindings map 1:1 to existing C functions:
- js_createElement    → xframe_node_create(type_enum)
- js_destroyElement   → xframe_node_destroy(node)
- js_appendChild      → xframe_node_add_child(parent, child)
- js_setStyle         → xframe_node_set_bg_color / set_text_color / set_opacity / etc.
- js_setLayout        → xframe_node_set_direction / set_align_items / etc.
- js_setText          → xframe_node_set_text(node, text)
- js_addEventListener → xframe_node_set_on_click / future event API
- js_removeEventListener → clear event handler

No intermediary is needed. Orange apps and XSHELL already call these C functions directly.
ts_bridge.c is not consumed by any caller in the current codebase.

## 7. Minimal Boot UI Sequence: "GEN.OS BOOT OK" on screen

### QEMU display flag
Use: -vga virtio  (NOT -vga std)
- -vga virtio: exposes /dev/dri/card0 via virtio-gpu. DRM ioctls work. xdisp_drm_init() succeeds.
- -vga std: VGA legacy framebuffer only. No DRM device node. xdisp_drm_init() fails with ENOENT.
- headless CI: -display none -nographic -serial stdio (already in qemu-boot.sh --headless mode)
- Current qemu-boot.sh uses -vga virtio in non-headless mode. This is correct.

### Minimum function call chain from power-on to "GEN.OS BOOT OK":

```
AetherBoot → XENOS kernel (entry.asm, main.c)
  → GENSD main (init/gensd/main.c)
    → xchain_run(handoff)
      → xchain_display(handoff)
          → xdisp_init(fb_phys, width, height, pitch, fmt)  [drm.c: opens /dev/dri/card0]
      → xchain_compositor()
          → xcomp_init()  [compositor.c: xdisp_drm_init → alloc back/front bufs → blank frame submit]
      → xchain_shell()
          → xshell_start()  [NEEDS REAL IMPL: xshell_init() + xshell_run()]
              → xshell_init()
                  → xframe_init()       [main.c: builds 2048 node free list]
                  → xframe_app_create(1920, 1200)  [NEEDS FIX: must call xcomp_surface_create]
                  → xframe_node_create(BOX) → scene_root
                  → xshell_greeter_create(&g_shell)  [builds login UI node tree]
                  → xshell_greeter_show(g_shell.greeter)  [MISSING: needs to be added]
              → xshell_run()
                  → xframe_run(g_shell.xframe_app)  [60fps loop]
                      → xframe_layout(app)
                      → xframe_paint(app, surface_buf, width, width, height)
                          [NEEDS FIX: buf must be real XCOMP surface buffer, not NULL]
                      → xcomp_surface_commit(surface)  [NEEDS FIX: needs to be called]
                      → xcomp_frame()  [blends surfaces into back buf → xdisp_buf_submit]
```

### Minimum "GEN.OS BOOT OK" text label path:
Since paint_text() is a console-log stub (no font rasterizer), text cannot render.
The minimum VISIBLE proof-of-display is a colored rectangle:
- A filled BOX node with bg_color=0xFF7C3AED (purple primary) at full screen size.
- paint_fill_rect() is REAL — it writes pixels. This will produce a solid color on screen.

To get actual text ("GEN.OS BOOT OK") requires one of:
a) Implement a minimal 8x16 bitmap font in paint_text() using a hardcoded glyph table.
b) Use xfont_render_string() from XCOMP fonts.h if that module is linked.
c) Blit a pre-rendered ARGB8888 splash screen image via XFRAME_NODE_IMAGE.

Option (a) is the minimum viable path. A 96-glyph (ASCII 32-127) 8x16 bitmap font
is ~1536 bytes of static data and can be inlined in paint.c with no external deps.

## 8. New Files Needed for Minimal Working Display

### Required new functions (not new files — add to existing files):

1. **xshell.c: add `pal_status_t xshell_start(void)`**
   Body: call xshell_init() + xshell_greeter_show() + xshell_run(). Return PAL_OK.
   This overrides the weak stub in boot_chain.c.

2. **xshell.c: add greeter_show() call inside xshell_init() or xshell_start()**
   After xshell_greeter_create(), call xshell_greeter_show(g_shell.greeter).

3. **main.c (xframe): fix xframe_app_create() — call xcomp_surface_create()**
   Replace synthetic surface_id with a real xcomp_surface_t pointer.
   Store xcomp_surface_t* in xframe_app_t (requires header change too).

4. **main.c (xframe): fix xframe_run() — obtain real pixel buffer and call surface_commit**
   In the paint step:
   ```c
   uint32_t *buf = xcomp_map_surface_buf(app->surface);
   xframe_paint(app, buf, app->width, app->width, app->height);
   xcomp_surface_commit(app->surface);
   ```

5. **paint.c: implement minimal bitmap font in paint_text()**
   8x16 glyph table, iterate text chars, blit each glyph row by row into buf.
   ~80 lines of C, no external deps.

### Optional new file (clean separation):

- **ui/xframe/runtime/font_bitmaps.h** — 8x16 glyph data as `static const uint8_t g_font8x16[96][16]`.
  Keeps paint.c clean. Inlined via #include — no linker dependency.
