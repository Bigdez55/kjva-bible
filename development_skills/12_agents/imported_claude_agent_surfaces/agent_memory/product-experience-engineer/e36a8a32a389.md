# Auto-Trading Bot UI Patterns

## Autonomous Scan Mode (2026-02-25)

The bot is fully autonomous — the backend scans the market and selects symbols. The frontend no longer maintains a user-editable symbol list.

### What was removed
- `customSymbols: string[]` state + localStorage persistence (`elson_bot_symbols`)
- `addCustomSymbol` / `removeCustomSymbol` handlers
- `symbolSearchInput` state + debounced `useEffect` + `useLazySearchSymbolsQuery`
- `deriveBotSymbols()` exported function
- `DEFAULT_BOT_SYMBOLS` constant
- `customSymbols` param from `startBot()`, `toggleBot()`, `UseAutoTradingReturn`

### What was added
- `scanned_symbols?: string[]` field on `AutoTradingStatus` in `autoTradingApi.ts`
- `scannedSymbols: string[]` on `UseAutoTradingReturn` (derived from `statusData?.scanned_symbols ?? []`)
- Read-only "Bot Currently Monitoring" chip display in bot tab (first 8 chips + "+N more" overflow)

### Start request pattern
```ts
// Both auto-start effect and startBot() now pass symbols: []
startMutation({
  portfolio_id: statusData.portfolio_id,
  strategy_names: [strategy],
  symbols: [], // Backend auto-scans when symbols is empty / None
  mode,
})
```

### Read-only scan display pattern
- Label: uppercase, `letterSpacing: '0.08em'`, `fontWeight: 600`
- Chips: `backgroundColor: \`${C.gold}12\``, `border: \`1px solid ${C.gold}30\``, `borderRadius: 20`
- Text inside chip: `Txt size={11} bold style={{ color: C.gold }}`
- Overflow chip: `rgba(255,255,255,0.05)` bg, `rgba(255,255,255,0.12)` border, `C.gray` text
- Empty state: italic `C.gray` text "Scanning market on start..."
- Show max 8 chips, then "+N more" badge

- `frontend/src/components/elson-v2/hooks/useAutoTrading.ts` — hook interface + implementation
- `frontend/src/pages/AIPage.tsx` — UI + state cleanup
