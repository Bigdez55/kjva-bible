## Key Files

- `devices/desktop/ui/xframe/runtime/module_loader.c` — implementation (~300 LOC)
- `devices/desktop/ui/xframe/runtime/module_loader.h` — public API (4 functions)
- `devices/desktop/apps/calculator/calculator_xframe.js` — proof-of-concept app JS
- `kernel/xenos/Makefile` — `module_loader.c` added to XFRAME_SRCS after `ts_bridge.c`

## Architecture

module_loader sits above ts_bridge:

```
xframe_load_app(app_dir)
  └→ ml_path_join(app_dir, "index.js") → full path
  └→ ml_registry_add(path)             → dedup cache
  └→ xframe_ts_eval_file(path)         → QuickJS evaluates
```

## require() Pattern

- `js_require` is a C QuickJS function registered as `globalThis.require`.
- It resolves specifiers relative to `g_current_app_dir` (set in `xframe_load_app`).
- Strips leading `./` from specifier before path join.
- Registers resolved path BEFORE eval to break recursive require() loops.
- Uses `g_require_buf[512KB]` static scratch — no heap allocation.

## Alias Injection

`ml_inject_globals()` evaluates a small JS bootstrapper string that wires:
  `createElement`, `destroyElement`, `appendChild`, `setStyle`, `setLayout`,
  `setText`, `addEventListener`, `removeEventListener`, `tokens`
as `globalThis.*` forwarding to `xframe.*`.

App JS code can thus omit the `xframe.` prefix entirely.

## Accessing QuickJS Context from Outside ts_bridge

ts_bridge does not expose `g_js_ctx` publicly. module_loader uses:

```c
extern JSContext *xframe_ts_get_context(void) __attribute__((weak));
JSContext *ctx = xframe_ts_get_context ? xframe_ts_get_context() : NULL;
```

**This means ts_bridge.c needs `xframe_ts_get_context()` added** when QuickJS
build is enabled. Without it, alias injection is deferred but the loader still
boots cleanly (module loading via eval_file still works).

## PAL_O_RDONLY Gotcha

`PAL_O_RDONLY` is used in ts_bridge.c and module_loader.c but is NOT defined in
`pal/include/pal.h` (as of Sprint 42). It resolves correctly at link time from
XVFS context (`XVFS_O_RDONLY = 0x0001`). Do NOT try to define it locally in
module_loader.c — match ts_bridge exactly. If the build gains a proper pal.h
constant later, it will transparently override.

## Why:

Migration goal is zero-Electron, JS apps running natively in XFRAME QuickJS.
Phase 1 establishes the loader layer so individual apps can be ported incrementally
by providing a `/opt/genos/apps/<name>/index.js` entry point.
