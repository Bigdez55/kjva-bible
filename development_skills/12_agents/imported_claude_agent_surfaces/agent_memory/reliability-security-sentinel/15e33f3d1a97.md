# GEN.OS Network & Security Originality Audit — 2026-03-21

**Scope:** All files in net/xnet/ and sec/xsec/ (crypto, tls, cert, audit)
**Method:** Full source read of every .c file

## VERDICT: NO EXTERNAL CODE COPIED

### Network Stack (net/xnet/) — ALL CLEAN
- tcp.c: original RFC 793 state machine. No lwIP/FreeBSD/Linux types or struct names.
- dns.c: original RFC 1035 stub resolver. No bind/dnsmasq/musl signatures.
- mdns.c: original RFC 6762 mDNS. No Apple Bonjour (no ObjC/NS types) or Avahi (no GLib types).
- vpn.c: WireGuard Noise_IKpsk2 **protocol pattern** implemented from spec. No wireguard.c kernel code. Sprint 35 skeleton with weak extern crypto refs — handshake not yet wired.
- All other xnet files (ip4, ipv6, arp, ethernet, icmp, udp, netif, socket, e1000, virtio_net, gossip): CLEAN.

### Crypto (sec/xsec/crypto/) — ALL CLEAN with one P1 flag
- chacha20.c: RFC 8439, GEN.OS PAL helpers. Not libsodium (different type names, different Poly1305 limb scheme).
- aes_gcm.c: FIPS 197 + NIST SP 800-38D. Constant-time S-box scan (volatile, full-table) — distinct from OpenSSL T-tables and mbedTLS FSM.
- sha256.c: FIPS 180-4. Uses xsec_ror32/xsec_be32 PAL helpers. CLEAN.
- hmac.c: RFC 2104 + RFC 5869 HKDF + RFC 8446 HKDF-Expand-Label. CLEAN.
- x25519.c: 5-limb 51-bit representation. Montgomery ladder. Sprint 40 low-order point rejection added. CLEAN.
- ecdsa_p256.c: Sprint 39. NIST P-256 4-limb uint64 representation. CLEAN.
- rsa.c + rsa_pss.c: Sprint 38. Original schoolbook modular exponentiation for RSA-2048 public key ops. CLEAN — but see P2 license issue.
- **ed25519.c (P1):** 10-limb 25/26-bit GF(2^255-19) field arithmetic structurally matches NaCl/SUPERCOP ref10 (Bernstein). ref10 is public domain so no license violation, but GEN.OS copyright may be overstated if derived. **Requires author declaration before ISO distribution.**

### TLS (sec/xsec/tls/) — ALL CLEAN
- tls13.c: RFC 8446 state machine. No OpenSSL/BoringSSL/mbedTLS/wolfSSL type names. Sprint 41 H9 inbuf staging is GEN.OS-specific.
- aes_ni.c: Intel AES-NI intrinsics from <wmmintrin.h> (compiler-provided). Not copied from any library.

### Cert (sec/xsec/cert/) — CLEAN with P2 license flag
- x509.c: Custom DER/ASN.1 cursor parser. No OpenSSL X509/d2i types. Sprint 21 CN null-byte rejection is GEN.OS addition.
- trust_anchors.c: Static CA fingerprint DB. Original. AGPL-3.0 header error.

### Audit (sec/xsec/audit/) — CLEAN with P3 duplication flag
- audit.c: Original ring buffer + spinlock audit log. No syslog/journald.
- causal_log.c: Migrated Sprint 24. Contains EMBEDDED SHA-256 (sha256_ctx_t, sha256_transform) — separate from canonical xsec_sha256_ctx_t in sha256.c. Both implementations are original but divergent. causal_log version lacks volatile S-box, xsec_secure_zero wipes.

## OPEN ACTION ITEMS

### P1 — ed25519.c field arithmetic originality declaration
- Author must declare: (a) independently derived from RFC 8032, or (b) derived from NaCl ref10 public domain with attribution added.
- Status: OPEN — blocks next public ISO build

### P2 — AGPL-3.0 license header mismatch
- Files: rsa.c, rsa_pss.c, trust_anchors.c — all declare AGPL-3.0 but should be LicenseRef-Proprietary.
- Most likely cause: copy-paste error in header template during Sprint 38/39 work.
- If error: correct to LicenseRef-Proprietary.
- If actual derivation: full legal analysis + AGPL source disclosure obligations apply.
- Status: OPEN — requires immediate investigation

### P3 — causal_log.c embedded SHA-256
- Consolidate to call xsec_sha256() from sha256.c
- Risk: causal_log SHA-256 does not have constant-time S-box or zeroization
- Status: OPEN — next sprint

### P3 — vpn.c Noise handshake skeleton
- Add compile-time or runtime guard: VPN tunnels must not activate without crypto symbols resolved
- Current weak-symbol pattern produces silent non-functional VPN
- Status: OPEN — next sprint

**Why:** Originality of cryptographic code is a legal and reputational requirement. The ed25519 field arithmetic concern is the highest risk because it touches the most legally sensitive boundary (DJB work, even if public domain). The AGPL headers are a distribution risk.

**How to apply:** Before any public ISO build or commercial distribution of GEN.OS, P1 and P2 items must be resolved. Record resolution in this file.
