### P1-CR4-UMIP-WRONG-BIT — FIXED 2026-03-31
```
File: kernel/xenos/core/main.c  Line: 3787 (was wrong, now fixed)
Before: cr4_sp |= (1UL << 11);   /* CR4.UMIP — ... */
After:  cr4_sp |= (1UL << 22);   /* CR4.UMIP — ... */
```
- CR4 bit 11 = VMXE (VMX Enable), NOT UMIP
- CR4 bit 22 = UMIP (correct)
- The post-VMM path does read-modify-write, so UMIP from Step 4.6 was preserved,
  but VMXE was being unexpectedly set, which on hardware supporting VMX but without
  proper VMX init can cause #GP or erratic behavior
- Assessment: CPUID-gated path at Step 4.6 fired first and set bit 22 correctly.
  The Step 5b path was adding bit 11 (VMXE) additionally. Net effect: UMIP was
  working but VMXE was being incorrectly set. Fix: change bit 11 to bit 22.
