Full audit of stubs, TODOs, deferred functions, and skeletal implementations.

**Why:** Team requested exhaustive research pass before Sprint 45+ planning. These items represent the gap between "code compiles and CI passes" and "feature actually works."

**How to apply:** Use this catalog to prioritize Sprint backlog. P0 and P1 items should not be deferred past Sprint 46. P0 item (VPN forward secrecy) is a security defect and must block any VPN feature claim.

## P0 — CRITICAL (1 item)

**STUB-NET-06** — `net/xnet/vpn/vpn.c:458`
VPN Noise handshake ephemeral key is derived deterministically from static private key + hash via SHA-256.
Real implementation requires `pal_random_bytes()` for ephemeral key generation.
Forward secrecy guarantee of WireGuard Noise IKpsk2 is broken entirely.
A passive observer with the static private key can decrypt all recorded sessions.

## P1 — BLOCKING (9 items)

**STUB-SEC-01** — `sec/xsec/tls/tls13.c:1101,1763`
TLS 1.3 application traffic keys use `app_hash_placeholder` (48 zero bytes) instead of the Finished transcript hash.
RFC 8446 §7.1 violation. Application layer traffic secrets derivation is not fully transcript-bound.
Occurs in both client path (line 1101) and server path (line 1763).

**STUB-NET-01** — `net/xnet/inet/ipv6.c:413–461`
ICMPv6 Echo Reply packet is fully constructed but never transmitted.
`(void)reply_total; (void)if_idx;` — the send call is absent. IPv6 ping never works.

**STUB-NET-05** — `net/xnet/vpn/vpn.c:271,692`
`xnet_tun_destroy()` declared as `__attribute__((weak))` extern with NO definition anywhere in repo.
Called unconditionally in `vpn_tunnel_destroy()`. NULL function pointer dereference at runtime.
TUN driver (`net/xnet/drv/tun.c`) does not exist.

**STUB-NET-07** — `net/xnet/vpn/vpn.c:440–631`
VPN computes transport keys but has no encrypt/decrypt data path.
All `xsec_chacha20_poly1305_encrypt()` calls wrapped in `(void)` — results discarded.
IPv6 AllowedIPs route removal logs "v6 del deferred" and does nothing.
No function exists that encrypts/decrypts actual VPN traffic.

**STUB-SEC-02** — `sec/xsec/tls/tls13.c:1852–1861`
`xsec_ecdsa_sign()` declared weak with no definition in sec/xsec/ or anywhere in repo.
TLS 1.3 server CertificateVerify path always returns `XSEC_ERR_INVAL`.
Server-side TLS authentication is impossible. XNET-hosted TLS services cannot authenticate.

**STUB-PKG-01** — `pkg/xpkg/format/pkg_format.c:566; ops/install.c:19`
All file content in built packages is `xpkg_memset(out_buf + pay_pos, 0, size)` — zero fill.
No real file data is ever stored in or extracted from XPKG archives.
The entire package install/extract path is a simulation.

**STUB-PKG-02** — `pkg/xpkg/ops/install.c:314–317,325–327`
XKABI capability grant/revoke in post_install_hook and post_remove_hook are `pal_console_printf()` only.
No XKABI handle is created. No capability table entry is created or revoked.
Installed packages have zero OS capabilities.

**STUB-PKG-05** — `pkg/xpkg/repo/repo_index.c:267–274`
XNET TCP functions redeclared as weak externs in repo_index.c without NULL guards.
If linked without full XNET stack (installer context), all four calls are NULL ptr dereferences.

## P2 — DEGRADED (9 items)

**STUB-NET-04** — `net/xnet/inet/ipv6.c:263–278`

**STUB-NET-08** — `net/xnet/drv/virtio_net.c:45; e1000.c:43`
Both NIC drivers polling-only. No IRQ handler. Deferred since Sprint 15.

**STUB-NET-09** — `net/xnet/app/dns.c:318`
No DNS TCP fallback on TC bit. Truncated responses from DNSSEC/large zones silently incomplete.

**STUB-NET-10** — `net/xnet/mesh/mdns.c:341`
mDNS sends to 224.0.0.251 without IGMP group join. Zero-conf discovery between GEN.OS nodes does not work.

**STUB-STORE-01** — `store/xstore/btree/btree.c:542`
`(void)snapshot_lsn` in btree_lookup. MVCC snapshot isolation not enforced. All transactions see latest version.

**STUB-STORE-02** — `store/xstore/txn/mvcc.c:340`
OCC conflict detection uses FNV-1a key hash mod total_pages, not actual modified leaf page. False conflicts and missed conflicts both possible.

**STUB-STORE-03** — `store/xstore/btree/btree.c:675`
B+ tree internal node split not implemented. Hard insert cap at order-128 internal node fill. Returns XSTORE_ERR_CAPACITY silently under high load.

**STUB-STORE-06** — `store/xstore/wal/wal.c:221–224`
WAL replay btree errors discarded with `(void)`. Crash recovery may leave store inconsistent with no diagnostic.

**STUB-PKG-03** — `pkg/xpkg/gate/refusal_gate.c:173–203`
Capability gate uses install_path prefix heuristic. System packages return WARN not FAIL. No capability manifest in metadata.
