### FLAGGED FILES

#### crypto/ed25519.c — P1 ATTRIBUTION REQUIRED
- Status: ADR-S46-02 CREATED (Sprint 46), ISO build UNBLOCKED.
- Finding: File header at line 14 states: "Field arithmetic and scalar reduction use the public-domain SUPERCOP/NaCl ref10 carry schedule (Bernstein, public domain)."
- The fe10 10-limb representation, fe10_reduce carry schedule, and fe10_mul 25-product schoolbook are structurally identical to the NaCl ref10 reference implementation.
- Disposition: Author declaration required. ADR-S46-02 exists. Public domain finding documented. No license incompatibility.
- Action: No further action needed for ISO builds. RESOLVED.

#### smb/smb_client.c — NOTE: HMAC-MD5 stub
- NTLMv2 authentication references HMAC-MD5 via a weak extern hook `xsec_ntlmv2_hash`.
- The weak stub (line 779) always returns -1 (stub/failure).
- No MD5 implementation exists in XSEC. The real implementation would need to be added to xsec if NTLMv2 auth is required.
- OPEN P2: MD5 is cryptographically weak. NTLMv2 over SMB2 requires HMAC-MD5 by protocol spec. If implemented, this introduces a weak hash for authentication. Acceptable only for SMB2 compatibility with existing Windows servers.
- No external code present — stub only.

## OPEN ITEMS AFTER THIS AUDIT
- P1-ED25519-ATTRIBUTION: RESOLVED via ADR-S46-02 (Sprint 46).
- P2-NTLM-MD5: xsec_ntlmv2_hash is a stub. If NTLMv2 auth is implemented, HMAC-MD5 must be added to XSEC with documented justification that MD5 is required only for SMB2 protocol compatibility.
- P2-VPN-STUB: vpn.c is a Sprint 35 skeleton. Noise_IKpsk2 handshake, ChaCha20-Poly1305 data path, and TUN interface not yet implemented. No ephemeral key generation exists.
- P3-CARPLAY-NAME: carplay_host.c/carplay_ui.c are named "carplay" but implement GEN.OS "AutoDrive" protocol. Rename to autodrive_host.c/autodrive_ui.c to avoid Apple trademark confusion risk.
