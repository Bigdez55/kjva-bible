### compositor/genos-shell/electron/main.ts — create-user uses res.ok

- Lines 161-175 (shell:create-user handler):
  ```typescript

  if (res.ok) {
    return { ok: true };
  }
  return { ok: false, error: "User creation failed" };
  ```
- Uses `res.ok` (truthy for 200-299, including 201) — CORRECT
- VERDICT: PASS

### HOWEVER — NEW FINDING P1: create-user missing email field

- identity/app.py UserCreate model (line 182-187) requires: username, email, password
  - email is EmailStr, required (no default)
- main.ts shell:create-user handler (lines 162-165) sends:
  ```typescript
  body: JSON.stringify({ username, password })
  ```
  - email field is MISSING from the POST body
- Identity service will reject this with HTTP 422 (Unprocessable Entity) from Pydantic validation
- FirstRunWizard.tsx calls `window.genosShell.createUser(state.username, state.password)` — no email collected from user
- The preload.ts createUser bridge also only accepts (username, password) — no email parameter
- RESULT: First-run user creation ALWAYS fails at identity service validation.
  The wizard shows "User creation failed" to every new user. This is a FUNCTIONAL BLOCKER.
- SEVERITY: P1 — blocks the primary user onboarding flow. System unusable for new installs.
- FIX OPTIONS:
  Option A (minimal): Generate synthetic email `${username}@genos.local` in main.ts before POST
    ```typescript
    body: JSON.stringify({ username, password, email: `${username}@genos.local` })
    ```
  Option B (UX-correct): Add email field to FirstRunWizard, thread it through preload + main.ts
  Option A is recommended for immediate fix — Option B is a follow-on UX improvement.
