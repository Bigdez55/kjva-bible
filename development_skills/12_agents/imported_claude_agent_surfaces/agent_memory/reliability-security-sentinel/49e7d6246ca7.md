## Files Produced

- shell/xshell/src/sticky_keys.c (~230 LOC): Standalone sticky-keys state machine (SK_IDLE/LATCHED/LOCKED per modifier). Defines its own keysym_to_mod_bit() to avoid dependency on keyboard_nav.c statics. Exposes xshell_sticky_keys_get_indicator_str() for topbar. Callbacks: on_latch_cb, on_lock_cb, on_release_cb.

- shell/xshell/src/filter_keys.c (~270 LOC): Three-filter model: Bounce Keys (debounce re-press after key-up), Slow Keys (min hold duration via tick callback), Repeat Keys (suppress auto-repeat). Per-key table 512 slots (keysym & 0x1FF). fk_parse_u32() marked __attribute__((unused)) — wired via XSTORE integration in Settings service layer. xshell_filter_keys_tick() must be called ~10ms for slow-key acceptance.

- shell/xshell/src/caret_browse.c (~290 LOC): Virtual text cursor for keyboard-only navigation. Uses xa11y_walk_tree() with correct xa11y_walk_fn signature (const xframe_node_t*, const xa11y_props_t*, int32_t depth, void*). Const-casts node pointer for caret_ctx_t storage (justified: xa11y_set_focus_node() requires non-const). F7 toggles on/off. Tab/Shift+Tab delegates to xa11y_focus_next/prev. xshell_caret_get_visual() for compositor cursor rendering. CB_TEXT_MAX == XFRAME_TEXT_MAX _Static_assert.

- init/gensd/backup.c (~430 LOC): Backup format: backup_hdr_t(64) + xstore_section_hdr_t(16) + KV pairs [u32 key_len | key | u32 val_len | val] + xblob_section_hdr_t(16) + blob entries [hash[32] | u32 data_len | data] + backup_footer_t(36 — SHA-256). Two-phase restore: validate hash first (constant-time comparison), then write. xblob_iter_read() only returns metadata — xblob_get() fetches actual blob data. bkp_write_u32() marked __attribute__((unused)). xstore_cursor_open() takes (store, txn, lower_key_t*, upper_key_t*, cursor*) — no separate len params.

- aetherboot/src/disk_encrypt.c (~900 LOC): UEFI pre-boot disk encryption prompt. EFI_NOT_READY, EFI_ABORTED, EFI_ACCESS_DENIED, EFI_SECURITY_VIOLATION defined locally (not in uefi_types.h minimal set). Uses `int` not `int32_t` (UEFI type env). serial_puts declared as file-scope weak extern (not in loop body). de_memcpy marked __attribute__((unused)). xsec_pbkdf2_sha256 + xsec_aes256gcm_decrypt declared as `int` return type with __attribute__((weak)). Password buffer zeroed via volatile write loop immediately after key derivation. Max 3 attempts before hard halt (cli; hlt). Minimal 8×16 bitmapped font embedded for GOP rendering.

## Critical API Discoveries

- xstore_cursor_open() signature: (store, txn, const xstore_key_t *lower, const xstore_key_t *upper, cursor*) — 5 args, not 7.
- xblob_iter_read() returns only: (blob, iter, out_hash[32], *out_len, *out_refcount) — NO blob data. Use xblob_get() separately.
- xa11y_walk_fn signature: (const xframe_node_t*, const xa11y_props_t*, int32_t depth, void*) — 4 args including props and depth.
- XFRAME_NODE_TEXT is the text node type for caret navigation.
- keyboard_nav.c keysym_to_mod_bit() is static — not exported. Each module that needs it must define its own copy.

**Why:** Saved to ensure future sprints wire these APIs correctly and avoid repeat compile failures.
**How to apply:** Before writing any xstore cursor or xblob iter code, use the exact signatures above.
