# GEN.OS Boot-Time Security Audit — 2026-03-29

**Files read:** kernel/xenos/core/main.c, kernel/xenos/mm/vmm.c,
devices/desktop/aetherboot/src/aetherboot.c, kernel/xenos/include/boot_handoff.h,
kernel/xenos/core/stack_guard.c, kernel/xenos/drv/iommu_vtd.c,
kernel/xenos/drv/elitebook_x360.c

## KEY CONFIRMED FACTS

- vmm.c line 16: `CR4.SMEP=0 assumed` — SMEP not set anywhere in kernel
- main.c kmain(): 14-step boot, NO CR4 write exists anywhere
- boot_handoff.h: ibrs/stibp/ssbd/umip/cet_ss/tsx_hle/tme ALL detected — NONE activated
- iommu_vtd.c: driver fully written — NOT called from kmain(). DMA completely unrestricted.
- stack_guard.c: canary = static 0xDEADC0DE5AFEC0DE — not randomized
- main.c PROC_USER_CODE_VA=0x400000 PROC_USER_STACK_VA=0x7FFF00000000 — FIXED, no ASLR
- -fno-stack-protector on all kernel C files (confirmed in compile comment)
- VERW: no VERW call in syscall.asm or entry.asm return paths
- MSR 0xC80 (JTAG lock): never written

## ALREADY CORRECT (verified)
- EFER.NXE set at step 4.5 before first CR3 load
- W^X in _vm_flags_to_pte (VM_WRITE|VM_EXEC silently drops VM_EXEC)
- KASLR kernel: RDRAND-based, 2MB-aligned, [0,64MB)
- TPM PCR[7] bootloader + PCR[9] kernel measured (Sprint 37/40)
- ECDSA P-256 kernel sig gate under GENOS_SECURE_BOOT (Sprint 40)

## NOT SUPPORTED ON i7-8565U (do not implement)
- Intel TME: Ice Lake+ only
- Intel PKU: absent from Whiskey Lake client
- Intel CET IBT/SHSTK: Tiger Lake+ only
- eIBRS: Comet Lake+ only (classic IBRS available, addresses Retbleed)

## SPRINT PLAN

### Sprint 47 (~116 LOC) — P0/P1 CPU hardware gates
- M01: SMEP CR4 bit 20 — main.c new cpu_security_features_init() after vmm_init
- M02: SMAP CR4 bit 21 + STAC/CLAC around user pointer accesses in syscall handlers
- M03: UMIP CR4 bit 11 — fold into same function
- M04/05/06: IBRS+STIBP+SSBD MSR 0x48 — in syscall_init() or new spectre_mitigations_init()
- M07: VERW before SYSRETQ and before IRET to CPL3
- M11: call iommu_vtd_init() from kmain after step 10a (ACPI), before USB/PCIe devices active
- M30: write MSR 0xC80 | (1ULL<<30) to lock JTAG debug interface

### Sprint 48 (~156 LOC) — P1 process isolation + entropy
- M08: seed stack canary from pal_random_bytes in stack_guard_init()
- M09: RDRAND slide for PROC_USER_CODE_VA and PROC_USER_STACK_VA per process
- M21: zero PFN field in non-present PTEs (L1TF mitigation)
- M22: disable TSX via MSR 0x122 if tsx_hle detected
- M23: SRBDS mitigation via MSR 0x123 bit 1 clear
- M31: FXSAVE64/FXRSTOR64 in context switch path (YMM registers from XJIT)

### Sprint 49 (~205 LOC) — P2 boot chain hardening
- M10: log IA32_UCODE_REV MSR at boot in AetherBoot
- M17: per-process syscall bitmask in xenos_process_t, checked at dispatch
- M26: FDE key wipe hook in panic.c + clean shutdown path
- M28: UEFI variable lock before ExitBootServices in aetherboot.c
- M29: TPM NV monotonic counter for anti-rollback in tpm.c + aetherboot.c

## IRON GATE CI CHECKS (GATE 9-15)
See full audit response for bash gate definitions.
Keys: CR4 bits 20/21/11 present in main.c, MSR_IA32_SPEC_CTRL written,
iommu_vtd_init called from main.c, VERW in syscall.asm, no static canary hex constants,
no fixed user VA constants, MSR 0xC80 written.

## Why: The IOMMU gap (M11) is the highest blast radius
Wi-Fi NIC (8086:A370 CNVi), NVMe, and Intel iGPU all perform unrestricted
DMA to arbitrary physical addresses. Compromised Wi-Fi firmware = kernel code overwrite.
iommu_vtd.c is fully written — this is a 10-LOC wiring call from kmain.

## How to apply
Sprint 47 is the mandatory security sprint before any production ISO build.
None of these features require new subsystems — all are MSR writes + one function call.
