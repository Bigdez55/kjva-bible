## Health Check P0 Bug (CONFIRMED PRESENT)
- File: `backend/app/api/api_v1/endpoints/monitoring.py` lines 138-145
- Bug: Returns HTTP 200 with `"status": "degraded"` when Redis or vLLM are down
- Current production state: Redis not provisioned → ALWAYS returns "degraded" with HTTP 200
- Fix: Add `if not all_ok: response.status_code = status.HTTP_503_SERVICE_UNAVAILABLE`
- CAUTION: Applying this fix immediately will trigger liveness probe restarts (Redis not ready)
- Must provision Redis FIRST, then apply the fix, or relax the `all_ok` definition

## JWT in localStorage (P1 — Wide Scope)
- JWT access token stored in localStorage in 20+ frontend service files
- authSlice.ts line 35: `token: localStorage.getItem('token')`
- auth.ts lines 46, 70, 108, 122: all write to localStorage
- httpOnly cookies ARE set on login (auth.py `_set_auth_cookies()`) but not relied on by frontend
- logout() does NOT call `response.delete_cookie()` — browser-side only cleanup
- Full migration requires removing ~20 localStorage references and switching to cookies: 'include'

## 2FA Not Enforced at Login (P1)
- 2FA IS implemented: `security.py` endpoint, pyotp TOTP, enable/disable/backup-codes
- 2FA is opt-in and NOT gated at the login endpoint
- `user.two_factor_enabled` is NOT checked during POST /auth/login
- Fix: add 2FA check in login endpoint: if user.two_factor_enabled, return partial token + require 2FA code

## SECRET_KEY Used for Both JWT and SSN Encryption (P1)
- `AES256Encryptor` defaults: `self.master_key = master_key or settings.SECRET_KEY`
- `auth.py` line 104: `encryptor = AES256Encryptor()` — uses JWT signing key to encrypt SSN
- Key rotation risk: rotating SECRET_KEY makes all stored SSNs unrecoverable
- Fix: create dedicated SSN_ENCRYPTION_KEY in Secret Manager, derived from ENCRYPTION_SALT

## VPC Firewall allow-vllm Still 0.0.0.0/0 (P1 — OPEN from 2026-02-26)
- Confirmed carry-forward from pre-implementation baseline
- Command to fix: `gcloud compute firewall-rules update allow-vllm --source-ranges=10.8.0.0/28,10.138.0.0/20`
- Blast radius: any compromised GCP compute can call vLLM directly, bypassing all auth

## CORS Wildcard Patterns Non-Functional (P2)
- FastAPI CORSMiddleware does NOT support glob/wildcard patterns
- `"https://elson-tb-2-*.vercel.app"`, `"https://*.us-west1.run.app"` are treated as LITERAL strings
- They never match — they're dead entries creating false sense of coverage
- No immediate security risk but creates config confusion

## Development Default SECRET_KEY Risk (P2)
- `config.py` line 413: `s.SECRET_KEY = "dev-only-secret-key-do-not-use-in-production-32chars"`
- Only activates if ENVIRONMENT == "development" AND SECRET_KEY is empty
- `cloudbuild.yaml` sets ENVIRONMENT=production, mitigating this in practice
- Add startup assertion: if ENVIRONMENT==production and DATABASE_URL starts with sqlite, raise RuntimeError

## SQLite Fallback Risk (P2)
- `config.py` line 99: `DATABASE_URL = os.getenv("DATABASE_URL", "sqlite:///./trading_platform.db")`
- Missing DATABASE_URL env var = silent SQLite fallback with data loss
- No startup assertion guards against this in production
