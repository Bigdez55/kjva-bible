# Reliability & Security Sentinel ‚Äî Persistent Memory

## Auth Chain Pattern (Confirmed 2026-02-18)

All AI endpoints (`aiTradingApi.ts`) use `baseQueryWithAuth` which reads from `localStorage.getItem('token')`.
This is a migration state ‚Äî httpOnly cookies are the long-term target but localStorage token is still active.
Any new API service file must use the same `baseQueryWithAuth` pattern ‚Äî never skip auth on AI/EFT endpoints.

## PII-to-EFT Compliance (Confirmed 2026-02-18)

Pattern is clean: proactive scan and portfolio analysis pass ZERO user identifiers to backend EFT endpoints.
User identity is derived server-side from JWT only. `portfolio_context` uses anonymous numeric aggregates.
`userName` prop passed to `useAIAdvisory` is display-only, not forwarded to LLM query parameters.

## XSS Risk: Txt Component (Confirmed 2026-02-18)

`src/components/elson-v2/ui/Text.tsx` renders via `<span>{children}</span>` ‚Äî React JSX text node escaping.
No `dangerouslySetInnerHTML` anywhere in the insight/analysis render paths.
Backend error messages are never surfaced verbatim from EFT hooks ‚Äî hooks return hardcoded error strings only.

## Known P2 Backlog Items

1. **Stale AI cache on shared-device logout** ‚Äî `getProactiveScan` and `getAIPortfolioAnalysis` have
   `keepUnusedDataFor: 300`. On logout, `aiTradingApi.util.resetApiState()` is not called. Previous
   user's insights visible for up to 5 minutes on shared devices. Non-blocking; no write risk.
   Fix: add `refetchOnMountOrArgChange: true` + dispatch resetApiState on logout.

2. **Bank account last4 in localStorage** ‚Äî `useWealthData.ts` stores `elson_linked_accounts` in
   localStorage (bank name + last4). Not full PAN; not PCI-in-scope. But should move server-side.
   Pre-existing pattern, not introduced in any recent sprint.

## useMarketTimer Safety (Confirmed 2026-02-18)

`useMarketTimer.ts` is purely client-side: no network, no Redux, no localStorage, no user data.
Imports: `useState`, `useEffect`, `useCallback` only.
Timer throttle: 60s intervals when >5min remaining, 1s when <=5min. Safe and efficient.

## npm Install Flag (Permanent)

This project requires `--legacy-peer-deps` for all npm operations.
Reason: react-scripts@5.0.1 declares `peerOptional typescript@"^3.2.1 || ^4"` but project uses typescript@5.x.
No `.npmrc` file ‚Äî flag must be passed explicitly. Never use `npm install` without `--legacy-peer-deps`.
