# Sprint 0 + Sprint 1 Security & Reliability Audit — 2026-03-02

## Verdict: CONDITIONAL — 6 P1 issues block Sprint 2 ISO release

Full report: docs/compliance/security-reliability-audit.md

## P1 Findings (All must be closed before Sprint 2 release ISO)

### P1-A: XKABI Handle Forgery — xkabi_capabilities.c
- Sprint 1 handles are bit-packed (slot|rights|generation), no HMAC.
- Any user-mode observer can forge handles for any slot/rights combo.
- FIX: Sprint 2 pal_sha256 HMAC over (slot||rights||gen||table_secret).
- GATE: NO handles to user-mode until HMAC is live.

### P1-B: PMM No Allocation Bitmap — pmm.c
- pmm_free_pages does not verify page was ever allocated.
- Caller with corrupt phys address can inject pages into buddy allocator.
- FIX: Sprint 2 per-zone allocation bitmap (4MB BSS for 128GB/4KB granularity).

### P1-C: Heap Large-Alloc Magic Forgery — heap.c kfree
- kfree checks (ptr - sizeof(large_hdr_t)) for LARGE_ALLOC_MAGIC.
- Attacker who can write before a slab slot can forge magic → controlled
  pmm_free_pages with attacker-chosen phys+order.
- FIX: Move large alloc metadata out-of-band (hash table by vaddr).

### P1-D: LCG CSPRNG Data Race — pal_aether.c
- LCG fallback uses plain static `st` variable, no atomic.
- On multi-CPU under future XENOS threading: data race, correlated output.
- FIX: Mark LCG non-threadsafe OR use __atomic for state. x86 RDRAND safe.

### P1-E: vDRAM FROZEN Eviction Loses Data — vdram.c
- demote_to_frozen releases COLD pages without NVMe persistence.
- vdram_freeze_region() returns PAL_ERR_NOT_SUPPORTED in Sprint 1.
- FIX: Wire nvme_write() before XDISP deploys persistent user data.
- GATE: Make FROZEN eviction fail hard until NVMe path is ready.

### P1-F: AetherBoot STORAGE_WRITE Overgrant — capability_init.c
- classify_protocol grants STORAGE_WRITE to all Block IO handles including ESP.
- FIX: Sprint 2 kernel re-classify block IO devices, restrict ESP writes to
  privileged kernel-only capability not derivable by user-mode.

## P2 Priority Findings (Sprint 2)

- P2-09 + P2-10: EFER.NXE set at step 9, AFTER vmm_init() at step 5. HHDM
  pages have no NX enforcement until syscall_init runs. Reorder: set NXE
  before vmm_init().
- P2-01: SFMASK missing IOPL bits (12-13). Add SFMASK_IOPL = (3u << 12).
- P2-08: UART serial_putc has no timeout. Hang deadlocks kernel.
- P2-04: Journal wire format has no per-entry sequence number.
