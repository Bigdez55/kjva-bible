Scout lane completed 2026-05-13. Key facts verified by reading source files:

**EXISTS:**
- Bootstrap fixed-point: `compiler/scc/tests/bootstrap/bootstrap_driver.sh` — scc1 SHA == scc2 SHA, no SKIP in v1.1 closure. Enforced in CI on every PR via `near-term-success.yml`.
- Deterministic tarball: `release/build_tarball.sh` uses fixed mtime + `--sort=name` + `--numeric-owner`.
- SHA-256 output: `tools/install/build_release.sh` writes `.sha256` file alongside tarball.
- Two-build byte-identity test: `tests/determinism/run_scc_reproducible.sh`.

**EXPLICITLY DEFERRED (comment in source):**
- GPG signing of tarballs: `tools/install/build_release.sh` line contains `# No GPG signing (DEFERRED — requires user key)`.
- No cosign, sigstore, in-toto, or SLSA tooling anywhere in the repo.

**MISSING:**
- No provenance JSON per release artifact.
- No `.asc` detached signature on SHA256SUMS.
- No SLSA L2 or L3 attestation documents.
- No property-based test runner, fuzz harness, or mutation harness.
- No bundle-level reproducibility proof (tarball bit-for-bit across two builds).

**Dependency surface:** Near-zero external deps. No Cargo.toml / package.json / requirements.txt / go.mod. Only external runtime link: `libSystem.B.dylib` (30 symbols, macOS only, G13 PARTIAL, Stage II resolution target). Only non-standard C include: `<stdarg.h>` in `scir_lint.c`.

**Report location:** `docs/reports/scout_v24_1_verification_supply_chain_readiness.md`

**Why:** Signing key ceremony is the single prerequisite that unblocks 4 of the 9 gates (artifact signing, provenance, SLSA L2, SLSA L3 design). Cannot be automated; must be a deliberate human action before any implementation sprint begins.
