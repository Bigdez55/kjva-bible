# Design Vision Stability Compliance Audit — 2026-03-02

## Scope
Audit of GEN.OS against the "reliable appliance" stability mandate: atomic updates, always-working
recovery, protected hardware paths, and CI regression coverage.

## Verdict Summary
- COMPLIANT: 6 areas
- PARTIAL: 4 areas (actionable gaps identified)
- MISSING: 2 areas (P1 defects requiring sprint work)
- VIOLATES: 0 areas

## KEY FINDINGS

### COMPLIANT
1. ISO integrity: SHA-256 + HMAC-based sig.json generated on every build (genos-build.py, release-gate.sh)
2. GRUB recovery entries: grub.cfg has 3 boot entries (normal, safe-graphics, rescue.target) — line 10-23
3. systemd watchdog: mesh-bridge (20s), entropy-bridge (15s), vdram-bridge (15s) all feed WATCHDOG=1
4. systemd restart policies: all 9 services have Restart=on-failure or Restart=always + StartLimitBurst
5. k3s deployment standard: update-service has startup/liveness/readiness probes, non-root, ReadOnlyRootFilesystem
6. Update service auth: hmac.compare_digest + os.environ["GENOS_SERVICE_TOKEN"] hard-fail — COMPLIANT

### PARTIAL
7. ISO signing: HMAC-SHA256 based (genos-build.py line 501) — NOT GPG. Provides tamper detection
   but not cryptographic non-repudiation. HMAC key is a local file, not HSM/GPG keyring.
   Mitigates: boot-from-tampered-ISO risk. Does NOT mitigate: build server compromise.
   File: build/genos-build.py lines 460-516; build/config/signing.key (secret, not committed).

8. GRUB A/B partition: MISSING from installed system. grub.cfg only covers live-ISO boot menu.
   genos_installer.py creates single-partition layout (EFI + one ext4 ROOT). No fallback/previous
   partition. Recovery relies on booting the live ISO again, not on an on-disk B partition.
   File: build/installer/genos_installer.py lines 98-105 (partition_disk).

9. CI regression coverage: CI covers Python (pytest), TypeScript (tsc + eslint), C (cppcheck + RFC-002
   test suites), k8s YAML validation. MISSING: no hardware-path regression tests (audio, USB, webcam,
   network, biometrics). Windows failure modes are not specifically tested in CI.
   Files: .github/workflows/ci.yml, foundation.yml, xenos.yml.

10. Update service implementation: platform/services/update/app.py is a STUB. /apply records
    operation in-memory dict but performs NO actual OS update steps (no squashfs swap, no partition
    write, no bootloader update, no state persistence). The "applied" field is set purely by comparing
    version strings. Comment at line 112 acknowledges this: "data is lost on pod restart".
    This is the most significant gap against the vision mandate.

### MISSING
11. A/B partition scheme (P1): Installed system has no dual-partition scheme for rollback. A failed
    update leaves the system in an unbootable or partially-updated state with no automatic fallback.
    The ONLY rollback path is to re-boot the live ISO and reinstall. This is equivalent to the Windows
    "Reset This PC" failure mode the vision explicitly rejects.
    Remediation: Implement A/B root partitions (slot A and slot B) in genos_installer.py. GRUB
    should write boot_slot variable; update service should write to inactive slot then set next-boot
    flag. On failed boot N times, GRUB reverts to previous slot automatically.

12. Pre-update state snapshot (P1): No mechanism captures system state before an update is applied.
    There is no btrfs snapshot, no overlayfs checkpoint, no squashfs swap with rollback pointer.
    If k3s service deployment fails mid-way, kubectl rollout undo handles the k3s layer, but the
    underlying OS layer has no equivalent.

## SPRINT RECOMMENDATIONS

### P1 — Must fix before GA
- UV-01: Implement A/B root partition layout in genos_installer.py. Add slot-aware GRUB config.
- UV-02: Implement actual OS update logic in platform/services/update/app.py (squashfs download,
  checksum verify, write to inactive slot, set boot flag, schedule reboot).
- UV-03: Add persistent operation store to update/app.py (SQLite Bronze tier, not in-memory dict).

### P2 — Sprint 2
- UV-04: Replace HMAC signing with GPG detached signature (using project GPG key, not HMAC).
  This enables cryptographic non-repudiation and revocation.
- UV-05: Add GRUB boot_slot + boot_tries counter logic to grub.cfg template.
- UV-06: Add hardware regression test matrix to CI: audio device enumeration, USB host detection,
  network interface detection, webcam v4l2 probe, fingerprint reader presence.
- UV-07: Add WatchdogSec to genos-shell.service (currently missing — labwc crash leaves no
  watchdog protection).
- UV-08: Add WatchdogSec and StartLimitBurst to genos-tablet-mode.service (currently missing
  both — unlike all other services).

### P3 — Backlog
- UV-09: Document deployment window policy in CI (NO FRIDAY AFTERNOON RELEASES). CI yml currently
  has no schedule block or time-of-week gate. Policy exists in agent manifest but not enforced
  in pipeline.
- UV-10: Add QEMU smoke-test step to CI pipeline (currently qemu-validate.sh exists but is not
  run in ci.yml — only in release-gate.sh --full mode).
