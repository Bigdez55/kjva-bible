## CRITICAL P1 BLOCKERS (must fix before next ISO)

### VULN-XSEC-01: RSA-2048 TLS 1.3 CertificateVerify — Functional Failure
- File: `sec/xsec/include/xsec.h` (`xsec_tls_ctx_t.peer_cert_pub_key[64]`), `sec/xsec/tls/tls13.c` Step 6
- `peer_cert_pub_key` is 64 bytes; RSA-2048 modulus is 256 bytes. Step 5 caps key copy at 64 bytes, stores `peer_cert_pub_key_len=64`. Step 6 RSA-PSS path checks `if (ctx->peer_cert_pub_key_len < 256u)` — always true for RSA → always aborts.
- All TLS 1.3 connections to RSA-keyed servers fail at CertificateVerify.
- Fix: Expand `peer_cert_pub_key[64]` to `[512]` and `xsec_x509_cert_t.pub_key[64]` to `[512]`.

### VULN-XSEC-02: Module-Static TLS Scratch Buffers — Race Condition
- File: `sec/xsec/tls/tls13.c`
- `tls_enc_scratch[16640]`, `s_recv_ct_buf[16640]`, `s_tls_hs_cert_chain[4]`, `s_client_hello_save[512]`, and 3 others are module-static (shared across all TLS contexts)
- Two concurrent TLS handshakes corrupt each other's crypto buffers → MITM-facilitation or crash
- Fix: Move all scratch into `xsec_tls_ctx_t` or a per-connection scratch struct.

### VULN-GOV-01: GateChainExecutor Default ALLOW for Unregistered Blocking Gates
- File: `Citadel/governance/decision_envelope.py` line 219
- `verdict = GateVerdict.ALLOW if not blocking else GateVerdict.ALLOW` — BOTH branches return ALLOW
- Correct: `verdict = GateVerdict.ALLOW if not blocking else GateVerdict.DENY`
- All blocking gates (Sarah, Esther, Magen, Ahki) default to ALLOW when evaluators not registered
- Race condition at startup / after member crash = unconditional governance bypass

### VULN-GOV-02: Magen Trust Score Spoofing
- File: `Citadel/governance/gate_evaluators.py` MagenGateEvaluator.TRUSTED_SOURCES
- `any(t in source for t in TRUSTED_SOURCES)` — substring match on "gen" and "genesys"
- `created_by="generated_by_attacker"` → "gen" in string → True → +0.2 unearned trust boost
- Fix: Replace with exact-set membership, remove "gen"/"genesys" from trusted set.

### VULN-GOV-03: SOUL_ISOLATION Invariant Is Dead Code
- File: `Citadel/council/runtime/invariants/invariant_engine.py` SoulRegistry
- Registers prefix `f"{agent_id}/"` (e.g., `"ruth/"`)
- `soul_manager._make_key()` format: `"soul:{agent}:{bucket}:{sub_path}"` (e.g., `"soul:ruth:persistent:key"`)
- `"soul:ruth:persistent:key".startswith("ruth/")` → False → invariant never triggers
- All Council members can read/write each other's soul state without violation
- Fix: Change registration prefix to `f"soul:{agent_id}:"` to match actual key format.

### VULN-GOV-04: governance_approved Self-Attestation — No Cryptographic Verification
- File: `Citadel/council/runtime/invariants/enforcement.py`
- `governance_approved = bool(context.get("governance_approved", False) or ...)`
- Any caller sets `context["governance_approved"] = True` → bypasses MEMORY_SAFETY 256MB guard
- No cryptographic proof of actual gate-chain approval required
- Fix: enforcement layer must receive a GovernanceVerdict object with envelope_id, verify it passed all blocking gates via the gate chain cache.

## P2 FINDINGS

### VULN-GOV-05: SoulManager Key Derivation — SHA-256 Not HKDF
- File: `Citadel/council/runtime/memory/soul_manager.py` `_derive_key()`
- `SHA-256(master_secret + agent_name)` — no salt, no domain separation, length-extension possible
- RFC 5869 HKDF required for proper key derivation
- Dev seed `SHA-256(b"GENOS_SOUL_MASTER_KEY_DEV_2026")` used when env var not set — deterministic, publicly derivable
- Fix: Replace with HKDF using `xsec_ffi.hmac_sha256` for PRK step.

### VULN-GOV-06: delete() Violates Member Reconstitution Doctrine
- File: `soul_manager.py` `delete()` method
- Fully functional: removes from in-memory dict + XSTORE B-tree + journal tombstone
- No governance gate, no SOUL_ISOLATION check
- Contradicts: "The memory is not recreated. The member is restored to it." (member_reconstitution_doctrine.md)
- Fix: Gate delete() behind 7-stage governance decision (min: Sarah + Esther blocking gates pass). Or soft-delete with tombstone preserving original data.

### VULN-GOV-07: Soul Journal in /tmp — Ephemeral and Unauthenticated
- File: `soul_manager.py` `_SOUL_JOURNAL_DIR = os.environ.get("SOUL_JOURNAL_DIR", "/tmp/genos/soul")`
- Journal lost on reboot. Journal entries have no HMAC — injectable by any /tmp writer.
- Fix: Move default to `/var/lib/genos/soul/journal`. Add HMAC-SHA-256 per entry using agent derived key.

### VULN-XSEC-03: HKDF Salt Size for SHA-384 Cipher Suite
- File: `sec/xsec/crypto/hmac.c` `xsec_hkdf_extract()`
- Default salt is 32 zero bytes. For TLS_AES_256_GCM_SHA384 (0x1302), RFC 8446 requires 48-byte zero salt.
- All TLS_AES_256_GCM_SHA384 sessions produce incorrect early_secret → handshake failure.
- Fix: Pass `hash_len` parameter to `xsec_hkdf_extract`; use 48 bytes for SHA-384.

### VULN-GOV-08: Covenant Pattern Matching — Unicode Bypass
- File: `Citadel/governance/covenant_enforcer.py`
- Plain ASCII substring match. Unicode confusables bypass ABSOLUTE covenants.
- Fix: Apply `unicodedata.normalize("NFKC", text)` + strip zero-width chars before matching.

## Architecture Notes — TLS 1.3 Handshake (Verified Correct)

9-step client flow verified: (1) ClientHello X25519 + cipher offers. (2) ServerHello cipher select. (3) X25519 shared + key schedule. (4) EncryptedExtensions. (5) Certificate + hostname verify. (6) CertificateVerify (Ed25519 works; RSA fails per VULN-XSEC-01). (7) Server Finished constant-time verify. (8) Client Finished sent. (9) App keys from post-Finished transcript (Sprint 46 NULL fix correct).

Key schedule verified: early_secret → handshake_secret → master_secret → app traffic per RFC 8446 §7.1.

Nonce construction verified: static IV XOR with 8-byte big-endian sequence number, right-aligned in 12-byte IV (RFC 8446 §5.3).
