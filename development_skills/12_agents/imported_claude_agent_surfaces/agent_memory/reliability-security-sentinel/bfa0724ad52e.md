# R1_PER Security and Evaluation Verification

**Date:** 2026-03-30
**Status:** DESIGN-PHASE ONLY — R1_PER/XCOG not yet implemented in live code

## Key Findings

### L5 Evaluation Gap (P1)
- L5.1 has NO R1_PER-specific metrics (no r1_per_encoding_fidelity, no xcog_opcode_accuracy)
- L5.2 anti-Goodhart cross-metric pairs do NOT cover encoding fidelity loss
- Systematic R1_PER encoder failures are invisible to the entire evaluation layer
- `r1_per_encoding_fidelity` should correlate with `task_completion_quality` (positive) and `revision_frequency_after_halt` (negative)
- Required: activate L5.4 (reserved slot) with perception fidelity metrics; add encoding_fidelity ↔ task_completion pair to L5.2

### Prompt Injection Immunity (Partially Valid)
- Core claim is architecturally sound — fixed opcode enumeration, no "execute arbitrary text" opcode in XISC
- RESIDUAL SURFACE 1: POS tagger misclassification to COMMAND if decision tree has fall-through default — must enforce fail-QUERY on ambiguity
- RESIDUAL SURFACE 2: XCOG_REFERENCE anaphora resolution can chain to TEXT_FRAGMENT entities that look like instructions — must be enforced as read-only data path
- RESIDUAL SURFACE 3: PAYLOAD field for freeform content is the classical injection surface — no sanitization spec exists
- RESIDUAL SURFACE 4: Fallback path bypassing XCOG encoding must be treated as full injection risk

### XKABI Rights Gap (P1 — INVALID claim)
- No XKABI_RIGHT_AI_INFER, no XKABI_RIGHT_AI_COMMAND, no RIGHT_EPISODIC_MEMORY defined
- Bit 11 is currently a gap in xkabi_rights.h (bits 0-10, 12-23 defined, 11 skipped)
- XKABI_RIGHT_ALL = 0x00FFFFFFu covers bits 0-23
- XCOG_INTENT(COMMAND) vs XCOG_INTENT(QUERY) must be different capability tiers before XMIND is wired to any shell endpoint
- Required: RIGHT_AI_INFER (bit 11), RIGHT_AI_COMMAND (bit 24+), RIGHT_EPISODIC_MEM (bit 25+)
- Adding bits 24-25 requires expanding XKABI_RIGHT_ALL — breaking change, needs capability migration

### SHA-256 Audit Hash (Conditionally Valid)
- input_hash[32] provides non-repudiation ONLY if computed over raw NL before any transformation
- Does NOT prove encoder behaved correctly — covers input, not the XCOG transformation
- Need xcog_sequence_hash[32] for full tamper detection of the encoding step
- 1024-entry XSEC audit ring is insufficient for AI inference audit volume — ring overwrites destroy forensic trail
- R1_PER audit events must route to XSTORE, not just the ring buffer

### Deterministic C Pipeline
- Shift-reduce POS tagger cannot produce out-of-grammar output but can produce incorrect in-grammar parses
- XISC_ANNOTATE (0x50) "embed policy annotation" is unconstrained — needs capability gate to prevent entity-type injection

## xkabi_rights.h Current State (read 2026-03-30)
- Bits 0-10 defined, bit 11 is a GAP, bits 12-23 defined
- CAMERA = bit 9, MICROPHONE = bit 10 (Sprint 45 additions, verified)
- XKABI_RIGHT_ALL = 0x00FFFFFFu (24 bits, 0-23)
- No AI/cognitive rights exist

## Iron Gate Requirements Before R1_PER Implementation
1. XKABI_RIGHT_AI_INFER + RIGHT_AI_COMMAND added to xkabi_rights.h
2. PAYLOAD sanitization spec documented (what is prohibited, is it training or code constraint)
3. L5.4 activated in unified_model_spec.json with perception fidelity metrics
4. Decision tree fallback documented as fail-QUERY (not fail-COMMAND)
5. R1_PER audit events routed to XSTORE not just 1024-entry ring

**Why:** XMIND inference is the AI backbone of the entire GENESYS system. A compromised perception layer (R1_PER) that misclassifies user intent has blast radius = 100% of all AI-mediated operations. Zero Trust requires these gates before any implementation commit.
