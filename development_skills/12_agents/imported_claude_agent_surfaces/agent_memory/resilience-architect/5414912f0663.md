# Sprint 3 P2 Hardening — Resilience Audit (2026-03-05)

## Summary

Audit of 5 error/failure paths introduced or affected by the Sprint 3 P2 hardening wave.

### 1. TLS CertificateVerify sig_len_cv != 64 Rejection (tls13.c:920-928)
- **Alert sent before return**: YES — `tls_send_alert(ctx, FATAL, DECODE_ERROR)` at line 924
- **Key material zeroed before return**: NO — returns immediately without zeroing cv_content or cv_transcript
- **Context state set to CLOSED**: NO — state remains at CERTIFICATE_VERIFY
- **Verdict**: FAIL — key material leak on error path. Only xsec_tls_close() zeros keys (line 1122-1131), but caller must know to call it. The early return at line 927 skips the `xsec_secure_zero(cv_content)` at line 953.

### 2. DNS pal_random_bytes() Return Value (dns.c:384)
- **Return value checked**: NO — `pal_random_bytes(qid_bytes, 2u)` return value is discarded
- **Fallback if RDRAND fails**: qid_bytes remains stack-uninitialized; QID is garbage from stack
- **pal_random_bytes behavior**: Returns PAL_ERR_NOT_SUPPORTED if RDRAND unavailable (pal_aether.c:806,812,821)
- **Same pattern in TCP**: `pal_random_bytes(rnd, 4)` at tcp.c:246 — also unchecked
- **Same pattern in gossip**: gossip.c:166,566 — also unchecked
- **Verdict**: UNSAFE — on non-x86 or RDRAND failure, QID is uninitialized stack data. Not cryptographically random, but also not predictably sequential. Partial mitigation: `if (qid == 0) qid = 1` prevents QID=0 but not predictability.

### 3. TCP Timer Failure Recovery (tcp.c:1264)
- **Timer model**: TCP does NOT use PAL timers. Uses polling via `xnet_tcp_timer_tick()` called by `xnet_tick()` at ~1Hz.
- **Timestamps**: `pal_time_now_ns()` used for all timing (RTO, TIME_WAIT, keepalive)
- **Failure mode**: If `xnet_tick()` stops being called, ALL TCP timers stall — no retransmissions, no TIME_WAIT cleanup, no keepalive. TCB pool fills up.
- **If pal_time_now_ns() fails**: Returns 0, causing immediate timeout of all timers — mass retransmit storm + immediate TIME_WAIT cleanup.
- **Verdict**: GRACEFUL for timer creation (no timers to create/fail), but SILENT FAILURE if tick stops. No watchdog on tick frequency.

### 4. Poly1305 AEAD Auth Failure (chacha20.c:458-478)
- **Returns XSEC_ERR_AUTH_FAIL**: YES — line 473
- **Plaintext zeroed on tag mismatch**: YES — `xsec_secure_zero(pt, ct_len)` at line 471
- **Computed tag zeroed**: YES — `xsec_secure_zero(computed_tag)` at line 472
- **Plaintext zeroed on core error**: YES — line 466
- **Constant-time comparison**: YES — `xsec_ct_eq()` at line 470
- **Verdict**: PASS — exemplary error handling. All sensitive material zeroed on every failure path.

### 5. X509 SAN Fallback (x509.c:750-803)
- **SAN parsing**: NOT IMPLEMENTED — no SubjectAltName (OID 2.5.29.17) parsing exists
- **CN-only validation**: hostname_match() only checks CN extracted from subject DN (line 796)
- **RFC 6125 compliance**: MISSING — RFC 6125 Section 6.4.4 states SAN dNSName MUST be checked before CN, and CN SHOULD NOT be used if SANs are present
- **Boundary fix quality**: The CN boundary fix (", CN=" prefix matching) is correct and prevents injection
- **Verdict**: MISSING — no SAN fallback. Certs using SANs (which is standard modern practice) will fail hostname validation even with correct SANs.

## New Bugs Found

1. **P1: TLS key material leak on CertVerify early return** — All 7 `return XSEC_ERR_TLS_HANDSHAKE` paths in steps 5-7 (lines 778-983) exit without zeroing handshake secrets (ecdhe_priv, shared_secret, hs_traffic keys). Only xsec_tls_close() does cleanup, but the caller receives an error and may not call close.

2. **P2: pal_random_bytes() unchecked in dns.c, tcp.c, gossip.c** — 4 call sites across 3 files ignore the return value. If RDRAND is unavailable (non-x86, or persistent RDRAND failure after 10 retries), QID/ISN/node-ID are uninitialized stack garbage.

3. **P2: No SAN support in x509 hostname verification** — Modern TLS certificates use SubjectAltName exclusively. Any cert with SANs but no matching CN will fail validation, breaking connections to most real-world servers.
