# Sprint 3 Telemetry Architecture — Signal Taxonomy & Hook Points

## Date: 2026-03-05

## Signal Count
- 10 raw network signals (NET-S01 to NET-S10)
- 7 derived network signals (NET-D01 to NET-D07)
- 7 raw security signals (SEC-S01 to SEC-S07)
- 4 derived security signals (SEC-D01 to SEC-D04)
- 7 raw package signals (PKG-S01 to PKG-S07)
- 4 derived package signals (PKG-D01 to PKG-D04)
- Total: 39 signals (31 raw + 8 derived, but some derived overlap)

## Key Architectural Decisions

### Three-Tier Aggregation
- Tier 0: Kernel C atomic counters + ring buffers (zero alloc)
- Tier 1: `xnet_telem_tick()` every 10s, snapshots + deltas, writes to journal
- Tier 2: Platform Python polls journal via IPC every 30s

### Memory Budget (new BSS): ~2.2KB total
- RTT histogram: 128B (16 exponential buckets)
- Tier 1 snapshot pair: ~2KB
- XPKG counters: 40B
- Existing rings (audit 80KB + journal 4MB) are reused

### CPU Budget: 1% of single core on HP EliteBook
- ~50 atomic_add/s at typical load
- Tier 1 tick: O(1) per 10s window
- Tier 2 poll: ~5ms per 30s cycle

### Critical Hook Points
- RTT histogram: inject after tcp.c:455 (tcp_update_rtt)
- Audit drain: new xsec_audit_drain_since() in audit.c
- Telem tick: add xnet_telem_tick() call at netif.c:478 (in xnet_tick)
- Journal reader: de-stub journal_read_recent() -> journal_drain_since()

### R-squared Baselines (Expected)
- XNET packets: 0.65-0.80 (stable)
- TCP RTT: 0.50-0.70 (moderate)
- Gossip: 0.45-0.55 (near CHAOS threshold)
- TLS: 0.40-0.55 (near CHAOS threshold)
- XPKG: N/A (too infrequent)

### Highest-Intensity Signals
- AUTH_TAG_MISMATCH: 0.90 (active attack indicator)
- DECRYPT_FAIL: 0.85 (crypto failure)
- Signature gate FAIL: 0.95 (supply chain compromise)
- Integrity gate FAIL: 0.90 (archive corruption)
- Handshake burst (>10/60s): 0.95 (coordinated attack)
