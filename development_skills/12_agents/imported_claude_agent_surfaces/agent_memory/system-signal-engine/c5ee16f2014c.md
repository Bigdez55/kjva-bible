## Detailed Findings

### P2-AUD-01: tls13.c silent rejection paths (MEDIUM)

Three CertificateVerify rejection paths lack xsec_audit_log():

1. Line 879: `pt_len < 8` minimum message length check — sends alert, no audit
2. Line 891: `cv_pos + sig_len_cv > pt_len` bounds overflow — sends alert, no audit
3. Line 941: `peer_cert_pub_key_len < 32` key too short — sends BAD_CERTIFICATE alert, no audit

Contrast with lines 925, 937, 948 which DO log HANDSHAKE_FAILED correctly.

### P2-AUD-02: dns.c zero audit integration (LOW)

- QID generated via pal_random_bytes (line 382-386), never logged
- No audit events for query, success, NXDOMAIN, or timeout
- Only logging: pal_console_printf for truncated response (line 306) — volatile
- Impact: Kaminsky-style attack forensics impossible

### P2-AUD-03: refusal_gate.c zero persistent audit (HIGH — REGRESSION)

- Originally flagged as S3-PKG-02 in batch SSE-S3-2026-0305-001
- Listed as P0 action item #1 in that batch — NOT REMEDIATED
- refusal_gate.c includes xsec.h (line 43) but has zero xsec_audit_log calls
- install.c includes xsec.h (line 48) but has zero xsec_audit_log calls
- All 9 gate results emitted only via pal_console_printf (volatile)
- Force-demotion path (FAIL->WARN, line 532-535) is especially dangerous unlogged
- Grep for xsec_audit_log across entire pkg/xpkg/ = zero matches

### P2-AUD-04: tcp.c zero state logging (LOW)

- All state transitions are direct assignment (e.g., t->state = TCP_ESTABLISHED)
- Zero pal_console_printf in tcp.c
- initcwnd = 10*MSS (line 464, RFC 6928) is static init — transparent to logs as expected
- Recommendation: aggregate counters in Tier 1 telemetry tick, not per-event audit

### P2-AUD-05: Audit ring capacity under TLS churn (MEDIUM)

- Ring: 1024 entries (XSEC_AUDIT_RING_SIZE)
- Per successful handshake+close: 3 events (CERT_VERIFIED + HANDSHAKE_COMPLETE + SESSION_CLOSED)
- 10 TLS/s = ring wraps in ~34 seconds
- 1 TLS/s = ring wraps in ~341 seconds (~5.7 minutes)
- dropped_events counter (S3-DI-H01 fix) tracks overflow but data is lost
- Recommendation: drain callback, configurable size, or priority eviction
