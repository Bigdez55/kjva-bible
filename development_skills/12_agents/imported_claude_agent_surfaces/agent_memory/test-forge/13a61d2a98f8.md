### Source Files Covered

- orange_docs.c (750 LOC) -- Docs: multi-doc, folders, search, formatting
- drive.c (751 LOC) + drive_blob.c (574 LOC) -- Drive + XBLOB content-addressed storage
- calendar.c (590 LOC) -- Calendar: month-view, events, Sakamoto algorithm
- mail.c (883 LOC) -- Mail: 3-panel, folders, compose, xwidget_input
- orange_sheets.c (882 LOC) -- Sheets: cells, formula eval (SUM/AVG/COUNT), CSV
- orange_photos.c (1,007 LOC) -- Photos/Slides: EXIF parser, albums, slideshow stub
- orange_store.c (521 LOC) -- Shared XSTORE injection layer

### Pattern

- stdlib-only Python (os, re, random, string, unittest, pathlib) -- NO external deps
- Mock objects simulate C app state machines (MockNote, MockDoc, MockDriveItem, etc.)
- White-box: source structure/include/constant validation via regex
- Black-box: behavioral tests via mock state machines
- Grey-box: cross-component wiring (XSTORE key schemas, injection functions)
- All hex PRNG seeds are valid Python int literals (no G-Z chars)
- Runtime: ~1.05s for all 503 tests (+ 77 pre-existing = 580 total in tests/orange/)
