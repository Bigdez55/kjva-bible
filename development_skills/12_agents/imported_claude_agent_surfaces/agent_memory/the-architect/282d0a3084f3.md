# Orange Suite Architectural Defect Register
# Review date: 2026-02-24
# Scope: go-notes, go-calendar, go-drive + orange/common/ (16 files)

## CRITICAL

### C-1: sandbox:true blocks process.env in preloads — TOKEN always empty string
- Files: all three electron/preload.ts
- Cause: Electron sandbox:true restricts Node.js env access in preload context
- Effect: GENOS_SESSION_TOKEN resolves to "" — every auth request silently fails
- Fix: Pass config via ipcMain.handle("app:config") from main.ts; preload calls ipcRenderer.invoke

### C-2: yjs in doc-model.ts not declared in any app package.json
- File: orange/common/doc-model.ts line 1
- Effect: Hard build failure when any bundler resolves doc-model.ts
- Fix: Either add yjs to each app's package.json, OR split into doc-types.ts (zero imports) and doc-model-yjs.ts

### C-3: IndexedDB openDB() called per operation — no singleton connection
- File: orange/go-notes/src/main.tsx lines 24-36, called in loadNotes/saveNote/deleteNote
- Effect: Connection storm at keystroke speed; O(n) open calls
- Fix: Module-level `let _db: IDBDatabase | null = null;` singleton with onversionchange cleanup

### C-4: TipTap onUpdate fires saveNote on every keystroke — no debounce
- File: orange/go-notes/src/main.tsx lines 95-104
- Effect: IDB write at typing speed + React re-renders entire notes array per character
- Fix: useRef timer debounce 500ms; also setContent(html, false) to suppress recursive update

### C-5: Calendar stores ALL events in single OrangeDocument body — O(n) payload on every write
- Files: go-calendar/src/main.tsx line 241, go-calendar/electron/preload.ts lines 25-30
- Effect: With 500 events, every save re-serializes and re-uploads entire history
- Fix: Store each event as individual OrangeDocument keyed by event.id; use tags["YYYY-MM"] for range queries

### C-6: FilePreview constructs image/PDF data URIs with btoa(res.text()) — breaks on binary
- File: orange/go-drive/src/main.tsx lines 19-41
- Cause: preload uses res.text() on binary response; btoa() throws DOMException on non-Latin1 bytes
- Fix: Preload returns arrayBuffer → Uint8Array → btoa(binary string); renderer uses value directly

### C-7: Download button uses stale `preview` state — delivers wrong file data
- File: orange/go-drive/src/main.tsx lines 159-167
- Effect: Downloading fileB without previewing it first downloads fileA or draftContent "hello"
- Fix: Each download handler fetches its own data via window.goDrive.download(item.name) on-demand

## HIGH

### H-1: common/ not copied to dist-electron/linux-unpacked/ — preload module resolution fails at runtime
- Files: all three package.json build scripts
- Effect: preload.js cannot resolve ../../common/sync-client at runtime
- Fix: Bundle preload via vite with rollupOptions entry, OR add ../../common to cp in build script

### H-2: Empty catch blocks in all preloads — sync failures are invisible
- Files: all three electron/preload.ts
- Fix: `console.error("[app:sync] failed:", err)` + return `{ ok: false, reason: string }`
- Renderers must inspect ok field and surface failure in UI

### H-3: Calendar load() casts events array without runtime validation
- File: go-calendar/src/main.tsx lines 209-212
- Effect: Corrupt/invalid events cause getEventDate() to return Invalid Date; events placed at hour 0
- Fix: isEventItem() type guard; filter(isEventItem) before setEvents

### H-4: share() sends username as uid without resolving to stable UID
- File: go-drive/electron/preload.ts lines 47-57
- Effect: Identity Service receives display username not UID; also filePath not included in grant body
- Fix: Add resource: filePath to grant body; expose resolveUser(username) backed by IdentityClient

### H-5: TipTap useEffect missing editor and active in dependency array
- File: go-notes/src/main.tsx lines 122-129
- Effect: Stale closure — editor init race; setContent without false flag causes infinite update loop
- Fix: deps = [editor, selected, dbReady, active?.body]; use setContent(html, false)

### H-6: reload() in go-drive not wrapped in useCallback — stale closure in useEffect
- File: go-drive/src/main.tsx lines 69-92
- Fix: useCallback(async () => {...}, [currentPath]); useEffect deps = [reload]

### H-7: Export produces .md extension with HTML content
- File: go-notes/src/main.tsx lines 281-290
- Fix: Export as .html, or install @tiptap/extension-markdown and use editor.storage.markdown.getMarkdown()

### H-8: AgendaView groups by toLocaleDateString() — locale-sensitive key collision
- File: go-calendar/src/main.tsx lines 155-167
- Fix: Use ev.time.slice(0,10) (ISO "YYYY-MM-DD") as grouping key; format separately for display

### H-9: WeekView renders 168 cells each with full events.filter() — O(events × 168) per render
- File: go-calendar/src/main.tsx lines 92-118 (MonthView same pattern)
- Fix: useMemo pre-index Map<"YYYY-MM-DD", EventItem[]>; cell lookup is O(1)
