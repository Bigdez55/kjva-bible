# Dev Environment Setup Notes

## Identity Service — Required Dev Dependencies (2026-02-26)

### Root Cause
The identity service test suite (`platform/services/identity/test_identity_app.py`) failed with
`ModuleNotFoundError: No module named 'bcrypt'` because the shared requirements file was not
installed in the dev environment.

### Single Requirements File
All platform service dependencies (identity, sync, provenance, etc.) are declared in ONE file:

```
platform/services/requirements.txt
```

This file is the single source of truth. There is NO separate per-service requirements file.
Install everything before running any platform service tests:

```bash
pip install -r platform/services/requirements.txt
pip install "pyjwt[crypto]==2.10.1"   # EdDSA algorithm support
```

### Why pyjwt[crypto] Is Separate
`requirements.txt` lists `pyjwt==2.10.1` (bare) but the identity service uses EdDSA signatures,
which require the `cryptography` extra. `cryptography==44.0.1` is already in requirements.txt
as a first-class dependency, so `pip install "pyjwt[crypto]"` is a no-op in practice — but it
is the explicit, correct install target that communicates intent.

### Dependency Chain That Must All Be Present
- `bcrypt==4.2.1` — password hashing (hashpw/checkpw in create_user + login)
- `pydantic[email]==2.10.6` → `email-validator>=2.0.0` — EmailStr field in UserCreate/UserInfo
- `sqlalchemy==2.0.38` — ORM for SQLite/PostgreSQL
- `cryptography==44.0.1` — Ed25519 key generation (pyjwt EdDSA backend)
- `pyjwt==2.10.1` — JWT encode/decode with EdDSA algorithm

### Permanent Prevention Rule
Before running ANY platform service test locally:
```bash
diff <(pip freeze | sort) <(sort platform/services/requirements.txt)
```
If there are uninstalled entries, install from requirements.txt first. Do NOT chase
`ModuleNotFoundError` one package at a time — install the whole file.
