## ADR-S26-01: GEN Shell Electron Architecture
File: `docs/adr/ADR-S26-01-gen-shell-electron-architecture.md`

Key recorded decisions:
- Electron is a PROCESS CONTAINER only — never a UI framework.
- `electron/` = main.ts, preload.ts, clipboard-monitor.ts. All Electron APIs live here.
- `src/renderer/` = XFRAME/React only. No `import from 'electron'` or `node:*` permitted.
- `contextIsolation: true`, `sandbox: true`, `nodeIntegration: false` — mandatory security settings.
- Renderer access to platform services flows exclusively through `window.genOS` (contextBridge).
- Renderer crash recovery handled via `render-process-gone` event + reload path in main.ts.
- Settings persistence: atomic write (tmp + rename) to `/run/genos/settings.json`.
- State watching: `fs.watch()` on PARENT DIRECTORY (not file) — survives inotify atomic renames.
- labwc: transitional Wayland compositor, QEMU/CI boot test only. Not production.
- Enforcement: ESLint `no-restricted-imports` on `src/renderer/**` recommended.

## ADR-S27-01: vDRAM Four-Tier Memory Hierarchy
File: `docs/adr/ADR-S27-01-vdram-tier-scoring.md`

Key recorded decisions:
- Four tiers: HOT (RW-mapped), WARM (RO-mapped), COLD (compressed), FROZEN (evicted+NVMe).
- Scoring function: idle time in ms vs two configurable thresholds (NOT LFU, NOT frequency decay).
  - `demote_threshold_ms` default = 5,000 ms (HOT → WARM)
  - `cold_threshold_ms` default = 30,000 ms (WARM → COLD)
- WARM tier exists specifically for O(1) write-fault promotion (single pal_map_protect call).
- Thermal override: `VDRAM_HINT_THERMAL` flag triggers immediate HOT→WARM under thermal pressure.
- `VDRAM_HINT_PIN_HOT`: region never demoted by scanner — use for XMIND active inference weights.
- Scanner holds `g_lock` for full 65,536-region scan — worst-case ~650 µs per 100 ms interval.
- `VDRAM_MAX_REGIONS = 65536` — 9 MB BSS, supports full Llama 3.2 3B Q4_0 + KV-cache.
- Memory budgets: HOT/WARM/COLD each default to 25% of total physical RAM (auto-computed).
- FROZEN→HOT currently returns zeroed pages (NVMe read path deferred to Sprint 30+).
- SHA-256 Merkle accumulator over integrity-verified COLD→HOT promotions (attestation chain).
- Lease mechanism: `vdram_lease()` prevents scanner from demoting regions under active access.
- Single lock (`g_lock` spinlock). No lock ordering problem — only one lock exists.
- `access_count` tracked but not used in scoring. Reserved for future LFU-hybrid evaluation.

## ADR-S38-01: XISC Linux Syscall Translation Design
File: `docs/adr/ADR-S38-01-linux-syscall-translation.md`

Key recorded decisions:
- Target: STATIC-LINKED Linux ELF64 x86-64 binaries only.
- Dispatch table covers 70+ routed syscall numbers (not the original "25" figure from the brief —
  the brief referenced the Sprint 38 baseline; Sprint 45 Wave 2 extended significantly).
- `LINUX_FD_MAX = 256` per fiber (upgraded from 64 in Sprint 39 after network tests hit limit).
- fd_table fields: handle, type (VNODE/CHANNEL/EPOLL), flags (FD_CLOEXEC), metadata (epoll_state*).
- `MAP_SHARED` → ENOSYS (VMO cross-process not wired).
- File-backed mmap (`fd != -1, !MAP_ANONYMOUS`) → ENOSYS.
- `fork`/`vfork`/`execve`/`wait4` → ENOSYS (Sprint 42+ target).
- 32-bit ELF, ARM64 ELF → rejected at load time (`e_machine != EM_X86_64`).
- Dynamic linking (PT_INTERP) → not followed, ENOSYS on execve.
- `SYS_stat` (legacy #4) routed to `linux_sys_newfstatat` with AT_FDCWD.
- All Linux ABI constants (syscall numbers, MAP_*, O_*, errno values, struct layouts) defined
  locally — zero Linux kernel headers anywhere in the file.
- XKABI→Linux errno translation table documented in the ADR.
- struct linux_stat = 144 bytes (x86-64 wire format). ARM64 port requires different layout.
- CI enforcement: `linux-header-scan` gate in sprint39.yml + xisc_ci.yml (dual enforcement).
