### IR Generator key contracts (xcc_irgen.h + xcc_irgen.c) — UPDATED Sprint 45
- PAL_TYPES_PROVIDED must be defined before including xjit.h in host builds — xjit.h force-sets PAL_FREESTANDING which fires manual typedef blocks conflicting with libc
- ABI PARAM COUNT: XCC_IRGEN_MAX_PARAMS is arch-conditional in xcc_irgen.h:
    `__aarch64__` / `__arm64__` → 8 (AAPCS64: X0-X7)
    default (x86-64)            → 6 (SysV AMD64: RDI/RSI/RDX/RCX/R8/R9)
- Vreg layout: 0..(MAX_PARAMS-1) = ABI param slots, vreg 200 = symbolic FP (XCC_IRGEN_RBP_VREG), next_vreg starts at (MAX_PARAMS+2)
- XJIT must pin vreg 200 to RBP (x86-64) or X29 (ARM64) — this is a cross-component contract
- Local vars: bump allocator growing downward (negative frame_top offset from FP); align to min(type_size, 8)
- Comparisons emit branch-materialise pattern: CMP + Jcc + MOV_RI 0 + JMP end + label + MOV_RI 1
- OP_DIV and OP_MOD are TODO stubs — XJIT has no DIV opcode yet (Sprint 47)
- xcc_type_sizeof() is a weak symbol in host builds; real impl provided by type checker TU
- Switch: two-pass compile (dispatch jump table first, then body with placed labels)
- Compile: `gcc -std=c11 -Wall -Wextra -pedantic -DXCC_HOST_BUILD -Itools/xcc/include -Idevices/desktop/jit/xjit/include -Ipal/include`

### Type checker key contracts (xcc_typeck.c + xcc_type.h) — UPDATED Sprint 45
- `xcc_typeck_init()` pre-allocates all 16 builtin types + file scope; registers fixed-width typedefs
- `xcc_typeck_check(ctx, root)` walks full AST, fills every expression node's `node->type` field
- `xcc_typeck_free(ctx)` uses `ctx->all_types[]` flat registry for single-pass cleanup — no ownership chains
- Freestanding: `tc_memset/memcpy/strlen/strcmp/strdup` are inline statics; `tc_strndup` tagged unused (reserved for IR gen / Sprint 46 designated initializers)
- `xcc_type_common()` = C11 §6.3.1.8 usual arithmetic conversions; `promote_integer()` = §6.3.1.1
- `compute_struct_layout()` = natural alignment; union = max_size rounded to max_align
- LONG DOUBLE: arch-conditional via XCC_LDOUBLE_SIZE / XCC_LDOUBLE_ALIGN macros in xcc_typeck.c:
    x86-64: size=10, align=16 (80-bit x87 extended; 16-byte ABI stack slot)
    ARM64:  size=16, align=16 (IEEE 754 binary128 / __float128)
- Error recovery: unknown identifiers fall back to t_int so later expressions still get a type; error count accumulated, never abort on first failure
- `tc_strndup` uses `__attribute__((unused))` — same pattern as parser helpers exposed only to tests

## USB HCD Weak-Extern Pattern (elitebook_x360.c §10a)

`hp_usb_enumerate_device()` is the gate: it must set `dev->present = 1u` before
`hp_usb_bulk_transfer()` can deliver data. The weak-extern triad is:
  `xhci_enumerate_devices` (scan ports) + `xhci_find_device` (VID:PID lookup) + `xhci_bulk_transfer` (TRB submit).
All three declared `__attribute__((weak))` — resolve to NULL in standalone builds; active when xhci.o linked.
Fast-path: if `dev->present` is already set, re-validate via `xhci_find_device` only (skip full port scan).
`xhci_enumerate_devices()` is idempotent (xHCI internally tracks `device_count`).

### Memory Budget (Decisive Constraint)
- `XSTORE_WAL_BUF_SIZE` = 65536 * 4384 bytes = **274 MB per store**
- Cache buffer = 1024 * 4112 bytes = **4 MB per store**
- Total per store: ~287 MB
- 4 separate stores = ~1.1 GB — ruled out on EliteBook x360

- `orange_ramdisk_ctx_t`: `{pal_handle_t pages_handle; pal_handle_t map_handle; uint8_t *base; uint32_t capacity_pages; pal_spinlock_t lock}`
- `read_page`: array index into mapped pages; zero-fill if page_no >= capacity (new page)
- `write_page`: grow via `orange_ramdisk_grow()` if page_no >= capacity (doubles up to 4096 pages max)
- `sync`: no-op for RAM disk; Sprint 43 upgrades to `pal_vfs_fsync`
- Initial allocation: 256 pages = 1 MB; grows by doubling up to 4096 pages = 16 MB
- Graceful degradation: on PAL alloc failure, inject NULL to all 4 apps (they check `if (!s_*_store) return`)

- `devices/console/remote/` — Serial management: SOH/CMD/LEN/PAYLOAD/CSUM framing + HMAC-SHA256 auth
- `devices/console/dev/` — XISC debugger: 22-opcode disasm, 16 breakpoints, hex dump, XJIT hooks
- `devices/console/boot/console_main.c` — GENSD entry: mode detect, banner, watchdog arm, 4 modes
- `devices/console/Makefile` — `make syntax` for CI; ARM64 cross via `make cross-arm64 CROSS=aarch64-linux-gnu-`
- `devices/console/genvirt/` — DEV ONLY (LLVM validation vehicle); `#error` blocks production builds
- Boot mode: kernel cmdline `console=recovery|remote|dev`, `genos.boot_mode=N`, or auto-detect
- All XJIT/XSEC/GENSD hooks are `__attribute__((weak))` — fail-closed or degraded-safe when absent
- Pitfall: static helper functions unused in a TU → `-Wunused-function` becomes `-Werror` failure
