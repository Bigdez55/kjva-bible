# GEN.OS Deployment Architecture — Docker to Real Hardware

## ADR Reference: ADR-DEPLOY-01 (2026-03-29)

## Key Findings from Codebase Audit

### BSS Bloat Root Cause
- AetherBoot `kern_span` limit raised to 256MB in Sprint 46 (was 64MB)
- Kernel BSS: 286MB — overflows by ~30MB
- Primary contributors: XSTORE WAL ring (65,536 × entry_size), XPKG registry arrays
  (g_registry with XPKG_MAX_INSTALLED=512), XMIND static weight buffers
- `vmm_bss_demand_zero_init()` already implemented (Sprint 46) — only first 64KB hot-mapped
- The 286MB file size is physical ELF memsz; runtime touch is far smaller

### Boot Chain
- AetherBoot: `devices/desktop/aetherboot/src/aetherboot.c` — raw UEFI, no EDK2
- WiFi firmware handoff: `xenos_boot_info_t.wifi_fw_phys` / `wifi_fw_size` — defined, stub driver
- gen_iso.c: `tools/iso/gen_iso.c` — ISO 9660 PVD + manifest library complete
- boot-test.sh: `tools/boot/boot-test.sh` — FAT32 superfloppy QEMU pipeline, BOOT VERIFIED

### Secure Boot Status
- XPKG_ROOT_FINGERPRINT: dev key auto-generated from kernel SHA-256 hash (sprint 22)
- `tpm.c` exists in AetherBoot — TPM measurement hooks present
- No PKCS#7 / sbsign chain yet — secure boot requires new work

### WiFi
- `kernel/xenos/drv/wifi.h` — stub driver, PCI VID:DID = 8086:A370 (Intel AC 9560, CNVi)
- `wifi_fw_phys` in boot_info_t — AetherBoot firmware handoff path defined
- `devices/desktop/apps/wifi/` driver created in Sprint 42 — HP-specific

### Docker Role
- `build/Dockerfile.buildenv` — dev/CI build environment (Ubuntu 24.04)
- `tools/docker/Dockerfile.toolchain` — pre-built toolchain at ghcr.io/bigdez55/genos-toolchain
- `devices/cloud/platform/docker/` — 10+ service Dockerfiles for cloud platform layer
- Docker has NO role on the bare-metal device itself

## Deployment Phase Summary

Phase 1: BSS fix → kern_span extension OR demand-zero-only path
Phase 2: ISO pipeline integration (gen_iso.c → boot-test.sh → genos.iso)
Phase 3: Secure Boot signing chain
Phase 4: HP EliteBook hardware bring-up (WiFi/touch/NVMe/audio)
Phase 5: Self-hosting build environment
Phase 6: OTA + XPKG update pipeline
Phase 7: Recovery/rollback via AetherBoot

## Critical Constraint
kern_span limit in aetherboot.c line 862: `if (kern_span > 256ULL * 1024 * 1024)`
Must be raised to 512MB OR BSS must be reduced below 256MB.
RECOMMENDED: raise to 512MB + keep demand-zero BSS — both fixes in parallel.
