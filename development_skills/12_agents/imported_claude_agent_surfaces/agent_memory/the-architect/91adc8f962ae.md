## Summary

## Opcode Space
- xisc.h defines opcodes 0x00-0x63 + 0xFF (HALT)
- Range 0x64-0xFE is UNOCCUPIED — 0x80-0x8F is AVAILABLE
- Cognitive opcodes must NOT collide with AI/HPC range (0x60-0x63)
- Encoding constraint: existing xisc_instr_t is 32-bit fixed-width (4 bytes)
  R1_PER proposal uses 8-byte encoding — requires NEW xcog_instr_t type, not reuse of xisc_instr_t
  _Static_assert(sizeof(xisc_instr_t) == 4) will NOT be violated if XCOG is a parallel type

## XMIND Tokenizer Impact
- tokenizer.c: BPE byte-level mode, 128K vocab, FNV-1a hash table
- xmind_generate() drives: tokenize → forward → sample pipeline
- Replacing tokenizer with semantic compiler requires:
  1. New entry point: xmind_infer_from_opcodes(xcog_instr_t *opcodes, uint32_t n)
  2. Embedding layer currently maps token_id → row of embed_weight matrix
     Cognitive opcodes have 8-byte structure; the model must be trained with
     opcode embeddings (256-dim per opcode field) — NOT a runtime patch
  3. xmind_rope_precompute() is position-independent — still valid for opcode sequences
  4. s_inference_lock (PAL spinlock) is already present — thread safety is covered

## Boot Sequence Placement
- kmain() 14-step sequence ends at Step 13 (gensd_start) → Step 14 (idle HLT)
- r1_per_init() MUST be called AFTER Step 7 (PAL-Aether init, heap available)
  and AFTER Step 8 (vDRAM PF handler) since context shards use vDRAM
- Correct placement: Step 12 area (after XKABI cap table, before GENSD)
  OR inside gensd_start() as a userland service init — preferred for isolation
- DO NOT place in kernel before Step 7 — no heap, no PAL alloc

## Security Claim Assessment
- Typed XISC input DOES structurally constrain the input space
- 16 opcodes × fixed 8-byte encoding = finite, enumerable instruction space
- Prompt injection requires free-form text in the input channel;
  typed opcodes eliminate that channel AT THE ADMISSION POINT (L3.1)
- Assessment: claim is ARCHITECTURALLY VALID — not a policy claim but a structural one

## Hard Constraint (OPEN ITEM)
- 16 opcodes (0x80-0x8F) for 7 primary Heptagon regions is TIGHT
  R1_PER alone has: INTENT, ENTITY, PREDICATE, MODIFIER, CONTEXT, QUERY, ASSERT,
  NEGATE, CONDITIONAL, TEMPORAL, CAUSAL, COMPARATIVE — 12 semantic categories
  for R1 alone, leaving 4 opcodes for R2-R7 coordination signals
  RECOMMENDATION: Reserve 0x70-0x8F (32 slots) not 0x80-0x8F (16 slots)
  0x70-0x7F = inter-region routing opcodes, 0x80-0x8F = semantic content opcodes
