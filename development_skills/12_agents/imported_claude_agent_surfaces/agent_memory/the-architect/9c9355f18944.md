## Critical API Facts

### XFRAME API (ui/xframe/runtime/xframe.h)
All setter functions exist as `static inline` in xframe.h — no external linkage needed:
- `xframe_node_set_direction()`, `set_align_items()`, `set_justify_content()`
- `set_size()`, `set_flex_grow()`, `set_gap()`, `set_padding()`, `set_margin()`
- `set_bg_color()`, `set_border_color()`, `set_border_width()`, `set_border_radius()`
- `set_text_color()`, `set_visible()`, `set_text()`
- CRITICAL: setters do NOT call xframe_node_mark_dirty() — callers must dirty manually

## Static Storage Pattern (no heap)
Every translation unit owns its own static storage variable — NEVER reference another TU's static:
- topbar.c:        `static xshell_topbar_t s_topbar;`
- dock.c:          `static xshell_dock_t s_dock_storage;`
- launcher.c:      `static xshell_launcher_t s_launcher_storage;`
- lockscreen.c:    `static xshell_lockscreen_t s_ls_storage;`
- greeter.c:       `static xshell_greeter_t s_greeter_storage;`
- context_menu.c:  `static xshell_context_menu_t s_cm_storage;`

## Algorithm Selections
- FNV-1a: offset=2166136261u, prime=16777619u (svc_registry.c)
- Kahn's topological sort: O(V+E) for XORCHESTRA dependency ordering

- Selection sort O(n^2): drive.c file list — acceptable at n<=512
- XOR PIN hash: lockscreen.c stub; keys={0x5A,0x3C,0x1F,0x72,0x4B,0x8E} — Sprint 6 replace
