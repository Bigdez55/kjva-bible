### XKABI Capability System (Ground Truth)
- Single system-wide handle table: 64K slots, static flat array (6MB BSS)
- Handle format: 64-bit HMAC-authenticated [slot:16|rights:16|gen:16|hmac:16]
- xkabi_cap_revoke(): marks single slot revoked (slot not freed — leaks)
- xkabi_cap_revoke_all(): bulk epoch increment — all old-generation handles invalid
- xkabi_cap_derive(): rights narrowing enforced, TOCTOU fixed in Sprint 21
- NO per-process capability tables (deferred to Sprint 2/16)
- NO capability endpoint objects (slots are object_id handles, not typed endpoints)

### Scheduler (Ground Truth)
- Cooperative fiber scheduler: 16 fiber slots, single-CPU
- Contra-rotation: thermal-driven phase switching (COMPUTE/COOLDOWN/LOAD)
- NO preemption. NO priority levels. NO WCET guarantees.
- contra_rotation.c comment at line 177: "Single-CPU cooperative (non-preemptive)"

### Confirmed Absent (zero grep matches)
- kdebug, kdb, ksyms
- PMC/PEBS performance counters
- kexec / hot-reboot
- core dump on panic
- ftrace/dtrace equivalent
- preempt_count / kernel preemption
- priority_inherit / priority ceiling
- subsystem heap isolation
- per-process capability tables

## Priority Matrix for Gaps

### P0 — Boot-blocking or security-critical (implement now)
1. Kernel preemption — cooperative scheduler is unsafe for real-time XMIND inference
2. Per-process capability tables — current single table is a security boundary violation
3. Capability slot reclamation — xkabi_cap_revoke() marks but never frees (memory leak at 6MB static + no slot recovery)
4. Core dump on panic — post-mortem debugging is impossible without this

### P1 — Production quality (next 2 sprints)
5. Kernel heap isolation — single heap means UAF in one subsystem corrupts all
6. Priority inheritance protocol — priority inversion will deadlock XSHELL under load
7. Kernel tracing infrastructure — observability metrics exist (Sprint 40) but no ring-buffer event trace
8. PMC access — needed for XMIND WCET profiling and AVX2 dot-product validation

### P2 — Certification / advanced capability (future sprints)
9. WCET analysis framework — required for any real-time claim
10. Hot-reboot / kexec — needed for zero-downtime kernel updates
11. Kernel debugging infrastructure (kdebug/GDB stub over serial)
12. IEC 61508 / ISO 26262 safety certification path
