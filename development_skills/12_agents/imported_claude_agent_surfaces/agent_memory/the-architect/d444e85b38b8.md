## API Contract Highlights

### Health endpoint consistency issue
All services: GET /healthz returns {"status": "healthy"} — 200 or 503
GENESYS AI: GET /healthz returns {"status": "ok"} — documented, matches agent-bridge.ts
Provenance: NO /health route (only /healthz) — inconsistent with all other services

### JWT architecture
Algorithm: EdDSA (Ed25519) throughout. Audience "genos", issuer "genos-identity".
Token types: access (30min TTL) and refresh (7day TTL) distinguished by "typ" claim.
Refresh rotation: revoked_at set atomically with new session INSERT in same commit.
Public key distribution: JWKS at /.well-known/jwks.json; services receive via SYNC_JWT_PUBLIC_KEY / PROVENANCE_JWT_PUBLIC_KEY env var.

### Service token authentication (gate, registry, artifact-store, update, genesys-ai)
All use HMAC constant-time comparison: hmac.compare_digest(presented, _SERVICE_TOKEN)
Empty-string guard: if not _SERVICE_TOKEN prevents compare_digest("", "") = True bypass
