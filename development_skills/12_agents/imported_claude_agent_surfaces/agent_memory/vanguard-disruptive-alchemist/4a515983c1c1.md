## GEN.OS Competitive Position (2026-03-29)

**Overall Score: 66/160 vs macOS 92, Linux 104, Windows 80, Fuchsia 76**

### Critical Gaps (Tier A — Block Real Deployment)
1. Process isolation: XKABI exists but cap_check NOT in syscall dispatch (~300 LOC fix)
2. No programmable kernel extensions (eBPF equivalent) — XJIT could serve with ~10 probe opcodes
3. No hardware security module integration (keys in BSS)
4. No live patching capability
5. Zero post-quantum crypto (ML-KEM/ML-DSA absent)
6. No async I/O primitive (no io_uring/kqueue/IOCP equivalent)
7. Syscall 29 collision: XK_SYS_CHAN_SEND == XK_SYS_EXEC (P0 ABI bug)

### Unique Structural Advantages (Cannot Be Replicated by Incumbents)
1. AI inference at ring 0 (XMIND in kernel address space) — enables predictive security
2. Capability security from first instruction (no ambient authority ever existed)
3. Sub-50ms full-stack boot (3 OOM faster than macOS)
4. ~55 syscalls — within formal verification envelope (seL4-class)
5. Single-author architectural coherence (PAL threading across all subsystems)
6. Zero legacy ABI burden — can break ABI anytime

### Century Architecture Decisions (Make NOW)
1. Wire XKABI enforcement into syscall dispatch (P0, ~300 LOC)
2. Implement ML-KEM-768 for post-quantum TLS hybrid (before 2027)
3. Start CBMC assertions (expand from 59 _Static_assert to 200+)
4. Capability-gated async I/O ring (~600 LOC kernel, 1/25th io_uring complexity)
5. ISA-agnostic intrinsic dispatch for all hardware paths (RISC-V prep)
6. Enable Intel CET (shadow stack + IBT) — ~100 LOC, eliminates ROP/JOP

**Why:** The competitive position is not about feature parity — it is about PROVABLE PROPERTIES that incumbents cannot achieve due to legacy.

**How to apply:** When prioritizing work, weight enforcement/verification/PQC over new features. The WIRING is the bottleneck, not the architecture.
