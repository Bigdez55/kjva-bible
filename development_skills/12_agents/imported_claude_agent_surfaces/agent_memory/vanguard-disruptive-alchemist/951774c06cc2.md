## CRITICAL BUG: mr_registry.c RCU Use-After-Free

Location: `kernel/linux-validation/vdram/mr_registry.c:119-138`

vdram_mr_free() does RCU_INIT_POINTER(NULL) then immediately kfree(mr).
No synchronize_rcu() or call_rcu() between pointer removal and deallocation.
Concurrent readers in vdram_mr_lookup_rkey() can atomic_inc on freed memory.

Fix: Embed struct rcu_head in vdram_mr, use call_rcu() for deferred free.

## ALGORITHM RECOMMENDATIONS

### SipHash-2-4 for XKABI handles
- 15-25ns vs 120-600ns for HMAC-SHA256 on 8-byte input
- linux/siphash.h in-tree since 4.13
- Eliminates crypto_alloc_shash lifecycle management

### CLZ-based entropy (no LUT)
- __builtin_clz compiles to single LZCNT instruction
- Correct for any packet length (LUT assumes 256-byte denominator)
- Saves 512 bytes icache/dcache

### SQLite WAL for provenance fallback
- Incremental flush: each record marked individually
- Crash-safe: committed transactions survive power loss
- No full-file rewrite on partial flush
