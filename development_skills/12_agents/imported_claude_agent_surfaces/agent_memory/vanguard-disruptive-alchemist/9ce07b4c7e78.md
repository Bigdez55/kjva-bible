# Radical Efficiency Audit (2026-03-02)

## Findings Summary

### 6 Eliminations (STOP DOING)
1. **E1: artifact-store** -- returns HTTP 501 on only endpoint, pure dead weight (128-512MB)
2. **E2: admission-webhook** -- duplicates gate service on single-node k3s (128-512MB)
3. **E3: merkle-collective** -- runs sleep(60) loop to answer health probes (128-512MB)
4. **E4: fabric-energy** -- simulates nonexistent BlueField-4 DPU (64-128MB)
5. **E5: update-service** -- in-memory dict comparing two strings (128-512MB)
6. **E6: 3 sysfs bridges** -- write constants to disk (sysfs absent), 3 Python processes (~135MB)

### 3 Consolidations
1. **C1: 7 FastAPI -> 1** -- identity/provenance/sync/gate as APIRouters (320-480MB saved)
2. **C2: 16 duplicate functions** -- decode_bearer(x4), _verify_token(x5), _atomic_write(x4), normalize_pem(x3)
3. **C3: thermal_guardian + predictor + dbus_bridge -> 1** -- IPC via JSON tmpfs is wrong (~60MB)

### Inversion Architecture (Zero Polling)
- 7 poll loops totaling 5.7 atomic writes/s, 16+ file reads/s, ~23 syscalls/s
- Phase 1: inotify replaces all file polling (4 hours)
- Phase 2: D-Bus signal subscriptions replace property polling (1 day)
- Phase 3: netlink from kernel modules (1 week, when modules ship)

### Total Savings: ~1.1-2.8 GB RAM, 13 processes eliminated, ~2 weeks effort

## Process Count Before/After
- Before: k3s + 13 pods + 5 Python daemons + Electron + PostgreSQL + MinIO = ~22 processes
- After: 1 platform pod + 1 AI pod + PostgreSQL + MinIO + 1 unified daemon + Electron = ~7 processes
