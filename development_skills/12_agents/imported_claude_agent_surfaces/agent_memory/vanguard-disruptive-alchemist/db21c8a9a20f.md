## Critical Findings

### P0: Supervisor manual byte-offset struct interpretation
- supervisor.c lines 256-296: interprets gsd_service_t via hardcoded offsets
- Any field change in gsd_service_t silently corrupts supervisor state
- Fix: shared typedef (copy struct def) or shared header (Sprint 3 planned)

### P1: Contra-rotation hysteresis defect at 60 deg C
- XENOS version: COOLDOWN->LOAD at temp <= 60 (line 233)
- Linux validation: COOLDOWN->LOAD at temp < 55 (5 deg C dead band)
- XENOS version has NO dead band = oscillation risk at exactly 60 deg C
- Comment on lines 228-232 documents discrepancy but defers to "spec"

### P1: Supervisor blocking sleep (up to 30s)
- supervisor_restart() calls pal_thread_sleep_ns(delay_ns) line 214
- Blocks ALL service monitoring during one service's backoff
- Documented in TODO lines 207-212

### P2: XKABI triple lock acquisition in derive
- xkabi_cap_derive acquires lock 4 times for one logical operation
- Lines: 358 (verify), 489 (read parent), 518 (write lineage), 558 (read child oid)
- Should be single critical section

### P2: Spinlock held during thermal read in contra_rotation
- cr_phase_advance called from cr_sched_tick while spinlock held (line 423)
- Linux version used mutex (sleepable); XENOS uses spinlock (non-sleepable)
- Regression if thermal read ever blocks

## Dead Code / Entropy
- contra_rotation: `idle_time_ns` field (never incremented), enterprise RDMA stubs (28 lines)
- xkabi: hand-rolled memset loops (should use __builtin_memset)
- supervisor: `probe_pending` field (never set true), `supervisor_probe_http` (stub, not static)
- pmm: DMA32 zone (unnecessary for EliteBook UMA target)
- pmm: duplicate inner loop in alloc fallback (lines 354-373) -- logic bug workaround

## Top 3 Proposals
1. Unified Tick Architecture: single heartbeat thread replacing N poll loops (-80/+40 lines)
2. Capabilities Without Handles: raw slot indices, no bit-packing (-40/+0 lines)
3. Event-Driven Supervisor: pal_event_wait replaces polling+sleeping (-50/+30 lines)

## PAL Gaps (not defects -- planned Sprint 2)
1. No pal_process_create/kill (supervisor, XKABI, syscall all need it)
2. No MMIO-only mapping (pal_map_pages requires pages_handle from alloc)
- Both fillable within existing 14-section structure
