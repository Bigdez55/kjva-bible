# RFC-002 v5.1 Red Team (2026-02-27)

## 5 Components Analyzed

### 1. XISC Opcode Space (5.25/10)
- 8-bit opcode, 24/256 used = 90.6% waste
- Proposal: 5-bit dense + 11-bit imm (fixes F-010 signed immediate)
- BUT: JIT excluded from build (xisc_llvm_jit.c). Premature optimization.
- Key file: xisc/include/xisc.h

### 2. XKABI Handle Table (7.5/10) -- HIGHEST PRIORITY
- 32-bit index, O(n) alloc, O(n) revocation (shim_handles.c:29, :216)
- Proposal: 128-bit SipHash-sealed capability tokens, O(1) validate+revoke
- Eliminates handle table, mutex contention (~15-25ns -> ~3ns per syscall)
- Key files: xkabi_types.h, shim_handles.c, shim_internal.h
- DO BEFORE ABI FREEZE

### 3. Thermal PROACTIVE_WINDOW_S=10s (5.75/10)
- Magic number, no empirical basis
- Proposal: auto-tuned via exponential tau estimation from EWMA history
- Critical: entire system runs on simulated data, never tested on EliteBook
- Key file: tools/thermal_guardian/thermal_predictor.py

### 4. Entropy LUT 256-entry (5.0/10)
- BUG: freq > 255 clamped, silent accuracy loss for packets > 255 bytes
- Proposal: coincidence index (sum freq[i]^2), SIMD-vectorizable
- SIMD claim in header comment is aspirational; no NEON implementation
- Key file: gateway/zc_gw/entropy_classifier.c

### 5. Provenance JSONL (5.25/10)
- 4 data-loss vectors: partial write, rotation destruction, duplicate replay, OOM flush
- Proposal: SQLite WAL, exactly-once delivery, crash-safe
- Critical: provenance SERVICE does not exist. No consumer of this data.
- Key file: tools/provenance_client.py

## Cross-Cutting Finding
All 5 components are architecturally sophisticated but operationally untested.
No component has been validated against real hardware or real workloads.
