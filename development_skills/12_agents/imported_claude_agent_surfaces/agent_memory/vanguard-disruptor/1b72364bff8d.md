# Git Hygiene & Build Pipeline Audit (2026-02-24)

## Findings Summary

### 1. .tar Removal: Necessary but Incomplete
- .tar files removed from working tree and added to .gitignore
- BUT: build/rootfs/ (2,374 files) and rootfs/ (2,374 files) are STILL TRACKED
- These contain debootstrap Debian system binaries (/bin/bash, /usr/lib/*, etc.)
- build/iso-work/boot/initrd.img (42MB) and vmlinuz (7.8MB) also tracked
- 83% of all tracked files (4,751 of 5,699) are build artifacts

### 2. .gitignore Gaps
Current .gitignore blocks: *.tar, *.iso, *.squashfs, build/images/, node_modules/
Missing: build/rootfs/, rootfs/, build/iso-work/, __pycache__/, .ruff_cache/, .pytest_cache/, .turbo/, *.egg-info/

### 3. Build Pipeline Architecture
- genos-build.py: docker build -> docker save -> .tar -> chroot-setup.sh copies into rootfs -> iso-assemble.py
- .tar files are intermediate artifacts needed only during ISO build inside Docker container
- They should never leave the Docker buildenv container
- k8s manifests reference genos/* with no imagePullPolicy or registry prefix

### 4. Electron App Duplication
- 7 Orange Suite apps + 4 other Electron apps = 11 total
- ALL share identical: devDependencies, build scripts, electron version, vite version
- Each downloads its own ~450MB electron binary
- No shared electron-builder config, no shared base package

### 5. CI/CD Gap
- ci.yml: Python lint (ruff) + pytest only
- Zero: Docker image builds, Electron builds, k8s manifest validation, .gitignore enforcement
- No artifact caching, no release pipeline
