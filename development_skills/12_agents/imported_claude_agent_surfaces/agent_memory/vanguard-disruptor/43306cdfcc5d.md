## Project Conventions
- Language Policy: Python, TypeScript, C ONLY (no Go, Rust, Java)
- 100% original OS: no Linux kernel, no third-party OS components
- Target hardware: HP EliteBook x360 (Intel i7, 16GB RAM, no discrete GPU)
- XMIND replaces Ollama (Q4_0 transformer, freestanding C, ~3,250 LOC)
- XJIT replaces LLVM (one-pass JIT, ~1,817 LOC)
- XFRAME: 11 widgets + CRDT sync + QuickJS bridge (stub, #ifdef QUICKJS_AVAILABLE)
- Orange Suite: 4 native C apps (notes, calendar, mail, drive)
- XPKG: 9-gate refusal (only 2 gates actually block: Signature+Integrity)
- Compile check: clang -target x86_64-unknown-none-elf -ffreestanding -Werror
- Sprint 0-13 COMPLETE, all compile checks pass

## Boot Chain (Sprint 13)
- AetherBoot: UEFI->GOP->ACPI->MemMap->TPM->TSC->ELF load->PageTables->ExitBS->CR3->Jump
- Page tables: PML4[0]=identity 4GB, PML4[273]=HHDM 4GB, PML4[511]=kernel -2GB
- Kernel entry: _xenos_start -> kmain (14-step init) -> gensd_start -> xchain_run
- Fiber scheduler: 16 slots, 64KB stacks, cooperative yield, no preemption
- LAPIC timer: 16 one-shot slots, div-16, vector 0x30 (not wired to preemption)

## Syscall Status (Post-Sprint 13)
- LIVE (15): PHYS_ALLOC/FREE, VIRT_MAP/UNMAP, THREAD_CREATE/YIELD/JOIN/SELF/SLEEP,
  TIMER_CREATE/SET/CANCEL, RANDOM, THREAD_SELF
- ENOSYS (13): PHYS_INFO, VIRT_PROTECT, EVENT_*, CHAN_*, DMA_ALLOC, SG_BUILD,
  PLATFORM, THERMAL, HANDLE_TYPE/CLOSE

## Critical Missing Hardware Support
- WiFi: XPCI detects AX201 at 0x02F0, zero driver code (needs 8-15K LOC)
- Audio: zero code, EliteBook has Realtek ALC236 (needs 3-6K LOC)
- USB: XPCI detects xHCI at 0x9D2F, zero driver code (needs 4-8K LOC)
- Bluetooth: zero code
- Real NIC driver: XNET has TCP/UDP but no hardware NIC driver

## Key Security Findings (Persistent)
- NEW P0: AetherBoot no kernel signature (EFI partition write = full compromise)
- NEW P0: No NX enforcement (HHDM is RWX), identity map persists after boot
- NEW P0: Fiber stacks have no guard page (silent corruption on overflow)
- XKABI handle forgery O(1): 128 guesses, gen=1, rights=chosen (P0, multiple audits)
- XKABI rights defined but NEVER enforced in syscall dispatch (P1)
- TLS no trust store: accepts ANY self-signed CA (P0, flagged 5+ times)
- Crypto primitives (field arith, Poly1305, ChaCha20, x25519 ladder) are CLEAN
