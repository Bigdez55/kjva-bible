# Phase 7 Red Team Audit (2026-02-24) -- Commit 3f38b7f

## Overall Verdict: 5.5/10 (up from 4/10 at Phase 6)

Phase 7 fixed the most egregious issues (hardcoded dev-token, PASS default, fail-open JWT)
but introduced new gaps and failed to address several fundamental architectural weaknesses.

## Attack Vectors Tested

### 1. HISTORY EXTRACTION -- SEVERITY: CRITICAL (P0)
- Ed25519 private key blob still extractable: `git show 394e107:platform/init/genos_platform_init.py`
- signing.key content still in history: `git show 394e107:build/config/signing.key`
- "PASS" admission token in history: Phase 7 removed it from code but not from git objects
- "dev-token" in history: same pattern
- All postgres/minio credentials with defaults in commit 394e107 diffs
- No `git filter-branch` or BFG Repo Cleaner has been run
- Pack file exists: pack-93bcb2ec70f3f796df677bf7b9f539a8635e1203.idx (objects are packed, not pruned)
- No refs/original/ directory (good -- no filter-branch residue)

### 2. TOKEN BYPASS (AI API verify_service_token) -- SEVERITY: MEDIUM (P1)
- Uses `auth[7:] != SERVICE_TOKEN` -- Python `!=` on strings is NOT timing-safe
- Vulnerable to timing side-channel attack to recover token character-by-character
- Empty string bypass: CLOSED (os.environ[] raises KeyError if unset)
- /status endpoint: leaks circuit breaker internals (calls count, failure count)
  but does NOT leak token -- acceptable info exposure

### 3. JWT BYPASS (sync/provenance fail-closed) -- SEVERITY: LOW (P2)
- Whitespace-only SYNC_JWT_PUBLIC_KEY: CLOSED (`not _raw_jwt_key.strip()` catches it)
- Malformed PEM: PyJWT raises PyJWTError -> 401 Unauthorized -- CLOSED
- Provenance service does NOT call normalize_pem before jwt.decode -- may fail
  on escaped PEM strings from k8s secrets, but fails closed (401), not open
- IDENTITY_ALLOW_DEV_KEYS=true generates ephemeral key -- documented, warns in logs

### 4. AUTOMATION BRIDGE -- SEVERITY: CRITICAL (P0)
- HTTP bridge on port 8787 has ZERO authentication
- Bound to 127.0.0.1 only (good), but ANY localhost process can:
  - POST /navigate -> navigate browser to any URL
  - POST /click -> click any CSS selector
  - POST /type -> type into any input
  - GET /content -> read full DOM (passwords, session tokens, banking data)
- Combined with hostNetwork on genesys-ai pod: the AI container shares the host
  network namespace, meaning the GENESYS AI agent can reach 127.0.0.1:8787
- Attack chain: compromise AI agent -> call bridge -> exfiltrate page content
- No request body size limits on bridge endpoints

### 5. ELECTRON XSS via executeJavaScript -- SEVERITY: MEDIUM (P1)
- Selectors: JSON.stringify(selector) is safe against quote escaping -- CLOSED
- automation:scroll: `y` is interpolated directly WITHOUT JSON.stringify:
  `window.scrollBy({top:${y},behavior:'smooth'});`
  If y is a string like `0},0);document.title=document.cookie//`
  it becomes valid JS injection
  TypeScript types y as `number` but IPC has no runtime validation -- the .js
  file has no type safety at all
- automation:fillForm: JSON.stringify(fields) is safe -- CLOSED
- automation:navigate: loadURL with no URL validation -- can load `javascript:` URLs
  or `file:///etc/passwd`

### 6. SERVICE TOKEN LEAKAGE -- SEVERITY: MEDIUM (P1)
- GENOS_SERVICE_TOKEN never appears in log/print statements -- GOOD
- Error detail at api.py:92 includes full exception message:
  `detail=f"Agent execution failed: {exc}"`
  If exc contains request details with Authorization header, token leaks in HTTP response
- Multiple preload files use `?? ""` fallback for GENOS_SERVICE_TOKEN:
  companion/preload.js, go-calendar/preload.js, go-notes/preload.js, go-docs/preload.ts
  Empty string = requests sent WITHOUT auth header = silent auth bypass to open endpoints
- Only browser/main.ts and api.py use os.environ[] (fail-hard); all other callers degrade silently

### 7. SIGNING KEY ON DISK -- SEVERITY: LOW (P2)
- build/config/signing.key deleted from tree in Phase 7 -- GOOD
- .gitignore blocks *.key AND explicit build/config/signing.key -- GOOD
- Key content still in git history at commit 394e107 -- SEE VECTOR 1

## Additional Findings

### A. Circuit Breaker Still Permanently Poisonable
- failures/calls are monotonic counters (never reset)
- After enough failures, error_rate permanently exceeds 0.10
- Circuit opens for 30s, but on next request error_rate is still >0.10 -> re-opens
- Only a process restart clears the counters

### B. genos-secrets Missing service-token Key
- 09_genesys-ai.yaml references secretKeyRef key: service-token
- genos_platform_init.py create_secret() never creates a "service-token" key
- Deployment will fail at pod startup with missing secret key error
- This is a functional gap, not a security gap, but blocks deployment

### C. hostNetwork: true on genesys-ai
- AI pod runs in host network namespace
- Can access any localhost port (including bridge 8787, Ollama 11434, k8s API 6443)
- Widens blast radius of any AI agent compromise

### D. Provenance Chain Seed Default
- PROVENANCE_CHAIN_SEED still defaults to "genos-seed" in provenance/app.py line 14
- This was _require_env'd in platform_init but the service itself still has a fallback
- Inconsistent: init requires it but service ignores the requirement
