# Vanguard Disruptor -- Agent Memory

## Verified P0 Vulnerabilities (2026-03-08)

### Kelly Bypass Chain (CONFIRMED)
- `backend/app/trading_engine/strategies/base.py:125` uses `0.02 * confidence * 1.5`
- `backend/app/trading_engine/risk/kelly_sizer.py` has full Kelly implementation but is never called from trade_executor
- `trade_executor.py:82` calls `strategy.calculate_position_size()` which hits the base class naive formula
- The Go risk-engine `cmd/server/main.go` has a correct `/kelly` endpoint but is not compiled by the Dockerfile

### Daily Loss Gate Disabled (CONFIRMED)
- `trade_executor.py:572` calls `portfolio.get_daily_drawdown()` without passing session
- `portfolio.py:124`: `if not session: return Decimal("0")` -- always returns zero
- The DAILY_LOSS circuit breaker type exists but can never trip through this path

### Circuit Breaker Persistence Not Wired (CONFIRMED)
- `main.py` startup never calls `get_circuit_breaker(db_session=..., db_session_factory=...)`
- `_db_session_factory` stays None on the singleton
- State only persists to ephemeral filesystem JSON file (deleted on Cloud Run restart)

### Go risk-engine Dockerfile Wrong Binary (CONFIRMED)
- `Dockerfile:4`: `go build ./cmd/risk` compiles the stub (only `/health`)
- Correct binary: `./cmd/server` (has `/kelly` + `/metrics`)
- `go.sum` file does not exist (Docker build will fail)

## Architecture Notes

### Regime Detection (regime_detection_service.py)
- 2-input classifier only: SPY SMA slope + VIX level
- Thresholds: +/-0.005 slope, VIX 20 boundary
- 1-hour cache (3600s) -- too long for market-open volatility
- Defaults to SIDEWAYS on any data failure
- FRED macro data fetched but NOT used in classification (only exposed via `get_macro_context()`)

### Signal Gate (signal_gate_service.py)
- Singleton pattern via `get_signal_gate()` / `_global_gate`
- Bootstrap from DB at startup (`bootstrap_signal_gate` in main.py)
- Isotonic calibration with 10-sample minimum
- Redis persistence attempted but Redis is unavailable (falls back to no-op)

### Trade Execution Chain
- auto_trading_service.py -> TradeExecutor -> strategy.calculate_position_size() -> base.py naive formula
- KellySizer exists as module singleton but sits unused in this chain
- The Go order-router is feature-flagged behind `USE_GO_ORDER_ROUTING` env var

### vLLM Configuration
- Primary: `VLLM_BASE_URL` (settings) -- centralized
- Fallback: `VLLM_FALLBACK_BASE_URL=http://10.138.0.4:8001` hardcoded in .env.template
- L4 GPU: 24GB VRAM, base model ~14GB, max 3-4 concurrent inferences
