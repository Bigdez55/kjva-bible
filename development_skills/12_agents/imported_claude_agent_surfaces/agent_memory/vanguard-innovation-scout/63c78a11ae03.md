## 1. KERNEL / SYSTEMS

### 1.1 eBPF for Zero-Overhead Kernel Telemetry
- Applicability: HIGH | Complexity: 4/5 | Gain: 400-800x latency reduction
- Replaces 3-layer polling chain (sysfs->bridge->JSON->dbus->JSON->Electron->IPC)
- Current end-to-end latency: 4-8 seconds. eBPF ringbuf: <10ms.
- Requires kernel 5.10+ (Debian 12 has 6.1), CAP_BPF for bridge process
- Risk: Not available inside standard k8s containers without explicit capabilities

### 1.2 io_uring for Bridge Daemons
- Applicability: MEDIUM | Complexity: 3/5 | Gain: 2-5x bridge CPU reduction
- All bridges use synchronous Path.read_text() and os.fsync() blocking asyncio
- io_uring batches multiple sysfs reads into single submission
- Risk: io_uring disabled in some container runtimes (bridges run as systemd, OK)

### 1.3 DAMON for vDRAM Tier Decisions
- Applicability: HIGH | Complexity: 3/5 | Gain: 20-40% fault latency reduction
- vdram_bridge reads static tier allocation; no dynamic page migration intelligence
- DAMON + DAMOS can auto-trigger tier migration based on measured access patterns
- Mainline since 5.15, maintained by Amazon. Zero kernel module changes needed.

### 1.4 EEVDF Scheduler
- Applicability: MEDIUM | Complexity: 1/5 (but kernel upgrade 4/5)
- Default in kernel 6.6+ (Debian 12 ships 6.1). Free with kernel upgrade.
- 5-15% frame time consistency improvement for compositor
- Risk: Kernel upgrade requires full regression of XISC modules and sysfs interfaces

## 2. AI INFERENCE

### 2.1 Continuous Batching
- Applicability: MEDIUM | Complexity: 2/5 | Gain: 2-4x multi-request throughput
- Ollama 0.3+ supports via OLLAMA_NUM_PARALLEL=4
- Requires converting model_manager.py to httpx.AsyncClient for concurrent submission
- Memory concern: 4 concurrent contexts = ~4GB (k8s limit is 4Gi, needs 6Gi)

### 2.2 Speculative Decoding
- Applicability: HIGH | Complexity: 2/5 | Gain: 1.5-2.5x TTFT
- Llama 3.2 1B as draft model (~600MB additional RAM)
- Same tokenizer as 3B target = ideal pairing
- Config-only change: OLLAMA_DRAFT_MODEL env var, no code changes

### 2.3 Flash Attention 2
- Applicability: LOW | Gain: <5% at 4K context on CPU
- NOT RECOMMENDED for current model/hardware

### 2.4 LoRA Fine-Tuning
- Applicability: HIGH | Complexity: 4/5 | Gain: 30-50% fewer hallucinations
- Directly addresses _HALLUCINATION_PATTERNS problem in api.py
- Requires external GPU training, ship adapter with ISO (~5-50MB)
- Runtime cost: ~5-10% additional inference latency per token
