# Security SOTA Audit -- Post Phase 7 (2026-02-24)

## Innovation Score: 49/100 (down from 52 in post-push audit)

Score decreased because stricter criteria applied: several "fixed" items were incomplete.

## Findings by Dimension

### 1. Secret Management (65/100)
- `_require_env()` pattern is adequate for airgapped single-node
- k3s etcd encryption NOT enabled (add `--secrets-encryption` flag)
- No secret rotation mechanism
- Recommended: SOPS + age for encrypted source-of-truth, NOT Vault/ESO (overkill for single-node)

### 2. API Authentication (55/100)
- JWT EdDSA pattern in identity/sync/provenance: GOOD
- Static string comparison in genesys-ai api.py + admission_webhook: BAD (timing attack, no rotation)
- ZERO auth on 4 services: gate, registry, artifact-store, update (critical for update service)
- Recommended: Extract shared decode_bearer to common/auth.py, add to all services

### 3. Electron Security (55/100)
- contextIsolation + sandbox: 11/11 apps (good)
- will-navigate + setWindowOpenHandler: 7/11 apps (missing: shell, companion, installer)
- CSP meta tags: 7/11 apps (no CSP on shell, browser, companion, installer)
- 9 executeJavaScript calls in browser with user-supplied selectors (XSS vector)
- BrowserView deprecated since Electron 30, must migrate to WebContentsView
- No IPC sender validation on any app

### 4. Git Security (40/100)
- signing.key in git history (2 commits), needs git-filter-repo purge
- 77 dist-electron/.turbo files still tracked
- 288 archive/ files still tracked
- No .pre-commit-config.yaml, no .gitattributes
- Recommended: gitleaks (not BFG which is abandoned)

### 5. CI/CD Pipeline (20/100)
- Only covers: language policy, ruff check, pytest
- Missing: TS lint, k8s validation, Docker lint, secret scan, dep audit, SAST, coverage
- Minimum viable: add gitleaks + pip-audit + npm audit + hadolint + kubeconform + eslint

### 6. k8s Defense-in-Depth (45/100)
- runAsNonRoot + readOnlyRootFilesystem: present on all pods
- ZERO NetworkPolicy, ZERO seccompProfile, ZERO capabilities drop
- hostNetwork: true on genesys-ai (critical: bypasses all network isolation)
- No startupProbe on most services (only genesys-ai has one)

## Key Decisions
- Vault/ESO: REJECTED for airgapped single-node (overkill)
- SOPS + age: RECOMMENDED for secret encryption
- mTLS/SPIFFE: REJECTED (too heavy for single-node)
- k8s native JWT service accounts: RECOMMENDED for inter-service auth
- git-filter-repo: RECOMMENDED over BFG (BFG abandoned since 2021)
- gitleaks: RECOMMENDED over git-secrets (broader pattern DB)
- WebContentsView: RECOMMENDED over BrowserView (deprecated)
- Semgrep: RECOMMENDED for SAST (free OSS rules)
- hadolint: RECOMMENDED for Dockerfile linting
- kubeconform: RECOMMENDED for k8s manifest validation
