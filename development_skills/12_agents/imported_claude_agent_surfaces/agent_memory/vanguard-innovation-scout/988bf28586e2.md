## Critical P0 Findings
- vDRAM: filp_open per page fault in T1-T2 migration (catastrophic latency)
- vDRAM: O(n) MR linear scan on every fault (pf_find_mr)
- Merkle: SHA-256 should be BLAKE3, padding vulnerability (leaf duplication)
- EP-RDMA: no hysteresis in state machine, arbitrary 50ms threshold

## Key Competitive Positions
- vDRAM: sits ABOVE CXL (complementary, not competing)
- XKABI: Zircon-inspired + hardware rights (novel). Gap: no formal verification (seL4 has proofs)
- XISC: not a compute ISA -- it's a policy distribution bus. Should be reframed.
- Merkle: Blue Ocean -- no competitor does verifiable collectives with per-rank attribution
- EP-RDMA: NCCL-event-driven is novel vs timer-based Intel DLW / Mellanox PM

## Competitors Monitored
