## Genuinely Novel Inventions (No Prior Art Found)

2. XKABI per-capability lineage tracking -- seL4 CDT is internal only
3. Boot-time capability seeding from EFI handles -- no other bootloader does this
4. PAL dual-backend (Linux-validation + Aether-native) -- not in Zephyr/UEFI/seL4

## Key Comparisons
- Contra-rotation vs EEVDF: orthogonal (thermal vs fairness)
- Contra-rotation vs seL4 MCS: no formal temporal guarantees
- XKABI vs seL4 caps: no CDT-style selective revocation
- XKABI vs Zircon: Zircon handles are kernel-internal (unforgeable)
- NVMe vs Linux: single queue is 10+ years behind per-CPU multiqueue
- AetherBoot vs Limine: Limine provides SMP + HHDM + ELF loading
- PAL vs Zephyr HAL: PAL simpler/cleaner, missing device enum + power mgmt
