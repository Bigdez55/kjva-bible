### Key Findings (Stable)
- `react-mosaic-component` is effectively unmaintained. Replace with `rc-dock` or `FlexLayout` for trading workspace.
- `d3rlpy` is stagnating. Replace with `CORL` (single-file offline RL) + `Minari` (standard dataset format).
- `lightweight-charts` v4 does NOT include built-in drawing tools. Requires plugin system.
- ONNX Runtime is correct for CPU inference on Cloud Run. Add int8 quantization for ~4x model size reduction.
- Gymnasium is the correct RL environment framework (Farama Foundation standard).
- DeFiLlama API is free and superior to Glassnode for DeFi/TVL/stablecoin metrics.
- Decision Transformers become viable at 50K+ trade decisions (currently 3,559). Architect for migration.

### Phase 3D Key Findings (2026-03-08)
- **DoRA WARNING:** FinLoRA benchmark (May 2025) shows DoRA underperforms vanilla LoRA on financial tasks when using same LR for magnitude+direction. Needs separate magnitude vector LR.
- **TradingAgents v0.2.0:** Sharpe >= 5.60, cumulative returns >= 23.21%. Bull/bear debate pattern worth adopting.
- **FLAG-TRADER:** 135M model outperforms larger models via gradient-based RL fusion. Signal fusion RL is the highest-leverage long-term improvement.
- **SSE is the standard:** FastAPI has built-in SSE support (EventSourceResponse since 0.135.0). 15s keep-alive, no-cache, no-buffering headers automatic.
- **RTK Query cannot stream SSE.** TanStack Query v5 streamedQuery is the solution. Add as parallel provider, do NOT migrate existing slices.
- **L4 GPU multi-LoRA:** Up to 50% throughput degradation with concurrent different LoRA adapters. Keep Ruth EFT calls sequential, not parallel.
- **WASI Preview 2:** Production-ready for simple compute but NOT for financial-grade numerical computing. Do not adopt for Elson in 2026.
- **Pyodide:** REJECT for production trading. 10-20MB browser payload, no GPU access, adds latency vs server-side computation.
- **Tab-within-tab UX:** ACCEPTABLE for Ruth (9 sub-tabs) per NN/g research, with visual clustering into 2 groups.
