## P0 Bugs Found
1. entropy_classifier.c line 71: LUT indexed by raw freq count, not freq/len. Wrong for all len!=256.
   Fix: `u32 lut_idx = min_t(u32, (u32)((u64)freq[i] * 256 / len), 255);`
2. mr_registry.c vdram_mr_free: kfree(mr) without synchronize_rcu() after RCU_INIT_POINTER(NULL).
   Use-after-free race with concurrent vdram_mr_lookup_rkey readers.
3. pf_handler.c pf_find_mr: O(n) list_for_each under spinlock on EVERY page fault.
   Replace with interval_tree or maple_tree.

## Key Competitive Positions
- XISC: Not a compute ISA. It's a tier-aware policy bus. Blue Ocean vs eBPF RFC 9669.
- XKABI: Unified sw+hw rights namespace is unique. Gap: no formal verification (seL4/CHERI have proofs).
- Thermal: Only open-source predictor with TTT+confidence. Gap: linear model, no workload class.
- Entropy: Novel application (fabric routing by entropy). Gap: P0 correctness bug.
- RCU/MR: rkey_table is textbook-correct. mr_list is O(n) anti-pattern.

## Recommendations
- XISC: Reframe as policy ISA, not compute ISA. Consider formal spec document.
- XKABI: KLEE symbolic execution for rights-only-narrow proof (weeks, not years).
- Thermal: Replace linear regression with 2-state Kalman filter (thermal RC model).
- Entropy: Fix LUT bug (1-line), add SIMD histogram (ARM NEON vcnt), adaptive thresholds.
- MR: Add synchronize_rcu(), replace list with maple_tree, two-phase rkey rotation.
