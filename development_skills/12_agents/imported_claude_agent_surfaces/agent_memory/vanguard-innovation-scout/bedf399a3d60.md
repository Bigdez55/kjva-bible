## Key Technology Decisions Made

### Adopted
- **QuickJS-NG** over QuickJS (2019): drop-in replacement, 1.3-1.8x faster, active maintenance
- **YATA** over RGA/Logoot/LSEQ for CRDT: identical merge behavior to RGA, matches Yjs compatibility
- **B+ tree** over LSM tree for XSTORE: correct for read-heavy single-user embedded workload
- **SQLite-style redo-only WAL** over ARIES: simpler, sufficient for single-writer
- **Copy-and-patch** over one-pass lowering for XJIT: 5-10x faster compilation, better code quality
- **Fork llama.cpp** over rewrite for XMIND: 95% value for 5% effort
- **QUIC-first** over TCP-first for XNET: matches browser-centric traffic profile

### Rejected with Rationale
- **Hermes (Meta)**: 2-3x faster than QuickJS-NG but C++ (language policy violation)
- **LibJS (SerenityOS)**: C++ and oversized (~200K LOC vs ~85K)
- **LSM tree (RocksDB-style)**: optimized for write-heavy servers, wrong for laptop workload
- **ARIES WAL**: redo+undo is overkill for single-writer embedded database
- **QBE for runtime JIT**: designed for AOT, not runtime JIT (file I/O, not memory I/O)
- **Full XMIND rewrite**: llama.cpp has ~180K stars, thousands of person-years of optimization
