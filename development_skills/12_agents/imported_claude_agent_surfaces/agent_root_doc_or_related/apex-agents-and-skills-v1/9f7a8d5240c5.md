# APEX Dashboard Engineering Agents & Reusable Skills Library

## Context

The KPI Dashboard project currently uses vanilla HTML/CSS/JS with Node.js backend processing. To build the most elite professional dashboards in the corporate space - across any framework and any data type - we need a system of 15 specialized APEX agents (Claude Code skills) plus 14 cross-cutting reusable skills. Each agent embodies a top 0.001% engineer in their domain. Together, they form a complete dashboard engineering platform reusable across ALL future projects.

**What gets created:** 29 skill folders in `.claude/skills/`, each with a `SKILL.md` file that Claude Code auto-discovers and invokes based on context.

**SKILL.md Size Requirement:** Each APEX agent SKILL.md must be a minimum of 10,000 characters (~2,500+ words). This ensures deep, comprehensive instructions covering persona, domain expertise, code patterns, architecture decisions, anti-patterns, testing requirements, and integration protocols.

**Model Mixture Strategy:** The 15 APEX agents use a deliberate Opus/Sonnet split based on task complexity:
- **Opus (deep reasoning, architecture, complex logic):** APEX-React (PRISM), APEX-D3 (CANVAS), APEX-AI (ORACLE), APEX-Enterprise (VAULT), APEX-DataOps (PIPELINE), APEX-Design (PRESTIGE), APEX-Performance (TURBO)
- **Sonnet (fast execution, code generation, scaffolding):** APEX-Vue (MOSAIC), APEX-Angular (FORTRESS), APEX-Svelte (VELOCITY), APEX-Python (JUPYTER), APEX-RealTime (PULSE), APEX-Accessibility (BEACON), APEX-Export (COURIER), APEX-Testing (SENTINEL)

Each SKILL.md will include a `preferred_model` directive so Claude Code routes to the right model tier.

## 15 APEX Agents

### Agent 1: APEX-React (Codename: PRISM)
- **Domain:** React 18+ / Next.js 14+ / Recharts / Nivo / TanStack Table / Zustand / shadcn/ui
- **Trigger:** User asks to build dashboard with React/Next.js, migrate to React, or use Recharts/Nivo
- **Capabilities:** Component architecture, RSC data loading, ISR for KPI pages, type-safe data layers, Suspense boundaries for loading states, chart composition with Recharts/Nivo

### Agent 2: APEX-Vue (Codename: MOSAIC)
- **Domain:** Vue 3 / Nuxt 3 / Vuetify 3 / Element Plus / Pinia / VueUse
- **Trigger:** User asks for Vue dashboard, Nuxt app, or Vuetify/Element Plus components
- **Capabilities:** Composition API patterns, Nuxt server routes, Vue reactive data binding, Vuetify data tables, Element Plus form builders, Pinia store architecture

### Agent 3: APEX-Angular (Codename: FORTRESS)
- **Domain:** Angular 17+ / Angular Material / PrimeNG / NgRx / RxJS / Angular CDK
- **Trigger:** User asks for Angular dashboard, enterprise Angular app, PrimeNG components
- **Capabilities:** Standalone components, Signals for reactivity, lazy-loaded feature modules, Angular CDK virtual scrolling, PrimeNG data grids, NgRx state management

### Agent 4: APEX-Svelte (Codename: VELOCITY)
- **Domain:** SvelteKit / Svelte 5 (runes) / Threlte / LayerCake / svelte-charts
- **Trigger:** User asks for Svelte dashboard, performance-critical UI, minimal JS bundle
- **Capabilities:** Runes reactivity, SvelteKit server load functions, zero-runtime CSS, sub-50KB bundle dashboards, streaming SSR, compile-time optimizations

### Agent 5: APEX-D3 (Codename: CANVAS)
- **Domain:** D3.js v7 / Observable Plot / SVG/Canvas rendering / topojson / d3-force
- **Trigger:** User needs custom/unique visualizations, publication-quality charts, animated transitions, geographic maps
- **Capabilities:** Custom force-directed graphs, Sankey diagrams, treemaps, geographic chloropleth maps, animated transitions, Canvas rendering for 100K+ datapoints, reusable D3 component patterns

### Agent 6: APEX-Python (Codename: JUPYTER)
- **Domain:** Plotly Dash / Streamlit / Panel (HoloViz) / Bokeh / Altair / FastAPI
- **Trigger:** User asks for Python dashboard, Streamlit app, Dash layout, data science dashboard

### Agent 7: APEX-DataOps (Codename: PIPELINE)
- **Domain:** DuckDB-WASM / Apache Arrow / Observable Framework / Papa Parse / SheetJS
- **Trigger:** User needs data processing, ETL pipeline, Excel parsing, CSV handling, in-browser analytics

## 14 Reusable Cross-Cutting Skills

### Skill 1: dashboard-scaffold
- **Purpose:** Scaffold a complete dashboard project in any framework
- **Supports:** React, Vue, Angular, Svelte, Python, Vanilla JS
- **Output:** Full project structure with build config, linting, testing, CI/CD

### Skill 2: data-pipeline
- **Purpose:** Design ETL pipelines with Bronze/Silver/Gold medallion architecture
- **Output:** Data layer configs, validation rules, transformation scripts, quality checks

### Skill 3: chart-builder
- **Purpose:** Generate chart configurations for any visualization library
- **Supports:** Recharts, ECharts, Chart.js, Plotly, D3, Nivo
- **Output:** Chart config objects, responsive wrappers, theme-aware palettes

### Skill 4: theme-engine
- **Purpose:** Generate complete design systems with dark/light mode
- **Output:** CSS custom properties, Tailwind config, color palettes, typography scales

### Skill 5: kpi-card-factory
- **Purpose:** Create KPI card components with status indicators, trends, penalties/incentives
- **Output:** Framework-specific card components with animations and responsive behavior

### Skill 6: table-master
- **Purpose:** Build advanced data tables (sorting, filtering, pagination, virtual scroll)
- **Compatible with:** TanStack Table, AG Grid, PrimeNG, Vuetify, vanilla
- **Output:** Table configurations, column definitions, filter logic

### Skill 7: export-suite
- **Purpose:** Add PDF, Excel, CSV, PNG export to any dashboard
- **Output:** Export functions, accessible PDF templates, Excel formatters

### Skill 8: alert-system
- **Purpose:** Threshold-based alerts with multi-channel notification
- **Channels:** Email, Slack, Teams, SMS, webhook
- **Output:** Alert rule configs, notification templates, escalation logic

### Skill 9: auth-guard
- **Purpose:** Add authentication/authorization to dashboards
- **Patterns:** RBAC, ABAC, JWT, session, SAML SSO, OAuth
- **Output:** Auth middleware, permission checks, login flows

### Skill 10: perf-profiler
- **Purpose:** Profile and optimize dashboard performance
- **Output:** Lighthouse CI config, Web Worker scripts, code splitting configs, performance budgets

### Skill 11: test-harness
- **Purpose:** Generate comprehensive test suites for dashboards
- **Output:** E2E test scripts, visual regression configs, snapshot baselines, CI gates

### Skill 12: ai-insights
- **Purpose:** Integrate AI-powered narratives and recommendations
- **Uses:** Claude API for insight generation, anomaly detection algorithms
- **Output:** AI analysis modules, prompt templates, insight formatters

### Skill 13: responsive-layout
- **Purpose:** Mobile-first responsive grid systems
- **Output:** Grid components, breakpoint configs, touch gesture handlers, print layouts

### Skill 14: deploy-pipeline
- **Purpose:** CI/CD templates for any hosting platform
- **Supports:** GitHub Pages, Vercel, Netlify, AWS, Azure
- **Output:** Workflow files, deploy configs, health check scripts
