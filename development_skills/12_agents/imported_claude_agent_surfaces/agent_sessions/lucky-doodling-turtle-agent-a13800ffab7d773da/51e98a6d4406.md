### Critical Constraints
- Language policy: Python (.py), TypeScript (.ts/.tsx), JavaScript (.js/.jsx), C (.c/.h), Java (.java) ONLY
- xisc.h uses `typedef enum : uint8_t` — C++ syntax, will FAIL gcc -std=c11/gnu11. Must be fixed before CI.
- genos_meshd.c references 4 companion files that don't exist in the POC at all: genos_meshd.h, discovery_mdns.c, identity_ed25519.c, wireguard_autoconfig.c, gossip.c — these must be authored
- kernel/security/xkabi_capabilities.c references `current->xkabi_ht` — requires task_struct extension, impossible without kernel patch or userspace shim layer
- Merkle Accumulator imports numpy — must be added to pyproject.toml
- EP-RDMA imports numpy — same dependency
- Existing CI has separate pytest invocations (platform/ shadow workaround)
- firmware/ is completely empty — ADR-003 in monorepo covers firmware strategy

## Risk Register

R1: MISSING MESHD COMPANION FILES — HIGH
  5 files referenced by genos_meshd.c don't exist anywhere. Must be authored.
  Mitigation: The-Architect authors stubs first, Reliability-Security-Sentinel reviews.

R2: XISC HEADER C++ SYNTAX — HIGH
  `typedef enum : uint8_t` is C++11, not C11. Will break gcc -std=gnu11.
  Mitigation: Rewrite as typedef + #define pattern compatible with gnu11.

R3: KERNEL TASK_STRUCT EXTENSION — HIGH
  xkabi_capabilities.c references `current->xkabi_ht`. Cannot modify task_struct
  without kernel patch or LSM hook. Must use LSM blob or prctl-based approach.
  Mitigation: Use security_blob approach or thread-local via prctl for POC phase.

R4: NUMPY NOT IN PYPROJECT.TOML — MEDIUM
  Both merkle_collective.py and fabric_energy_controller.py import numpy.
  Will fail in CI immediately.
  Mitigation: T1 adds numpy>=1.26 to pyproject.toml before any Python work.

R5: CI SHADOW MODULE COLLISION — MEDIUM
  Existing note: platform/ shadows stdlib platform module. New kernel/ and gateway/
  directories could create similar collisions if test files are added.
  Mitigation: Extend pytest.ini_options testpaths explicitly; never use wildcard collection.
