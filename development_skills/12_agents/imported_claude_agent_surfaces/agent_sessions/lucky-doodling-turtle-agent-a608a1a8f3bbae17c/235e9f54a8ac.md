## Key Findings Summary

### CRITICAL (P0) — 4 findings
- C1: xkabi_capabilities.c:95 — 16-bit MAC truncation. Only 2 bytes of HMAC used for handle integrity. 2^16 = 65536 guesses to forge any capability handle.
- C2: xkabi_capabilities.c:194 — Object ID truncation to 24 bits for idr_find lookup. 64-bit object_id & 0xFFFFFF. Only 16M unique IDs before collision. Two capabilities with same low 24 bits will validate against the wrong cap object.
- C3: rdma_attestation.c:146 — Fixed-length SHA-256 call: crypto_sha256(peer_cert, 256, digest). peer_cert pointer treated as always being 256 bytes. No length parameter passed from call sites. Truncates or overreads variable-length cert chains.
- C4: mr_registry.c:28 — rkey_table hash collision: rkey % MR_MAX_PER_NODE. Two different rkeys that share the same modulus value will silently overwrite each other in the table. mr_generate_rkey() loop only checks rkey_table[rkey % MR_MAX_PER_NODE] != NULL but does not check that the occupant's rkey matches. Allows one MR to usurp another's rkey slot.

### HIGH (P1) — 8 findings
- H1: xkabi_capabilities.c:181 — Generation epoch check reads from untrusted handle bytes AFTER HMAC verification but WITHOUT re-reading from the cap object. handle[10] generation is attacker-controlled post-decode.
- H2: xkabi_capabilities.c:249-255 — Nested spinlock: spin_lock(&ht->lock) then spin_lock(&cap->lock). Lock ordering is inconsistent with revocation (revoke also acquires ht->lock then cap->lock). If another path acquires cap->lock first, deadlock.
- H3: rdma_attestation.c:90-96 — spdm_get_certificate(): cert_chain is stack-allocated 4096-byte buffer, resp is also 4096. min_t(u32, ret - 8, 2048) guards the memcpy destination but ret can be < 8, making ret-8 wrap to a huge u32, bypassing min_t.
- H4: rdma_attestation.c:149-160 — session_cache_lock held during ktime_get() call (clock read inside spinlock). Also: sess->last_used = ktime_get() inside lock at line 155, with another ktime_get() at 154 - two separate clock reads introduce TOCTOU on session age check.
- H5: genos_meshd.c:111 — snprintf(wg_if, sizeof(wg_if), "genos%d", meshd.n_peers). Interface name is generated BEFORE acquiring meshd.lock. n_peers read races with concurrent discovery thread. Two threads can generate the same wg_if name.
- H6: genos_meshd.c:128 — strncpy(p->endpoint, peers[i].endpoint, sizeof(p->endpoint)) — strncpy does not guarantee null termination if source equals or exceeds buffer length. endpoint field is 64 bytes; if peers[i].endpoint is exactly 64 bytes, no null terminator.
- H7: tier_migration.c:119 — pos = (pm - mr->pages) << PAGE_SHIFT where mr is obtained via pm->lru_node.prev cast. This is casting a list_head pointer to a vdram_mr pointer directly, which is undefined behavior and can produce wrong offset if lru_node is not the first member of vdram_mr.
- H8: mr_registry.c:162-173 — vdram_mr_lookup_rkey() uses rcu_read_lock() but rkey_table is written under spinlock (not RCU-safe). RCU requires call_rcu/synchronize_rcu on the write side, not just a spinlock. Readers can see torn pointer writes.

### MEDIUM (P2) — 10 findings
- M1: xkabi_capabilities.c:314-319 — audit_log_format logs rights in hex and event_type as integer. No process context (pid, uid) in audit record. Audit record insufficient for forensics.
- M2: rdma_attestation.c:132-136 — KDF: crypto_kdf256_expand(session_key_out, digest, 32, nonce, 32). Session key derived from SHA-256(cert) || nonce. No peer's response nonce incorporated. Attacker who controls the nonce replay can influence session key.
- M3: rdma_attestation.c:240 — sess->n_bound_rkeys used as array index without bounds check before increment: *(u32 *)&sess->bound_rkeys[sess->n_bound_rkeys++] = rkey. bound_rkeys array is [64][4]. If n_bound_rkeys reaches 64 before check, writes out of bounds.
- M4: genos_meshd.c:169-171 — identity_ed25519_sign called with sizeof(my_state) - 64. If gossip_state_msg is <= 64 bytes, this underflows to a huge size_t, signing garbage memory.
- M5: genos_meshd.c:291-307 — genos_meshd_best_t3_peer() returns p->endpoint (pointer into meshd.peers array) after releasing meshd.lock. Caller holds a pointer into the array without the lock — use-after-free if a concurrent thread removes the peer.
- M6: pf_handler.c:265-266 — fault_va calculated as vmf->vma->vm_start + (vmf->pgoff << PAGE_SHIFT). pgoff is user-controlled. No validation that computed VA is within vma bounds before passing to pf_find_mr. An adversarial pgoff could produce a VA outside the VMA.
- M7: tier_migration.c:113-114 — filp_open(t2->nvme_path, O_RDWR | O_CREAT, 0600) in kernel context. nvme_path is derived from node configuration but is never validated to be an absolute path or to be within expected boundaries. Path injection possible if node config is user-influenced.
- M8: entropy_classifier.c:65 — payload_offset = min_t(u32, 40, pkt_len / 4). If pkt_len < 160 the offset becomes pkt_len/4. Then score = entropy_classifier_score(pkt + payload_offset, pkt_len - payload_offset). If pkt_len == 0, both are 0 but entropy_classifier_score guards this. If pkt_len < 40 but > 0: pkt_len - payload_offset = pkt_len - pkt_len/4 which is fine. Edge case: pkt_len=1, offset=0, len=1 — will score only 1 byte and route to UET every time.
- M9: fabric_energy_controller.py:165 — _write_lane_register opens device path via f-string: open(f"{self.device_path}/lanes", "w"). device_path is user-supplied CLI argument. No path sanitization — directory traversal possible if attacker controls --device argument.
- M10: thermal_guardian.py:109 — Path("/tmp/genos_thermal_state.json").write_text() writes to a world-writable /tmp path. Symlink attack: attacker pre-creates /tmp/genos_thermal_state.json as a symlink to an arbitrary file. Process running as any elevated user will overwrite it.

### LOW (P3) — 8 findings
- L1: xkabi_capabilities.c:301-308 — xkabi_ht_free() does not drain active capabilities before freeing. If concurrent operations hold references into the idr, use-after-free on the idr memory.
- L2: xkabi_capabilities.c:71 — global_revocation_epoch is a plain u32 with no overflow protection. After 4 billion epoch advances, wraps to 0 and all revoked caps become valid again.
- L3: rdma_attestation.c:51-53 — Session cache is a hashtable with no eviction policy and no maximum size. DoS: attacker can exhaust kernel memory by triggering enough unique SPDM sessions, each consuming sizeof(spdm_session) ~= 400+ bytes.
- L4: genos_meshd.c:250-251 — pr_info("genos-meshd: pubkey=%*phN\n", 8, meshd.my_pubkey) logs the first 8 bytes of our Ed25519 public key to dmesg. Public keys should not be logged without explicit operational intent — they can be used to target a node.
- L5: qp_relay.c:167 — translate_ib_to_roce(NULL, dst_gid, dst_qpn, entry->cached_roce_udp) — ib_bth is NULL. Inside translate_ib_to_roce, memcpy(roce_hdr_out + 8, ib_bth, 12) will NULL-dereference on first access.
- L6: qp_relay.c:281-287 — pkt + 12 passed to fpga/doca forward without length validation. If pkt_len < 12, this produces a pointer past the end of the packet buffer.
- L7: test_xkabi_barriers.c:63 — rdma_mr_register uses rand() for rkey generation: (uint32_t)(rand() & 0xFFFFFFFF). rand() is not cryptographically secure and is seeded with a predictable default. Production kernel code must use get_random_bytes() — the test stub does not reflect the actual kernel behavior and creates false confidence.
- L8: merkle_collective.py:56-58 — MerkleCommitment.deserialize() uses assert len(data) == 64. In Python, assertions can be disabled with -O flag. Production deserialization must use explicit if/raise, not assert.
