# RFC-002 v5 Integration Architecture Plan

## Status: PROPOSAL (Plan Mode — No Edits Made)

## 10 Technologies Being Integrated
1. vDRAM — 4-tier unified memory engine (kernel module)
2. Contra-Rotation Scheduler — thermal-driven cooperative scheduler (kernel module)
3. RDMA Attestation — SPDM + per-QP AES-256-GCM (kernel module)
4. Zero-Copy Gateway — QP relay without payload buffering (C userspace)
5. XISC JIT — LLVM IR with policy annotation pass (C, LLVM dependency)
6. EP-RDMA — Energy-proportional fabric controller (Python tool)
7. Merkle Verifiable Compute — in-fabric commitment (Python tool + C tests)
8. genos-meshd — OS-native mesh daemon (kernel module)
9. XKABI Capabilities — 128-bit handle table (existing xisc/ extended)
10. GEN.VIRT — XISC-native hypervisor (stubs only, Phase 1)

## Key Observations from Code Review
- POC xisc/include/xisc.h conflicts with existing xisc/include/xkabi.h namespace
- xisc_llvm_jit.c uses linux/slab.h + LLVM-C headers simultaneously — JIT lives in userspace wrapper, not in-kernel
- vdram.h uses linux/mm.h kernel headers — true kernel module
- EP-RDMA (fabric_energy_controller.py) and Merkle (merkle_collective.py) are pure Python tools
- merkle_collective.py imports numpy — new Python dependency required
- xisc_llvm_jit.c includes #include "../vdram/vdram.h" — cross-directory coupling
- Existing CI has 4 jobs; need to add 5+ new jobs
- Dockerfile.buildenv missing: cmake (only build-essential), llvm headers, rdma-core, linux-headers
- packages.list missing: wireguard-tools, linux-headers-amd64, rdma-core
- pyproject.toml missing: numpy in dev deps
