## Key Dependency Graph (raw)
- vDRAM PF Handler depends on: mesh (T3 transport), JIT (hints), attestation (MR gate), XKABI (T2 cap), Contra-Rotation (compute/cooldown callbacks)
- Contra-Rotation depends on: vDRAM (prefetch trigger), mesh (gossip thermal), Thermal Guardian (temp feed)
- JIT depends on: vDRAM (hint injection), XKABI (security enforcement), vDRAM MR Registry
- XKABI depends on: nothing upstream (root security primitive)
- genos-meshd depends on: Ed25519 identity, WireGuard, vDRAM (T3 availability query), Contra-Rotation (phase gossip)
- ZC Gateway depends on: RDMA Attestation (QPN verification), FPGA/DPU hardware
- RDMA Attestation depends on: SPDM transport, TPM (PCR extend), vDRAM (MR revocation)
- Merkle depends on: RDMA Attestation (commit gates), vDRAM (commit point), NCCL
- EP-RDMA depends on: Contra-Rotation (cooldown signal), NCCL events, DPU (BlueField-4)
- Thermal Guardian depends on: Contra-Rotation (feeds thermal data), sysfs

## Failure Mode Categories
- vDRAM: SIGBUS/SIGSEGV propagation to all processes using vDRAM VMAs
- RDMA Attestation session cache corruption: all T3 access denied
- genos-meshd crash: T3 tier collapses, prefetch disabled
- XKABI epoch advance: ALL T2 capabilities invalidated simultaneously
- JIT cache eviction bug: silent random execution of stale native code
- Contra-Rotation thread death: no cooldown phases, thermal runaway risk
- EP-RDMA: graceful degrade (full power, no savings)
- Merkle: silent (integrity auditing lost, compute continues)
- ZC Gateway: enterprise fabric drops, consumer unaffected
- Thermal Guardian: monitoring blind, Contra-Rotation uses stale temps
