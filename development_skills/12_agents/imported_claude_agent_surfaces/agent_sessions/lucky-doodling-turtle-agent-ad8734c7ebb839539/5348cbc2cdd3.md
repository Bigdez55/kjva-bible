## Key Findings Summary (pre-report)

### CRITICAL Issues
1. pf_handler.c:335 — `vdram_tier_resolve_page()` called but NEVER DEFINED anywhere in the POC
2. pf_handler.c:141 — `iommu_domain_check_va()` called but not defined; `iommu_domain_check_va` takes mnt_ns as arg (wrong API)
3. tier_migration.c:119,171 — Pointer arithmetic `pm - mr->pages` to recover index from lru_node.prev pointer is UNDEFINED BEHAVIOR (invalid cast)
4. tier_migration.c:232 — `rdma_post_write_wr()` called for RDMA READ operation (IB_WR_RDMA_READ) — wrong function name
5. tier_migration.c:337 — Uses `-ENOTSUP` which is POSIX-only, not a standard Linux kernel errno
6. mr_registry.c:28 — `mr_generate_rkey()` busy-loop has potential infinite loop if table is nearly full
7. mr_registry.c:152 — `crypto_kdf256_expand()` called but is NOT a real Linux kernel crypto API function
8. mr_registry.c:164-174 — `vdram_mr_lookup_rkey()` uses `rcu_read_lock()` but table access is via spinlock-protected array, not RCU-managed — lock mismatch
9. xkabi_capabilities.c:170 — `current->xkabi_ht` — `xkabi_ht` field does NOT exist in Linux `task_struct`; requires kernel patch
10. pf_handler.c:257-258 — `vdram_fault_handler` return type declared as `int` but `vm_fault_t` is required by Linux mm
11. contra_rotation.c:89 — `thermal_zone_get_zone_by_name(NULL)` — incorrect API call, requires name string not NULL
12. contra_rotation.c:78 — `g_cr_sched` global with no synchronization for `contra_rotation_compute_active()` lockless reads
13. genos_meshd.c:179,185 — `gossip_send()` called inside `spin_lock()` — DEADLOCK risk (network I/O under spinlock in kernel)
14. rdma_attestation.c:135 — `crypto_kdf256_expand()` — same fake API, does not exist in kernel
15. rdma_attestation.c:134 — `crypto_sha256()` is not a standard kernel API; should be `crypto_shash` with allocated tfm
16. xisc_llvm_jit.c — LLVM C API used inside a Linux kernel module — this is architecturally impossible; LLVM cannot run in ring-0
17. xisc.h:42 — `typedef enum : uint8_t` — C++ syntax, not valid C11/GNU11
18. prefetch_engine.c:100 — `node->prefetch_state` field does not exist in `vdram_node` struct defined in vdram.h

- `ci.yml`: No C compilation step for RFC-002 POC test suite; kernel module builds not integrated
- `ci.yml`: xkabi conformance step runs Python pytest against `xisc/conformance/` but the RFC-002 conformance suite is pure C, not Python
