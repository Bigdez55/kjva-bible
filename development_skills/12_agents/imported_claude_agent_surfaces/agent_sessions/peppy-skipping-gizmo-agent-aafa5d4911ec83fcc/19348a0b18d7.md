## CHECKPOINT 6: DEMO GATE VALIDATION

### Demo 9: EP-RDMA Energy Savings (>20%)

**Status: FAIL -- P0**

**Gate Claim:** >20% power savings via EP-RDMA SerDes lane scaling.

**Evidence Examined:**
- `/tmp/rfc002_analysis/gen-os/tools/ep_rdma/fabric_energy_controller.py` -- Well-designed controller with async control loop, DPU interface, state machine, energy accounting.
- CI job `demo-gates` step "Demo 9" -- Only runs `py_compile` (syntax check).

**Findings:**
1. The Python controller calculates savings as: `1.0 - actual_energy / baseline_energy` where `baseline_energy` is always `LANES_MAX * POWER_PER_LANE_W * dt` and `actual_energy` uses `dpu.read_power_watts()` which in simulation mode returns `sim_lanes * 50.0`.
2. If the controller runs with NCCL simulation (30% idle ratio), the mathematical model yields: 30% idle * 75% reduction during idle = 22.5% average savings. This exceeds the 20% gate.
3. **However:** This is a mathematical tautology, not an empirical measurement. The simulation produces the exact savings the formula predicts. No actual hardware power measurement validates the claim.
4. The CI never executes the controller at all -- it only syntax-checks the file.

**Verdict:** The Demo 9 gate is not validated by CI. The controller's simulation would pass mathematically but this is not a genuine gate check. The CI step is misleading.

### Demo 10: Merkle Verifiable Compute (100% Detection)

**Status: PASS**

**Gate Claim:** 100% tamper detection rate.

**Evidence Examined:**
- `/tmp/rfc002_analysis/gen-os/tests/verifiable/test_merkle.c` function `test_100_percent_tamper_detection_rate()` (lines 117-146) -- Runs 1000 trials with random 8-leaf trees, random leaf/byte/bit tamper, verifies 100% detection.
- CI job `demo-gates` step "Demo 10" -- Runs `./build/test_merkle` which executes the 1000-trial gate test.
- Uses `srand(42)` for deterministic reproduction.

**Findings:**
1. The test uses a non-cryptographic hash (FNV-1a variant) but the collision properties are sufficient for the POC demo gate.
2. The 1000-trial test with random single-bit flips across random positions provides strong statistical evidence.
3. The Python `merkle_collective.py` `demo10_tamper_detection_test()` provides an independent validation path using real SHA-256.
4. CI actually executes the binary and checks its exit code.

**Verdict:** Demo 10 gate is genuinely validated. The C test suite runs 1000 random tamper trials and requires 100% detection. The CI correctly executes this binary and would fail on non-zero exit.
