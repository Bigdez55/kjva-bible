## EXECUTIVE SUMMARY

Five deliverables, sequenced for maximum value extraction with minimal risk. Total timeline: **30 weeks** (buffered from raw 25 weeks). The critical insight: these deliverables have natural parallelism -- the CRA-to-Vite migration is entirely frontend, the Go services are entirely backend, and the Rust WASM expansion touches only the `frontend/src/wasm/` subtree. However, the single-developer constraint means we serialize execution, with small overlap windows during review/stabilization phases.

**Sequencing Rationale (Value vs Risk):**

| Order | Deliverable | Why This Order |
|-------|------------|----------------|
| 1 | CRA -> Vite | Fastest ROI. Unblocks DX. Zero production risk (same output). |
| 2 | Go Market Data Gateway | Highest user-visible perf gain (500ms -> 50ms). Read-only service. |
| 3 | Go Risk Engine | Builds on Gateway infra patterns. Compute-heavy, no write operations. |
| 4 | Go Order Router | SAFETY CRITICAL -- real money. Must be last Go service (most infra maturity). |
| 5 | Rust WASM Expansion | Lowest urgency. Frontend-only. Can run as cooldown after Order Router intensity. |

### Consumer-Driven Contract Testing

**Pattern:**
1. Python backend defines expected response schemas as Pydantic models
2. These schemas are exported as JSON Schema files in `backend/contracts/`
3. Go services load these schemas in tests and validate their responses against them
4. If a Go service changes its response, Python contract tests fail in CI
5. This prevents breaking changes from Go -> Python

### Shadow Traffic Validation

**Protocol for each Go service:**

1. **Setup (Sprint N.4):**
   - Python backend sends request to both Python handler AND Go service
   - Go service response is NOT returned to the client
   - Both responses are logged with a correlation ID

2. **Comparison (continuous):**
   - Background job compares Go vs Python responses
   - Discrepancies logged with full context (request, both responses, timestamp)
   - Alert if discrepancy rate > 0.1%

3. **Duration:**
   - Market Data Gateway: 1 week shadow minimum
   - Risk Engine: 1 week shadow minimum
   - Order Router: 2 weeks shadow minimum (paper trades only -- NEVER shadow live orders)

4. **Exit Criteria:**
   - 0 discrepancies for last 48 hours
   - `reliability-security-sentinel` sign-off
   - `fintech-integrity-auditor` sign-off (Order Router only)
