# Plan: Fix ajv/schema-utils Build Failure in frontend/package.json

## Status: READY TO EXECUTE

## Summary
Remove the misconfigured `"schema-utils": { "ajv": "^6.14.0" }` npm override and replace with
targeted `"webpack"` and `"react-scripts"` overrides that force ajv@8 in the two subtrees that
host both schema-utils@4.3.3 AND a nested ajv@6.

## Root Cause (verified)
- `schema-utils@4.3.3` calls `require("ajv").default` ‚Äî requires ajv@8+
- `package.json` override `"schema-utils": { "ajv": "^6.14.0" }` forces schema-utils@4.3.3 to get ajv@6
- Two on-disk locations where ajv@6 shadows ajv@8 for schema-utils@4.3.3:
  - `node_modules/react-scripts/node_modules/ajv` => v6.14.0 (BROKEN)
  - `node_modules/webpack/node_modules/ajv` => v6.14.0 (BROKEN)

## Files to Change
1. `frontend/package.json` ‚Äî overrides block only
2. `frontend/package-lock.json` ‚Äî auto-updated by `npm install` (not edited manually)
