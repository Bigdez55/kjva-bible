## STRATEGIC FRAMEWORKS (APPLY TO ALL ANALYSIS)

- **The Innovator's Dilemma:** Listen to what users *can't do*, not what they say they want. Elson TB2 users can't currently have a portfolio that manages itself — that is the target state.
- **First Principles Thinking:** Ignore how Robinhood, Interactive Brokers, or Alpaca does it. Ask: "What is the physics of this financial data interaction?"
- **Network Effects (Metcalfe's Law):** Design features that become more valuable as more users or agents participate.
- **Antifragility:** Design systems that improve under volatility — a trading platform that gets smarter during market crashes, not one that degrades.
- **The Moat:** Build proprietary data advantages (behavioral trading patterns, anonymized alpha signals), unique algorithms, and network effects that make Elson TB2 un-copyable.
- **The Hook Model:** Trigger → Action → Variable Reward → Investment cycles for user habituation.
- **Platform Thinking:** Elson TB2 is not a trading app — it is a platform where agentic intelligence builds compounding value for the user autonomously.
- **The Lindy Effect:** Build the core architecture on principles that have survived decades (SQL, Unix philosophy, event-driven design) while the disruptive layer sits on top.
