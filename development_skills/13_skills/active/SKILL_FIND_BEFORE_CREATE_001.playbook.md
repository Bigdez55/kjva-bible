# Playbook: Find Before Create

## Skill ID
SKILL_FIND_BEFORE_CREATE_001

## Purpose
Force a structured search for canonical existing locations **before** any file or folder is created. Hard enforcement: if a candidate exists, the agent MUST reuse it OR leave a redirect stub at the abandoned location before creating elsewhere. This prevents the orphan-folder failure mode where an agent finds an existing artifact, decides not to use it, and creates a parallel duplicate elsewhere — leaving the original empty and the project drifting.

## When to invoke

Invoke this skill via [/apex:check_path](../../37_command_protocol/slash_commands/apex_check_path.md) **before**:
- Creating any new file via `Write`.
- Creating any new directory via `mkdir` or via writing a file under a non-existent parent.
- Writing the first content into a previously-empty `.gitkeep`-only directory.
- Adding a new entry to a registry (skills, schemas, templates, ADRs, slices, etc.).
- Creating a new branch dedicated to a topic that may already have work in progress.

Do **not** invoke for:
- Edits to a file you have already read and confirmed exists.
- Routine appends to ledgers/registries that are auto-generated by `sync_registries.py`.

## Inputs

- `target_path` — string, required. The path the agent intends to create or write to. Absolute or repo-relative.
- `artifact_kind` — string, optional. One of `file | directory | registry_entry | skill | adr | spec | diagram | template | schema`. When supplied, expands the search to the artifact's canonical home (e.g. `adr` → `04_architecture/adrs/`).
- `repo_root` — string, optional. Defaults to the nearest git root of `target_path`.

## Steps

### 1. Normalize the target

- Resolve `target_path` to an absolute path.
- Compute case-insensitive, hyphen/underscore-collapsed token (e.g. `super-c-academy` and `super_c_academy` both collapse to `superCAcademy`).

### 2. Search

Run **all six** of the following searches and concatenate results:

1. **Exact path** — does `target_path` already exist (file, dir, or symlink)?
2. **Case/separator variant** — search for any sibling whose collapsed token equals the normalized target token.
3. **Content fingerprint** — if creating a file with planned content, hash the planned bytes and look for any existing file under `repo_root` with the same hash (`git ls-files | xargs -I{} sha256sum`).
4. **Artifact-kind locality** — if `artifact_kind` is supplied, list every existing artifact under that kind's canonical home (see table below) and check whether the new artifact's purpose overlaps any.
5. **Frontmatter/ID match** — for skill yaml, ADR md, spec yaml, etc., the new artifact has a planned `id`. Search every same-kind artifact for that `id` already in use.
6. **OneDrive ghost check** — if `repo_root` is under a OneDrive path, list dehydrated-but-present file names (xattr `com.apple.fileprovider.fpsd@`-flagged) under the target's parent directory; these would not show up in standard `find` if not materialized.

#### Artifact-kind canonical homes

| `artifact_kind` | Canonical home |
|---|---|
| `skill` | `13_skills/active/` |
| `adr` | `04_architecture/adrs/` |
| `spec` | `03_specs/<category>/` |
| `diagram` | `04_architecture/diagrams/source/` |
| `template` | `14_templates/<class>/` |
| `schema` | `26_schemas/<class>/` |
| `slice` | `22_vertical_slices/` |
| `evidence` | `23_evidence/evidence_packets/` |
| `runbook` | `10_operations/runbooks/` |
| `policy` | `15_governance/policies/` |
| `intake_packet` | `00_intake/intake_packets/` |
| `starter_packet` | `30_repo_starter/starter_packets/` |
| `repo_twin` | `39_repo_twins/twins/<NAME>/` |

### 3. Decide

- If **search 1 hits** → decision: `REUSE_CANDIDATE`. The artifact exists. Do not create. Read it and either edit in place or surface to the caller.
- If **search 2/3/5 hits** (variant or duplicate ID) → decision: `REUSE_CANDIDATE` or `REDIRECT_REQUIRED`. If the existing path is canonical, reuse. If the new path is more correct (e.g. matches updated naming), the agent must:
  1. Move the canonical content from the old path to the new path via `git mv`.
  2. Leave a `MOVED.md` stub at the old location pointing to the new one (template below).
  3. Update every reference to the old path (registries, links, ADR rollback paths, etc.).
- If **search 4 hits** (artifact-kind locality has overlapping artifacts) → decision: `REUSE_CANDIDATE` if any overlap is substantive; else `OK_TO_CREATE` with `rationale` explaining why none of the existing artifacts in the canonical home cover the new one.
- If **search 6 hits** (OneDrive ghost) → decision: `HALT`. Do not create until the ghost is materialized (open in Finder) or removed. Creating alongside a ghost causes silent collision on next sync.
- Otherwise → decision: `OK_TO_CREATE`.

### 4. Enforce

- `REUSE_CANDIDATE` — the agent does not call `Write`/`mkdir` on `target_path`. Continues by reading and editing the candidate.
- `REDIRECT_REQUIRED` — before any new content is written at `target_path`, the agent writes the redirect stub at the old canonical location and updates every reference. Only then does work continue.
- `HALT` — surface the OneDrive ghost or other blocker to the caller; await resolution.
- `OK_TO_CREATE` — the agent records the `rationale` in the new artifact (see "rationale capture" below) and proceeds.

### 5. Rationale capture (for `OK_TO_CREATE` only)

Every artifact created after a non-trivial check (i.e. searches 2–6 returned candidates that the agent decided not to reuse) must carry a `creation_rationale` field:

- For YAML artifacts: top-level `creation_rationale: "<one sentence>"` field.
- For markdown artifacts: a `## Creation rationale` section near the top.
- For files where adding metadata is impractical (binary, generated): record the rationale in [18_registry/change_ledger.yaml](../../18_registry/change_ledger.yaml) on the same row that introduces the file.

This is the audit trail. If a future drift report shows orphan candidates, the rationale tells the next agent (or a human) why the choice was made.

### 6. Redirect stub template

When `decision == REDIRECT_REQUIRED`, write the following at the old canonical path (named `MOVED.md` if the old path was a directory; `<old_filename>.MOVED.md` if it was a file):

```markdown
# MOVED

This path was superseded on <YYYY-MM-DD> per [SKILL_FIND_BEFORE_CREATE_001](../../13_skills/active/SKILL_FIND_BEFORE_CREATE_001.yaml).

- **Old path:** `<old/path>`
- **New canonical path:** `<new/path>`
- **Reason:** <one sentence>
- **Change ledger row:** `<CL-NNNN>`

Do not author new content here. Open the new path.
```

## Outputs

- `decision` — one of `OK_TO_CREATE | REUSE_CANDIDATE | REDIRECT_REQUIRED | HALT`.
- `candidates` — list of existing paths that matched any of searches 1–6.
- `rationale` — captured per step 5 when `decision == OK_TO_CREATE`.
- `redirect_record` — the `MOVED.md` content written per step 6 when `decision == REDIRECT_REQUIRED`.

## Failure modes

| Failure | Detection | Remediation |
|---|---|---|
| Agent skips the check and creates a duplicate | post-hoc drift report finds two paths with same canonical token | Run `/apex:check_path` retroactively; pick a canonical, leave `MOVED.md` at the loser. Add a row to mistake ledger; if recurrent, bump skill `improvement_history`. |
| Agent finds candidate but creates anyway with weak rationale | review of `creation_rationale` strings during audit | Reject the rationale on the next slice review; revert to candidate. |
| Agent abandons folder with no redirect | empty directory present + parallel content elsewhere | `git mv` content back or forward; write `MOVED.md` retroactively; flag in mistake ledger. |
| Search misses case/separator variant | post-hoc grep with collapsed-token comparison | Add the collapsed-token check to the agent's pre-write hook; verify once per session at start. |
| Search misses OneDrive ghost | next sync recreates the ghost path, causing collision | Materialize ghosts before any session that may create files in the same parent (open the parent dir in Finder, scroll once). Alternatively, mark the parent "Always keep on this device". |
| `creation_rationale` field missing on artifact created after a non-trivial check | schema validation against [26_schemas/](../../26_schemas/) | Required-field check; reject the artifact at PR review until rationale is added. |

## Hard enforcement

This skill enforces the FIND-before-CREATE rule **hard**: an agent that does not call `/apex:check_path` and produces a duplicate path violates the skill, and the violation is recorded as a mistake. Two violations of the same kind across slices triggers a skill improvement proposal per [SKILL_REPO_ONBOARDING_001](../../13_skills/active/SKILL_REPO_ONBOARDING_001.yaml) Tier 4 step 3.
