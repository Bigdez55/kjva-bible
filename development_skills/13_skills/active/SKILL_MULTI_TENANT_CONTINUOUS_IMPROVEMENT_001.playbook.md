# Skill: Continuous Skill Improvement

## Purpose

Use this skill to keep the multi-tenant platform skill stack improving over time.

- improve the skill

## Core Doctrine

Every missed requirement, failed build, repeated bug, user correction, or new architectural insight must strengthen the skill stack.

## Required Outputs

When invoked, produce:

1. Miss/gap description.
2. Affected skill(s).
3. Root cause.
4. New trigger phrase.
5. New checklist item.
6. New gate or test.
7. New anti-pattern.
8. Version update note.
9. Regression prompt.
10. Cross-skill routing update.

## Improvement Record Template

```markdown

## Date

<skill path/name>

<what was missed>

## Root Cause

<trigger/checklist/gate/test/doc change>
```

## Required Continuous Improvement Mechanisms

Each skill should support:

- miss logging
- failure-mode updates
- trigger refinement
- checklist strengthening
- test prompt additions
- regression cases
- cross-skill routing updates
- versioned improvement notes

## Anti-Patterns

Avoid:

- letting user corrections disappear
- updating only the current answer but not the skill
- no regression prompt
- no gate for repeated failure
- no cross-skill update
